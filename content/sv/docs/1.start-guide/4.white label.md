## Introduktion

White label förvandlar Blue till en flexibel projekt- och processhanteringsplattform som ser ut och känns som din egen produkt — utan bördan av att hantera teknik, utvecklare, servrar eller drifttid.

Med Blues White Label kan du:

- **Visa upp ditt varumärke:** Ersätt Blues namn, logotyp och färger med dina egna för att skapa en sömlös varumärkesupplevelse.
- **Köra på din domän:** Driva Blue på din egen webbdomän för en professionell, sammanhängande närvaro.
- **Använda din e-postinfrastruktur:** Skicka e-post från din adress för personlig kommunikation.


För byråer stärker White Label-tillägget ditt varumärke, förbättrar serviceerbjudanden och effektiviserar kundsamarbete. Genom att integrera Blue i dina arbetsflöden under ditt varumärke skapar du en sömlös, professionell upplevelse som hjälper dig att sluta fler kunder och behålla dem genom att leverera en unik blandning av tjänster och programvara skräddarsydd för deras behov.


## Prissättning

White label-tillägget kostar $20/månad per företag eller $200/år per organisation, utöver din grundläggande Blue-prenumeration. Du kan läsa mer om prissättning [här](/pricing).

Om du är intresserad av att köpa white label i bulk för flera organisationer, kontakta sales@blue.cc
 

## Anpassade domäner

För att White Label ska fungera måste du först sätta upp en anpassad domän för webbapplikationen. 

Anpassade domäner kan konfigureras för följande:

- Webbapplikation (krävs)
- [Formulär](/docs/projects/forms) (valfritt)
- [Fildelningslänkar](/docs/projects/file-management) (valfritt)
- Dokumentation (kommer snart)


För att konfigurera en anpassad domän, lägg till CNAME-poster i dina DNS-inställningar för varje subdomän. Observera att endast subdomäner stöds, inte rotdomäner. Så `app.yourcompany.com` stöds, men `yourcompany.com` gör det inte.


| Domän-typ                  | CNAME-post                   | Pekar på                   |
|----------------------------|------------------------------|----------------------------|
| Webbapplikation            | `app.yourcompany.com`        | `wl-app.onrender.com`      |
| Formulär                   | `forms.yourcompany.com`      | `wl-forms.onrender.com`    |
| Fildelningslänkar          | `files.yourcompany.com`      | `wl-files.onrender.com`    |


Tabellen ovan visar föreslagna CNAME-poster, men du kan använda vilken subdomän du vill. 

Till exempel kan huvudwebbapplikationen vara `projects.yourcompany.com` och fildelningslänkar kan vara `media.yourcompany.com`.


## Varumärkesanpassning

Anpassa följande element för att anpassa Blue till ditt varumärke:

- **Logotyp**: Ladda upp din logotyp för att ersätta Blues varumärke över hela plattformen.
- **Favicon**: Lägg till din egen favicon som visas i webbläsarflikar och bokmärken.
- **Primär gränssnittsfärg**: Ställ in en primär färg för gränssnittet som tillämpas konsekvent för alla användare.
- **Appnamn**: Anpassa appnamnet som visas i webbläsarflikar och i skrivbordsapplikationen.
- **Typsnitt**: (Kommer snart) Anpassa typsnitten som används över plattformen för att matcha ditt varumärke från en utvald lista av Google-typsnitt.

Blue tillhandahåller en förhandsvisning av inloggningssidan och appikonen medan du konfigurerar dina varumärkesinställningar.

Din varumärkesanpassning kommer även att gälla för [skrivbordsapplikationen](/docs/start-guide/download-apps#desktop-app), vilket säkerställer en helt white-labelad upplevelse.

Detta är den rekommenderade konfigurationen för logotyp och favicon:

| Tillgång | Rekommenderad form | Bakgrund      | Bildförhållande |
|----------|-------------------|---------------|-----------------|
| Logotyp  | Rektangel         | Transparent   | 1:1.5           |
| Favicon  | Fyrkant           | Transparent   | 1:1             |


## E-postnotifikationer

Du kan använda SMTP för att skicka e-postnotifikationer från din egen e-postserver med din egen domän istället för vår e-postadress.

Du behöver ange följande information till Blue:

| Inställning                           | Beskrivning                                                                                                                                        |
|---------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------|
| **SMTP-server**                       | Adressen till servern som hanterar sändning av dina e-postmeddelanden.                                                                            |
| **SMTP-port**                         | Portnumret som används för att ansluta till SMTP-servern, vanligtvis 587 för TLS eller 465 för SSL.                                              |
| **Användarnamn**                      | Användarnamnet för autentisering med din SMTP-server, vanligtvis din e-postadress.                                                               |
| **Lösenord (lagras i krypterat format)** | Lösenordet associerat med användarnamnet, som kommer att lagras säkert i krypterat format för att skydda dina inloggningsuppgifter.        |
| **Avsändarnamn**                      | Namnet som visas som avsändare i e-postnotifikationerna och hjälper mottagare att identifiera källan till e-postmeddelandet.                    |
| **Avsändar-e-postadress**            | E-postadressen som visas som avsändare, vilket säkerställer att svar går till rätt adress. Observera att detta måste vara samma domän som SMTP-servern. |

Blue kräver användning av TLS/SSL-kryptering för SMTP-anslutningen.

Du kan sedan testa anslutningen till din SMTP-server genom att klicka på knappen "Testa anslutning". Blue kommer automatiskt att testa anslutningen till din SMTP-server och meddela dig om den lyckas eller inte via en notifikation i appen.

### Överväganden 

Alla transaktions-, notifikations- och automationsmeddelanden kommer att skickas via din SMTP-server. Det enda undantaget är e-postmeddelanden som innehåller OTP (One Time Passwords) som skickas via en generisk e-postadress (notifications@process-system.app) för att säkerställa säkerheten.

Säkerställ att din SMTP-server:

- Kan hantera stora e-postvolymer.
- Has ett starkt sändningsrykte för att undvika spamfilter.
- Har tillräcklig kvot för att hantera den förväntade e-postbelastningen.


### Anpassade inbjudningsmeddelanden

Du kan också anpassa det första inbjudningsmeddelandet som skickas till nya användare när de registrerar sig för din Blue-instans. 

### Anpassade introduktionsmeddelanden (Kommer snart)

Utan white labeling skickar Blue en serie introduktionsmeddelanden till nya användare när de går med i ett företag för att hjälpa dem komma igång. När white-label är aktiverat skickas inte dessa meddelanden, och i framtiden kommer du att kunna skapa din egen uppsättning introduktionsmeddelanden för att välkomna nya användare.

## Vanliga frågor


**Vad ingår i white label-tillägget?**

White label-tillägget inkluderar tre huvudfunktioner: anpassad domänintegration för webbapplikation, formulär och fildelning; anpassad varumärkesanpassning (logotyp, favicon, färger och appnamn); och anpassade e-postnotifikationer med din egen SMTP-server.

**Hur mycket kostar white label?**

White label-tillägget kostar $20/månad/företag eller $200/år/företag utöver din grundläggande Blue-prenumeration. Bulkrabatter kommer att finnas tillgängliga i framtiden för konton med flera företag.

**Påverkar white-labeling Blues funktionalitet?**

Nej, alla kärnfunktioner och funktionalitet i Blue förblir exakt densamma. White-labeling ändrar bara utseendet och varumärkesanpassningen av plattformen.


**Kan jag använda en rotdomän istället för en subdomän?**

Nej, Blue stöder endast subdomäner för anpassad domänkonfiguration.

**Behöver jag sätta upp alla anpassade domäner?**

Nej, endast webbapplikationsdomänen krävs. Formulär- och fildelningsdomäner är valfria. Dokumentationsdomänstöd kommer snart.

**Kan jag använda andra subdomännamn än de föreslagna?**

Ja, du kan använda vilka subdomännamn du föredrar. Till exempel kan du använda `projects.yourcompany.com` istället för `app.yourcompany.com`.

**Vilka varumärkeselement kan jag anpassa?**

Du kan anpassa din logotyp, favicon, primär gränssnittsfärg och appnamn.

**Kommer skrivbordsapplikationen att spegla min varumärkesanpassning?**

Ja, skrivbordsapplikationen kommer att vara helt white-labelad med din varumärkesanpassning.

**Kan olika användare ha olika färgscheman?**

Nej, den primära färgen du väljer kommer att tillämpas för alla användare av din Blue-instans.

**Vilka e-postkonfigurationsdetaljer behöver jag ange?**

Du behöver ange SMTP-serveradress, SMTP-port, användarnamn, lösenord, avsändarnamn och avsändar-e-postadress. SMTP-lösenordet lagras i krypterat format för att skydda dina inloggningsuppgifter.

**Kan jag testa SMTP-konfigurationen innan jag går live?**

Ja, Blue tillhandahåller en "Testa anslutning"-knapp som verifierar din SMTP-serverkonfiguration och meddelar dig om resultatet.

**Vilka typer av e-postmeddelanden skickas genom min SMTP-server?**

Alla e-postmeddelanden kommer att skickas genom din SMTP-server, inklusive transaktionsmeddelanden, notifikationsmeddelanden, e-postautomationer. OTP (One-Time Password) e-postmeddelanden skickas via en generisk e-postadress (notifications@process-system.app) för att säkerställa säkerheten.


**Kan jag white-labela mobilapplikationerna?**

Blue stöder för närvarande inte white-labeling av mobilapplikationer. Men om du behöver en varumärkesversion på Apple App Store eller Google Play Store kan du kontakta sales@blue.cc för att diskutera alternativ.

**Kommer white-label-stöd för mobilappar att finnas tillgängligt i framtiden?**

Det finns för närvarande inga planer på att stödja white-labeling av mobilapplikationerna.

**Hur kan jag få hjälp med white-label-konfiguration?**

För allmän hjälp kan du kontakta support@blue.cc. För säljrelaterade frågor om anpassade mobilapplikationer, kontakta sales@blue.cc.

**Kan jag testa white-label-funktionerna innan jag implementerar dem?**

Ja, Blue tillhandahåller en förhandsvisning av inloggningssidan och appikonen medan du konfigurerar dina varumärkesinställningar.

## DNS-inställningar

Vi har bekvämt samlat instruktioner för hur man sätter upp white label-domäner för vanliga DNS (Domain Name Service) system:

#### GoDaddy

1. Logga in på ditt GoDaddy-konto och öppna din produkt.
2. Navigera till Domain Control Center.
3. På sidan Domain Settings, hitta och välj ditt domännamn.
4. Scrolla ner till avsnittet Additional Settings och välj Manage DNS.
5. På sidan DNS Management, scrolla ner till botten av avsnittet Records och klicka på Add.
6. I Type-rullgardinsmenyn, välj CNAME och fyll i följande obligatoriska fält:
    - Host - Värdnamnet eller prefixet för CNAME. Du kan använda en punkt (.), men inte som första eller sista tecken. På varandra följande punkter (..) är inte tillåtna. Värden får inte överstiga 25 tecken eller vara @-symbolen.
    - Points to - Sätt detta till `wl-app.onrender.com`
    - TTL - Ange hur länge servern ska cache-lagra information.
7. Klicka på Save.

#### Bluehost

1. Logga in på ditt BlueHost-konto.
2. Klicka på Domains-menyn och sedan på Zone Editor. En ny sida visas.
3. Navigera till avsnittet Add DNS Record.
4. I fältet Host Record, ange subdomänen för adressen du vill använda för din anpassade domän.
5. I Type-rullgardinsmenyn, välj CNAME-alternativet.
6. I avsnittet Points To, ange `wl-app.onrender.com`
7. Klicka på Add Record.

#### Cloudflare

1. Logga in på ditt Cloudflare-konto.
2. I rullgardinsmenyn, lokalisera och välj din domän.
3. Klicka på fliken DNS Settings.
4. Klicka på "Add Record":
5. Välj CNAME-alternativet.
6. Sätt "Name" till din subdomän
7. Sätt "Target" till `wl-app.onrender.com`
8. Sätt Cloudflare Proxy Toggle-ikonen till ett grått moln.

#### Domain.com

1. Logga in på ditt Domain.com-konto.
2. Lokalisera din domän och klicka på Manage.
3. På menyn till vänster, klicka på DNS and Nameservers.
4. Klicka på Add DNS record för att lägga till CNAME-posten.
5. Sätt target till `wl-app.onrender.com`
6. Klicka på Update DNS för att slutföra processen.

#### Dreamhost

1. Logga in på ditt DreamHost-konto.
2. Till vänster på sidan, klicka på Domains-länken.
3. I Domains-rullgardinsmenyn, välj Manage Domains.
4. Lokalisera din domän och klicka på DNS-länken under din domän.
5. Scrolla ner till avsnittet Add a custom DNS record to (your custom domain) och redigera följande fält:
    - Name - skriv subdomänen du vill använda
    - Type - välj CNAME-alternativet.
    - Value - ange `wl-app.onrender.com`
6. Klicka på Add Record Now! för att lägga till CNAME-posten.

#### Google Domains

1. Logga in på ditt Google Domains-konto.
2. Hitta och klicka på ditt domännamn.
3. På menyn till vänster, klicka på DNS.
4. Navigera till avsnittet Custom Resource Records.
5. I Name-fältet, ange subdomänen för adressen du vill använda för din anpassade domän.
6. I Type-rullgardinsmenyn, välj CNAME-alternativet.
7. I Data-fältet, ange `wl-app.onrender.com`
8. Klicka på Add för att slutföra processen.

#### Hostgator

1. Logga in på ditt HostGator-konto.
2. På vänster sida av sidan, klicka på CPanel.
3. Scrolla ner till avsnittet Domains och klicka på Zone Editor.
4. Hitta din domän och klicka på CName Record-länken bredvid domänen.
5. I dialogrutan Add a CName Record, fyll i följande tomma fält:
    - Name - ange ditt valda subdomän/domännamn.
    - CNAME - ange `wl-app.onrender.com`
6. Klicka på Add a CName Record. Ett meddelande från HostGator visas som anger att du framgångsrikt har lagt till en ny CNAME till ditt HostGator-konto.

#### NameCheap

1. Logga in på ditt NameCheap-konto.
2. I Account-rullgardinsmenyn, välj Dashboard.
3. Klicka på Domain List och hitta din domän.
4. Klicka på Manage bredvid din domän.
5. Klicka på Advanced DNS och navigera till avsnittet Host Records.
6. Klicka på Add New Record.
7. I Type-rullgardinsmenyn, välj CNAME Record och ange din subdomän som värd.
8. Ange sedan `wl-app.onrender.com`
9. Klicka på Save all changes för att slutföra processen.
