Formelbaserade anpassade fält låter dig skapa beräkningar baserade på dina befintliga data i Blue. Detta är utmärkt för att beräkna vinstmarginaler, summor, skattesatser och provisioner.


![Formula 1](/docs/Formula_1.png "Formula 1")

Du kan se en videoöversikt av formelbaserade anpassade fält i videon nedan:

<youtube url="https://www.youtube.com/watch?v=RsXnOc2JU9k" />

Formler kan använda data från följande anpassade fält inom samma post:

- [Valuta](/sv/docs/custom-fields/currency)
- [Stjärnbetyg](/sv/docs/custom-fields/star-rating)
- [Nummer](/sv/docs/custom-fields/number)
- [Procent](/sv/docs/custom-fields/percent)

Skriv helt enkelt en krullparentes `{` så ger systemet dig en rullgardinslista med fält som du kan referera till.

## Stödda Operationer

Du kan sedan utföra operationer på dessa värden med hjälp av funktioner:

- **AVERAGE**: Beräkna medelvärdet av en uppsättning siffror.
- **AVERAGEA**: Bestäm medelvärdet, behandla text som noll.
- **COUNT**: Räkna antalet poster i ett nummerfält.
- **COUNTA**: Räkna antalet icke-tomma värden.
- **MAX**: Hitta det maximala värdet i en uppsättning.
- **MIN**: Identifiera det minimala värdet i en uppsättning.
- **MINUS**: Subtrahera ett värde från ett annat.
- **MULTIPLY**: Multiplicera två värden.
- **SUM**: Lägg ihop en serie siffror

Du kan använda dessa operationer genom att skicka det anpassade fältnamnet till funktionen. Till exempel, om du hade ett anpassat fält som heter `Cost` kunde du använda `AVERAGE(Cost)` för att beräkna den genomsnittliga kostnaden för en uppsättning poster.

För att skicka flera anpassade fält till en funktion kan du använda kommatecknet `,` för att separera varje fält. Till exempel, om du hade anpassade fält som heter `Cost` och `Profit` kunde du använda `AVERAGE(Cost, Profit)` för att beräkna den genomsnittliga kostnaden och vinsten för en uppsättning poster.

Du kan också använda följande matematiska operatorer:

| Operator | Beskrivning |
|:--------:|:-----------:|
| **+**    | Addition    |
| **-**    | Subtraktion |
| **\***    | Multiplikation |
| **/**    | Division    |
| **%**    | Procent  |
| **^**    | Upphöjning |

Se ett exempel nedan på att skapa en vinstberäkning baserad på försäljningspris och kostnad:

Det finns inga begränsningar för komplexiteten eller antalet operationer i dina formler. De kommer att valideras korrekt så länge syntaxen är giltig. Formler refererar endast till fält inom samma post, inte externa data.

## Konfigurera Visning
Det sista steget för att skapa ett formelbaserat anpassat fält är att bestämma hur data ska visas.

Formelfält erbjuder tre visningsalternativ:

1. **Nummer**: Detta alternativ visar ett enkelt numeriskt värde. Du kan också välja hur många decimaler som ska visas, vilket ger dig kontroll över detaljnivån i dina data.
2. **Valuta**: Använd detta alternativ för att visa värden i penningformat. Det är utmärkt för att spåra budgetar, intäkter, kostnader och andra finansiella data. Valutan kommer att formateras enligt lokala standarder och stöder olika globala valutor, med [stöd för många globala valutor](/sv/docs/custom-fields/currency).
3. **Procent**: Detta format visar värden som procentsatser, vilket är användbart för att visa slutförandegrad, tillväxtmått eller andra förhållanden. Det konverterar automatiskt det beräknade värdet till en procentsats, vilket gör det lätt att förstå proportioner.

Användare kan visa den underliggande formellogiken genom att hovra över fältet.
