## 개요

태그는 Blue에서 레코드를 분류하고 조직하는 강력한 방법입니다. 태그 시스템은 태그를 생성, 관리 및 필터링하는 포괄적인 기능을 제공하며, 대량 작업 및 고급 필터링을 지원합니다.

## 태그 목록

유연한 필터링 및 정렬 옵션으로 프로젝트에서 태그를 검색합니다.

### 기본 예제

```graphql
query ListTags {
  tagList(
    filter: { 
      projectIds: ["project-123"] 
      excludeArchivedProjects: false 
    }
    first: 50
    orderBy: title_ASC
  ) {
    items {
      id
      uid
      title
      color
      project {
        id
        name
      }
      createdAt
      updatedAt
    }
    pageInfo {
      totalPages
      totalItems
      page
      perPage
      hasNextPage
      hasPreviousPage
    }
    totalCount
  }
}
```

### 고급 필터링 예제

```graphql
query FilterTags {
  tagList(
    filter: {
      projectIds: ["project-123", "project-456"]
      search: "urgent"
      colors: ["#ff0000", "#ffaa00"]
      excludeArchivedProjects: true
    }
    orderBy: createdAt_DESC
    first: 20
  ) {
    items {
      id
      title
      color
      todos {
        id
        title
      }
    }
    totalCount
  }
}
```

## 태그 생성

프로젝트 내에서 새로운 태그를 생성합니다.

```graphql
mutation CreateTag {
  createTag(
    input: {
      title: "High Priority"
      color: "#ff0000"
    }
  ) {
    id
    uid
    title
    color
    project {
      id
      name
    }
    createdAt
  }
}
```

## 태그 업데이트

기존 태그 속성을 수정합니다.

```graphql
mutation UpdateTag {
  editTag(
    input: {
      id: "tag-123"
      title: "Critical Priority"
      color: "#cc0000"
    }
  ) {
    id
    title
    color
    updatedAt
  }
}
```

## 태그 삭제

시스템에서 태그를 제거합니다. 이는 모든 관련 레코드에서 태그를 제거합니다.

```graphql
mutation DeleteTag {
  deleteTag(id: "tag-123")
}
```

## 레코드에 태그 지정

기존 태그를 사용하거나 새로운 태그를 생성하여 레코드(할 일)와 태그를 연결합니다.

### 기존 태그 사용

```graphql
mutation TagRecord {
  setTodoTags(
    input: {
      todoId: "todo-123"
      tagIds: ["tag-456", "tag-789"]
    }
  )
}
```

### 태그 지정 중 태그 생성

태그 제목을 제공하여 즉석에서 새로운 태그를 생성할 수 있습니다:

```graphql
mutation TagRecordWithNewTags {
  setTodoTags(
    input: {
      todoId: "todo-123"
      tagIds: ["existing-tag-id"]
      tagTitles: ["New Tag", "Another New Tag"]
    }
  )
}
```

## AI 태그 제안

레코드에 대한 AI 기반 태그 제안을 받습니다:

```graphql
mutation GetTagSuggestions {
  aiTag(
    input: {
      todoId: "todo-123"
      suggestionsCount: 5
    }
  ) {
    suggestions {
      title
      confidence
      color
    }
  }
}
```

## 입력 매개변수

### TagListFilter

| 매개변수 | 유형 | 필수 | 설명 |
|-----------|------|----------|-------------|
| `projectIds` | [String!] | 아니요 | 프로젝트 ID 또는 슬러그로 태그 필터링 |  |
| `excludeArchivedProjects` | Boolean | 아니요 | 아카이브된 프로젝트의 태그 제외 |  |
| `search` | String | 아니요 | 태그 제목 검색(대소문자 구분 없음) |  |
| `titles` | [String!] | 아니요 | 특정 태그 제목으로 필터링 |  |
| `colors` | [String!] | 아니요 | 특정 색상으로 필터링(16진수 형식) |  |
| `tagIds` | [String!] | 아니요 | 특정 태그 ID로 필터링 |  |

### CreateTagInput

| 매개변수 | 유형 | 필수 | 설명 |
|-----------|------|----------|-------------|
| `title` | String | 아니요 | 태그 제목(제공되지 않으면 자동 생성) |  |
| `color` | String! | ✅ 예 | 16진수 형식의 태그 색상(예: "#ff0000") |  |

### EditTagInput

| 매개변수 | 유형 | 필수 | 설명 |
|-----------|------|----------|-------------|
| `id` | String! | ✅ 예 | 업데이트할 태그의 ID |  |
| `title` | String | 아니요 | 새로운 태그 제목 |  |
| `color` | String | 아니요 | 새로운 태그 색상(16진수 형식) |  |

### SetTodoTagsInput

| 매개변수 | 유형 | 필수 | 설명 |
|-----------|------|----------|-------------|
| `todoId` | String! | ✅ 예 | 태그할 레코드의 ID |  |
| `tagIds` | [String!] | 아니요 | 적용할 기존 태그의 ID |  |
| `tagTitles` | [String!] | 아니요 | 생성하고 적용할 새로운 태그의 제목 |  |

### TagOrderByInput 값

| 값 | 설명 |
|-------|-------------|
| `id_ASC` / `id_DESC` | 태그 ID로 정렬 |  |
| `uid_ASC` / `uid_DESC` | 고유 식별자로 정렬 |  |
| `title_ASC` / `title_DESC` | 제목별로 알파벳 순으로 정렬 |  |
| `color_ASC` / `color_DESC` | 색상 값으로 정렬 |  |
| `createdAt_ASC` / `createdAt_DESC` | 생성 날짜로 정렬 |  |
| `updatedAt_ASC` / `updatedAt_DESC` | 마지막 업데이트 날짜로 정렬 |  |

## 응답 필드

### 태그 유형

| 필드 | 유형 | 설명 |
|-------|------|-------------|
| `id` | ID! | 태그의 고유 식별자 |  |
| `uid` | String! | 사용자 친화적인 고유 식별자 |  |
| `title` | String! | 태그 제목/이름 |  |
| `color` | String! | 16진수 형식의 태그 색상 |  |
| `project` | Project! | 이 태그가 속한 프로젝트 |  |
| `todos` | [Todo!]! | 이 태그와 관련된 레코드 |  |
| `createdAt` | DateTime! | 태그가 생성된 시점 |  |
| `updatedAt` | DateTime! | 태그가 마지막으로 업데이트된 시점 |  |

### TagPagination 응답

| 필드 | 유형 | 설명 |
|-------|------|-------------|
| `items` | [Tag!]! | 태그 레코드 배열 |  |
| `pageInfo` | PageInfo! | 페이지 정보 |  |
| `totalCount` | Int! | 필터와 일치하는 태그의 총 수 |  |

### PageInfo 필드

| 필드 | 유형 | 설명 |
|-------|------|-------------|
| `totalPages` | Int | 총 페이지 수 |  |
| `totalItems` | Int | 모든 페이지의 총 항목 수 |  |
| `page` | Int | 현재 페이지 번호 |  |
| `perPage` | Int | 페이지당 항목 수 |  |
| `hasNextPage` | Boolean! | 다음 페이지가 있는지 여부 |  |
| `hasPreviousPage` | Boolean! | 이전 페이지가 있는지 여부 |  |

## 필수 권한

### 태그 관리

| 작업 | 필수 역할 |
|--------|--------------|
| Create tags | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |
| Edit tags | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |
| Delete tags | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |
| Apply tags to records | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |

### 태그 가시성

태그는 태그가 생성된 프로젝트에 접근할 수 있는 모든 사용자에게 표시됩니다. 특정 작업에 대해 역할 기반 필터링이 적용될 수 있습니다.

## 오류 응답

### 태그를 찾을 수 없음
```json
{
  "errors": [{
    "message": "Tag not found",
    "extensions": {
      "code": "NOT_FOUND"
    }
  }]
}
```

### 잘못된 색상 형식
```json
{
  "errors": [{
    "message": "Color must be in hex format (e.g., #ff0000)",
    "extensions": {
      "code": "BAD_USER_INPUT"
    }
  }]
}
```

### 권한 거부
```json
{
  "errors": [{
    "message": "You do not have permission to modify tags in this project",
    "extensions": {
      "code": "FORBIDDEN"
    }
  }]
}
```

### 레코드를 찾을 수 없음
```json
{
  "errors": [{
    "message": "Todo not found",
    "extensions": {
      "code": "NOT_FOUND"
    }
  }]
}
```

## 모범 사례

1. **일관된 색상 사용**: 팀을 위한 색상 코딩 시스템을 설정합니다.
2. **설명적인 제목**: 팀원이 이해할 수 있는 명확하고 설명적인 태그 이름을 사용합니다.
3. **배치 작업**: `setTodoTags`를 사용하여 여러 태그를 한 번에 적용합니다.
4. **검색 및 필터링**: 새로운 태그를 생성하기 전에 기존 태그를 찾기 위해 검색 기능을 사용합니다.
5. **태그 관리**: 사용하지 않는 태그를 정기적으로 검토하고 정리합니다.
6. **색상 규칙**: 일반 태그 유형에 대해 표준 색상을 사용하는 것을 고려합니다(긴급 태그는 빨간색, 완료된 태그는 초록색 등).

## 고급 기능

### 태그 생성 기본값

- `setTodoTags`를 통해 생성된 새로운 태그는 `tagTitles`의 기본 색상으로 설정됩니다.
- 지정되지 않은 색상의 태그는 `#00a0d2`로 대체됩니다.
- 태그 제목은 자동으로 공백이 제거됩니다.

### 자동화 통합

태그는 다음과 같이 사용할 수 있습니다:
- 자동화 규칙에 의해 자동으로 적용됩니다.
- 자동화 워크플로의 트리거로 사용됩니다.
- 자동화 작업을 통해 수정됩니다.

### 실시간 업데이트

태그 변경 사항은 GraphQL 구독을 통해 방송되어 연결된 클라이언트에서 실시간 업데이트를 가능하게 합니다.

## 중요 사항

- 태그를 삭제하면 모든 관련 레코드에서 제거됩니다.
- 태그는 프로젝트 범위로 설정되며 프로젝트 간에 공유할 수 없습니다.
- 색상 값은 유효한 16진수 코드여야 합니다(예: "#ff0000", "#00aa00").
- 태그 제목은 대소문자를 구분합니다.
- 단일 레코드에 최대 100개의 태그를 적용할 수 있습니다.
- 태그는 제목을 기준으로 전체 텍스트 검색을 지원합니다.
