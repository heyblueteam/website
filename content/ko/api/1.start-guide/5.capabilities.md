Blue API는 Blue의 데이터와 기능에 프로그래밍 방식으로 상호작용할 수 있는 강력한 도구입니다. Blue API로 달성할 수 있는 몇 가지 기능은 다음과 같습니다:

Blue API는 Blue의 데이터와 기능에 프로그래밍 방식으로 상호작용할 수 있는 다양한 강력한 기능을 제공합니다.

이 기능들을 더 자세히 살펴보겠습니다.

## 데이터 읽기

API를 사용하면 유연하고 정밀한 스키마를 사용하여 단일 쿼리에서 특정 데이터를 가져올 수 있습니다. 이는 간단한 데이터 포인트에서 복잡한 중첩 구조에 이르기까지 필요한 정보를 정확히 요청할 수 있음을 의미하며, 모든 것을 하나의 효율적인 쿼리로 처리할 수 있습니다. 이 기능은 여러 관련 엔터티에서 데이터를 검색해야 할 때 특히 유용합니다.

## 데이터 쓰기

Blue API를 사용하면 변이를 통해 서버의 데이터를 수정할 수 있습니다. 이러한 변이를 사용하면 새 레코드를 생성하거나 기존 레코드를 업데이트하거나 필요에 따라 데이터를 삭제할 수 있습니다. 이를 통해 Blue 인스턴스의 데이터에 대한 완전한 제어를 할 수 있으며, 프로세스를 자동화하거나 작업 흐름에서 Blue를 다른 시스템과 통합할 수 있습니다.

대량 작업을 위해 API는 `createCustomFieldOptions`, `deleteFiles` 및 `uploadFiles`와 같은 특정 변이를 제공합니다. 모든 엔터티에 대한 일반적인 대량 생성/업데이트 작업은 사용할 수 없지만, 단일 GraphQL 요청에서 여러 변이를 보내어 여러 작업을 효율적으로 처리할 수 있습니다.

## 실시간 업데이트

Blue API의 가장 강력한 기능 중 하나는 구독을 지원하는 것입니다. 이를 통해 클라이언트는 지속적인 폴링 없이 실시간으로 데이터 변경 사항을 들을 수 있습니다. 특정 데이터가 변경될 때 즉시 알림을 받을 수 있도록 구독을 설정할 수 있으며, 이를 통해 항상 최신 정보를 표시하는 반응형 애플리케이션을 구축할 수 있습니다.

## 효율적인 데이터 가져오기

API는 데이터의 과도한 가져오기를 줄이도록 설계되었습니다. 필요 이상의 데이터를 받을 수 있는 전통적인 REST API와 달리, Blue API를 사용하면 검색할 필드를 정확히 지정할 수 있습니다. 이는 네트워크를 통해 전송되는 데이터 양을 줄일 뿐만 아니라 불필요한 데이터 처리 최소화를 통해 애플리케이션의 성능을 개선합니다.

API에는 성능 문제 및 과도하게 중첩된 쿼리로 인한 잠재적 남용을 방지하기 위해 쿼리 깊이 제한(최대 10 수준)이 포함되어 있습니다. 이는 복잡한 데이터 검색을 허용하면서도 최적의 성능을 보장합니다.

## 스키마 탐색

Blue API는 그 기능을 동적으로 발견하고 탐색할 수 있는 기능을 제공합니다. 이는 API 자체에 쿼리하여 사용 가능한 쿼리, 변이 및 유형을 이해할 수 있음을 의미합니다. 이 기능은 개발자에게 매우 유용하며, API의 기능을 쉽게 탐색할 수 있게 해주고 강력하고 미래 지향적인 통합을 구축하는 데 도움을 줍니다.
