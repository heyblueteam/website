Blue의 레코드는 유연하며 고객 상호작용 추적, 영업, 프로젝트 마일스톤 또는 재고 관리와 같은 다양한 목적으로 사용할 수 있습니다. 이러한 적응성을 통해 Blue를 비즈니스 요구에 맞게 맞춤 설정할 수 있어 중요한 정보를 구성하는 데 유용한 도구가 됩니다.

## 레코드 이름 변경

Blue에서는 필요에 맞게 레코드 이름을 쉽게 변경할 수 있습니다. 프로젝트 설정으로 이동하여 이름을 작업이나 할당과 같이 원하는 대로 변경하기만 하면 됩니다. 이 변경 사항은 프로젝트 전체에 적용됩니다.

다음은 레코드에 사용할 수 있는 이름의 몇 가지 예시입니다:

- 작업 / 작업
- 할당 / 할당
- 단계 / 단계
- 이슈 / 이슈
- 티켓 / 티켓
- 리드 / 리드
- 거래 / 거래
- 케이스 / 케이스
- 기회 / 기회
- 클라이언트 / 클라이언트
- 제품 / 제품
- 주문 / 주문

선택한 이름의 단수형과 복수형을 모두 포함하는 것을 잊지 마세요. 이렇게 하면 시스템이 상황에 따라 올바른 용어를 표시하는 데 도움이 됩니다.

레코드 이름을 맞춤 설정하면 업계나 개인 선호도에 맞는 용어를 사용할 수 있어 팀이 플랫폼을 더 쉽고 직관적으로 사용할 수 있습니다.

![Records 1](/docs/Record_1.png "Records 1")

레코드는 다음과 같은 다양한 엔티티를 나타낼 수 있습니다:

- 고객 상호작용
- 영업 기록
- 프로젝트 마일스톤
- 재고 항목

이러한 유연성으로 Blue는 다양한 비즈니스 프로세스와 운영 요구 사항에 맞게 조정될 수 있어 중요한 데이터를 구성하고 관리하는 강력한 도구가 됩니다.

![Records 2](/docs/Record_2.png "Records 2")
