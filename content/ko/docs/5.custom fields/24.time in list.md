목록 체류 시간 사용자 정의 필드는 레코드가 각 목록에 머무른 시간을 자동으로 추적하여, 워크플로 진행 상황과 병목 지점에 대한 상세한 인사이트를 제공합니다.

목록 체류 시간 필드는 워크플로 효율성을 분석하고 프로세스 개선 사항을 식별하는 데 매우 유용합니다. 몇 가지 예시는 다음과 같습니다:

- 지원 티켓이 각 상태(신규, 진행 중, 고객 대기, 해결됨)에서 소요되는 시간 모니터링
- 영업 파이프라인 진행 상황을 추적하여 거래가 정체되는 지점 파악
- 제조 워크플로 단계를 분석하여 생산 일정 최적화
- 초안에서 게시까지 콘텐츠 승인 프로세스 측정
- 지원에서 채용까지 채용 워크플로의 병목 지점 식별
- 프로젝트 단계를 추적하여 지연이 일반적으로 발생하는 부분 파악
- 서비스 제공 단계를 모니터링하여 고객 경험 개선

목록 체류 시간 필드는 수동 개입 없이 자동으로 실시간 추적을 제공합니다. 각 목록에서 소요된 시간을 과거 및 현재 모두 계산하고, 레코드가 가장 많은 시간을 보내는 곳의 백분율 분석을 보여줍니다. 이를 통해 데이터 기반 프로세스 최적화가 가능하고, 비효율성을 식별하며, 워크플로 성능에 대한 명확한 가시성을 제공합니다.

## 작동 방식

목록 체류 시간 추적은 프로젝트에 목록 체류 시간 사용자 정의 필드를 추가하면 자동으로 시작됩니다. 시스템은:

1. **입장 및 퇴장 시간 추적**: 항목이 각 목록에 들어가고 나가는 정확한 타임스탬프를 기록
2. **지속 시간 계산**: 입장과 퇴장 시간의 차이를 사용하여 소요 시간을 자동 계산
3. **현재 상태 처리**: 현재 목록에 있는 항목의 경우, 입장 시점부터 현재까지의 시간 계산
4. **이력 보존**: 목록이 이름이 변경되거나 삭제되어도 완전한 시간 추적을 유지
5. **백분율 분석 표시**: 전체 시간 중 각 목록에서 소요된 시간의 비율 표시

## 표시 옵션

목록 체류 시간 데이터는 여러 위치에서 나타납니다:

- **레코드 카드**: 직관적인 아이콘과 함께 현재 목록 시간 및 총 프로젝트 시간 표시
- **레코드 세부 보기**: 각 목록에 대한 진행률 막대가 있는 포괄적인 분석 표시
- **데이터베이스 보기**: 분석을 위해 테이블 형태로 시간 데이터 표시 가능
- **사용자 정의 필드 통합**: 고급 추적을 위해 TIME_DURATION 사용자 정의 필드와 함께 작동

## 설정

목록 체류 시간 필드는 설정이 필요하지 않으며 모든 목록 이동을 자동으로 추적합니다. 하지만 표시 옵션을 제어할 수 있습니다:

- **현재 목록에서의 시간 표시**: 레코드 카드에서 현재 목록에서 소요된 시간 표시 토글
- **프로젝트에서의 시간 표시**: 프로젝트의 모든 목록에서 소요된 총 시간 표시 토글
- **사용자 정의 필드 가시성**: 시간 추적이 레코드 세부 정보에 나타나는지 제어

## 주요 이점

- **설정 불필요**: 필드가 추가되면 자동으로 추적 시작
- **완전한 이력**: 목록 변경 및 삭제를 통해서도 추적 유지
- **실시간 업데이트**: 현재 항목에 대한 시간 계산이 지속적으로 업데이트
- **시각적 진행률 표시기**: 진행률 막대가 목록 간 상대적 시간 분포 표시
- **프로세스 최적화**: 워크플로가 느려지거나 막히는 지점 식별
- **성능 지표**: 팀 생산성 및 프로세스 효율성에 대한 인사이트 생성
