Blue에서 "댓글 전용 사용자"는 프로젝트 논의에 참여하면서 다른 기능에 대한 접근이 제한된 독특한 사용자 역할입니다. 이 역할은 전체 프로젝트 접근 없이 대화에 기여하고 피드백을 제공해야 하는 사람들에게 이상적입니다.

## 역할 및 권한

Blue의 댓글 전용 사용자는 주로 프로젝트 내에서의 커뮤니케이션에 집중합니다. 그들의 권한에는 다음이 포함됩니다:

- **댓글 추가**: 기존 기록이나 작업에 댓글을 추가하여 논의에 적극적으로 참여할 수 있습니다.
- **읽기 접근**: 모든 댓글을 읽고 프로젝트 내의 세부 정보를 볼 수 있는 능력이 있어, 그들을 정보에 통달하고 참여하게 합니다.

## 제한 사항

댓글 전용 사용자가 논의에 참여할 수 있지만, 그들의 접근은 여러 면에서 제한됩니다:

- **편집 기능 없음**: 기록이나 작업을 생성하거나 편집할 수 없습니다.
- **제한된 접근**: 프로젝트 설정을 변경하거나 기록을 추가하거나 수정할 수 없는 댓글 및 보기로만 제한됩니다.

**이상적인 사용 사례**

이 역할은 다음과 같은 시나리오에서 특히 유용합니다:

- **클라이언트 피드백**: 클라이언트가 특정 작업이나 프로젝트에 대한 의견을 제공할 수 있도록 하면서 전체 접근을 허용하지 않습니다.
- **외부 협업**: 진행 중인 작업에 대해 댓글을 달아야 하지만 전체 프로젝트 접근이 필요하지 않은 외부 파트너나 이해관계자와의 참여.
- **검토 및 승인 프로세스**: 다양한 팀원으로부터 입력이 필요한 피드백 루프를 촉진하지만 전체 프로젝트 참여는 필요하지 않은 경우.

## 설정

Blue에서 댓글 전용 사용자를 설정하려면, 프로젝트 관리자가 다음을 수행해야 합니다:

1. 프로젝트 설정에서 '사람' 탭으로 이동합니다.
2. 개인을 사용자로 추가하고 '댓글 전용' 역할을 할당합니다.

**결론**

댓글 전용 사용자는 Blue의 협업 환경에서 필수적인 부분으로, 논의 및 피드백 프로세스에 대한 통제된 참여를 허용합니다. 이러한 사용자를 프로젝트에 통합함으로써, 모든 필요한 당사자가 대화에 참여하도록 보장하면서 프로젝트의 데이터와 워크플로우의 무결성과 보안을 유지할 수 있습니다.
