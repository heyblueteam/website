## 개요

커버 이미지는 기록 카드에 이미지 미리보기를 직접 표시하여 기록에 시각적 맥락을 추가할 수 있는 기능입니다. 이 기능은 기록 내의 다양한 소스에서 이미지를 자동으로 추출하여 썸네일로 표시함으로써, 시각적 콘텐츠를 한눈에 식별하고 정리하기 쉽게 만듭니다.

디자인 프로젝트, 콘텐츠 캘린더, 제품 카탈로그 또는 시각적 자산이 포함된 모든 워크플로를 관리하든, 커버 이미지는 기록을 빠르게 스캔하고 찾고 있는 것을 찾는 데 도움이 됩니다. 이 기능에는 실시간 진행률 추적과 특정 워크플로 요구사항에 맞는 유연한 표시 옵션이 포함되어 있습니다.

## 주요 기능

- **자동 이미지 추출**: Blue가 선택한 소스에서 이미지를 자동으로 찾고 표시합니다
- **다중 이미지 소스**: 기록 설명, 댓글 또는 사용자 정의 파일 필드에서 이미지를 가져옵니다
- **표시 맞춤 옵션**: Cover 또는 Contain으로 이미지가 표시되는 방식을 선택합니다
- **스마트 선택**: 소스에서 첫 번째 또는 마지막 이미지를 사용할지 구성합니다
- **실시간 업데이트**: 기록 콘텐츠가 변경되면 커버 이미지가 자동으로 업데이트됩니다
- **실시간 진행률 추적**: 실시간 진행률 구독으로 대량 처리를 모니터링합니다

## 작동 방식

### 커버 이미지 활성화

커버 이미지는 프로젝트 관리자가 프로젝트 수준에서 구성합니다:

1. 프로젝트를 열고 **프로젝트 설정**으로 이동합니다
2. **기록 설정**을 선택합니다
3. **이미지 커버 옵션** 섹션을 찾습니다
4. **커버 이미지 활성화** 토글을 활성화하여 기능을 켭니다

### 구성 옵션

#### 이미지 소스
Blue가 이미지를 찾을 위치를 선택합니다:
- **설명**: 기록 설명에 포함된 이미지를 추출합니다 (기본값)
- **댓글**: 기록 댓글에서 첫 번째/마지막 이미지를 가져옵니다
- **사용자 정의 필드**: 특정 파일 형식 사용자 정의 필드의 파일을 사용합니다 (모든 파일 형식 지원)

#### 이미지 선택
여러 이미지를 사용할 수 있을 때 표시할 이미지를 선택합니다:
- **첫 번째 이미지**: 찾은 첫 번째 이미지를 사용합니다 (기본값)
- **마지막 이미지**: 가장 최근 이미지를 사용합니다

#### 표시 맞춤
미리보기 영역에서 이미지가 표시되는 방식을 제어합니다:
- **Cover**: 이미지가 전체 미리보기 영역을 채우며, 종횡비를 유지하기 위해 가장자리가 잘릴 수 있습니다 (기본값)
- **Contain**: 미리보기 영역 내에서 완전한 이미지를 표시하며, 빈 공간이 있을 수 있습니다

## 사용 사례

**디자인 프로젝트** - 태스크 카드에 목업과 디자인 반복을 직접 표시하여, 개별 기록을 열지 않고도 각 디자인 태스크의 현재 상태를 쉽게 볼 수 있습니다.

**콘텐츠 계획** - 콘텐츠 캘린더와 편집 워크플로에서 주요 이미지나 썸네일을 표시하여 콘텐츠 파이프라인을 한눈에 시각화할 수 있습니다.

**제품 관리** - 재고 기록이나 기능 요청에 제품 사진을 표시하여, 토론 중에 팀이 항목을 빠르게 식별할 수 있도록 도와줍니다.

**부동산** - 목록 기록에 부동산 사진을 표시하여, 에이전트가 시각적으로 부동산을 빠르게 탐색할 수 있게 합니다.

**이벤트 계획** - 계획 태스크에 장소 사진, 디자인 컨셉 또는 이벤트 레이아웃을 표시하여 더 나은 시각적 조직화를 제공합니다.

## 커버 이미지 설정

### 1단계: 이미지 소스 선택

워크플로에서 이미지가 자연스럽게 존재하는 위치를 고려하세요:
- 이미지가 태스크 설명이나 브리프의 일부인 경우 **설명**을 사용하세요
- 이미지가 팀 협업을 통해 추가되는 경우 **댓글**을 사용하세요
- 이미지 전용 필드가 있는 경우 **사용자 정의 필드**를 사용하세요

### 2단계: 표시 설정 구성

1. 이미지 유형에 따라 선호하는 **이미지 맞춤**을 선택하세요:
   - 자르기가 허용되는 사진과 디자인의 경우 **Cover**를 사용하세요
   - 모든 것을 볼 필요가 있는 다이어그램이나 스크린샷의 경우 **Contain**을 사용하세요

2. 워크플로에 따라 **이미지 선택**을 선택하세요:
   - 기본 이미지가 처음에 추가되는 워크플로의 경우 **첫 번째**를 사용하세요
   - 가장 최근 이미지가 가장 관련성이 높은 워크플로의 경우 **마지막**을 사용하세요

### 3단계: 기존 기록 처리

커버 이미지를 처음 활성화하거나 구성을 변경할 때, Blue가 자동으로 기록을 처리합니다:
1. 실시간 진행률 업데이트가 처리 상태를 표시합니다
2. 최적의 성능을 위해 기록이 배치 단위로 처리됩니다 (배치당 50개 기록)
3. 백그라운드에서 처리가 진행되는 동안 계속 작업할 수 있습니다
4. 처리가 완료되면 목록이 자동으로 새로고침됩니다
5. 구성 변경사항이 저장될 때 처리가 자동으로 시작됩니다

![커버 처리 진행률](/docs/records/cover-images/progress.png "커버 처리 진행률")



## 기술적 세부사항

### 지원되는 형식
- 설명 또는 댓글 소스를 사용할 때:
  - JPEG/JPG 이미지
  - PNG 이미지 (투명도 포함)
  - GIF 이미지 (정적 표시)
- 사용자 정의 필드 소스를 사용할 때:
  - 모든 파일 형식을 표시할 수 있습니다 (이미지, PDF, 문서)
  - 비이미지 파일은 적절한 파일 형식 아이콘을 표시합니다
- 모든 파일은 Blue에 업로드되어야 합니다 (외부 URL은 지원되지 않음)

### 제한사항
- 대량 커버 생성을 위한 프로젝트당 최대 6,000개 기록
- 파일 형식 사용자 정의 필드만 소스로 사용할 수 있습니다
- 커버 이미지는 보기 전용이며 직접 편집할 수 없습니다

### 자동 업데이트
다음과 같은 경우 커버가 자동으로 재생성됩니다:
- 기록 설명이 편집될 때 (설명 소스를 사용하는 경우)
- 이미지가 포함된 새 댓글이 추가될 때 (댓글 소스를 사용하는 경우)
- 사용자 정의 필드의 파일이 업데이트될 때 (사용자 정의 필드 소스를 사용하는 경우)
- 커버 구성 설정이 변경될 때
- 표시 맞춤 옵션이 수정될 때

### API 액세스
커버 이미지는 GraphQL API와 완전히 통합됩니다:
- `setTodoCoverConfig` 뮤테이션을 통해 구성
- Todo 유형의 `cover` 필드를 통해 커버 데이터 액세스
- `subscribeToCoverGenerationProgress` 구독으로 진행률 모니터링
- `/uploads/{uid}/thumbnail/cover`에서 썸네일 URL 자동 생성


## 다른 기능과의 통합

커버 이미지는 다른 Blue 기능과 원활하게 작동합니다:
- **[양식](/ko/docs/projects/forms)**: 제출된 이미지가 자동으로 커버가 될 수 있습니다
- **[API](/ko/docs/integrations/api)**: 구성 및 데이터 액세스를 위한 완전한 GraphQL API 지원
- **[사용자 정의 필드](/ko/docs/projects/custom-fields)**: 모든 파일 형식 필드가 커버 소스로 사용될 수 있습니다
