CSV(Comma Separated Value) 파일을 가져올 수 있습니다. Excel 파일이 있는 경우, [다음 링크에서 CSV 파일로 변환할 수 있습니다](https://cloudconvert.com/xls-to-csv).

## CSV 가져오기

Blue에서 CSV 파일을 가져오는 것은 간단한 과정입니다. CSV 파일의 각 열은 Blue의 기본 또는 [사용자 정의 필드](/docs/custom-fields/introduction)에 해당합니다.

시작하려면, 데이터를 가져오려는 프로젝트의 프로젝트 관리자여야 합니다. 프로젝트 설정 아이콘(톱니바퀴 아이콘)을 클릭하고 "가져오기"를 선택합니다.

![CSV 가져오기 1](/docs/CSVimport1.png "CSV Import 1")

가져오기 기능은 프로젝트 관리자용 프로젝트 설정에서 사용할 수 있습니다.

위 스크린샷에서 "CSV 템플릿 다운로드"라는 또 다른 옵션도 볼 수 있습니다. 이는 Blue 프로젝트에서 이미 [사용자 정의 필드](/docs/custom-fields/introduction)를 생성한 경우 매우 유용합니다. 모든 사용자 정의 필드에 대한 열이 포함된 CSV 파일을 다운로드할 수 있기 때문입니다! 이는 가져오기 전에 CSV의 정확한 구조를 보장하고 싶을 때 좋습니다.

다음으로, CSV 파일을 Blue로 드래그 앤 드롭하면 됩니다:

![CSV 가져오기 2](/docs/CSVimport2.png "CSV Import 2")

데이터를 업로드하려면 드래그 앤 드롭하세요.

파일 크기에 따라 업로드 및 Blue의 분석에 몇 초가 걸릴 수 있습니다. Blue가 데이터를 분석하면 데이터의 처음 10개 행을 미리 볼 수 있습니다:

![CSV 가져오기 3](/docs/CSVimport3.png "CSV Import 3")

그런 다음 CSV의 각 열을 Blue의 필드에 매핑할 수 있습니다.

![CSV 가져오기 4](/docs/CSVimport4.png "CSV Import 4")

## 허용되는 날짜 형식

Blue는 가져오기를 위한 다양한 날짜 형식을 지원합니다. 지원되는 모든 형식은 다음과 같습니다:

- **유닉스 타임스탬프**:
  ```plaintext
  1511648546 (seconds)
  1620021848429 (milliseconds)
  1620024872717915000 (nanoseconds)
  ```
  
- **RFC 표준**:
  ```plaintext
  RFC 3339: 2021-05-01T01:17:02.604456Z
  RFC 2822: Wed, 02 Jun 2021 06:31:39 GMT
  ```

- **데이터베이스 형식**:
  ```plaintext
  PostgreSQL: 
  2019-11-29 08:08-08
  2019-11-29 08:08:05-08
  2021-05-02 23:31:36.0741-07
  ```

- **일반 형식**:
  ```plaintext
  Year-Month-Day: 
  2021-04-30 21:14
  2021-02-21
  2021-Feb-21
  
  Month/Day/Year:
  3/31/2014 22:05
  04/02/2014 03:00:51
  
  Day-Month-Year:
  12 Feb 2006, 19:17
  03 February 2013
  ```

- **시간대**:
  ```plaintext
  2014-04-26 13:13:43 +0800
  2017-11-25 13:31 PST
  2021-02-21 UTC
  ```

- **자연어**:
  ```plaintext
  May 6 at 9:24 PM
  September 17, 2012 at 10:09am PST
  October 7, 1970
  ```

- **국제 형식**:
  ```plaintext
  Chinese: 2014年04月08日
  Japanese: 2014/04/08 22:05
  ```

- **시간만**:
  ```plaintext
  01:06:06
  4:00pm PST
  6:00 AM UTC
  ```

모든 날짜는 가져오는 동안 UTC 시간대로 변환됩니다. 시간 구성 요소는 선택 사항입니다 - 제공되지 않으면 시간은 00:00:00 UTC로 기본 설정됩니다.

## 데이터의 쉼표 처리

데이터에 쉼표가 포함된 경우(예: 주소나 회사 이름에서) 가져오기 오류를 방지하기 위해 해당 값을 큰따옴표로 감싸야 합니다.

**예시:**
```
Company,Address
Acme Inc,123 Main St, New York
```

**다음과 같이 변경되어야 합니다:**
```
Company,Address
"Acme Inc","123 Main St, New York"
```

이렇게 하면 "123 Main St, New York"의 쉼표가 열 구분자가 아닌 데이터의 일부로 처리됩니다.
