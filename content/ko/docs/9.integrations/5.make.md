[Make.com](https://www.make.com)은 다양한 웹 앱을 연결하고 워크플로를 자동화하는 데 도움을 주는 통합 플랫폼입니다. Blue는 **공식적으로** Make.com과 통합되어 2,000개 이상의 앱에서 원활한 자동화를 가능하게 합니다—코딩이 필요 없습니다!

![Blue Make.com 모듈 개요](/docs/blue_make_modules_overview.png "Overview of available Blue modules in Make.com")

## 사용 사례

다음은 Make.com 통합을 통해 Blue의 기능을 확장하는 몇 가지 강력한 방법입니다:
Blue의 Make.com 통합은 다양한 비즈니스 기능에서 강력한 자동화를 가능하게 합니다. 고객 경험을 위해 Blue를 Intercom, HubSpot 및 Zendesk와 연결하여 포괄적인 고객 기록을 생성하고, 상호작용을 동기화하며, Slack 및 Microsoft Teams를 통해 실시간 커뮤니케이션을 유지할 수 있습니다. 프로젝트 관리에서는 Blue가 Jira, GitHub, Trello 및 Monday.com과 통합되어 작업을 동기화하고, 진행 상황을 추적하며, 알림을 자동화합니다.

마케팅 및 판매를 위해 Blue는 Mailchimp, Stripe 및 Calendly와 연결하여 캠페인을 추적하고, 결제를 관리하며, 회의 워크플로를 자동화합니다. 문서 관리는 Google Drive, Dropbox, OneDrive 및 SharePoint와의 통합을 통해 간소화되며, Slack, Gmail 및 WhatsApp과 같은 커뮤니케이션 도구는 Blue의 중앙 작업 관리 시스템을 통해 통합됩니다. 데이터 분석 기능은 Google Sheets, Power BI, Airtable 및 Tableau와의 연결을 통해 향상되며, 금융 서비스, 전자 상거래 및 원격 팀 관리를 위한 산업별 솔루션이 제공됩니다.

## Blue 계정 연결 및 인증

Blue를 Make.com과 안전하게 연결하려면:

1. Blue 계정에 로그인하고 프로필을 열어 **API** 탭으로 이동합니다.
2. **토큰 생성**을 클릭하고 이름(및 선택적 만료일)을 입력합니다.
3. 다음 두 가지를 복사합니다:
   - **토큰 ID** (`x‑bloo‑token‑id`)
   - **비밀 키** (`x‑bloo‑token‑secret`)

## 사용 가능한 Blue 모듈

시나리오를 구성할 때 다음 모듈을 삽입합니다:

### 레코드 생성
사용자 정의 필드 및 메타데이터로 새 Blue 항목을 추가합니다.

![레코드 생성 모듈](/docs/create_a_record.png "Create a Record module configuration")

### GraphQL 쿼리 실행
고급 데이터 조작을 위한 권한 있는 GraphQL 작업을 실행합니다.

![GraphQL 쿼리 실행 모듈](/docs/execute_a_graphql_query.png "Execute GraphQL Query module configuration")

### 레코드 가져오기
ID로 레코드를 검색하고 모든 관련 데이터를 가져옵니다.

![레코드 가져오기 모듈](/docs/get_a_record.png "Get a Record module configuration")

### 프로젝트에 사용자 초대
팀원에게 프로젝트 초대장을 자동으로 보냅니다.

![사용자 초대 모듈](/docs/invite_user_to_a_project.png "Invite User to Project module configuration")

### 모든 프로젝트 목록
모든 Blue 프로젝트와 그 세부 정보를 가져옵니다.

![프로젝트 목록 모듈](/docs/list_all_projects.png "List All Projects module configuration")

### 사용자 정의 필드 목록
Blue 프로젝트의 필드 메타데이터를 검색합니다.

![사용자 정의 필드 목록 모듈](/docs/list_custom_fields.png "List Custom Fields module configuration")

### 프로젝트의 목록 목록
Blue 프로젝트 내의 목록/파이프라인에 접근합니다.

![목록 목록 모듈](/docs/list_lists_of_a_project.png "List Lists of a Project module configuration")

### 프로젝트 내 태그 목록
프로젝트 전반에 걸친 태그 세부정보 및 사용량을 가져옵니다.

![태그 목록 모듈](/docs/list_tags_within_projects.png "List Tags Within Projects module configuration")

### 프로젝트의 레코드 검색
다양한 기준에 따라 레코드를 필터링하고 찾습니다.

![레코드 검색 모듈](/docs/search_records_of_a_project.png "Search Records of a Project module configuration")

### 이벤트 감시
레코드 생성, 업데이트 또는 삭제 시 트리거됩니다.

![이벤트 감시 모듈](/docs/watch_events.png "Watch Events module configuration")

## 모듈 매개변수 참조

### 이벤트 감시
- 이벤트 (선택 사항): 다중 선택 이벤트 유형
- 회사 ID (선택 사항): 다중 선택 회사 ID 또는 슬러그
- 프로젝트 ID (선택 사항): 다중 선택 프로젝트 ID 또는 슬러그

### 레코드 생성
- 회사 ID: 회사 ID 또는 슬러그
- 프로젝트 ID: 프로젝트 ID 또는 슬러그
- 목록: 목록 ID
- 레코드 이름: 레코드의 이름
- 기본 필드: 기본 값
- 단일 날짜: 기한
- 날짜 범위: 시작 및 기한 날짜
- 담당자: 다중 선택 사용자
- 태그: 다중 선택 태그
- 설명: 텍스트 필드, HTML 지원
- 체크리스트: 체크리스트 제목 배열

**지원되는 사용자 정의 필드:**
| 필드 유형 | 설명 |
|------------|-------------|
| 단일 행 텍스트 | 선택적 유효성 검사 및 문자 제한이 있는 단일 행 텍스트 필드 |
| 다중 행 텍스트 | 리치 텍스트 지원 및 서식 옵션이 있는 다중 행 텍스트 필드 |
| 단일 선택 | 단일 옵션 선택 및 사용자 정의 옵션이 있는 드롭다운 |
| 다중 선택 | 다중 옵션 선택 및 사용자 정의 옵션이 있는 드롭다운 |
| 위치 | 지도 보기 및 주소 조회가 있는 위도 및 경도 |
| 체크박스 | 선택적 기본 값 및 사용자 정의 레이블이 있는 참/거짓 전환 |
| 국가 | 국가 이름이 포함된 두 글자 ISO 국가 코드 선택기 |
| 통화 | 통화 기호 및 환율 지원이 있는 숫자 필드 (예: USD200, 200USD, KHR4000) |
| 날짜 | 선택적 시간 구성 요소 및 시간대 지원이 있는 날짜 선택기 |
| 전화 | E.164, 국제 형식 또는 국가 코드 + 번호를 지원하는 전화 번호 필드 |
| 이메일 | 형식 유효성 검사 및 도메인 검증이 있는 이메일 필드 |
| 별점 | 시각적 별 및 사용자 정의 레이블이 있는 1-5의 숫자 등급 |
| 백분율 | 백분율 기호 (0-100) 및 소수 정밀도가 있는 숫자 필드 |
| URL | 형식 유효성 검사 및 미리 보기가 있는 웹 주소 필드 |
| 숫자 | 선택적 소수 자릿수 및 단위 표시가 있는 숫자 필드 |
| 수식 | 수학 연산을 기반으로 다른 필드에서 계산된 필드 |
| 고유 ID | 사용자 정의 접두사 및 형식이 있는 자동 생성 식별자 |
| 참조 | 검색 및 필터 옵션이 있는 다른 레코드에 대한 링크 |
| 조회 | 매핑 옵션이 있는 참조된 레코드에서 데이터를 가져오는 동적 필드 |
| 시간 지속 | 사용자 정의 표시 형식이 있는 시간 범위 (시간 및 분) |

### 레코드 업데이트
- 회사 ID: 회사 ID 또는 슬러그
- 프로젝트 ID: 프로젝트 ID 또는 슬러그
- 목록: 목록 ID
- 레코드 ID: 업데이트할 레코드의 ID
- 레코드 이름: 레코드의 이름
- 기본 필드: 기본 값
- 단일 날짜: 기한
- 날짜 범위: 시작 및 기한 날짜
- 담당자: 다중 선택 사용자
- 태그: 다중 선택 태그
- 설명: 텍스트 필드, HTML 지원
- 체크리스트: 체크리스트 제목 배열

**지원되는 사용자 정의 필드:** (레코드 생성과 동일)

### 레코드 검색
- 회사 ID: 회사 ID 또는 슬러그
- 프로젝트 ID: 프로젝트 ID 또는 슬러그
- 목록: 목록 ID
- 검색/쿼리: 고급 필터링 옵션이 있는 검색 텍스트

### 레코드 가져오기
- 회사 ID: 회사 ID 또는 슬러그
- 프로젝트 ID: 프로젝트 ID 또는 슬러그
- 목록: 목록 ID
- 레코드 ID: 모든 관련 데이터와 함께 검색할 레코드의 ID

### 사용자 초대
- 회사 ID: 회사 ID 또는 슬러그
- 프로젝트 ID: 프로젝트 ID 또는 슬러그
- 이메일 주소: 유효성 검사가 있는 사용자의 이메일 주소
- 역할 ID: 권한이 있는 역할 ID (기본 또는 사용자 정의)

### 사용자 정의 필드 목록
- 회사 ID: 회사 ID 또는 슬러그
- 프로젝트 ID: 프로젝트 ID 또는 슬러그
- 제한: 페이지 매김과 함께 반환할 최대 필드 수

### 목록 목록
- 회사 ID: 회사 ID 또는 슬러그
- 프로젝트 ID: 프로젝트 ID 또는 슬러그
- 제한: 정렬 옵션과 함께 반환할 최대 목록 수

### 모든 프로젝트 목록
- 회사 ID: 다중 선택 회사 ID 또는 슬러그
- 제한: 필터링과 함께 반환할 최대 프로젝트 수

### 태그 목록
- 회사 ID: 회사 ID 또는 슬러그
- 프로젝트 ID: 프로젝트 ID 또는 슬러그
- 제한: 사용 통계와 함께 반환할 최대 태그 수

### GraphQL 실행
- 헤더: 인증 및 사용자 정의 헤더를 위한 키-값 쌍 배열
- 메서드: 요청 옵션이 있는 GET 또는 POST
- 쿼리: 변수 지원이 있는 GraphQL 쿼리 또는 변형
- 작업 이름: 추적을 위한 작업의 이름
- 변수 데이터 소스: 원시, 양식 또는 유형 검사가 있는 컬렉션
- 변수: 유효성 검사가 있는 GraphQL 변수 내용
