Blue는 기본 OpenStreetMap 대신 구글 지도를 지도 보기로 사용할 수 있습니다. 구글 지도는 특정 지역의 더 자세한 지도와 더 정확한 위치 데이터를 제공하는 경우가 많습니다. 그러나 이를 위해서는 구글 지도 API 키가 필요하며, 회사 소유자가 회사 설정에 입력해야 합니다.

구글 지도로 전환하면 회사 지도, [프로젝트의 지도 보기](/docs/views/map), [위치 사용자 정의 필드](/docs/custom-fields/location-map)를 포함한 Blue의 모든 지도 보기에도 영향을 미칩니다.

Blue에서 구글 지도를 사용하려면 Google Cloud Console에서 다음 API를 활성화해야 합니다:
- [지도 타일 API](https://console.cloud.google.com/apis/library/tile.googleapis.com): 지도 타일을 렌더링하는 데 필요합니다.
- [장소 API](https://console.cloud.google.com/apis/library/places.googleapis.com): 장소를 검색하고 그 좌표를 얻는 데 필요합니다.
- [지오코딩 API](https://console.cloud.google.com/apis/library/geocoding-backend.googleapis.com): 장소의 좌표를 얻는 데 필요합니다.

[구글 클라우드 계정](https://console.cloud.google.com/)에 가입하고 프로젝트를 생성하여 API 키를 얻으려면 [여기](https://developers.google.com/maps/documentation/embed/get-api-key)의 지침을 따르세요.

## API 키 사용 제한

보안상의 이유로, 구글 지도 API 키가 승인된 도메인에서만 작동하도록 제한하는 것이 중요합니다. 이는 API 키가 노출될 경우 무단 사용을 방지합니다. API 키를 다음과 같이 제한해야 합니다:

- app.blue.cc - 주요 Blue 애플리케이션
- forms.blue.cc - 공개 양식용
- [화이트 라벨](/docs/start-guide/white-label)을 사용하는 경우 귀하의 맞춤 도메인

API 키를 제한하려면:

1. [Google Cloud Console 자격 증명 페이지](https://console.cloud.google.com/apis/credentials)로 이동합니다.
2. API 키를 찾아 "편집" (연필 아이콘)을 클릭합니다.
3. "응용 프로그램 제한"에서 "HTTP 참조자(웹사이트)"를 선택합니다.
4. 다음 참조자를 추가합니다:
   - `*.app.blue.cc/*`
   - `*.forms.blue.cc/*`
   - `*.yourdomain.com/*` (화이트 라벨을 사용하는 경우)
5. "저장"을 클릭합니다.

**구글 지도 요금에 대한 중요 참고 사항**<br><br>

구글은 Blue 구독과 별도로 구글 지도 API 사용에 대해 요금을 청구합니다. 구글 지도 계정 및 API 사용과 관련된 모든 요금, 비용 및 수수료는 조직 관리자에게 전적으로 책임이 있습니다. Blue는 구글 지도 API 사용으로 인해 발생하는 모든 요금에 대해 책임을 지지 않으며, 여기에는 제3자 통합, API 사용 또는 저장 비용이 포함되지만 이에 국한되지 않습니다.
