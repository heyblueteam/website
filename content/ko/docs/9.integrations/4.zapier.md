Zapier는 다양한 웹 앱을 연결하고 워크플로우를 자동화하는 데 도움이 되는 통합 플랫폼입니다.

**사용 사례**

다음은 Blue가 Zapier를 통해 다른 앱과 통합되는 가장 인기 있는 방법 중 일부입니다:

**_Blue 레코드를 Google Sheets와 동기화하기_**

Blue 레코드가 생성되거나 업데이트될 때, 이를 자동으로 Google Sheet의 새로운 행으로 추가하여 중앙 집중식 보고를 수행합니다.

**_Blue 레코드에서 주문 확인 이메일 보내기_**

Blue에서 특정 사용자 정의 필드를 선택하면 고객에게 주문을 확인하는 이메일을 자동으로 발송합니다.

**_Blue 댓글을 Slack에 추가하기_**

중요한 레코드 업데이트를 팀이 인지할 수 있도록 적절한 Slack 채널에 실시간 Blue 알림을 게시합니다.

## 주요 이점

**도구 간 자동화**: Zapier에서 생성된 Zaps는 Blue와 수천 개의 다른 앱 간에 데이터를 자동으로 전송하고 수신할 수 있습니다. 이는 시스템 간 데이터 이동에 대한 수작업을 없애줍니다.

**실시간 통합**: Blue의 웹훅 트리거를 사용하여 Zaps는 새로운 레코드, 추가된 댓글, 상태 변경 등 Blue의 이벤트에 즉시 반응합니다. 이는 실시간 동기화 및 프로세스 자동화를 가능하게 합니다.

**쉽게 구축하고 유지 관리**: Zaps는 도구를 통합하는 사용자 친화적인 무코드 방법을 제공합니다. 개발 작업이 필요하지 않으며, 지속적인 유지 관리 없이도 계속 작동합니다.

## Blue에서 웹훅 설정하기

</video>

1. 프로필 설정에서 Blue 계정에 로그인합니다.
2. 프로필 설정으로 가서 "Webhooks"를 클릭합니다.
3. "Add Webhook"을 클릭합니다.
4. 웹훅에 이름을 지정하고 Zapier 웹훅 URL을 입력합니다.
5. 웹훅을 트리거할 이벤트를 선택합니다.
6. "Create Webhook"을 클릭합니다.

## Blue에서 Zapier로

1. Zapier 계정에 로그인합니다.
2. "Make a Zap"을 클릭합니다.
3. 첫 번째 단계에서 ["Webhooks by Zapier" 트리거 앱](https://zapier.com/apps/webhook)을 검색하고 선택합니다.
4. Blue 웹훅 URL에서 페이로드를 수신하도록 구성합니다.
5. 두 번째 단계에서 데이터를 보낼 대상 앱을 검색하고 선택합니다.
6. JSON 페이로드 필드를 대상 앱의 입력 필드에 매핑합니다.
7. Zap을 켭니다.

이제 Blue에서 지정된 이벤트가 발생하면 웹훅 페이로드가 Zapier로 전송되어 Zap 설정에 따라 통합된 앱으로 데이터를 자동으로 전달합니다.

## Zapier에서 Blue로

다른 앱에서 Blue로 데이터를 푸시하려면 다음 단계를 따릅니다:

**1. Blue API 자격 증명 얻기**
- Blue 설정 > API 토큰으로 이동합니다.
- 새 토큰을 생성하거나 기존 자격 증명을 사용합니다:
  - 토큰 ID
  - 토큰 비밀
  - 회사 ID
  - 프로젝트 ID

**2. Zapier에서 Zap 생성하기**
1. 트리거 앱을 설정합니다 (예: Google Sheets, Slack 등).
2. "Webhooks by Zapier"를 액션 앱으로 추가합니다.
3. "Custom Request" 이벤트 유형을 선택합니다.

**3. 웹훅 구성하기**
- 방법: POST
- URL: `https://api.blue.cc/graphql` (필요한 경우 베타 엔드포인트 사용)
- 데이터 패스스루: FALSE
- 언플래튼: NO

**4. 헤더 설정:**
```json
{
  "Content-Type": "application/json",
  "X-Bloo-Token-ID": "Your_Token_ID",
  "X-Bloo-Token-Secret": "Your_Token_Secret", 
  "X-Bloo-Company-ID": "Your_Company_ID",
  "X-Bloo-Project-ID": "Your_Project_ID"
}
```

**5. 쿼리 구성 (예제 변형):**
```json
{
  "query": "mutation CreateTodo($input: CreateTodoInput!) {\n  createTodo(input: $input) {\n    id\n    title\n    customFields {\n      value\n    }\n  }\n}",
  "variables": {
    "input": {
      "todoListId": "your-todo-list-id",
      "title": "New task from Zapier",
      "customFields": [
        {
          "customFieldId": "priority-field-id",
          "value": "High"
        },
        {
          "customFieldId": "due-date-field-id",
          "value": "2024-03-01"
        }
      ]
    }
  }
}
```

**6. 테스트 및 활성화**
- 먼저 Blue의 API Playground를 사용하여 변형을 테스트합니다.
- Zapier의 데이터 필드를 사용하여 트리거 앱의 동적 값을 매핑합니다.
- 성공적인 테스트 후 Zap을 활성화합니다.
