## 소개

White Label은 Blue를 완전히 귀사의 제품처럼 보이고 느껴지는 유연한 프로젝트 및 프로세스 관리 플랫폼으로 변환시켜드립니다 — 기술, 개발자, 서버 또는 가동 시간을 관리하는 부담 없이 말이죠.

Blue의 White Label을 통해 다음이 가능합니다:

- **브랜드 강조:** Blue의 이름, 로고, 색상을 귀사의 것으로 교체하여 완벽한 브랜드 경험을 만듭니다.
- **자체 도메인 운영:** 전문적이고 일관된 존재감을 위해 귀사의 웹 도메인에서 Blue를 운영합니다.
- **자체 이메일 인프라 활용:** 개인화된 커뮤니케이션을 위해 귀사의 주소에서 이메일을 발송합니다.


에이전시의 경우, White Label 애드온은 브랜드를 강화하고 서비스 제공을 향상시키며 클라이언트 협업을 간소화합니다. Blue를 귀사의 브랜딩으로 워크플로우에 통합함으로써 원활하고 전문적인 경험을 만들어 더 많은 클라이언트를 확보하고, 그들의 필요에 맞춘 서비스와 소프트웨어의 독특한 조합을 제공하여 고객을 유지할 수 있습니다.


## 가격

White Label 애드온은 기본 Blue 구독료에 추가로 회사당 월 $50 또는 조직당 연간 $500입니다. 가격에 대한 자세한 내용은 [여기](/pricing)에서 확인할 수 있습니다.

여러 조직을 위한 White Label 대량 구매에 관심이 있으시면 sales@blue.cc로 문의해 주세요.
 

## 사용자 정의 도메인

White Label이 작동하려면 먼저 웹 애플리케이션용 사용자 정의 도메인을 설정해야 합니다.

다음에 대해 사용자 정의 도메인을 구성할 수 있습니다:

- 웹 애플리케이션 (필수)
- [양식](/docs/projects/forms) (선택사항)
- [파일 공유 링크](/docs/projects/file-management) (선택사항)
- 문서 (곧 제공 예정)


사용자 정의 도메인을 구성하려면 각 하위 도메인에 대해 DNS 설정에 CNAME 레코드를 추가하세요. 루트 도메인이 아닌 하위 도메인만 지원된다는 점을 유의하세요. 따라서 `app.yourcompany.com`은 지원되지만 `yourcompany.com`은 지원되지 않습니다.


| 도메인 유형                | CNAME 레코드                  | 가리키는 대상              |
|----------------------------|-------------------------------|----------------------------|
| 웹 애플리케이션            | `app.yourcompany.com`         | `wl-app.onrender.com`    |
| 양식                       | `forms.yourcompany.com`       | `wl-forms.onrender.com`    |
| 파일 공유 링크             | `files.yourcompany.com`       | `wl-files.onrender.com`    |


위 표는 권장 CNAME 레코드를 제공하지만, 원하는 하위 도메인을 사용할 수 있습니다.

예를 들어, 메인 웹 애플리케이션은 `projects.yourcompany.com`이 될 수 있고 파일 공유 링크는 `media.yourcompany.com`이 될 수 있습니다.


## 브랜딩

Blue를 귀사의 브랜드에 맞추기 위해 다음 요소를 사용자 정의하세요:

- **로고**: 플랫폼 전체에서 Blue의 브랜딩을 대체할 로고를 업로드합니다.
- **파비콘**: 브라우저 탭과 북마크에 표시될 자체 파비콘을 추가합니다.
- **기본 인터페이스 색상**: 모든 사용자에게 일관되게 적용되는 인터페이스의 기본 색상을 설정합니다.
- **앱 이름**: 브라우저 탭과 데스크톱 애플리케이션에 표시되는 앱 이름을 사용자 정의합니다.
- **폰트**: (곧 제공 예정) 선택된 Google 폰트 목록에서 브랜드에 맞는 폰트로 플랫폼 전체의 폰트를 사용자 정의합니다.

Blue는 브랜딩 설정을 구성할 때 로그인 페이지와 앱 아이콘의 실시간 미리보기를 제공합니다.

귀사의 브랜딩은 [데스크톱 애플리케이션](/docs/start-guide/download-apps#desktop-app)에도 적용되어 완전한 White Label 경험을 보장합니다.

로고와 파비콘에 대한 권장 구성은 다음과 같습니다:

| 자산    | 권장 모양 | 배경        | 종횡비 |
|---------|----------|------------|--------|
| 로고    | 직사각형  | 투명       | 1:1.5  |
| 파비콘  | 정사각형  | 투명       | 1:1    |


## 이메일 알림

SMTP를 사용하여 저희 이메일 주소 대신 귀사의 도메인으로 자체 이메일 서버에서 이메일 알림을 보낼 수 있습니다.

Blue에 다음 정보를 제공해야 합니다:

| 설정                              | 설명                                                                                                                                               |
|-----------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------|
| **SMTP 서버**                     | 이메일 전송을 처리할 서버의 주소입니다.                                                                                                            |
| **SMTP 포트**                     | SMTP 서버에 연결하는 데 사용되는 포트 번호로, 일반적으로 TLS의 경우 587, SSL의 경우 465입니다.                                                      |
| **사용자명**                      | SMTP 서버 인증을 위한 사용자명으로, 일반적으로 이메일 주소입니다.                                                                                  |
| **비밀번호 (암호화된 형식으로 저장)** | 사용자명과 연결된 비밀번호로, 자격 증명을 보호하기 위해 암호화된 형식으로 안전하게 저장됩니다.                                                      |
| **발신자 이름**                   | 이메일 알림에서 발신자로 표시될 이름으로, 수신자가 이메일의 출처를 식별하는 데 도움이 됩니다.                                                        |
| **발신자 이메일 주소**            | 발신자로 표시될 이메일 주소로, 회신이 올바른 주소로 가도록 합니다. 이는 SMTP 서버와 동일한 도메인이어야 합니다.                                      |

Blue는 SMTP 연결에 TLS/SSL 암호화 사용을 강제합니다.

그런 다음 "연결 테스트" 버튼을 클릭하여 SMTP 서버에 대한 연결을 테스트할 수 있습니다. Blue는 자동으로 SMTP 서버에 대한 연결을 테스트하고 앱 내 알림을 통해 성공 여부를 알려드립니다.

### 고려사항 

모든 트랜잭션, 알림 및 자동화 이메일은 귀사의 SMTP 서버를 통해 전송됩니다. 유일한 예외는 보안을 보장하기 위해 일반 이메일 주소(notifications@process-system.app)를 통해 전송되는 OTP(일회용 비밀번호)가 포함된 이메일입니다.

SMTP 서버가 다음을 충족하는지 확인하세요:

- 대량의 이메일을 처리할 수 있어야 합니다.
- 스팸 필터를 피하기 위해 강력한 발신 평판을 가져야 합니다.
- 예상되는 이메일 부하를 수용할 수 있는 충분한 할당량이 있어야 합니다.


### 사용자 정의 초대 이메일

새로운 사용자가 Blue 인스턴스에 가입할 때 전송되는 초기 초대 이메일을 사용자 정의할 수도 있습니다.

### 사용자 정의 온보딩 이메일 (곧 제공 예정)

White Label이 없으면 Blue는 새로운 사용자가 회사에 가입할 때 시작을 돕기 위해 일련의 온보딩 이메일을 전송합니다. White Label이 켜지면 이러한 이메일은 전송되지 않으며, 향후 새로운 사용자를 환영하기 위한 자체 온보딩 이메일 세트를 만들 수 있게 됩니다.

## 모바일 애플리케이션

현재 모바일 애플리케이션의 White Label을 지원할 계획은 없지만, Apple Appstore 및 Google Playstore 계정에서 자체 브랜드 버전의 모바일 애플리케이션을 원하신다면 sales@blue.cc로 연락하여 논의할 수 있습니다.

## FAQ

**White Label 애드온에는 무엇이 포함되나요?**

White Label 애드온에는 세 가지 주요 기능이 포함됩니다: 웹 애플리케이션, 양식 및 파일 공유를 위한 사용자 정의 도메인 통합; 사용자 정의 브랜딩(로고, 파비콘, 색상 및 앱 이름); 자체 SMTP 서버를 사용한 사용자 정의 이메일 알림.

**White Label 비용은 얼마인가요?**

White Label 애드온은 기본 Blue 구독료에 추가로 회사당 월 $50 또는 회사당 연간 $500입니다. 향후 여러 회사가 있는 계정에 대해 대량 할인이 제공될 예정입니다.

**White Label이 Blue의 기능에 영향을 미치나요?**

아니요, Blue의 모든 핵심 기능과 기능성은 정확히 동일하게 유지됩니다. White Label은 플랫폼의 모양과 브랜딩만 변경합니다.


**하위 도메인 대신 루트 도메인을 사용할 수 있나요?**

아니요, Blue는 사용자 정의 도메인 설정에 하위 도메인만 지원합니다.

**모든 사용자 정의 도메인을 설정해야 하나요?**

아니요, 웹 애플리케이션 도메인만 필수입니다. 양식 및 파일 공유 도메인은 선택 사항입니다. 문서 도메인 지원은 곧 제공될 예정입니다.

**제안된 것과 다른 하위 도메인 이름을 사용할 수 있나요?**

예, 원하는 하위 도메인 이름을 사용할 수 있습니다. 예를 들어, `app.yourcompany.com` 대신 `projects.yourcompany.com`을 사용할 수 있습니다.

**어떤 브랜딩 요소를 사용자 정의할 수 있나요?**

로고, 파비콘, 기본 인터페이스 색상 및 앱 이름을 사용자 정의할 수 있습니다.

**데스크톱 애플리케이션이 내 브랜딩을 반영하나요?**

예, 데스크톱 애플리케이션은 귀사의 브랜딩으로 완전히 White Label됩니다.

**다른 사용자가 다른 색상 구성표를 가질 수 있나요?**

아니요, 선택한 기본 색상은 Blue 인스턴스의 모든 사용자에게 적용됩니다.

**어떤 이메일 구성 세부 정보를 제공해야 하나요?**

SMTP 서버 주소, SMTP 포트, 사용자명, 비밀번호, 발신자 이름 및 발신자 이메일 주소를 제공해야 합니다. SMTP 비밀번호는 자격 증명을 보호하기 위해 암호화된 형식으로 저장됩니다.

**라이브로 전환하기 전에 SMTP 구성을 테스트할 수 있나요?**

예, Blue는 SMTP 서버 구성을 확인하고 결과를 알려주는 "연결 테스트" 버튼을 제공합니다.

**어떤 유형의 이메일이 내 SMTP 서버를 통해 전송되나요?**

트랜잭션 이메일, 알림 이메일, 이메일 자동화를 포함한 모든 이메일이 귀사의 SMTP 서버를 통해 전송됩니다. OTP(일회용 비밀번호) 이메일은 보안을 보장하기 위해 일반 이메일 주소(notifications@process-system.app)를 통해 전송됩니다.


**모바일 애플리케이션을 White Label할 수 있나요?**

Blue는 현재 모바일 애플리케이션의 White Label을 지원하지 않습니다. 그러나 Apple App Store 또는 Google Play Store에서 브랜드 버전이 필요하신 경우 sales@blue.cc로 연락하여 옵션을 논의할 수 있습니다.

**향후 모바일 앱에 대한 White Label 지원이 제공될 예정인가요?**

현재 모바일 애플리케이션의 White Label을 지원할 계획은 없습니다.

**White Label 설정에 대한 도움을 어떻게 받을 수 있나요?**

일반적인 도움이 필요하시면 support@blue.cc로 문의하실 수 있습니다. 사용자 정의 모바일 애플리케이션에 대한 영업 관련 질문은 sales@blue.cc로 문의하세요.

**White Label 기능을 구현하기 전에 테스트할 수 있나요?**

예, Blue는 브랜딩 설정을 구성할 때 로그인 페이지와 앱 아이콘의 미리보기를 제공합니다.

## DNS 설정

일반적인 DNS(도메인 네임 서비스) 시스템에서 White Label 도메인을 설정하는 방법에 대한 지침을 편리하게 모았습니다:

#### GoDaddy

1. GoDaddy 계정에 로그인하고 제품을 엽니다.
2. 도메인 제어 센터로 이동합니다.
3. 도메인 설정 페이지에서 도메인 이름을 찾아 선택합니다.
4. 추가 설정 섹션으로 스크롤하여 DNS 관리를 선택합니다.
5. DNS 관리 페이지에서 레코드 섹션의 하단으로 스크롤하여 추가를 클릭합니다.
6. 유형 드롭다운 메뉴에서 CNAME을 선택하고 다음 필수 필드를 완성합니다:
    - 호스트 - CNAME의 호스트 이름 또는 접두사. 마침표(.)를 사용할 수 있지만 첫 번째 또는 마지막 문자로는 사용할 수 없습니다. 연속된 마침표(..)는 허용되지 않습니다. 호스트는 25자를 초과하거나 @ 기호가 될 수 없습니다.
    - 가리키는 대상 - `wl-app.onrender.com`으로 설정
    - TTL - 서버가 정보를 캐시해야 하는 시간을 지정합니다.
7. 저장을 클릭합니다.

#### Bluehost

1. BlueHost 계정에 로그인합니다.
2. 도메인 메뉴를 클릭한 다음 Zone Editor를 클릭합니다. 새 페이지가 나타납니다.
3. DNS 레코드 추가 섹션으로 이동합니다.
4. 호스트 레코드 필드에 사용자 정의 도메인에 사용할 주소의 하위 도메인을 입력합니다.
5. 유형 드롭다운 메뉴에서 CNAME 옵션을 선택합니다.
6. 가리키는 대상 섹션에 `wl-app.onrender.com`을 입력합니다.
7. 레코드 추가를 클릭합니다.

#### Cloudflare

1. Cloudflare 계정에 로그인합니다.
2. 드롭다운 메뉴에서 도메인을 찾아 선택합니다.
3. DNS 설정 탭을 클릭합니다.
4. "레코드 추가"를 클릭합니다:
5. CNAME 옵션을 선택합니다.
6. "이름"을 하위 도메인으로 설정합니다.
7. "대상"을 `wl-app.onrender.com`으로 설정합니다.
8. Cloudflare 프록시 토글 아이콘을 회색 구름으로 설정합니다.

#### Domain.com

1. Domain.com 계정에 로그인합니다.
2. 도메인을 찾아 관리를 클릭합니다.
3. 왼쪽 메뉴에서 DNS 및 네임서버를 클릭합니다.
4. DNS 레코드 추가를 클릭하여 CNAME 레코드를 추가합니다.
5. 대상을 `wl-app.onrender.com`으로 설정합니다.
6. DNS 업데이트를 클릭하여 프로세스를 완료합니다.

#### Dreamhost

1. DreamHost 계정에 로그인합니다.
2. 페이지 왼쪽에서 도메인 링크를 클릭합니다.
3. 도메인 드롭다운 메뉴에서 도메인 관리를 선택합니다.
4. 도메인을 찾아 도메인 아래의 DNS 링크를 클릭합니다.
5. (사용자 정의 도메인)에 사용자 정의 DNS 레코드 추가 섹션으로 스크롤하여 다음 필드를 편집합니다:
    - 이름 - 사용할 하위 도메인을 입력합니다.
    - 유형 - CNAME 옵션을 선택합니다.
    - 값 - `wl-app.onrender.com`을 입력합니다.
6. 지금 레코드 추가!를 클릭하여 CNAME 레코드를 추가합니다.

#### Google Domains

1. Google Domains 계정에 로그인합니다.
2. 도메인 이름을 찾아 클릭합니다.
3. 왼쪽 메뉴에서 DNS를 클릭합니다.
4. 사용자 정의 리소스 레코드 섹션으로 이동합니다.
5. 이름 필드에 사용자 정의 도메인에 사용할 주소의 하위 도메인을 입력합니다.
6. 유형 드롭다운 메뉴에서 CNAME 옵션을 선택합니다.
7. 데이터 필드에 `wl-app.onrender.com`을 입력합니다.
8. 추가를 클릭하여 프로세스를 완료합니다.

#### Hostgator

1. HostGator 계정에 로그인합니다.
2. 페이지 왼쪽에서 CPanel을 클릭합니다.
3. 도메인 섹션으로 스크롤하여 Zone Editor를 클릭합니다.
4. 도메인을 찾아 도메인 옆의 CName 레코드 링크를 클릭합니다.
5. CName 레코드 추가 대화 상자에서 다음 빈 필드를 완성합니다:
    - 이름 - 선택한 하위 도메인/도메인 이름을 입력합니다.
    - CNAME - `wl-app.onrender.com`을 입력합니다.
6. CName 레코드 추가를 클릭합니다. HostGator 계정에 새 CNAME을 성공적으로 추가했다는 HostGator의 메시지가 나타납니다.

#### NameCheap

1. NameCheap 계정에 로그인합니다.
2. 계정 드롭다운 메뉴에서 대시보드를 선택합니다.
3. 도메인 목록을 클릭하고 도메인을 찾습니다.
4. 도메인 옆의 관리를 클릭합니다.
5. 고급 DNS를 클릭하고 호스트 레코드 섹션으로 이동합니다.
6. 새 레코드 추가를 클릭합니다.
7. 유형 드롭다운 메뉴에서 CNAME 레코드를 선택하고 하위 도메인을 호스트로 입력합니다.
8. 그런 다음 `wl-app.onrender.com`을 입력합니다.
9. 모든 변경 사항 저장을 클릭하여 프로세스를 완료합니다.
