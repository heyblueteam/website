## 简介

白标功能将 Blue 转变为一个灵活的项目和流程管理平台，看起来和使用起来完全像您自己的产品——无需承担管理技术、开发人员、服务器或正常运行时间的负担。

通过 Blue 的白标功能，您可以：

- **展示您的品牌：** 用您自己的名称、标识和颜色替换 Blue 的品牌，打造无缝的品牌体验。
- **在您的域名上运行：** 在您自己的网络域名上运营 Blue，营造专业、统一的形象。
- **利用您的邮箱基础设施：** 从您的地址发送邮件，进行个性化沟通。


对于代理机构而言，白标附加服务可以强化您的品牌、增强服务产品，并简化客户协作。通过在您的品牌下将 Blue 嵌入到您的工作流程中，您可以打造无缝、专业的体验，帮助您赢得更多客户，并通过提供针对客户需求量身定制的独特服务和软件组合来留住客户。


## 定价

白标附加服务费用为每公司每月 $20 或每组织每年 $200，需在您的核心 Blue 订阅基础上额外支付。您可以在[这里](/pricing)了解更多定价信息。

如果您有兴趣为多个组织批量购买白标服务，请联系 sales@blue.cc

 

## 自定义域名

要使白标功能正常工作，您首先需要为网络应用程序设置自定义域名。

自定义域名可以配置以下内容：

- 网络应用程序（必需）
- [表单](/docs/projects/forms)（可选）
- [文件共享链接](/docs/projects/file-management)（可选）
- 文档（即将推出）


要配置自定义域名，请为每个子域名在您的 DNS 设置中添加 CNAME 记录。请注意，仅支持子域名，不支持根域名。因此支持 `app.yourcompany.com`，但不支持 `yourcompany.com`。


| 域名类型                | CNAME 记录                  | 指向                  |
|----------------------------|-------------------------------|----------------------------|
| 网络应用程序            | `app.yourcompany.com`         | `wl-app.onrender.com`    |
| 表单                      | `forms.yourcompany.com`       | `wl-forms.onrender.com`    |
| 文件共享链接         | `files.yourcompany.com`       | `wl-files.onrender.com`    |


上表提供了建议的 CNAME 记录，但您可以使用任何您想要的子域名。

例如，主网络应用程序可以是 `projects.yourcompany.com`，文件共享链接可以是 `media.yourcompany.com`。


## 品牌设置

自定义以下元素以使 Blue 与您的品牌保持一致：

- **标识**：上传您的标识以替换平台上 Blue 的品牌。
- **网站图标**：添加您自己的网站图标，这将出现在浏览器标签页和书签中。
- **主界面颜色**：为界面设置主颜色，在所有用户中一致应用。
- **应用名称**：自定义在浏览器标签页和桌面应用程序中显示的应用名称。
- **字体**：（即将推出）从精选的 Google 字体列表中自定义平台上使用的字体以匹配您的品牌。

Blue 在您配置品牌设置时提供登录页面和应用图标的实时预览。

您的品牌设置也将应用到[桌面应用程序](/docs/start-guide/download-apps#desktop-app)，确保完全的白标体验。

以下是推荐的标识和网站图标配置：

| 资产   | 推荐形状 | 背景      | 宽高比 |
|---------|------------------|-----------------|-------------|
| 标识    | 矩形        | 透明     | 1:1.5       |
| 网站图标 | 正方形           | 透明     | 1:1         |


## 邮件通知

您可以使用 SMTP 从您自己的邮件服务器发送邮件通知，使用您自己的域名而不是我们的邮箱地址。

您需要向 Blue 提供以下信息：

| 设置                          | 描述                                                                                                                                         |
|-----------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------|
| **SMTP 服务器**                   | 处理发送邮件的服务器地址。                                                                              |
| **SMTP 端口**                     | 用于连接 SMTP 服务器的端口号，TLS 通常为 587，SSL 通常为 465。                                                           |
| **用户名**                      | 用于向 SMTP 服务器认证的用户名，通常是您的邮箱地址。                                                                  |
| **密码（以加密格式存储）** | 与用户名关联的密码，将以加密格式安全存储以保护您的凭据。              |
| **发件人姓名**                   | 将在邮件通知中显示为发件人的姓名，帮助收件人识别邮件来源。                            |
| **发件人邮箱地址**          | 将显示为发件人的邮箱地址，确保回复邮件发送到正确的地址。请注意，这必须与 SMTP 服务器使用相同的域名。 |

Blue 强制对 SMTP 连接使用 TLS/SSL 加密。

然后，您可以通过点击"测试连接"按钮来测试与 SMTP 服务器的连接。Blue 将自动测试与您的 SMTP 服务器的连接，并通过应用内通知告知您是否成功。

### 注意事项 

所有交易、通知和自动化邮件都将通过您的 SMTP 服务器发送。唯一的例外是包含 OTP（一次性密码）的邮件，这些邮件通过通用邮箱地址（notifications@process-system.app）发送以确保安全。

确保您的 SMTP 服务器：

- 能够处理大量邮件。
- 具有良好的发送声誉以避免垃圾邮件过滤器。
- 有足够的配额来满足预期的邮件负载。


### 自定义邀请邮件

您还可以自定义发送给新用户注册您的 Blue 实例时的初始邀请邮件。

### 自定义入门邮件（即将推出）

如果没有白标功能，Blue 会向加入公司的新用户发送一系列入门邮件来帮助他们开始使用。当开启白标功能后，这些邮件不会发送，将来您将能够创建自己的入门邮件集合来欢迎新用户。

## 常见问题


**白标附加服务包含什么？**

白标附加服务包含三个主要功能：网络应用程序、表单和文件共享的自定义域名集成；自定义品牌（标识、网站图标、颜色和应用名称）；以及使用您自己的 SMTP 服务器的自定义邮件通知。

**白标功能的费用是多少？**

白标附加服务在您的核心 Blue 订阅基础上的费用为每公司每月 $20 或每公司每年 $200。对于拥有多个公司的账户，将来会提供批量折扣。

**白标功能是否会影响 Blue 的功能？**

不会，Blue 的所有核心功能和特性保持完全相同。白标功能仅改变平台的外观和品牌。


**我可以使用根域名而不是子域名吗？**

不可以，Blue 仅支持子域名进行自定义域名设置。

**我需要设置所有的自定义域名吗？**

不需要，只有网络应用程序域名是必需的。表单和文件共享域名是可选的。文档域名支持即将推出。

**我可以使用与建议不同的子域名吗？**

可以，您可以使用任何您喜欢的子域名。例如，您可以使用 `projects.yourcompany.com` 而不是 `app.yourcompany.com`。

**我可以自定义哪些品牌元素？**

您可以自定义您的标识、网站图标、主界面颜色和应用名称。

**桌面应用程序会反映我的品牌吗？**

是的，桌面应用程序将完全使用您的品牌进行白标处理。

**不同用户可以有不同的颜色方案吗？**

不可以，您选择的主颜色将在您的 Blue 实例的所有用户中强制使用。

**我需要提供哪些邮件配置详细信息？**

您需要提供 SMTP 服务器地址、SMTP 端口、用户名、密码、发件人姓名和发件人邮箱地址。SMTP 密码以加密格式存储以保护您的凭据。

**我可以在正式使用前测试 SMTP 配置吗？**

可以，Blue 提供"测试连接"按钮，将验证您的 SMTP 服务器配置并通知您结果。

**哪些类型的邮件将通过我的 SMTP 服务器发送？**

所有邮件都将通过您的 SMTP 服务器发送，包括交易邮件、通知邮件、邮件自动化。OTP（一次性密码）邮件通过通用邮箱地址（notifications@process-system.app）发送以确保安全。


**我可以对移动应用程序进行白标处理吗？**

Blue 目前不支持移动应用程序的白标功能。但是，如果您需要在 Apple App Store 或 Google Play Store 上发布品牌版本，您可以联系 sales@blue.cc 讨论选项。

**移动应用的白标支持将来会推出吗？**

目前没有支持移动应用程序白标功能的计划。

**我如何获得白标设置的帮助？**

如需一般帮助，您可以联系 support@blue.cc。如需关于自定义移动应用程序的销售相关问题，请联系 sales@blue.cc。

**我可以在实施前测试白标功能吗？**

可以，Blue 在您配置品牌设置时提供登录页面和应用图标的预览。

## DNS 设置

我们已经方便地收集了常见 DNS（域名系统）系统的白标域名设置说明：

#### GoDaddy

1. 登录您的 GoDaddy 账户，并打开您的产品。
2. 导航到域名控制中心。
3. 在域名设置页面，找到并选择您的域名。
4. 向下滚动到附加设置部分，选择管理 DNS。
5. 在 DNS 管理页面，向下滚动到记录部分的底部，点击添加。
6. 在类型下拉菜单中，选择 CNAME，并完成以下必填字段：
    - 主机 - CNAME 的主机名或前缀。您可以使用句号（.），但不能作为第一个或最后一个字符。不允许连续的句号（..）。主机不能超过 25 个字符或为 @ 符号。
    - 指向 - 设置为 `wl-app.onrender.com`
    - TTL - 指定服务器必须缓存信息的时间量。
7. 点击保存。

#### Bluehost

1. 登录您的 BlueHost 账户。
2. 点击域名菜单，然后点击区域编辑器。将出现新页面。
3. 导航到添加 DNS 记录部分。
4. 在主机记录字段中，输入您想用于自定义域名的地址的子域名。
5. 在类型下拉菜单中，选择 CNAME 选项。
6. 在指向部分，输入 `wl-app.onrender.com`
7. 点击添加记录。

#### Cloudflare

1. 登录您的 Cloudflare 账户。
2. 在下拉菜单中，找到并选择您的域名。
3. 点击 DNS 设置标签页。
4. 点击"添加记录"。
5. 选择 CNAME 选项。
6. 将"名称"设置为您的子域名
7. 将"目标"设置为 `wl-app.onrender.com`
8. 将 Cloudflare 代理切换图标设置为灰色云朵。

#### Domain.com

1. 登录您的 Domain.com 账户。
2. 找到您的域名并点击管理。
3. 在左侧菜单中，点击 DNS 和名称服务器。
4. 点击添加 DNS 记录以添加 CNAME 记录。
5. 将目标设置为 `wl-app.onrender.com`
6. 点击更新 DNS 完成过程。

#### Dreamhost

1. 登录您的 DreamHost 账户。
2. 在页面左侧，点击域名链接。
3. 在域名下拉菜单中，选择管理域名。
4. 找到您的域名并点击您域名下的 DNS 链接。
5. 向下滚动到添加自定义 DNS 记录到（您的自定义域名）部分并编辑以下字段：
    - 名称 - 键入您想使用的子域名
    - 类型 - 选择 CNAME 选项。
    - 值 - 输入 `wl-app.onrender.com`
6. 点击立即添加记录！以添加 CNAME 记录。

#### Google Domains

1. 登录您的 Google Domains 账户。
2. 找到并点击您的域名。
3. 在左侧菜单中，点击 DNS。
4. 导航到自定义资源记录部分。
5. 在名称字段中，输入您想用于自定义域名的地址的子域名。
6. 在类型下拉菜单中，选择 CNAME 选项。
7. 在数据字段中，输入 `wl-app.onrender.com`
8. 点击添加完成过程。

#### Hostgator

1. 登录您的 HostGator 账户。
2. 在页面左侧，点击 CPanel。
3. 向下滚动到域名部分，点击区域编辑器。
4. 找到您的域名并点击域名旁边的 CName 记录链接。
5. 在添加 CName 记录对话框中，完成以下空白字段：
    - 名称 - 输入您选择的子域名/域名。
    - CNAME - 输入 `wl-app.onrender.com`
6. 点击添加 CName 记录。将出现来自 HostGator 的消息，说明您已成功向您的 HostGator 账户添加了新的 CNAME。

#### NameCheap

1. 登录您的 NameCheap 账户。
2. 在账户下拉菜单中，选择仪表板。
3. 点击域名列表并找到您的域名。
4. 点击您域名旁边的管理。
5. 点击高级 DNS 并导航到主机记录部分。
6. 点击添加新记录。
7. 在类型下拉菜单中，选择 CNAME 记录并输入您的子域名作为主机。
8. 然后输入 `wl-app.onrender.com`
9. 点击保存所有更改完成过程。
