Zapier 是一个集成平台，帮助连接不同的网络应用程序并自动化工作流程。

**用例**

以下是 Blue 通过 Zapier 与其他应用程序集成的一些最受欢迎的方式：

**_将 Blue 记录同步到 Google Sheets_**

当 Blue 记录被创建或更新时，自动将其附加为 Google 表格中的新行，以便集中报告。

**_从 Blue 记录发送订单确认_**

当您在 Blue 上选择某个自定义字段时，自动向您的客户发送电子邮件以确认订单。

**_将 Blue 评论添加到 Slack_**

实时获取 Blue 通知，并发布到正确的 Slack 渠道，以便您的团队了解重要的记录更新。

## 主要好处

**跨工具自动化**：在 Zapier 中创建的 Zaps 可以自动在 Blue 和成千上万的其他应用程序之间发送和接收数据。这消除了在系统之间移动数据的手动工作。

**实时集成**：使用 Blue 的 webhook 触发器，Zaps 会立即对 Blue 中的事件（如新记录、添加评论、状态变化等）做出反应。这使得实时同步和流程自动化成为可能。

**易于构建和维护**：Zaps 提供了一种用户友好的无代码方式来集成工具。无需开发工作。并且它们在没有持续维护的情况下继续工作。

1. 登录到您的 Blue 账户，在个人资料设置下
2. 转到个人资料设置并点击“Webhooks”
3. 点击“添加 Webhook”
4. 给 webhook 命名并输入 Zapier webhook URL
5. 选择哪些事件应触发 webhook
6. 点击“创建 Webhook”

## 从 Blue 到 Zapier

1. 登录到您的 Zapier 账户
2. 点击创建 Zap  
3. 在第一步中，搜索并选择 ["Webhooks by Zapier" 触发应用](https://zapier.com/apps/webhook)
4. 配置它以接受来自您的 Blue webhook URL 的有效负载
5. 在第二步中，搜索并选择要发送数据的目标应用
6. 将 JSON 有效负载字段映射到目标应用的输入字段
7. 启用 Zap

现在，当在 Blue 中发生指定事件时，webhook 有效负载将被发送到 Zapier，Zapier 会根据 Zap 设置自动将数据传递给您集成的应用。

## 从 Zapier 到 Blue

要将数据从其他应用程序推送到 Blue，请按照以下步骤操作：

**1. 获取 Blue API 凭据**
- 转到 Blue 设置 > API 令牌
- 创建新令牌或使用现有凭据：
  - 令牌 ID
  - 令牌密钥
  - 公司 ID
  - 项目 ID

**2. 在 Zapier 中创建 Zap**
1. 设置您的触发应用（例如，Google Sheets、Slack 等）
2. 添加“Webhooks by Zapier”作为操作应用
3. 选择“自定义请求”事件类型

**3. 配置 Webhook**
- 方法：POST
- URL：`https://api.blue.cc/graphql`（如有需要可使用 beta 端点）
- 数据传递：FALSE
- 取消扁平化：NO

**4. 设置头部：**
```json
{
  "Content-Type": "application/json",
  "X-Bloo-Token-ID": "Your_Token_ID",
  "X-Bloo-Token-Secret": "Your_Token_Secret", 
  "X-Bloo-Company-ID": "Your_Company_ID",
  "X-Bloo-Project-ID": "Your_Project_ID"
}
```

**5. 配置查询（示例变更）：**
```json
{
  "query": "mutation CreateTodo($input: CreateTodoInput!) {\n  createTodo(input: $input) {\n    id\n    title\n    customFields {\n      value\n    }\n  }\n}",
  "variables": {
    "input": {
      "todoListId": "your-todo-list-id",
      "title": "New task from Zapier",
      "customFields": [
        {
          "customFieldId": "priority-field-id",
          "value": "High"
        },
        {
          "customFieldId": "due-date-field-id",
          "value": "2024-03-01"
        }
      ]
    }
  }
}
```

**6. 测试并激活**
- 首先使用 Blue 的 API Playground 测试变更
- 使用 Zapier 的数据字段映射触发应用的动态值
- 测试成功后启用 Zap
