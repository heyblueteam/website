您可以导入 CSV（逗号分隔值）文件。如果您有 Excel 文件，[您可以在以下链接将其转换为 CSV 文件](https://cloudconvert.com/xls-to-csv)。

## 导入 CSV

在 Blue 中导入 CSV 文件是一个简单的过程。您的 CSV 文件中的每一列对应于 Blue 中的一个默认或 [自定义字段](/docs/custom-fields/introduction)。

首先，您必须是要导入数据的项目的项目管理员。单击项目设置图标（齿轮图标），然后选择“导入”。

![CSV 导入 1](/docs/CSVimport1.png "CSV Import 1")

导入功能在项目设置下对项目管理员可用。

在上面的截图中，您还可以看到另一个选项，称为“下载 CSV 模板”。如果您已经在 Blue 项目中创建了 [自定义字段](/docs/custom-fields/introduction)，这非常有用，因为您可以下载一个包含所有自定义字段列的 CSV 文件！这在您希望确保在导入之前具有精确的 CSV 结构时非常好。

接下来，只需将您的 CSV 文件拖放到 Blue 中：

![CSV 导入 2](/docs/CSVimport2.png "CSV Import 2")

拖放以上传您的数据。

根据您的文件大小，这可能需要几秒钟的时间来上传和让 Blue 进行分析。一旦 Blue 分析完数据，您将看到数据前十行的预览：

![CSV 导入 3](/docs/CSVimport3.png "CSV Import 3")

然后，您可以将 CSV 中的每一列映射到 Blue 中的一个字段。

![CSV 导入 4](/docs/CSVimport4.png "CSV Import 4")

## 接受的日期格式

Blue 支持多种日期格式进行导入。以下是所有支持的格式：

- **Unix 时间戳**：
  ```plaintext
  1511648546 (seconds)
  1620021848429 (milliseconds)
  1620024872717915000 (nanoseconds)
  ```
  
- **RFC 标准**：
  ```plaintext
  RFC 3339: 2021-05-01T01:17:02.604456Z
  RFC 2822: Wed, 02 Jun 2021 06:31:39 GMT
  ```

- **数据库格式**：
  ```plaintext
  PostgreSQL: 
  2019-11-29 08:08-08
  2019-11-29 08:08:05-08
  2021-05-02 23:31:36.0741-07
  ```

- **常见格式**：
  ```plaintext
  Year-Month-Day: 
  2021-04-30 21:14
  2021-02-21
  2021-Feb-21
  
  Month/Day/Year:
  3/31/2014 22:05
  04/02/2014 03:00:51
  
  Day-Month-Year:
  12 Feb 2006, 19:17
  03 February 2013
  ```

- **时区**：
  ```plaintext
  2014-04-26 13:13:43 +0800
  2017-11-25 13:31 PST
  2021-02-21 UTC
  ```

- **自然语言**：
  ```plaintext
  May 6 at 9:24 PM
  September 17, 2012 at 10:09am PST
  October 7, 1970
  ```

- **国际格式**：
  ```plaintext
  Chinese: 2014年04月08日
  Japanese: 2014/04/08 22:05
  ```

- **仅时间**：
  ```plaintext
  01:06:06
  4:00pm PST
  6:00 AM UTC
  ```

所有日期将在导入期间转换为 UTC 时区。时间组件是可选的 - 如果未提供，时间将默认为 00:00:00 UTC。

## 处理数据中的逗号

如果您的数据包含逗号（例如，在地址或公司名称中），您需要将这些值用双引号括起来，以防止导入错误。

**示例：**
```
Company,Address
Acme Inc,123 Main St, New York
```

**应变为：**
```
Company,Address
"Acme Inc","123 Main St, New York"
```

这确保了“123 Main St, New York”中的逗号被视为数据的一部分，而不是列分隔符。
