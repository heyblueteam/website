Webhook是从一个应用程序自动发送到另一个应用程序的消息，当某些事情发生时。例如，当在Blue中创建新记录时，可以配置Webhook将该记录的数据发送到另一个系统。

<youtube url="https://www.youtube.com/watch?v=bdVF2vZ8lRA" /> 

Blue Webhooks会在事件发生后的毫秒内立即触发。这种实时特性使它们在将Blue与其他业务系统集成时变得无价。

**Webhooks**

- 跨系统的实时数据同步
- 能够响应并对Blue事件采取行动
- 无需重复轮询以获取更改；即时推送
- 创建自定义集成和工作流程

**前所未有的控制**

Blue Webhooks让您对以下内容进行细粒度控制：

- 确切哪些事件应触发Webhook以及来自哪个项目
- 过滤器以缩小记录的范围
- 签名验证以增强安全性

**创建Webhook**

在Blue中，项目管理员可以在个人设置下创建Webhooks。您需要指定：

- Webhook名称
- 发送数据的有效负载URL
- 可选的签名密钥

**触发器**

Webhook可以由各种Blue事件触发，包括：

- 记录创建/更新/删除
- 评论添加/编辑
- 受托人更改
- 检查列表状态更改
- 自定义字段值更改
- 还有更多

**Webhook有效负载**

触发时，Blue会向您指定的URL发送包含事件所有细节的JSON有效负载。

以下是一个示例有效负载：

```
{
  "event": "RECORD_CREATED",
  "record": {
    "id": "1234",
    "title": "New record"
    ...
  },
  "company": {
    "id": "5678"  
  },
  "project": {
    "id": "91011"
  },
  "user": {
    "id": "1213" 
  }
}
```

**签名**

为了验证有效负载确实来自Blue，可以检查可选的签名。与OAuth类似，签名是通过使用您选择的密钥对有效负载内容进行哈希生成的。

**Node.js中的示例验证**

```
const crypto = require('crypto');
const key = 'secret'; 
const body = {...webhookPayload} 
const hash = crypto.createHmac('sha256', key).update(body);
const signature = hash.digest('hex');
```

**其他集成**

除了自定义Webhooks，Blue还提供与工具如[Zapier](/docs/integrations/zapier)和[Pabbly](/docs/integrations/pabbly-connect) Connect的集成，以简化将Blue数据与数百个流行应用程序连接的过程。
