Blue 允许您使用谷歌地图作为地图视图，而不是默认的 OpenStreetMap。谷歌地图通常在某些地区提供更详细的地图和更准确的位置数据。然而，这需要一个谷歌地图 API 密钥，并且必须由公司所有者在公司设置中输入。

当您切换到谷歌地图时，这会影响 Blue 中的所有地图视图，包括公司地图、[项目中的地图视图](/docs/views/map) 和 [位置自定义字段](/docs/custom-fields/location-map)。

要在 Blue 中使用谷歌地图，需要在谷歌云控制台中启用以下 API：
- [地图切片 API](https://console.cloud.google.com/apis/library/tile.googleapis.com)：这是渲染地图切片所必需的。
- [地点 API](https://console.cloud.google.com/apis/library/places.googleapis.com)：这是搜索地点并获取其坐标所必需的。
- [地理编码 API](https://console.cloud.google.com/apis/library/geocoding-backend.googleapis.com)：这是获取地点坐标所必需的。

您需要注册一个 [谷歌云账户](https://console.cloud.google.com/) 并创建一个项目以获取 API 密钥，具体说明请参见 [这里](https://developers.google.com/maps/documentation/embed/get-api-key)。

## 限制 API 密钥使用

出于安全考虑，限制您的谷歌地图 API 密钥仅在授权域名上工作至关重要。这可以防止您的 API 密钥被泄露后被未经授权的使用。您应该将 API 密钥限制为仅在以下域名上工作：

- app.blue.cc - 主要的 Blue 应用程序
- forms.blue.cc - 用于公共表单
- 如果您使用 [白标](/docs/start-guide/white-label)，则为您的自定义域名

要限制您的 API 密钥：

1. 转到 [谷歌云控制台凭据页面](https://console.cloud.google.com/apis/credentials)
2. 找到您的 API 密钥并点击“编辑”（铅笔图标）
3. 在“应用程序限制”下，选择“HTTP 引用来源（网站）”
4. 添加以下引用来源：
   - `*.app.blue.cc/*`
   - `*.forms.blue.cc/*`
   - `*.yourdomain.com/*`（如果使用白标）
5. 点击“保存”

**关于谷歌地图费用的重要说明**<br><br>

谷歌将单独向您收取使用谷歌地图 API 的费用，与您的 Blue 订阅无关。与您的谷歌地图账户和 API 使用相关的所有费用、开支和费用均由组织管理员独自承担。Blue 对您使用谷歌地图 API 产生的任何费用不承担责任，包括但不限于第三方集成、API 使用或存储费用。
