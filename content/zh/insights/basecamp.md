## Basecamp 简介

Basecamp 是最早的项目管理系统之一——它于 2004 年推出！

该平台最初是由网页设计公司 [37signals](https://37signals.com/) 创建的内部项目管理工具，该公司由 [Jason Fried](https://world.hey.com/jason) 和 [David Heinemeier Hansson](https://dhh.dk/) 共同创立。

David 还是[开源网络框架 Ruby on Rails](https://rubyonrails.org/) 的创建者，[GitHub](https://github.com/)、[Shopify](https://www.shopify.com/)、[Airbnb](https://www.airbnb.com/)、[Square](https://squareup.com/)、[Coinbase](https://www.coinbase.com/) 和 [Zendesk](https://www.zendesk.com/) 等公司都在使用该框架。

![](/insights/basecamp-home.png)
<p class="text-sm text-gray-500 text-center -mt-8">Image description goes here</p>


Basecamp 的开发是为了满足公司自身在项目管理和客户沟通方面的需求。这实际上[与 Blue 的故事非常相似！](/about)

推出后，Basecamp 迅速获得了人气，促使创始人完全专注于它，并最终在 2014 年将公司更名为 Basecamp，标志着产品首次发布十周年。他们专注于与 [Blue](/) 类似的理念：*"令人耳目一新的简单项目管理。"*

多年来，Basecamp 运营着一个名为 [Signal vs Noise](https://signalvnoise.com/) 的优秀博客，值得一读。

Basecamp 团队还出版了几本有趣的书籍，其中包含许多逆向思维。

- **[Shape Up](https://basecamp.com/books/shapeup)**：为团队提供工具，以应对产品开发过程中每个阶段的风险和未知因素。
- **[It Doesn't Have to Be Crazy at Work](https://basecamp.com/books/calm)**：认为平静、专注的工作环境比混乱和高压的环境更有效率和可持续，强调工作与生活平衡的重要性。
- **[Rework](https://basecamp.com/books/rework)**：挑战传统的商业实践，倡导更简单、更高效、更少传统的创业和工作方法。
- **[Remote](https://basecamp.com/books/remote)**：推广远程工作的好处和可行性，强调企业和员工如何在不受传统办公室环境限制的情况下蓬勃发展。
- **[Getting Real](https://basecamp.com/books/getting-real)**：提供构建成功的基于网络的应用程序的实用建议，专注于简单、速度和效率，而不是传统的复杂方法。

## Blue 简介

Blue 由 [Emanuele "Manny" Faja](https://www.linkedin.com/in/efaja/) 于 2018 年创立，使命明确：

> 通过构建地球上最好的项目管理平台来组织世界的工作——简单、强大、灵活且人人都能负担得起。

有趣的是，Manny 进入项目管理领域的旅程始于使用 Basecamp 管理他的[产品设计公司](/insights/agency-success-guide)。他喜欢 Basecamp 的简单性（您可以在 [Basecamp 网站上找到他的 Basecamp 评论！](https://basecamp.com/customers)），但随着他的机构发展，他需要更强大的东西，于是他转向了 [Asana](https://asana.com)。

然而，简单性消失了，对于他的大多数客户来说太复杂了！

Manny 想要的是*像 Basecamp 一样易于使用*，*像 Asana 一样强大*的东西。

与 Basecamp 一样，Blue 自豪地采用自力更生模式。我们在这个行业中属于少数，大多数竞争对手都筹集了数亿美元的风险投资资金。这种方法使 Blue 能够忠于其愿景，并保持对用户需求和产品质量的强烈关注，而不必在投资者需求和客户需求之间周旋。

### Basecamp 计划分解

Basecamp 有两个计划。

1. **Basecamp**：每用户收费 $15，不包括[时间表](https://3.basecamp-help.com/article/897-timesheet)和[管理专业包](https://3.basecamp-help.com/article/687-admin-pro-pack)。这些附加组件每个每月 $50。
2. **Basecamp Pro Unlimited**：每月 $349，适用于无限用户和所有功能。

让我们进一步深入了解这两个计划。

| 功能 | Basecamp（按用户计费） | Basecamp Pro Unlimited |
|---------|---------------------|------------------------|
| 费用 | 每用户每月 $15 | 每月 $299（年度计费）<br>每月 $349（月度计费） |
| 适合 | 自由职业者、初创公司或小型团队 | 成长中的企业和大型团队 |
| 用户 | 按用户付费 | 固定费率无限用户 |
| 文件存储 | 500 GB | 5 TB |
| 项目 | 无限 | 无限 |
| 访客协作者 | 免费（不收费） | 免费（不收费） |
| 客户支持 | 标准 | 优先 24/7 |
| 入职协助 | 无 | 一对一协助 |
| 计费选项 | 可按月计费 | 可以通过支票按年支付 |
| 附加功能 | 访问所有基本项目管理功能 | Basecamp 计划的所有功能 |


因此，Blue 的按用户费用比 Basecamp 便宜一半以上（$7 对比 $15），并且 Blue 默认包含所有功能——没有额外的付费附加组件或追加销售。

与 Basecamp 的 Pro Unlimited 计划相比，如果您有 49 个用户或更少，Blue 是更好的选择。对于拥有 50 个或更多用户的团队，Basecamp Pro Unlimited 在纯粹的每用户价格基础上变得更具成本效益。但是，重要的是要注意 Blue 为某些用户类型（客户、仅评论、仅查看）提供免费账户，这可以使其对大型团队具有成本效益，具体取决于用户类型的组合。

## 文件存储

在文件存储方面，Blue 和 Basecamp 选择了不同的路径。Blue 通过在所有计划中提供慷慨的文件存储来保持简单。这意味着无论您的团队有多大或您的项目有多广泛，您永远不必担心空间不足。此外，每个文件 5GB 的慷慨上传限制，Blue 可以满足大多数业务需求。

然而，Basecamp 根据您选择的计划选择分层存储系统。他们的基本计划为您提供 500GB 的可靠存储空间，这对于中小型团队或项目来说应该绰绰有余。如果您的业务有更大的需求，那么他们的 Pro Unlimited 计划真的很出色，提供高达 5TB 的存储空间。这种灵活的方法让团队可以随着增长扩展其存储。

需要考虑的一件事是 Basecamp 没有明确说明上传的最大文件大小。与 Blue 不同，Blue 将其限制指定为 5GB，Basecamp 使这个细节有些模糊。如果您经常处理大文件，这种遗漏可能会引起一些注意。

您在 Blue 和 Basecamp 之间的选择可能取决于您的需求。如果您更喜欢无限存储和明确文件大小限制的保证，Blue 可能是您的选择。有了不会达到上限的安心感和合理的文件大小限制，Blue 为大多数场景提供了直接的解决方案。

另一方面，如果您认为您的存储需求将符合 Basecamp 的框架，他们的计划可能适合您。基本计划上的 500GB 对许多团队来说相当慷慨，而 Pro Unlimited 计划上跃升到 5TB 提供了充足的增长空间。如果您经常处理超过 5GB 的文件，请务必检查 Basecamp 的上传限制。

归根结底，Blue 和 Basecamp 都提供了强大的存储选项。


## API 和 Webhook

### API

Basecamp 有一个[文档齐全的 Rest API](https://github.com/basecamp/api)，然而，即使是 Basecamp 也承认它并不全面：

> Basecamp API 并不旨在成为您在我们所有应用程序中可以执行的所有操作的完整模型。事实是，只有不到百分之零点五的最终用户利用与 Basecamp 集成的应用程序或编写自己的集成。如此低的使用率意味着继续更新 API 以与所有新功能保持同步在 Basecamp 并不是高优先级。当我们为那不到百分之零点五的用户工作时，我们没有帮助另外 99.5% 的用户。

只有不到百分之零点五的最终用户利用 API 的事实可能是因为他们的目标受众是寻找简单东西的典型小企业。

[Blue GraphQL API](/platform/api) 对 Blue 中的所有功能具有 100% 的覆盖率。

因此，您可以通过我们的界面执行的*任何*操作都可以通过编程触发。这是我们首次创建 API 时的有意识设计选择，我们希望为开发人员提供对所有内容的完全访问权限。

每次我们[发布新功能](/platform/changelog)时，API 都会更新以支持这些新功能。

### Webhook


Webhook 是当特定事件发生时从一个应用程序发送到另一个应用程序的自动消息。在项目管理软件的背景下，webhook 很有用，因为它们允许不同工具之间的无缝通信。例如，在项目管理软件中，webhook 可以触发诸如更新任务状态、创建通知或与其他软件实时同步数据等操作。这使项目管理工具与其他系统之间能够更好地集成，简化工作流程并提高效率。

Basecamp 曾经在 [Basecamp 3 中有 webhook](https://github.com/basecamp/bc3-api/blob/master/sections/webhooks.md)，但网上没有提到 Basecamp 4 有 webhook，我们在软件测试中也找不到任何 webhook。

但是，[Basecamp 确实与第三方软件有一些集成](https://basecamp.com/extras)。作为比较，[您还可以查看 Blue 提供的集成列表。](/platform/integrations)

## 公司结构的差异

当您选择项目管理工具时，重要的是不仅要看其功能。产品背后的公司也很重要。他们的公司结构和重点可能会影响工具的发展方式及其长期可持续性。

让我们深入了解 Basecamp 和 Blue 在这方面的不同之处。

Basecamp 由 37signals 创建，这是一家主要由其创始人拥有的公司，但有一个有趣的例外：亚马逊创始人 Jeff Bezos 拥有少量股份。[虽然 37signals 喜欢说他们没有投资者](https://37signals.com/01)，但由于 Bezos 的参与，这并不完全正确。因此，虽然他们享有很大程度的独立性，但他们仍然必须考虑外部利益相关者的利益。

37signals 和 Blue 之间的主要区别是他们的产品重点。37signals 不仅仅专注于 Basecamp；他们还创建和维护其他产品，如 [Hey](https://hey.com)（电子邮件服务）和 Ruby on Rails（流行的网络框架）。最近，他们添加了 [Once](https://once.com)，其中包括两个新产品：[Campfire](https://once.com/campfire)（聊天工具）和 [Writebook](https://once.com/writebook)（用于协作写作）。

这种广泛的关注意味着 37signals 必须在各种项目和行业之间分配其注意力、资源和专业知识。

这种分散的注意力确实影响了 Basecamp 的发展方式。主要更新通常每隔几年才会发生一次。虽然这种稳定的方法有其好处，但它可能无法跟上项目管理需求的快速变化或行业的新趋势。

许多 Basecamp 用户发现他们长时间使用相同的工具，即使他们的工作流程和挑战发生了变化。

在 Blue，我们相信改进是 1% 一次复合的。我们对项目管理平台有着专一的关注。这使团队能够将所有精力投入到增强 Blue 的功能中。Blue 独立运营，没有外部投资者将他们推向不同的方向。这种独立性与他们的专一关注相结合，帮助 Blue 快速响应用户反馈和市场变化。

Blue 的奉献精神体现在其开发周期中。该平台[每月都会定期更新和增强](/platform/changelog)。这种敏捷的方法意味着 Blue 可以迅速适应用户建议并满足新兴需求。用户可以期待频繁的改进、新功能和精细调整的工具，确保平台保持新鲜和有效。

因此，在 Basecamp 和 Blue 之间做出决定时，请考虑您是想要一个随时间变化不大的平台，还是一个不断适应您需求的平台。如果您重视定期更新和对增强项目管理体验的坚定承诺，您可能会发现 Blue 更适合。

最终，您的决定应该反映您团队的独特需求和偏好。在当今快节奏的商业世界中，拥有一个不断发展并持续增加价值的项目管理工具真的可以让您与众不同。

## Basecamp 的替代方案？

在仔细查看了 Blue 和 Basecamp 之后，很明显 Blue 作为一个出色的替代方案脱颖而出，特别是对于想要简单性*和*强大功能相结合的团队。

Basecamp 已经存在了一段时间，在项目管理领域享有盛名。然而，Blue 带来了几个显著的优势：

- **定价：** Blue 提供了一个具有成本效益的解决方案，每用户每月仅 $7，而 Basecamp 每用户每月为 $15。对于拥有多达 49 个用户的团队，这可以节省大量费用。
- **功能集：** Blue 拥有广泛的功能，包括 Basecamp 不提供的基本视图，如时间线、仪表板和表格视图。此外，它还提供高级工具，如 AI 集成、可定制表单、任务依赖关系和跨项目引用。
- **灵活性：** Blue 提供各种查看选项，并允许您自定义权限以适应您团队的需求。对于发现 Basecamp 更严格的设置限制的团队来说，这种灵活性可能很有吸引力。
- **存储：** Blue 在所有计划中提供慷慨的文件存储，对单个文件有明确的 5GB 限制。这种直接的方法可能比 Basecamp 的分层存储系统更容易管理，后者可能不适合每个团队。
