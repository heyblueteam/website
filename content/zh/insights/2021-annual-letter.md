以下是2021年的一些有趣数据和我们取得的成就，以及对2022年的展望。

2021年有趣的数据统计及同比增长

* 注册公司数：503 → 4,585
* Blue使用国家数：63 → 122
* 可用语言数：3 → 15
* 创建的项目数：1,357 → 16,874
* 创建的待办事项数：29,676 → 329,543
* Blue中存储的文件数：10,297 → 145,145
* Blue中的存储量：265GB → 3,502GB
* 提交的功能请求：400+
* 收入增长：5.6倍（并实现盈亏平衡——呼！）
* 我们没有扩大团队，仍然只有5个人。
* 作为对比，其他公司的团队规模要大得多：
	+ Asana → 900名员工
	+ ClickUp → 800名员工
	+ Monday → 800名员工
	+ Wrike → 950名员工
	+ Atlassian → 5,700名员工
* 客户支持
	+ 支持邮件：约6,000封
	+ 我们的满意度评分是89/100，这意味着89%的支持请求被客户评为"优秀"！
	+ 51%的支持请求仅需一次回复即可解决。
	+ 我们的平均响应时间是4.5小时，相比2020年的11.5小时有了显著改善。
	+ 我们与客户进行了300多次一对一通话，了解他们的使用案例，回答支持问题，帮助他们充分利用Blue。
* 我们部署了41个新版本的Blue，几乎每周一个新版本！
* 我们还进行了1,452次代码提交，为Blue添加了112,755行代码。
* 2021年Blue的总停机时间为5.5小时，正常运行时间达到99.93%。
* 未解决的bug数量达到了112个的历史新高。虽然这听起来很糟糕，但实际上这是因为有这么多人在使用平台……更多的眼睛帮我们发现需要修复的问题！

## 不会改变的事情

易用性。一般来说，随着时间的推移，事物会变得更加复杂和难以使用。我们的目标是相反的，实际上让事物变得更容易使用。我们将始终专注于出色的用户体验。我们将始终在功能和灵活性之间做出明智的权衡。


人性化的方法。当您需要时，您始终能够与我们取得联系，您永远不会收到来自Blue的noreply@邮件。


慢即是快。我们始终在准备好时才发布，绝不提前。我们的功能平均在Beta测试阶段花费两个月。这让客户在我们向所有人发布功能之前提供反馈。这不会改变。它帮助我们将技术债务保持在最低限度。这意味着我们将有限的时间用于创新，而不是解决混乱。


专注于小型组织。小型企业使用由拥有数千名员工的公司开发的软件，这很奇怪。这些公司了解小团队和企业主的需求吗？我们将保持小规模，只专注于小型组织。如果我们制作一个试图为所有人服务的产品——它最终实际上对任何人都不起作用！
