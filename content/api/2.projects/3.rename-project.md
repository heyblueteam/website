## Rename a Project

Updates the name of an existing project. The project slug will be automatically regenerated based on the new name.

### Example

```graphql
mutation RenameProject {
  editProject(input: {
    projectId: "project_abc123"
    name: "Q2 Marketing Campaign"
  }) {
    id
    name
    slug
  }
}
```

## Input Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `projectId` | String! | ✅ Yes | The ID of the project to rename |
| `name` | String! | ✅ Yes | The new name for the project (max 50 characters) |

## Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | String! | Project ID |
| `name` | String! | Updated project name |
| `slug` | String! | Auto-generated URL-friendly slug |

## Required Permissions

| Role | Can Rename Project |
|------|-------------------|
| `OWNER` | ✅ Yes |
| `ADMIN` | ✅ Yes |
| `MEMBER` | ❌ No |
| `CLIENT` | ❌ No |

## Error Responses

### Project Not Found
```json
{
  "errors": [{
    "message": "Project not found",
    "extensions": {
      "code": "PROJECT_NOT_FOUND"
    }
  }]
}
```

### Name Too Long
```json
{
  "errors": [{
    "message": "Project name cannot exceed 50 characters",
    "extensions": {
      "code": "PROJECT_NAME_TOO_LONG"
    }
  }]
}
```

## Important Notes

- **Slug Generation**: The project slug is automatically regenerated from the new name
- **Uniqueness**: Project names don't need to be unique within a company
- **Special Characters**: HTML tags are automatically stripped from names
- **Real-time Updates**: Connected clients receive updates via subscriptions
