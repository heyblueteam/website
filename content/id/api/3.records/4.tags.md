## Ikhtisar

Tag adalah cara yang kuat untuk mengkategorikan dan mengorganisir catatan di Blue. Sistem tag menyediakan fungsionalitas komprehensif untuk membuat, mengelola, dan memfilter tag, dengan dukungan untuk operasi massal dan pemfilteran lanjutan.

## Daftar Tag

Ambil tag dari proyek dengan opsi pemfilteran dan pengurutan yang fleksibel.

### Contoh Dasar

```graphql
query ListTags {
  tagList(
    filter: { 
      projectIds: ["project-123"] 
      excludeArchivedProjects: false 
    }
    first: 50
    orderBy: title_ASC
  ) {
    items {
      id
      uid
      title
      color
      project {
        id
        name
      }
      createdAt
      updatedAt
    }
    pageInfo {
      totalPages
      totalItems
      page
      perPage
      hasNextPage
      hasPreviousPage
    }
    totalCount
  }
}
```

### Contoh Pemfilteran Lanjutan

```graphql
query FilterTags {
  tagList(
    filter: {
      projectIds: ["project-123", "project-456"]
      search: "urgent"
      colors: ["#ff0000", "#ffaa00"]
      excludeArchivedProjects: true
    }
    orderBy: createdAt_DESC
    first: 20
  ) {
    items {
      id
      title
      color
      todos {
        id
        title
      }
    }
    totalCount
  }
}
```

## Buat Tag

Buat tag baru dalam proyek.

```graphql
mutation CreateTag {
  createTag(
    input: {
      title: "High Priority"
      color: "#ff0000"
    }
  ) {
    id
    uid
    title
    color
    project {
      id
      name
    }
    createdAt
  }
}
```

## Perbarui Tag

Modifikasi properti tag yang ada.

```graphql
mutation UpdateTag {
  editTag(
    input: {
      id: "tag-123"
      title: "Critical Priority"
      color: "#cc0000"
    }
  ) {
    id
    title
    color
    updatedAt
  }
}
```

## Hapus Tag

Hapus tag dari sistem. Ini juga akan menghapus tag dari semua catatan terkait.

```graphql
mutation DeleteTag {
  deleteTag(id: "tag-123")
}
```

## Tag Catatan

Asosiasikan tag dengan catatan (todos) menggunakan tag yang ada atau buat yang baru.

### Menggunakan Tag yang Ada

```graphql
mutation TagRecord {
  setTodoTags(
    input: {
      todoId: "todo-123"
      tagIds: ["tag-456", "tag-789"]
    }
  )
}
```

### Buat Tag Saat Menandai

Anda dapat membuat tag baru secara langsung dengan memberikan judul tag:

```graphql
mutation TagRecordWithNewTags {
  setTodoTags(
    input: {
      todoId: "todo-123"
      tagIds: ["existing-tag-id"]
      tagTitles: ["New Tag", "Another New Tag"]
    }
  )
}
```

## Saran Tag AI

Dapatkan saran tag yang didukung AI untuk sebuah catatan:

```graphql
mutation GetTagSuggestions {
  aiTag(
    input: {
      todoId: "todo-123"
      suggestionsCount: 5
    }
  ) {
    suggestions {
      title
      confidence
      color
    }
  }
}
```

## Parameter Input

### TagListFilter

| Parameter | Tipe | Diperlukan | Deskripsi |
|-----------|------|------------|-----------|
| `projectIds` | [String!] | Tidak | Filter tag berdasarkan ID proyek atau slug |  |
| `excludeArchivedProjects` | Boolean | Tidak | Kecualikan tag dari proyek yang diarsipkan |  |
| `search` | String | Tidak | Cari judul tag (tidak sensitif huruf besar/kecil) |  |
| `titles` | [String!] | Tidak | Filter berdasarkan judul tag tertentu |  |
| `colors` | [String!] | Tidak | Filter berdasarkan warna tertentu (format hex) |  |
| `tagIds` | [String!] | Tidak | Filter berdasarkan ID tag tertentu |  |

### CreateTagInput

| Parameter | Tipe | Diperlukan | Deskripsi |
|-----------|------|------------|-----------|
| `title` | String | Tidak | Judul tag (dihasilkan secara otomatis jika tidak diberikan) |  |
| `color` | String! | ✅ Ya | Warna tag dalam format hex (misalnya, "#ff0000") |  |

### EditTagInput

| Parameter | Tipe | Diperlukan | Deskripsi |
|-----------|------|------------|-----------|
| `id` | String! | ✅ Ya | ID tag yang akan diperbarui |  |
| `title` | String | Tidak | Judul tag baru |  |
| `color` | String | Tidak | Warna tag baru dalam format hex |  |

### SetTodoTagsInput

| Parameter | Tipe | Diperlukan | Deskripsi |
|-----------|------|------------|-----------|
| `todoId` | String! | ✅ Ya | ID catatan yang akan diberi tag |  |
| `tagIds` | [String!] | Tidak | ID tag yang ada untuk diterapkan |  |
| `tagTitles` | [String!] | Tidak | Judul tag baru untuk dibuat dan diterapkan |  |

### TagOrderByInput Values

| Nilai | Deskripsi |
|-------|-----------|
| `id_ASC` / `id_DESC` | Urutkan berdasarkan ID tag |  |
| `uid_ASC` / `uid_DESC` | Urutkan berdasarkan pengenal unik |  |
| `title_ASC` / `title_DESC` | Urutkan secara alfabetis berdasarkan judul |  |
| `color_ASC` / `color_DESC` | Urutkan berdasarkan nilai warna |  |
| `createdAt_ASC` / `createdAt_DESC` | Urutkan berdasarkan tanggal pembuatan |  |
| `updatedAt_ASC` / `updatedAt_DESC` | Urutkan berdasarkan tanggal pembaruan terakhir |  |

## Bidang Respons

### Tipe Tag

| Bidang | Tipe | Deskripsi |
|--------|------|-----------|
| `id` | ID! | Pengenal unik untuk tag |  |
| `uid` | String! | Pengenal unik yang ramah pengguna |  |
| `title` | String! | Judul/nama tag |  |
| `color` | String! | Warna tag dalam format hex |  |
| `project` | Project! | Proyek yang dimiliki tag ini |  |
| `todos` | [Todo!]! | Catatan yang terkait dengan tag ini |  |
| `createdAt` | DateTime! | Waktu tag dibuat |  |
| `updatedAt` | DateTime! | Waktu tag terakhir diperbarui |  |

### TagPagination Respons

| Bidang | Tipe | Deskripsi |
|--------|------|-----------|
| `items` | [Tag!]! | Array catatan tag |  |
| `pageInfo` | PageInfo! | Informasi pagination |  |
| `totalCount` | Int! | Jumlah total tag yang sesuai dengan filter |  |

### Bidang PageInfo

| Bidang | Tipe | Deskripsi |
|--------|------|-----------|
| `totalPages` | Int | Jumlah total halaman |  |
| `totalItems` | Int | Jumlah total item di semua halaman |  |
| `page` | Int | Nomor halaman saat ini |  |
| `perPage` | Int | Jumlah item per halaman |  |
| `hasNextPage` | Boolean! | Apakah ada halaman berikutnya |  |
| `hasPreviousPage` | Boolean! | Apakah ada halaman sebelumnya |  |

## Izin yang Diperlukan

### Manajemen Tag

| Tindakan | Peran yang Diperlukan |
|----------|-----------------------|
| Create tags | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |
| Edit tags | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |
| Delete tags | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |
| Apply tags to records | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |

### Visibilitas Tag

Tag terlihat oleh semua pengguna yang memiliki akses ke proyek di mana tag dibuat. Penyaringan berbasis peran mungkin berlaku untuk operasi tertentu.

## Praktik Terbaik

1. **Gunakan Warna yang Konsisten**: Tetapkan sistem pengkodean warna untuk tim Anda
2. **Judul Deskriptif**: Gunakan nama tag yang jelas dan deskriptif yang akan dipahami anggota tim
3. **Operasi Massal**: Gunakan `setTodoTags` untuk menerapkan beberapa tag sekaligus
4. **Cari dan Filter**: Gunakan fungsi pencarian untuk menemukan tag yang ada sebelum membuat yang baru
5. **Manajemen Tag**: Tinjau dan bersihkan tag yang tidak digunakan secara teratur
6. **Konvensi Warna**: Pertimbangkan untuk menggunakan warna standar untuk jenis tag umum (merah untuk mendesak, hijau untuk selesai, dll.)

## Fitur Lanjutan

### Default Pembuatan Tag

- Tag baru yang dibuat melalui `setTodoTags` dengan `tagTitles` secara default berwarna `#4a9fff`
- Tag tanpa warna yang ditentukan akan kembali ke `#00a0d2`
- Judul tag secara otomatis dipangkas dari spasi

### Integrasi Otomatisasi

Tag dapat:
- Diterapkan secara otomatis oleh aturan otomatisasi
- Digunakan sebagai pemicu untuk alur kerja otomatisasi
- Dimodifikasi melalui tindakan otomatisasi

### Pembaruan Waktu Nyata

Perubahan tag disiarkan melalui langganan GraphQL, memungkinkan pembaruan waktu nyata di klien yang terhubung.

## Catatan Penting

- Menghapus tag menghapusnya dari semua catatan terkait
- Tag bersifat proyek-spesifik dan tidak dapat dibagikan antar proyek
- Nilai warna harus merupakan kode hex yang valid (misalnya, "#ff0000", "#00aa00")
- Judul tag bersifat sensitif terhadap huruf besar/kecil
- Maksimum 100 tag dapat diterapkan pada satu catatan
- Tag mendukung pencarian teks penuh di seluruh judul
