## Ikhtisar

Gambar sampul memungkinkan Anda menambahkan konteks visual pada record dengan menampilkan pratinjau gambar langsung pada kartu record. Fitur ini secara otomatis mengekstrak gambar dari berbagai sumber dalam record Anda dan menampilkannya sebagai thumbnail, sehingga memudahkan identifikasi dan pengorganisasian konten visual dengan sekali pandang.

Baik Anda mengelola proyek desain, kalender konten, katalog produk, atau alur kerja apa pun yang melibatkan aset visual, gambar sampul membantu Anda memindai record dengan cepat dan menemukan apa yang Anda cari. Fitur ini mencakup pelacakan progres real-time dan opsi tampilan fleksibel untuk mencocokkan kebutuhan alur kerja spesifik Anda.

## Fitur Utama

- **Ekstraksi Gambar Otomatis**: Blue secara otomatis menemukan dan menampilkan gambar dari sumber yang Anda pilih
- **Sumber Gambar Beragam**: Tarik gambar dari deskripsi record, komentar, atau field file kustom
- **Opsi Penyesuaian Tampilan**: Pilih bagaimana gambar muncul dengan Cover atau Contain
- **Seleksi Cerdas**: Konfigurasikan apakah akan menggunakan gambar pertama atau terakhir dari sumber Anda
- **Pembaruan Real-time**: Gambar sampul diperbarui secara otomatis ketika konten record berubah
- **Pelacakan Progres Live**: Pantau pemrosesan massal dengan langganan progres real-time

## Cara Kerja

### Mengaktifkan Gambar Sampul

Gambar sampul dikonfigurasi di level proyek oleh Administrator Proyek:

1. Buka proyek Anda dan navigasi ke **Pengaturan Proyek**
2. Pilih **Pengaturan Record** 
3. Temukan bagian **Opsi Sampul Gambar**
4. Aktifkan **Enable Cover Images** untuk mengaktifkan fitur

### Opsi Konfigurasi

#### Sumber Gambar
Pilih di mana Blue harus mencari gambar:
- **Description**: Mengekstrak gambar yang tertanam dalam deskripsi record (default)
- **Comments**: Menarik gambar pertama/terakhir dari komentar record
- **Custom Field**: Menggunakan file dari field kustom tipe file tertentu (mendukung semua jenis file)

#### Seleksi Gambar
Ketika beberapa gambar tersedia, pilih mana yang akan ditampilkan:
- **First Image**: Menggunakan gambar pertama yang ditemukan (default)
- **Last Image**: Menggunakan gambar terbaru

#### Penyesuaian Tampilan
Kontrol bagaimana gambar muncul di area pratinjau:
- **Cover**: Gambar mengisi seluruh area pratinjau, mungkin memotong tepi untuk mempertahankan rasio aspek (default)
- **Contain**: Menampilkan gambar lengkap dalam area pratinjau, mungkin ada ruang kosong

## Kasus Penggunaan

**Proyek Desain** - Tampilkan mockup dan iterasi desain langsung pada kartu tugas, sehingga mudah melihat status terkini setiap tugas desain tanpa membuka record individual.

**Perencanaan Konten** - Untuk kalender konten dan alur kerja editorial, tampilkan gambar unggulan atau thumbnail untuk memvisualisasikan pipeline konten Anda dengan sekali pandang.

**Manajemen Produk** - Tampilkan foto produk pada record inventaris atau permintaan fitur, membantu tim mengidentifikasi item dengan cepat selama diskusi.

**Real Estate** - Tampilkan foto properti pada record listing, memungkinkan agen menjelajahi properti secara visual dengan cepat.

**Perencanaan Acara** - Tampilkan foto venue, konsep desain, atau layout acara pada tugas perencanaan untuk organisasi visual yang lebih baik.

## Menyiapkan Gambar Sampul

### Langkah 1: Pilih Sumber Gambar Anda

Pertimbangkan di mana gambar secara alami ada dalam alur kerja Anda:
- Jika gambar adalah bagian dari deskripsi tugas atau brief, gunakan **Description**
- Jika gambar ditambahkan melalui kolaborasi tim, gunakan **Comments**
- Jika Anda memiliki field khusus untuk gambar, gunakan **Custom Field**

### Langkah 2: Konfigurasikan Pengaturan Tampilan

1. Pilih **Image Fit** yang diinginkan berdasarkan jenis gambar Anda:
   - Gunakan **Cover** untuk foto dan desain di mana pemotongan dapat diterima
   - Gunakan **Contain** untuk diagram atau screenshot di mana Anda perlu melihat semuanya

2. Pilih **Image Selection** berdasarkan alur kerja Anda:
   - Gunakan **First** untuk alur kerja di mana gambar utama ditambahkan awalnya
   - Gunakan **Last** untuk alur kerja di mana gambar terbaru paling relevan

### Langkah 3: Proses Record yang Ada

Ketika Anda pertama kali mengaktifkan gambar sampul atau mengubah konfigurasi, Blue secara otomatis memproses record Anda:
1. Pembaruan progres real-time menunjukkan status pemrosesan 
2. Record diproses dalam batch (50 record per batch) untuk performa optimal
3. Anda dapat terus bekerja sementara pemrosesan terjadi di latar belakang
4. Daftar akan secara otomatis disegarkan ketika pemrosesan selesai
5. Pemrosesan secara otomatis dipicu ketika perubahan konfigurasi disimpan

## Detail Teknis

### Format yang Didukung
- Ketika menggunakan sumber Description atau Comments:
  - Gambar JPEG/JPG
  - Gambar PNG (termasuk transparansi)
  - Gambar GIF (tampilan statis)
- Ketika menggunakan sumber Custom Field:
  - Semua jenis file dapat ditampilkan (gambar, PDF, dokumen)
  - File non-gambar menampilkan ikon jenis file yang sesuai
- Semua file harus diunggah ke Blue (URL eksternal tidak didukung)

### Keterbatasan
- Maksimum 6.000 record per proyek untuk generasi sampul massal
- Hanya field kustom tipe file yang dapat digunakan sebagai sumber
- Gambar sampul hanya dapat dilihat dan tidak dapat diedit langsung

### Pembaruan Otomatis
Sampul secara otomatis regenerasi ketika:
- Deskripsi record diedit (jika menggunakan sumber Description)
- Komentar baru dengan gambar ditambahkan (jika menggunakan sumber Comments)  
- File dalam field kustom diperbarui (jika menggunakan sumber Custom Field)
- Pengaturan konfigurasi sampul diubah
- Opsi penyesuaian tampilan dimodifikasi

### Akses API
Gambar sampul terintegrasi penuh dengan GraphQL API:
- Konfigurasikan melalui mutasi `setTodoCoverConfig`
- Akses data sampul melalui field `cover` pada tipe Todo
- Pantau progres dengan langganan `subscribeToCoverGenerationProgress`
- URL thumbnail secara otomatis dihasilkan di `/uploads/{uid}/thumbnail/cover`
