Anda dapat mengimpor file CSV (Comma Separated Value). Jika Anda memiliki file Excel, [Anda dapat mengonversinya menjadi file CSV di tautan berikut](https://cloudconvert.com/xls-to-csv).

## Mengimpor CSV

Mengimpor file CSV di Blue adalah proses yang sederhana. Setiap kolom dalam file CSV Anda sesuai dengan field default atau [custom field](/docs/custom-fields/introduction) di Blue.

Untuk memulai, Anda harus menjadi Administrator Proyek dari proyek tempat Anda ingin mengimpor data. Klik pada ikon pengaturan proyek (ikon roda gigi) dan pilih "Impor".

![Impor CSV 1](/docs/CSVimport1.png "CSV Import 1")

Fitur Impor tersedia di bawah Pengaturan Proyek untuk Administrator Proyek.

Dalam tangkapan layar di atas, Anda juga dapat melihat opsi lain yang disebut "Unduh Template CSV". Ini sangat berguna jika Anda telah membuat [custom fields](/docs/custom-fields/introduction) di proyek Blue Anda karena Anda dapat mengunduh file CSV yang mencakup kolom untuk semua custom fields Anda! Ini sangat baik ketika Anda ingin memastikan bahwa Anda memiliki struktur yang tepat untuk CSV Anda sebelum mengimpor.

Selanjutnya, cukup seret dan lepas file CSV Anda ke dalam Blue:

![Impor CSV 2](/docs/CSVimport2.png "CSV Import 2")

Seret dan lepas untuk mengunggah data Anda.

Tergantung pada ukuran file Anda, ini mungkin memerlukan beberapa detik untuk Anda unggah dan untuk Blue menganalisis. Setelah Blue menganalisis data, Anda akan melihat pratinjau dari sepuluh baris pertama data Anda:

![Impor CSV 3](/docs/CSVimport3.png "CSV Import 3")

Anda kemudian dapat memetakan setiap kolom dalam CSV Anda ke field di Blue.

![Impor CSV 4](/docs/CSVimport4.png "CSV Import 4")

## Format Tanggal yang Diterima

Blue mendukung berbagai format tanggal untuk impor. Berikut adalah semua format yang didukung:

- **Timestamp Unix**:
  ```plaintext
  1511648546 (seconds)
  1620021848429 (milliseconds)
  1620024872717915000 (nanoseconds)
  ```
  
- **Standar RFC**:
  ```plaintext
  RFC 3339: 2021-05-01T01:17:02.604456Z
  RFC 2822: Wed, 02 Jun 2021 06:31:39 GMT
  ```

- **Format Database**:
  ```plaintext
  PostgreSQL: 
  2019-11-29 08:08-08
  2019-11-29 08:08:05-08
  2021-05-02 23:31:36.0741-07
  ```

- **Format Umum**:
  ```plaintext
  Year-Month-Day: 
  2021-04-30 21:14
  2021-02-21
  2021-Feb-21
  
  Month/Day/Year:
  3/31/2014 22:05
  04/02/2014 03:00:51
  
  Day-Month-Year:
  12 Feb 2006, 19:17
  03 February 2013
  ```

- **Zona Waktu**:
  ```plaintext
  2014-04-26 13:13:43 +0800
  2017-11-25 13:31 PST
  2021-02-21 UTC
  ```

- **Bahasa Alami**:
  ```plaintext
  May 6 at 9:24 PM
  September 17, 2012 at 10:09am PST
  October 7, 1970
  ```

- **Format Internasional**:
  ```plaintext
  Chinese: 2014年04月08日
  Japanese: 2014/04/08 22:05
  ```

- **Hanya Waktu**:
  ```plaintext
  01:06:06
  4:00pm PST
  6:00 AM UTC
  ```

Semua tanggal akan dikonversi ke zona waktu UTC selama impor. Komponen waktu bersifat opsional - jika tidak disediakan, waktu akan default ke 00:00:00 UTC.

## Menangani Koma dalam Data Anda

Jika data Anda mengandung koma (misalnya, dalam alamat atau nama perusahaan), Anda perlu membungkus nilai-nilai tersebut dalam tanda kutip ganda untuk mencegah kesalahan impor.

**Contoh:**
```
Company,Address
Acme Inc,123 Main St, New York
```

**Harus menjadi:**
```
Company,Address
"Acme Inc","123 Main St, New York"
```

Ini memastikan koma dalam "123 Main St, New York" diperlakukan sebagai bagian dari data daripada pemisah kolom.
