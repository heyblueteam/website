[Make.com](https://www.make.com) adalah platform integrasi yang membantu menghubungkan berbagai aplikasi web dan mengotomatiskan alur kerja. Blue **secara resmi terintegrasi** dengan Make.com, memungkinkan otomatisasi yang mulus di lebih dari 2.000 aplikasi—tanpa perlu coding!

![Blue Make.com Modules Overview](/docs/blue_make_modules_overview.png "Overview of available Blue modules in Make.com")

## Kasus Penggunaan

Berikut adalah beberapa cara kuat untuk memperluas kemampuan Blue dengan integrasi Make.com:
Integrasi Make.com Blue memungkinkan otomatisasi yang kuat di berbagai fungsi bisnis. Untuk pengalaman pelanggan, Anda dapat menghubungkan Blue dengan alat seperti Intercom, HubSpot, dan Zendesk untuk membuat catatan pelanggan yang komprehensif, menyinkronkan interaksi, dan mempertahankan komunikasi waktu nyata melalui Slack dan Microsoft Teams. Dalam manajemen proyek, Blue terintegrasi dengan Jira, GitHub, Trello, dan Monday.com untuk menyinkronkan tugas, melacak kemajuan, dan mengotomatiskan notifikasi.

Untuk pemasaran dan penjualan, Blue terhubung dengan Mailchimp, Stripe, dan Calendly untuk melacak kampanye, mengelola pembayaran, dan mengotomatiskan alur kerja pertemuan. Manajemen dokumen dipermudah melalui integrasi dengan Google Drive, Dropbox, OneDrive, dan SharePoint, sementara alat komunikasi seperti Slack, Gmail, dan WhatsApp disatukan melalui sistem manajemen tugas pusat Blue. Kemampuan analitik data ditingkatkan melalui koneksi dengan Google Sheets, Power BI, Airtable, dan Tableau, dan solusi spesifik industri tersedia untuk layanan keuangan, e-commerce, dan manajemen tim jarak jauh.

## Hubungkan & Autentikasi Akun Blue Anda

Untuk menghubungkan Blue dengan Make.com secara aman:

1. Masuk ke akun Blue Anda dan buka profil Anda → pergi ke tab **API**
2. Klik **Generate a Token**, berikan nama (dan masa berlaku opsional)
3. Salin keduanya:
   - **Token ID** (`x‑bloo‑token‑id`)
   - **Secret Key** (`x‑bloo‑token‑secret`)

::callout
 ---
icon: lightbulb
---
**Catatan Penting** <br/>
Kunci rahasia hanya ditampilkan sekali—simpan dengan aman!
::

5. Di Make.com, saat membuat koneksi baru, masukkan:
   - **Token ID**
   - **Secret Key**

## Referensi Parameter Modul

### Pantau Peristiwa
- Peristiwa (opsional): Jenis peristiwa multi-pilih
- ID Perusahaan (opsional): ID perusahaan atau slug multi-pilih
- ID Proyek (opsional): ID proyek atau slug multi-pilih

### Buat Catatan
- ID Perusahaan: ID perusahaan atau slug
- ID Proyek: ID proyek atau slug
- Daftar: ID daftar
- Nama Catatan: Nama catatan
- Bidang Default: Nilai default
- Tanggal Tunggal: Tanggal jatuh tempo
- Rentang Tanggal: Tanggal mulai dan jatuh tempo
- Penugasan: Pengguna multi-pilih
- Tag: Tag multi-pilih
- Deskripsi: Bidang teks, mendukung HTML
- Checklist: Array judul checklist

**Bidang Kustom yang Didukung:**
| Tipe Bidang | Deskripsi |
|-------------|-----------|
| Teks Satu Baris | Bidang teks satu baris dengan validasi dan batas karakter opsional |
| Teks Beberapa Baris | Bidang teks beberapa baris dengan dukungan teks kaya dan opsi pemformatan |
| Pilihan Tunggal | Dropdown dengan pemilihan satu opsi dan opsi kustom |
| Pilihan Ganda | Dropdown dengan pemilihan beberapa opsi dan opsi kustom |
| Lokasi | Garis lintang dan bujur dengan tampilan peta dan pencarian alamat |
| Kotak Centang | Toggle benar/salah dengan nilai default opsional dan label kustom |
| Negara | Pemilih kode negara ISO dua huruf dengan nama negara |
| Mata Uang | Bidang numerik dengan simbol mata uang dan dukungan nilai tukar (misalnya, USD200, 200USD, KHR4000) |
| Tanggal | Pemilih tanggal dengan komponen waktu opsional dan dukungan zona waktu |
| Telepon | Bidang nomor telepon yang mendukung format E.164, internasional, atau kode negara + nomor dengan validasi |
| Email | Bidang email dengan validasi format dan verifikasi domain |
| Peringkat Bintang | Peringkat numerik dari 1-5 dengan bintang visual dan label kustom |
| Persentase | Bidang numerik dengan simbol persentase (0-100) dan presisi desimal |
| URL | Bidang alamat web dengan validasi format dan pratinjau |
| Nomor | Bidang numerik dengan tempat desimal opsional dan tampilan unit |
| Formula | Bidang terhitung berdasarkan bidang lain dengan operasi matematika |
| ID Unik | Pengenal yang dihasilkan secara otomatis dengan awalan dan format kustom |
| Referensi | Tautan ke catatan lain dengan opsi pencarian dan filter |
| Lookup | Bidang dinamis yang menarik data dari catatan yang direferensikan dengan opsi pemetaan |
| Durasi Waktu | Rentang waktu dalam jam dan menit dengan format tampilan kustom |

### Perbarui Catatan
- ID Perusahaan: ID perusahaan atau slug
- ID Proyek: ID proyek atau slug
- Daftar: ID daftar
- ID Catatan: ID catatan yang akan diperbarui
- Nama Catatan: Nama catatan
- Bidang Default: Nilai default
- Tanggal Tunggal: Tanggal jatuh tempo
- Rentang Tanggal: Tanggal mulai dan jatuh tempo
- Penugasan: Pengguna multi-pilih
- Tag: Tag multi-pilih
- Deskripsi: Bidang teks, mendukung HTML
- Checklist: Array judul checklist

**Bidang Kustom yang Didukung:** (Sama dengan "Buat Catatan")

### Cari Catatan
- ID Perusahaan: ID perusahaan atau slug
- ID Proyek: ID proyek atau slug
- Daftar: ID daftar
- Pencarian/Kueri: Teks pencarian dengan opsi penyaringan lanjutan

### Ambil Catatan
- ID Perusahaan: ID perusahaan atau slug
- ID Proyek: ID proyek atau slug
- Daftar: ID daftar
- ID Catatan: ID catatan yang akan diambil dengan semua data terkait

### Undang Pengguna
- ID Perusahaan: ID perusahaan atau slug
- ID Proyek: ID proyek atau slug
- Alamat Email: Alamat email pengguna dengan validasi
- ID Peran: ID peran (default atau kustom) dengan izin

### Daftar Bidang Kustom
- ID Perusahaan: ID perusahaan atau slug
- ID Proyek: ID proyek atau slug
- Batas: Jumlah maksimum bidang yang akan dikembalikan dengan pagination

### Daftar Daftar
- ID Perusahaan: ID perusahaan atau slug
- ID Proyek: ID proyek atau slug
- Batas: Jumlah maksimum daftar yang akan dikembalikan dengan opsi pengurutan

### Daftar Semua Proyek
- ID Perusahaan: ID perusahaan atau slug multi-pilih
- Batas: Jumlah maksimum proyek yang akan dikembalikan dengan penyaringan

### Daftar Tag
- ID Perusahaan: ID perusahaan atau slug
- ID Proyek: ID proyek atau slug
- Batas: Jumlah maksimum tag yang akan dikembalikan dengan statistik penggunaan

### Eksekusi GraphQL
- Header: Array pasangan kunci-nilai untuk otentikasi dan header kustom
- Metode: GET atau POST dengan opsi permintaan
- Kuery: Kuery atau mutasi GraphQL dengan dukungan variabel
- Nama Operasi: Nama operasi untuk pelacakan
- Sumber Data Variabel: Mentah, Form, atau Koleksi dengan pemeriksaan tipe
- Variabel: Konten variabel GraphQL dengan validasi
