## Pendahuluan

White label mengubah Blue menjadi platform manajemen proyek dan proses yang fleksibel yang terlihat dan terasa sepenuhnya seperti produk Anda sendiri â€” tanpa beban mengelola teknologi, developer, server, atau uptime.

Dengan White Label Blue, Anda dapat:

- **Tampilkan Merek Anda:** Ganti nama, logo, dan warna Blue dengan milik Anda sendiri, menciptakan pengalaman merek yang mulus.
- **Jalankan di Domain Anda:** Operasikan Blue di domain web Anda sendiri untuk tampilan yang profesional dan kohesif.
- **Manfaatkan Infrastruktur Email Anda:** Kirim email dari alamat Anda untuk komunikasi yang dipersonalisasi.


Untuk agensi, add-on White Label memperkuat merek Anda, meningkatkan penawaran layanan, dan memperlancar kolaborasi klien. Dengan menanamkan Blue ke dalam alur kerja Anda di bawah merek Anda, Anda menciptakan pengalaman yang mulus dan profesional yang membantu Anda mendapatkan lebih banyak klien dan mempertahankan mereka dengan memberikan perpaduan unik antara layanan dan perangkat lunak yang disesuaikan dengan kebutuhan mereka.


## Harga

Add-on white-label dikenakan biaya $50/bulan per perusahaan atau $500/tahun per organisasi, di samping langganan Blue inti Anda. Anda dapat mempelajari lebih lanjut tentang harga [di sini](/pricing).

Jika Anda tertarik untuk membeli white label dalam jumlah besar untuk beberapa organisasi, silakan hubungi sales@blue.cc
 

## Domain Kustom

Agar White Label berfungsi, Anda perlu terlebih dahulu menyiapkan domain kustom untuk aplikasi web. 

Domain kustom dapat dikonfigurasi untuk hal-hal berikut:

- Aplikasi web (wajib)
- [Formulir](/docs/projects/forms) (opsional)
- [Tautan berbagi file](/docs/projects/file-management) (opsional)
- Dokumentasi (segera hadir)


Untuk mengonfigurasi domain kustom, tambahkan catatan CNAME ke pengaturan DNS Anda untuk setiap subdomain. Harap diperhatikan bahwa hanya subdomain yang didukung, bukan domain root. Jadi `app.yourcompany.com` didukung, tetapi `yourcompany.com` tidak.


| Jenis Domain               | Catatan CNAME                 | Mengarah ke                |
|----------------------------|-------------------------------|----------------------------|
| Aplikasi Web               | `app.yourcompany.com`         | `wl-app.onrender.com`      |
| Formulir                   | `forms.yourcompany.com`       | `wl-forms.onrender.com`    |
| Tautan Berbagi File        | `files.yourcompany.com`       | `wl-files.onrender.com`    |


Tabel di atas memberikan catatan CNAME yang disarankan, tetapi Anda dapat menggunakan subdomain apa pun yang Anda inginkan. 

Misalnya, aplikasi web utama bisa menjadi `projects.yourcompany.com` dan tautan berbagi file bisa menjadi `media.yourcompany.com`.


## Branding

Sesuaikan elemen-elemen berikut untuk menyelaraskan Blue dengan merek Anda:

- **Logo**: Unggah logo Anda untuk menggantikan branding Blue di seluruh platform.
- **Favicon**: Tambahkan favicon Anda sendiri, yang akan muncul di tab browser dan bookmark.
- **Warna Antarmuka Utama**: Tetapkan warna utama untuk antarmuka, diterapkan secara konsisten di semua pengguna.
- **Nama Aplikasi**: Sesuaikan nama aplikasi yang ditampilkan di tab browser dan di aplikasi desktop.
- **Font**: (Segera hadir) Sesuaikan font yang digunakan di seluruh platform agar sesuai dengan merek Anda dari daftar Google fonts yang dipilih.

Blue menyediakan pratinjau langsung halaman login dan ikon aplikasi saat Anda mengonfigurasi pengaturan branding Anda.

Branding Anda juga akan berlaku untuk [aplikasi desktop](/docs/start-guide/download-apps#desktop-app), memastikan pengalaman white-label yang lengkap.

Ini adalah konfigurasi yang direkomendasikan untuk logo dan favicon:

| Aset    | Bentuk yang Direkomendasikan | Latar Belakang  | Rasio Aspek |
|---------|------------------------------|-----------------|-------------|
| Logo    | Persegi panjang              | Transparan      | 1:1.5       |
| Favicon | Persegi                      | Transparan      | 1:1         |


## Notifikasi Email

Anda dapat menggunakan SMTP untuk mengirim notifikasi email dari server email Anda sendiri dengan domain Anda sendiri alih-alih alamat email kami.

Anda perlu memberikan informasi berikut ke Blue:

| Pengaturan                                  | Deskripsi                                                                                                                                           |
|---------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------|
| **Server SMTP**                             | Alamat server yang akan menangani pengiriman email Anda.                                                                                          |
| **Port SMTP**                               | Nomor port yang digunakan untuk terhubung ke server SMTP, biasanya 587 untuk TLS atau 465 untuk SSL.                                             |
| **Username**                                | Username untuk autentikasi dengan server SMTP Anda, biasanya alamat email Anda.                                                                   |
| **Password (disimpan dalam format terenkripsi)** | Password yang terkait dengan username, yang akan disimpan dengan aman dalam format terenkripsi untuk melindungi kredensial Anda.            |
| **Nama Pengirim**                           | Nama yang akan muncul sebagai pengirim dalam notifikasi email, membantu penerima mengidentifikasi sumber email.                                  |
| **Alamat Email Pengirim**                   | Alamat email yang akan ditampilkan sebagai pengirim, memastikan bahwa balasan dikirim ke alamat yang benar. Perhatikan bahwa ini harus domain yang sama dengan server SMTP. |

Blue memberlakukan penggunaan enkripsi TLS/SSL untuk koneksi SMTP.

Anda kemudian dapat menguji koneksi ke server SMTP Anda dengan mengklik tombol "Test Connection". Blue akan secara otomatis menguji koneksi ke server SMTP Anda dan memberi tahu Anda apakah berhasil atau tidak melalui notifikasi dalam aplikasi.

### Pertimbangan 

Semua email transaksional, notifikasi, dan otomasi akan dikirim melalui server SMTP Anda. Satu-satunya pengecualian adalah email yang berisi OTP (One Time Passwords) yang dikirim melalui alamat email generik (notifications@process-system.app) untuk memastikan keamanan.

Pastikan bahwa server SMTP Anda:

- Dapat menangani volume email yang tinggi.
- Memiliki reputasi pengiriman yang kuat untuk menghindari filter spam.
- Memiliki kuota yang cukup untuk mengakomodasi beban email yang diharapkan.


### Email Undangan Kustom

Anda juga dapat menyesuaikan email undangan awal yang dikirim ke pengguna baru saat mereka mendaftar untuk Blue Anda. 

### Email Onboarding Kustom (Segera Hadir)

Tanpa white labeling, Blue mengirimkan serangkaian email onboarding kepada pengguna baru saat mereka bergabung dengan perusahaan untuk membantu mereka memulai. Ketika white-label diaktifkan, email-email ini tidak dikirim, dan di masa depan Anda akan dapat membuat rangkaian email onboarding Anda sendiri untuk menyambut pengguna baru.

## FAQ

**Apa yang termasuk dalam add-on white label?**

Add-on white label mencakup tiga fitur utama: integrasi domain kustom untuk aplikasi web, formulir, dan berbagi file; branding kustom (logo, favicon, warna, dan nama aplikasi); dan notifikasi email kustom menggunakan server SMTP Anda sendiri.

**Berapa biaya white label?**

Add-on white label dikenakan biaya $50/bulan/perusahaan atau $500/tahun/perusahaan di atas langganan Blue inti Anda. Diskon untuk pembelian dalam jumlah besar akan tersedia di masa depan untuk akun dengan beberapa perusahaan.

**Apakah white-labeling mempengaruhi fungsionalitas Blue?**

Tidak, semua fitur dan fungsionalitas inti Blue tetap sama persis. White-labeling hanya mengubah tampilan dan branding platform.


**Bisakah saya menggunakan domain root alih-alih subdomain?**

Tidak, Blue hanya mendukung subdomain untuk pengaturan domain kustom.

**Apakah saya perlu menyiapkan semua domain kustom?**

Tidak, hanya domain aplikasi web yang wajib. Domain formulir dan berbagi file bersifat opsional. Dukungan domain dokumentasi akan segera hadir.

**Bisakah saya menggunakan nama subdomain yang berbeda dari yang disarankan?**

Ya, Anda dapat menggunakan nama subdomain apa pun yang Anda sukai. Misalnya, Anda bisa menggunakan `projects.yourcompany.com` alih-alih `app.yourcompany.com`.

**Elemen branding apa yang dapat saya sesuaikan?**

Anda dapat menyesuaikan logo, favicon, warna antarmuka utama, dan nama aplikasi Anda.

**Apakah aplikasi desktop akan mencerminkan branding saya?**

Ya, aplikasi desktop akan sepenuhnya white-labeled dengan branding Anda.

**Bisakah pengguna yang berbeda memiliki skema warna yang berbeda?**

Tidak, warna utama yang Anda pilih akan diberlakukan di semua pengguna Blue Anda.

**Detail konfigurasi email apa yang perlu saya berikan?**

Anda perlu memberikan alamat server SMTP, port SMTP, username, password, nama pengirim, dan alamat email pengirim. Password SMTP disimpan dalam format terenkripsi untuk melindungi kredensial Anda.

**Bisakah saya menguji konfigurasi SMTP sebelum go live?**

Ya, Blue menyediakan tombol "Test Connection" yang akan memverifikasi konfigurasi server SMTP Anda dan memberi tahu Anda hasilnya.

**Jenis email apa yang akan dikirim melalui server SMTP saya?**

Semua email akan dikirim melalui server SMTP Anda, termasuk email transaksional, email notifikasi, otomasi email. Email OTP (One-Time Password) dikirim melalui alamat email generik (notifications@process-system.app) untuk memastikan keamanan.


**Bisakah saya white-label aplikasi mobile?**

Blue saat ini tidak mendukung white-labeling aplikasi mobile. Namun, jika Anda memerlukan versi bermerek di Apple App Store atau Google Play Store, Anda dapat menghubungi sales@blue.cc untuk mendiskusikan opsi.

**Apakah dukungan white-label untuk aplikasi mobile akan tersedia di masa depan?**

Saat ini tidak ada rencana untuk mendukung white-labeling aplikasi mobile.

**Bagaimana saya bisa mendapatkan bantuan dengan pengaturan white-label?**

Untuk bantuan umum, Anda dapat menghubungi support@blue.cc. Untuk pertanyaan terkait penjualan tentang aplikasi mobile kustom, hubungi sales@blue.cc.

**Bisakah saya menguji fitur white-label sebelum menerapkannya?**

Ya, Blue menyediakan pratinjau halaman login dan ikon aplikasi saat Anda mengonfigurasi pengaturan branding Anda.

## Pengaturan DNS

Kami telah dengan mudah mengumpulkan instruksi tentang cara mengatur domain white label untuk sistem DNS (Domain Name Service) umum:

#### GoDaddy

1. Masuk ke akun GoDaddy Anda, dan buka produk Anda.
2. Navigasi ke Domain Control Center.
3. Pada halaman Domain Settings, temukan dan pilih nama domain Anda.
4. Gulir ke bawah ke bagian Additional Settings, dan pilih Manage DNS.
5. Pada halaman DNS Management, gulir ke bawah ke bagian bawah Records, dan klik Add.
6. Di menu drop-down Type, pilih CNAME, dan lengkapi field wajib berikut:
    - Host - Nama host atau prefix untuk CNAME. Anda dapat menggunakan titik (.), tetapi tidak sebagai karakter pertama atau terakhir. Titik berturut-turut (..) tidak diperbolehkan. Host tidak boleh melebihi 25 karakter atau menjadi simbol @.
    - Points to - Atur ini ke `wl-app.onrender.com`
    - TTL - Tentukan jumlah waktu server harus menyimpan cache informasi.
7. Klik Save.

#### Bluehost

1. Masuk ke akun BlueHost Anda.
2. Klik menu Domains kemudian klik Zone Editor. Halaman baru muncul.
3. Navigasi ke bagian Add DNS Record.
4. Di field Host Record, masukkan subdomain dari alamat yang ingin Anda gunakan untuk domain kustom Anda.
5. Di menu drop-down Type, pilih opsi CNAME.
6. Di bagian Points To, masukkan `wl-app.onrender.com`
7. Klik Add Record.

#### Cloudflare

1. Masuk ke akun Cloudflare Anda.
2. Di menu drop-down, cari dan pilih domain Anda.
3. Klik tab DNS Settings.
4. Klik "Add Record:
5. Pilih opsi CNAME.
6. Atur "Name" ke subdomain Anda
7. Atur "Target" ke `wl-app.onrender.com`
8. Atur ikon Cloudflare Proxy Toggle ke awan abu-abu.

#### Domain.com

1. Masuk ke akun Domain.com Anda.
2. Cari domain Anda dan klik Manage.
3. Di menu di sebelah kiri, klik DNS and Nameservers.
4. Klik Add DNS record untuk menambahkan catatan CNAME.
5. Atur target ke `wl-app.onrender.com`
6. Klik Update DNS untuk menyelesaikan proses.

#### Dreamhost

1. Masuk ke akun DreamHost Anda.
2. Di sebelah kiri halaman, klik tautan Domains.
3. Di menu drop-down Domains, pilih Manage Domains.
4. Cari domain Anda dan klik tautan DNS di bawah domain Anda.
5. Gulir ke bawah ke bagian Add a custom DNS record to (your custom domain) dan edit field berikut:
    - Name - ketik subdomain yang ingin Anda gunakan
    - Type - pilih opsi CNAME.
    - Value - masukkan `wl-app.onrender.com`
6. Klik Add Record Now! untuk menambahkan catatan CNAME.

#### Google Domains

1. Masuk ke akun Google Domains Anda.
2. Temukan dan klik nama domain Anda.
3. Di menu di sebelah kiri, klik DNS.
4. Navigasi ke bagian Custom Resource Records.
5. Di field Name, masukkan subdomain dari alamat yang ingin Anda gunakan untuk domain kustom Anda.
6. Di menu drop-down Type, pilih opsi CNAME.
7. Di field Data, masukkan `wl-app.onrender.com`
8. Klik Add untuk menyelesaikan proses.

#### Hostgator

1. Masuk ke akun HostGator Anda.
2. Di sisi kiri halaman, klik CPanel.
3. Gulir ke bawah ke bagian Domains, dan klik Zone Editor.
4. Temukan domain Anda dan klik tautan CName Record di sebelah domain.
5. Di kotak dialog Add a CName Record, lengkapi field kosong berikut:
    - Name - masukkan subdomain/nama domain pilihan Anda.
    - CNAME - masukkan `wl-app.onrender.com`
6. Klik Add a CName Record. Pesan dari HostGator muncul, menyatakan bahwa Anda telah berhasil menambahkan CNAME baru ke akun HostGator Anda.

#### NameCheap

1. Masuk ke akun NameCheap Anda.
2. Di menu drop-down Account, pilih Dashboard.
3. Klik Domain List dan temukan domain Anda.
4. Klik Manage di sebelah domain Anda.
5. Klik Advanced DNS dan navigasi ke bagian Host Records.
6. Klik Add New Record.
7. Di menu drop-down Type, pilih CNAME Record dan masukkan subdomain Anda sebagai host.
8. Kemudian masukkan `wl-app.onrender.com`
9. Klik Save all changes untuk menyelesaikan proses.
