Możesz importować pliki CSV (Comma Separated Value). Jeśli masz plik Excel, [możesz go przekonwertować na plik CSV pod tym linkiem](https://cloudconvert.com/xls-to-csv).

## Importowanie CSV

Importowanie plików CSV w Blue jest prostym procesem. Każda kolumna w twoim pliku CSV odpowiada domyślnemu lub [niestandardowemu polu](/docs/custom-fields/introduction) w Blue.

Aby rozpocząć, musisz być Administratorem Projektu w projekcie, do którego chcesz importować dane. Kliknij ikonę ustawień projektu (ikona zębatki) i wybierz "Importuj".

![CSV Import 1](/docs/CSVimport1.png "CSV Import 1")

Funkcja importu jest dostępna w ustawieniach projektu dla Administratorów Projektu.

Na zrzucie ekranu powyżej możesz również zobaczyć inną opcję o nazwie "Pobierz szablon CSV". Jest to niezwykle przydatne, jeśli już utworzyłeś swoje [niestandardowe pola](/docs/custom-fields/introduction) w swoim projekcie Blue, ponieważ możesz pobrać plik CSV, który zawiera kolumny dla wszystkich twoich niestandardowych pól! To świetne, gdy chcesz upewnić się, że masz dokładną strukturę dla swojego CSV przed importem.

Następnie po prostu przeciągnij i upuść swój plik CSV do Blue:

![CSV Import 2](/docs/CSVimport2.png "CSV Import 2")

Przeciągnij i upuść, aby przesłać swoje dane.

W zależności od rozmiaru pliku, może to zająć kilka sekund, aby przesłać i aby Blue przeanalizował dane. Gdy Blue przeanalizuje dane, zobaczysz podgląd pierwszych dziesięciu wierszy swoich danych:

![CSV Import 3](/docs/CSVimport3.png "CSV Import 3")

Następnie możesz przypisać każdą kolumnę w swoim CSV do pola w Blue.

![CSV Import 4](/docs/CSVimport4.png "CSV Import 4")

## Akceptowane formaty dat

Blue obsługuje szeroki wachlarz formatów dat do importu. Oto wszystkie obsługiwane formaty:

- **Znaczniki czasu Unix**:
  ```plaintext
  1511648546 (seconds)
  1620021848429 (milliseconds)
  1620024872717915000 (nanoseconds)
  ```
  
- **Standardy RFC**:
  ```plaintext
  RFC 3339: 2021-05-01T01:17:02.604456Z
  RFC 2822: Wed, 02 Jun 2021 06:31:39 GMT
  ```

- **Formaty baz danych**:
  ```plaintext
  PostgreSQL: 
  2019-11-29 08:08-08
  2019-11-29 08:08:05-08
  2021-05-02 23:31:36.0741-07
  ```

- **Powszechne formaty**:
  ```plaintext
  Year-Month-Day: 
  2021-04-30 21:14
  2021-02-21
  2021-Feb-21
  
  Month/Day/Year:
  3/31/2014 22:05
  04/02/2014 03:00:51
  
  Day-Month-Year:
  12 Feb 2006, 19:17
  03 February 2013
  ```

- **Strefy czasowe**:
  ```plaintext
  2014-04-26 13:13:43 +0800
  2017-11-25 13:31 PST
  2021-02-21 UTC
  ```

- **Język naturalny**:
  ```plaintext
  May 6 at 9:24 PM
  September 17, 2012 at 10:09am PST
  October 7, 1970
  ```

- **Formaty międzynarodowe**:
  ```plaintext
  Chinese: 2014年04月08日
  Japanese: 2014/04/08 22:05
  ```

- **Tylko czas**:
  ```plaintext
  01:06:06
  4:00pm PST
  6:00 AM UTC
  ```

Wszystkie daty zostaną przekształcone na strefę czasową UTC podczas importu. Składniki czasu są opcjonalne - jeśli nie są podane, czas domyślnie wynosi 00:00:00 UTC.

## Obsługa przecinków w danych

Jeśli twoje dane zawierają przecinki (na przykład w adresach lub nazwach firm), musisz owinąć te wartości w podwójne cudzysłowy, aby zapobiec błędom importu.

**Przykład:**
```
Company,Address
Acme Inc,123 Main St, New York
```

**Powinno stać się:**
```
Company,Address
"Acme Inc","123 Main St, New York"
```

To zapewnia, że przecinek w "123 Main St, Nowy Jork" jest traktowany jako część danych, a nie jako separator kolumn.
