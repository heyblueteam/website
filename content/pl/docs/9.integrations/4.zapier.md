Zapier to platforma integracyjna, która pomaga łączyć różne aplikacje internetowe i automatyzować przepływy pracy.

**Przykłady użycia**

Oto niektóre z najpopularniejszych sposobów, w jakie Blue jest zintegrowany z innymi aplikacjami za pośrednictwem Zapier:

**_Synchronizuj rekordy Blue z Google Sheets_**

Gdy rekordy Blue są tworzone lub aktualizowane, automatycznie dodawaj je jako nowe wiersze w arkuszu Google do centralnego raportowania.

**_Wyślij potwierdzenie zamówienia z rekordów Blue_**

Gdy wybierzesz określone pole niestandardowe w Blue, automatycznie wyślij e-mail, aby potwierdzić zamówienie z klientem.

**_Dodaj komentarze Blue do Slack_**

Otrzymuj powiadomienia Blue w czasie rzeczywistym publikowane w odpowiednich kanałach Slack, aby Twój zespół był świadomy ważnych aktualizacji rekordów.

## Kluczowe korzyści

**Automatyzacja w różnych narzędziach**: Zaps utworzone w Zapier mogą automatycznie wysyłać i odbierać dane między Blue a tysiącami innych aplikacji. Eliminuje to ręczną pracę przy przenoszeniu danych między systemami.

**Integracje w czasie rzeczywistym**: Korzystając z wyzwalaczy webhooków Blue, Zaps reagują natychmiast na zdarzenia w Blue, takie jak nowy rekord, dodany komentarz, zmiana statusu itp. Umożliwia to synchronizację w czasie rzeczywistym i automatyzację procesów.

**Łatwość budowy i utrzymania**: Zaps zapewniają przyjazny dla użytkownika sposób integracji narzędzi bez kodowania. Nie jest potrzebna praca deweloperska. I działają bez potrzeby ciągłej konserwacji.

## Blue do Zapier

1. Zaloguj się na swoje konto Zapier
2. Kliknij Utwórz Zap  
3. W pierwszym kroku wyszukaj i wybierz aplikację wyzwalającą ["Webhooks by Zapier"](https://zapier.com/apps/webhook)
4. Skonfiguruj ją, aby akceptowała ładunki z adresu URL webhooka Blue
5. W drugim kroku wyszukaj i wybierz aplikację docelową, do której chcesz wysłać dane
6. Mapuj pola ładunku JSON do pól wejściowych aplikacji docelowej
7. Włącz Zap

Teraz, gdy wystąpią określone zdarzenia w Blue, ładunek webhooka zostanie wysłany do Zapier, który automatycznie przekaże dane do zintegrowanej aplikacji zgodnie z ustawieniami Zapa.

## Zapier do Blue

Aby przesłać dane z innych aplikacji do Blue, wykonaj następujące kroki:

**1. Uzyskaj dane uwierzytelniające API Blue**
- Przejdź do Ustawienia Blue > Tokeny API
- Utwórz nowy token lub użyj istniejących danych:
  - ID tokena
  - Sekret tokena
  - ID firmy
  - ID projektu

**2. Utwórz Zap w Zapier**
1. Skonfiguruj swoją aplikację wyzwalającą (np. Google Sheets, Slack itp.)
2. Dodaj "Webhooks by Zapier" jako aplikację akcji
3. Wybierz typ zdarzenia "Custom Request"

**3. Skonfiguruj Webhook**
- Metoda: POST
- URL: `https://api.blue.cc/graphql` (lub punkt końcowy beta, jeśli to konieczne)
- Przekazywanie danych: FAŁSZ
- Rozwiń: NIE

**4. Ustaw nagłówki:**
```json
{
  "Content-Type": "application/json",
  "X-Bloo-Token-ID": "Your_Token_ID",
  "X-Bloo-Token-Secret": "Your_Token_Secret", 
  "X-Bloo-Company-ID": "Your_Company_ID",
  "X-Bloo-Project-ID": "Your_Project_ID"
}
```

**5. Skonfiguruj zapytanie (Przykład mutacji):**
```json
{
  "query": "mutation CreateTodo($input: CreateTodoInput!) {\n  createTodo(input: $input) {\n    id\n    title\n    customFields {\n      value\n    }\n  }\n}",
  "variables": {
    "input": {
      "todoListId": "your-todo-list-id",
      "title": "New task from Zapier",
      "customFields": [
        {
          "customFieldId": "priority-field-id",
          "value": "High"
        },
        {
          "customFieldId": "due-date-field-id",
          "value": "2024-03-01"
        }
      ]
    }
  }
}
```

**6. Testuj i aktywuj**
- Użyj API Playground Blue, aby najpierw przetestować mutacje
- Mapuj dynamiczne wartości z aplikacji wyzwalających, korzystając z pól danych Zapier
- Włącz Zap po pomyślnym teście
