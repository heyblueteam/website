## Przegląd

Tagi to potężny sposób na kategoryzowanie i organizowanie rekordów w Blue. System tagów zapewnia kompleksową funkcjonalność do tworzenia, zarządzania i filtrowania tagów, z obsługą operacji masowych i zaawansowanego filtrowania.

## Lista Tagów

Pobierz tagi z projektów z elastycznymi opcjami filtrowania i sortowania.

### Podstawowy Przykład

```graphql
query ListTags {
  tagList(
    filter: { 
      projectIds: ["project-123"] 
      excludeArchivedProjects: false 
    }
    first: 50
    orderBy: title_ASC
  ) {
    items {
      id
      uid
      title
      color
      project {
        id
        name
      }
      createdAt
      updatedAt
    }
    pageInfo {
      totalPages
      totalItems
      page
      perPage
      hasNextPage
      hasPreviousPage
    }
    totalCount
  }
}
```

### Przykład Zaawansowanego Filtrowania

```graphql
query FilterTags {
  tagList(
    filter: {
      projectIds: ["project-123", "project-456"]
      search: "urgent"
      colors: ["#ff0000", "#ffaa00"]
      excludeArchivedProjects: true
    }
    orderBy: createdAt_DESC
    first: 20
  ) {
    items {
      id
      title
      color
      todos {
        id
        title
      }
    }
    totalCount
  }
}
```

## Tworzenie Tagów

Twórz nowe tagi w ramach projektu.

```graphql
mutation CreateTag {
  createTag(
    input: {
      title: "High Priority"
      color: "#ff0000"
    }
  ) {
    id
    uid
    title
    color
    project {
      id
      name
    }
    createdAt
  }
}
```

## Aktualizacja Tagów

Modyfikuj istniejące właściwości tagów.

```graphql
mutation UpdateTag {
  editTag(
    input: {
      id: "tag-123"
      title: "Critical Priority"
      color: "#cc0000"
    }
  ) {
    id
    title
    color
    updatedAt
  }
}
```

## Usuwanie Tagów

Usuń tagi z systemu. Spowoduje to również usunięcie tagu ze wszystkich powiązanych rekordów.

```graphql
mutation DeleteTag {
  deleteTag(id: "tag-123")
}
```

## Tagowanie Rekordów

Powiąż tagi z rekordami (zadaniami) za pomocą istniejących tagów lub stwórz nowe.

### Używanie Istniejących Tagów

```graphql
mutation TagRecord {
  setTodoTags(
    input: {
      todoId: "todo-123"
      tagIds: ["tag-456", "tag-789"]
    }
  )
}
```

### Tworzenie Tagów Podczas Tagowania

Możesz tworzyć nowe tagi w locie, podając tytuły tagów:

```graphql
mutation TagRecordWithNewTags {
  setTodoTags(
    input: {
      todoId: "todo-123"
      tagIds: ["existing-tag-id"]
      tagTitles: ["New Tag", "Another New Tag"]
    }
  )
}
```

## Sugestie Tagów AI

Uzyskaj sugestie tagów napędzane przez AI dla rekordu:

```graphql
mutation GetTagSuggestions {
  aiTag(
    input: {
      todoId: "todo-123"
      suggestionsCount: 5
    }
  ) {
    suggestions {
      title
      confidence
      color
    }
  }
}
```

## Parametry Wejściowe

### TagListFilter

| Parametr | Typ | Wymagany | Opis |
|-----------|------|----------|-------------|
| `projectIds` | [String!] | Nie | Filtrowanie tagów według identyfikatorów projektów lub slugów |  |
| `excludeArchivedProjects` | Boolean | Nie | Wyklucz tagi z archiwalnych projektów |  |
| `search` | String | Nie | Wyszukiwanie tytułów tagów (niezależnie od wielkości liter) |  |
| `titles` | [String!] | Nie | Filtrowanie według konkretnych tytułów tagów |  |
| `colors` | [String!] | Nie | Filtrowanie według konkretnych kolorów (format hex) |  |
| `tagIds` | [String!] | Nie | Filtrowanie według konkretnych identyfikatorów tagów |  |

### CreateTagInput

| Parametr | Typ | Wymagany | Opis |
|-----------|------|----------|-------------|
| `title` | String | Nie | Tytuł tagu (automatycznie generowany, jeśli nie podano) |  |
| `color` | String! | ✅ Tak | Kolor tagu w formacie hex (np. "#ff0000") |  |

### EditTagInput

| Parametr | Typ | Wymagany | Opis |
|-----------|------|----------|-------------|
| `id` | String! | ✅ Tak | ID tagu do aktualizacji |  |
| `title` | String | Nie | Nowy tytuł tagu |  |
| `color` | String | Nie | Nowy kolor tagu w formacie hex |  |

### SetTodoTagsInput

| Parametr | Typ | Wymagany | Opis |
|-----------|------|----------|-------------|
| `todoId` | String! | ✅ Tak | ID rekordu do oznaczenia |  |
| `tagIds` | [String!] | Nie | ID istniejących tagów do zastosowania |  |
| `tagTitles` | [String!] | Nie | Tytuły nowych tagów do stworzenia i zastosowania |  |

### TagOrderByInput Values

| Wartość | Opis |
|-------|-------------|
| `id_ASC` / `id_DESC` | Sortuj według ID tagu |  |
| `uid_ASC` / `uid_DESC` | Sortuj według unikalnego identyfikatora |  |
| `title_ASC` / `title_DESC` | Sortuj alfabetycznie według tytułu |  |
| `color_ASC` / `color_DESC` | Sortuj według wartości koloru |  |
| `createdAt_ASC` / `createdAt_DESC` | Sortuj według daty utworzenia |  |
| `updatedAt_ASC` / `updatedAt_DESC` | Sortuj według daty ostatniej aktualizacji |  |

## Pola Odpowiedzi

### Typ Tagu

| Pole | Typ | Opis |
|-------|------|-------------|
| `id` | ID! | Unikalny identyfikator tagu |  |
| `uid` | String! | Przyjazny użytkownikowi unikalny identyfikator |  |
| `title` | String! | Tytuł/nazwa tagu |  |
| `color` | String! | Kolor tagu w formacie hex |  |
| `project` | Project! | Projekt, do którego należy ten tag |  |
| `todos` | [Todo!]! | Rekordy powiązane z tym tagiem |  |
| `createdAt` | DateTime! | Kiedy tag został utworzony |  |
| `updatedAt` | DateTime! | Kiedy tag został ostatnio zaktualizowany |  |

### Odpowiedź TagPagination

| Pole | Typ | Opis |
|-------|------|-------------|
| `items` | [Tag!]! | Tablica rekordów tagów |  |
| `pageInfo` | PageInfo! | Informacje o paginacji |  |
| `totalCount` | Int! | Całkowita liczba tagów pasujących do filtrów |  |

### Pola PageInfo

| Pole | Typ | Opis |
|-------|------|-------------|
| `totalPages` | Int | Całkowita liczba stron |  |
| `totalItems` | Int | Całkowita liczba elementów we wszystkich stronach |  |
| `page` | Int | Numer bieżącej strony |  |
| `perPage` | Int | Liczba elementów na stronie |  |
| `hasNextPage` | Boolean! | Czy jest następna strona |  |
| `hasPreviousPage` | Boolean! | Czy jest poprzednia strona |  |

## Wymagane Uprawnienia

### Zarządzanie Tagami

| Akcja | Wymagana Rola |
|--------|--------------|
| Create tags | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |
| Edit tags | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |
| Delete tags | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |
| Apply tags to records | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |

### Widoczność Tagów

Tagi są widoczne dla wszystkich użytkowników, którzy mają dostęp do projektu, w którym tag został utworzony. Filtrowanie oparte na rolach może mieć zastosowanie w przypadku niektórych operacji.

## Najlepsze Praktyki

1. **Używaj Spójnych Kolorów**: Ustal system kodowania kolorów dla swojego zespołu
2. **Opisowe Tytuły**: Używaj jasnych, opisowych nazw tagów, które będą zrozumiałe dla członków zespołu
3. **Operacje Masowe**: Używaj `setTodoTags` do zastosowania wielu tagów jednocześnie
4. **Wyszukiwanie i Filtrowanie**: Używaj funkcji wyszukiwania, aby znaleźć istniejące tagi przed utworzeniem nowych
5. **Zarządzanie Tagami**: Regularnie przeglądaj i porządkuj nieużywane tagi
6. **Konwencje Kolorów**: Rozważ użycie standardowych kolorów dla typowych tagów (czerwony dla pilnych, zielony dla ukończonych itd.)

## Zaawansowane Funkcje

### Domyślne Ustawienia Tworzenia Tagów

- Nowe tagi tworzone za pomocą `setTodoTags` z `tagTitles` domyślnie mają kolor `#4a9fff`
- Tagi bez określonych kolorów przyjmują kolor `#00a0d2`
- Tytuły tagów są automatycznie przycinane z białych znaków

### Integracja Automatyzacji

Tagi mogą być:
- Automatycznie stosowane przez zasady automatyzacji
- Używane jako wyzwalacze dla przepływów pracy automatyzacji
- Modyfikowane przez działania automatyzacji

### Aktualizacje w Czasie Rzeczywistym

Zmiany tagów są transmitowane za pomocą subskrypcji GraphQL, co pozwala na aktualizacje w czasie rzeczywistym w połączonych klientach.

## Ważne Uwagi

- Usunięcie tagu usuwa go ze wszystkich powiązanych rekordów
- Tagi są ograniczone do projektów i nie mogą być dzielone między projektami
- Wartości kolorów muszą być poprawnymi kodami hex (np. "#ff0000", "#00aa00")
- Tytuły tagów są wrażliwe na wielkość liter
- Maksymalnie 100 tagów może być zastosowanych do jednego rekordu
- Tagi wspierają pełnotekstowe wyszukiwanie w tytułach
