Zapier គឺជាវេទិកាដែលជួយភ្ជាប់កម្មវិធីវេបផ្សេងៗ និងស្វ័យប្រវត្តិកម្មវិធីការងារ។

**ករណីប្រើប្រាស់**

នេះគឺជាវិធីដែលពេញនិយមបំផុតដែល Blue ត្រូវបានភ្ជាប់ជាមួយកម្មវិធីផ្សេងទៀតតាមរយៈ Zapier:

**_សម្របសម្រួលកំណត់ត្រា Blue ទៅ Google Sheets_**

នៅពេលដែលកំណត់ត្រា Blue ត្រូវបានបង្កើត ឬអាប់ដេត អ្នកអាចបន្ថែមវាជួរថ្មីនៅក្នុង Google Sheet សម្រាប់ការរាយការណ៍មជ្ឈមណ្ឌល។

**_ផ្ញើការបញ្ជាក់ការបញ្ជាទិញពីកំណត់ត្រា Blue_**

នៅពេលដែលអ្នកជ្រើសវាលកំណត់ខ្លះនៅលើ Blue អ្នកអាចផ្ញើអ៊ីមែលដើម្បីបញ្ជាក់ការបញ្ជាទិញជាមួយអតិថិជនរបស់អ្នកដោយស្វ័យប្រវត្តិ។

**_បន្ថែមមតិយោបល់ Blue ទៅ Slack_**

ទទួលបានការជូនដំណឹង Blue នៅពេលវេលាពិតដែលបានបង្ហោះទៅឆានែល Slack ដែលត្រឹមត្រូវ ដើម្បីឱ្យក្រុមរបស់អ្នកដឹងពីការអាប់ដេតកំណត់ត្រាសំខាន់ៗ។

## អត្ថប្រយោជន៍សំខាន់

**ស្វ័យប្រវត្តិកម្មនៅក្នុងឧបករណ៍**: Zaps ដែលបានបង្កើតនៅក្នុង Zapier អាចផ្ញើ និងទទួលទិន្នន័យដោយស្វ័យប្រវត្តិរវាង Blue និងកម្មវិធីផ្សេងទៀតរាប់ពាន់។ នេះបង្កើនការងារដោយដៃក្នុងការផ្លាស់ប្តូរទិន្នន័យរវាងប្រព័ន្ធ។

**ការភ្ជាប់នៅពេលវេលាពិត**: ដោយប្រើប្រាស់ការជូនដំណឹង webhook របស់ Blue Zaps នឹងឆ្លើយតបភ្លាមៗទៅនឹងព្រឹត្តិការណ៍នៅក្នុង Blue ដូចជាកំណត់ត្រាថ្មី មតិយោបល់ដែលបានបន្ថែម ការផ្លាស់ប្តូរស្ថានភាព និងអ្វីៗផ្សេងទៀត។ នេះអនុញ្ញាតឱ្យមានការសម្របសម្រួលនៅពេលវេលាពិត និងស្វ័យប្រវត្តិកម្មដំណើរការ។

**ងាយស្រួលក្នុងការបង្កើត និងថែរក្សា**: Zaps ផ្តល់នូវវិធីដែលមិនចាំបាច់កូដដើម្បីភ្ជាប់ឧបករណ៍។ មិនចាំបាច់ការងារអភិវឌ្ឍន៍ទេ។ ហើយវានឹងបន្តធ្វើការជាមួយការថែរក្សាដែលមិនមាន។

4. កំណត់វាឱ្យទទួល payloads ពី URL webhook របស់ Blue របស់អ្នក
5. សម្រាប់ជំហានទីពីរ សូមស្វែងរក និងជ្រើសកម្មវិធីគោលដៅដើម្បីផ្ញើទិន្នន័យទៅ
6. គូសវាល JSON payload ទៅវាលបញ្ចូលនៃកម្មវិធីគោលដៅ
7. បើក Zap

## Zapier ទៅ Blue

ដើម្បីផ្ញើទិន្នន័យពីកម្មវិធីផ្សេងៗទៅ Blue សូមអនុវត្តន៍ជំហានទាំងនេះ:

**1. ទទួលបានសញ្ញាប័ត្រ Blue API**
- ទៅកាន់ការកំណត់ Blue > API Tokens
- បង្កើតសញ្ញាប័ត្រថ្មី ឬប្រើសញ្ញាប័ត្រដែលមានស្រាប់:
  - Token ID
  - Token Secret
  - Company ID
  - Project ID

**2. បង្កើត Zap នៅក្នុង Zapier**
1. កំណត់កម្មវិធីជំរុញរបស់អ្នក (ឧ. Google Sheets, Slack, ល។)
2. បន្ថែម "Webhooks by Zapier" ជាកម្មវិធីសកម្ម
3. ជ្រើសប្រភេទព្រឹត្តិការណ៍ "Custom Request"

**3. កំណត់ Webhook**
- វិធីសាស្ត្រ: POST
- URL: `https://api.blue.cc/graphql` (ឬ beta endpoint ប្រសិនបើចាំបាច់)
- Data Pass-Through: FALSE
- Unflatten: NO

**4. កំណត់ Headers:**
```json
{
  "Content-Type": "application/json",
  "X-Bloo-Token-ID": "Your_Token_ID",
  "X-Bloo-Token-Secret": "Your_Token_Secret", 
  "X-Bloo-Company-ID": "Your_Company_ID",
  "X-Bloo-Project-ID": "Your_Project_ID"
}
```

**5. កំណត់ Query (ឧទាហរណ៍ Mutation):**
```json
{
  "query": "mutation CreateTodo($input: CreateTodoInput!) {\n  createTodo(input: $input) {\n    id\n    title\n    customFields {\n      value\n    }\n  }\n}",
  "variables": {
    "input": {
      "todoListId": "your-todo-list-id",
      "title": "New task from Zapier",
      "customFields": [
        {
          "customFieldId": "priority-field-id",
          "value": "High"
        },
        {
          "customFieldId": "due-date-field-id",
          "value": "2024-03-01"
        }
      ]
    }
  }
}
```

**6. សាកល្បង និងបើកใช้งาน**
- ប្រើប្រាស់ API Playground របស់ Blue ដើម្បីសាកល្បង mutations មុន
- គូសតម្លៃដំណាមពីកម្មវិធីជំរុញដោយប្រើវាលទិន្នន័យរបស់ Zapier
- បើក Zap បន្ទាប់ពីការសាកល្បងជោគជ័យ
