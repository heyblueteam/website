អ្នកអាចនាំចូលឯកសារ CSV (Comma Separated Value)។ ប្រសិនបើអ្នកមានឯកសារ Excel, [អ្នកអាចបំលែងវាទៅជាឯកសារ CSV នៅលីងដូចខាងក្រោម](https://cloudconvert.com/xls-to-csv)។

## ការនាំចូល CSV

ការនាំចូលឯកសារ CSV នៅក្នុង Blue គឺជាដំណើរការងាយស្រួល។ សូម្បីតែជួរដេកមួយក្នុងឯកសារ CSV របស់អ្នកត្រូវនឹងជួរដេកលំនាំឬ [ជួរដេកផ្ទាល់ខ្លួន](/docs/custom-fields/introduction) នៅក្នុង Blue។

ដើម្បីចាប់ផ្តើម អ្នកត្រូវតែជាអ្នកគ្រប់គ្រងគម្រោងនៃគម្រោងដែលអ្នកចង់នាំចូលទិន្នន័យ។ ចុចលើរូបតំណាងកំណត់រចនាសម្ព័ន្ធគម្រោង (រូបតំណាងកង់) ហើយជ្រើស "នាំចូល"

![CSV Import 1](/docs/CSVimport1.png "CSV Import 1")

លក្ខណៈនាំចូលមានស្រាប់នៅក្រោមការកំណត់គម្រោងសម្រាប់អ្នកគ្រប់គ្រងគម្រោង

នៅក្នុងរូបថតខាងលើ អ្នកក៏អាចមើលឃើញជម្រើសផ្សេងទៀតដែលហៅថា "ទាញយកគំរូ CSV"។ នេះមានប្រយោជន៍ខ្លាំងប្រសិនបើអ្នកបានបង្កើត [ជួរដេកផ្ទាល់ខ្លួន](/docs/custom-fields/introduction) របស់អ្នកនៅក្នុងគម្រោង Blue របស់អ្នក ពីព្រោះអ្នកអាចទាញយកឯកសារ CSV ដែលមានជួរដេកសម្រាប់ជួរដេកផ្ទាល់ខ្លួនទាំងអស់របស់អ្នក! នេះគឺល្អពេលដែលអ្នកចង់ធានាថាអ្នកមានរចនាសម្ព័ន្ធត្រឹមត្រូវសម្រាប់ CSV របស់អ្នកមុនពេលនាំចូល។

បន្ទាប់មក គ្រាន់តែទាញនិងទម្លាក់ឯកសារ CSV របស់អ្នកចូលទៅក្នុង Blue:

![CSV Import 2](/docs/CSVimport2.png "CSV Import 2")

ទាញនិងទម្លាក់ដើម្បីបញ្ចូលទិន្នន័យរបស់អ្នក

អាស្រ័យលើទំហំឯកសាររបស់អ្នក វាអាចចំណាយពេលប៉ុន្មានវិនាទីសម្រាប់អ្នកដើម្បីបញ្ចូល និងសម្រាប់ Blue ដើម្បីវិភាគ។ បន្ទាប់ពី Blue បានវិភាគទិន្នន័យ អ្នកនឹងមើលឃើញការមើលឃើញនៃជួរដេកដំបូងដប់ជួរដេករបស់អ្នក:

![CSV Import 3](/docs/CSVimport3.png "CSV Import 3")

អ្នកអាចបញ្ជាក់ជួរដេកនីមួយៗនៅក្នុង CSV របស់អ្នកទៅជាជួរដេកនៅក្នុង Blue។

## ទ្រង់ទ្រាយថ្ងៃដែលគាំទ្រ

Blue គាំទ្រទ្រង់ទ្រាយថ្ងៃជាច្រើនសម្រាប់នាំចូល។ នេះគឺជាទ្រង់ទ្រាយដែលគាំទ្រទាំងអស់:

- **Unix Timestamps**:
  ```plaintext
  1511648546 (seconds)
  1620021848429 (milliseconds)
  1620024872717915000 (nanoseconds)
  ```
  
- **RFC Standards**:
  ```plaintext
  RFC 3339: 2021-05-01T01:17:02.604456Z
  RFC 2822: Wed, 02 Jun 2021 06:31:39 GMT
  ```

- **Database Formats**:
  ```plaintext
  PostgreSQL: 
  2019-11-29 08:08-08
  2019-11-29 08:08:05-08
  2021-05-02 23:31:36.0741-07
  ```

- **Common Formats**:
  ```plaintext
  Year-Month-Day: 
  2021-04-30 21:14
  2021-02-21
  2021-Feb-21
  
  Month/Day/Year:
  3/31/2014 22:05
  04/02/2014 03:00:51
  
  Day-Month-Year:
  12 Feb 2006, 19:17
  03 February 2013
  ```

- **Time Zones**:
  ```plaintext
  2014-04-26 13:13:43 +0800
  2017-11-25 13:31 PST
  2021-02-21 UTC
  ```

- **Natural Language**:
  ```plaintext
  May 6 at 9:24 PM
  September 17, 2012 at 10:09am PST
  October 7, 1970
  ```

- **International Formats**:
  ```plaintext
  Chinese: 2014年04月08日
  Japanese: 2014/04/08 22:05
  ```

- **Time Only**:
  ```plaintext
  01:06:06
  4:00pm PST
  6:00 AM UTC
  ```

ថ្ងៃទាំងអស់នឹងត្រូវបានបំលែងទៅជាអត្រាពេល UTC ក្នុងអំឡុងពេលនាំចូល។ ធាតុពេលវេលាជាជម្រើស - ប្រសិនបើមិនបានផ្តល់ វាពេលវេលានឹងលំនាំទៅ 00:00:00 UTC។

## ការដោះស្រាយកម្រងក្នុងទិន្នន័យរបស់អ្នក

ប្រសិនបើទិន្នន័យរបស់អ្នកមានកម្រង (ឧ. នៅក្នុងអាសយដ្ឋាន ឬឈ្មោះក្រុមហ៊ុន) អ្នកត្រូវតែដាក់តម្លៃទាំងនោះក្នុងសញ្ញាដូចគ្នាដើម្បីចៀសវាងកំហុសនាំចូល។

**ឧទាហរណ៍:**
```
Company,Address
Acme Inc,123 Main St, New York
```

**គួរតែប្តូរទៅជា:**
```
Company,Address
"Acme Inc","123 Main St, New York"
```

នេះធានាថាកម្រងនៅក្នុង "123 Main St, New York" ត្រូវបានចាត់ទុកថាជាផ្នែកមួយនៃទិន្នន័យមិនមែនជាការបំបែកជួរដេកទេ។
