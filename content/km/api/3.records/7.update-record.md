## កែប្រែព័ត៌មានកំណត់ត្រា

ដើម្បីកែប្រែលក្ខណៈសំខាន់ៗនៃកំណត់ត្រា សូមប្រើ `editTodo` mutation:

```graphql
mutation UpdateRecordDetails {
  editTodo(
    input: {
      todoId: "YOUR RECORD ID"
      todoListId: "RECORD LIST ID TO MOVE THE RECORD TO"
      position: "NEW RECORD POSITION IN NUMBER"
      title: "NEW RECORD TITLE"
      html: "NEW RECORD DESCRIPTION IN HTML (MUST MATCH TEXT)"
      text: "NEW RECORD DESCRIPTION IN TEXT (MUST MATCH HTML)"
      startedAt: "NEW RECORD DUE DATE (START)"
      duedAt: "NEW RECORD DUE DATE (END)"
      color: "RECORD COLOR CODE"
    }
  ) {
    id
    title
    position
    html
    text
    color
  }
}
```

### ឯកសារវាលបញ្ចូល
| វាល | ប្រភេទ | ព័ត៌មានពិពណ៌នា |
|-------|------|-------------|
| todoId | String | (ត្រូវការពិត) អត្តសញ្ញាណនៃកំណត់ត្រាដែលត្រូវកែប្រែ |  |
| todoListId | String | អត្តសញ្ញាណបញ្ជីថ្មីប្រសិនបើបញ្ជីកំណត់ត្រាត្រូវបានផ្លាស់ទី |  |
| position | Float | ទីតាំងថ្មីក្នុងបញ្ជី |  |
| title | String | ចំណងជើងកំណត់ត្រាដែលបានកែប្រែ |  |
| html/text | String | ព័ត៌មានពិពណ៌នាដែលបានកែប្រែ (ត្រូវតែស្របគ្នានៅក្នុងទាំងពីរវាល) |  |
| startedAt/duedAt | DateTime | ថ្ងៃខែឆ្នាំចាប់ផ្តើម/បញ្ចប់ដែលបានកែប្រែក្នុងទ្រង់ទ្រាយ ISO 8601 |  |
| color | String | កូដពណ៌ពីជម្រើសដែលមាន |  |

## កែប្រែវាលផ្ទាល់ខ្លួន
ដើម្បីកែប្រែតម្លៃវាលផ្ទាល់ខ្លួន សូមប្រើ `setTodoCustomField` mutation ជាមួយប៉ារ៉ាម៉ែត្រដែលមានលក្ខណៈពិសេស:

### វាលផ្អែកលើអត្ថបទ
```graphql
mutation {
  setTodoCustomField(
    input: {
      customFieldId: "YOUR CUSTOM FIELD ID"
      todoId: "YOUR RECORD ID"
      text: "VALUE"
    }
  )
}
```
អនុវត្តទៅ: `TEXT_SINGLE`, `TEXT_MULTI`, `URL`, `EMAIL`

### វាលលេខ
```graphql
mutation {
  setTodoCustomField(
    input: {
      customFieldId: "YOUR CUSTOM FIELD ID"
      todoId: "YOUR RECORD ID" 
      number: "NUMERIC_VALUE"
    }
  )
}
```
អនុវត្តទៅ: `NUMBER`, `CURRENCY`, `PERCENT`, `RATING`

### វាលជ្រើស
```graphql
mutation {
  setTodoCustomField(
    input: {
      customFieldId: "YOUR CUSTOM FIELD ID"
      todoId: "YOUR RECORD ID"
      customFieldOptionIds: ["OPTION_ID_1", "OPTION_ID_2"]
    }
  )
}
```
អនុវត្តទៅ: `SELECT_SINGLE`, `SELECT_MULTI`

### វាលពិសេស
**លេខទូរស័ព្ទ:**
```graphql
mutation {
  setTodoCustomField(
    input: {
      customFieldId: "YOUR CUSTOM FIELD ID"
      todoId: "YOUR RECORD ID"
      text: "+33642526644"
      regionCode: "FR"
    }
  )
}
```

**ប្រទេស:**
```graphql
mutation {
  setTodoCustomField(
    input: {
      customFieldId: "YOUR CUSTOM FIELD ID"
      todoId: "YOUR RECORD ID"
      countryCodes: ["AF", "AL", "DZ"]
      text: "Afghanistan, Albania, Algeria"
    }
  )
}
```

**ទីតាំង:**
```graphql
mutation {
  setTodoCustomField(
    input: {
      customFieldId: "YOUR CUSTOM FIELD ID"
      todoId: "YOUR RECORD ID"
      latitude: 42.2923323
      longitude: 12.126621199999999
      text: "Via Cassia, Querce d'Orlando, Capranica, Italy"
    }
  )
}
```

**ប្រអប់ត្រួតពិនិត្យ:**
```graphql
mutation {
  setTodoCustomField(
    input: {
      customFieldId: "YOUR CUSTOM FIELD ID"
      todoId: "YOUR RECORD ID"
      checked: true
    }
  )
}
```

## អាជ្ញាប័ណ្ណដែលត្រូវការ

អ្នកប្រើត្រូវតែមានការចូលដំណើរការប្រព័ន្ធគម្រោងដែលសមស្របដើម្បីកែប្រែកំណត់ត្រា:

| កម្រិតចូល | អាចកែប្រែកំណត់ត្រា |
|--------------|-------------------|
| `OWNER` | ✅ បាទ |  |
| `ADMIN` | ✅ បាទ |  |
| `MEMBER` | ✅ បាទ |  |
| `CLIENT` | ✅ បាទ |  |
| `COMMENT_ONLY` | ❌ ទេ |  |
| `VIEW_ONLY` | ❌ ទេ |  |

អាជ្ញាប័ណ្ណវាលផ្ទាល់ខ្លួនបន្ថែមអាចត្រូវអនុវត្តសម្រាប់ `setTodoCustomField` ដោយផ្អែកលើការកំណត់តួនាទី។

## តម្លៃត្រឡប់

- `editTodo` ត្រឡប់មកវិញនូវវត្ថុ `Todo` ដែលបានកែប្រែពេញលេញ
- `setTodoCustomField` ត្រឡប់មកវិញ `Boolean!` ដែលបញ្ជាក់ពីភាពជោគជ័យ

### កំណត់ចំណាំ
1. អត្តសញ្ញាណវាលផ្ទាល់ខ្លួនអាចរកឃើញបានដោយប្រើការស្នើសុំ [បញ្ជីវាលផ្ទាល់ខ្លួន](/api/custom-fields/list-custom-fields)
2. លេខទូរស័ព្ទត្រូវតែមានក្នុងទ្រង់ទ្រាយ E.164 នៅពេលប្រើ API ដោយផ្ទាល់
3. វាលទីតាំងគឺល្អបំផុតក្នុងការគ្រប់គ្រងតាមរយៈអន្តរកម្មកម្មវិធី Blue
4. វាល `html` និង `text` ត្រូវបានសមស្របដោយស្វ័យប្រវត្តិនៅពេល `html` ត្រូវបានផ្តល់
5. ប្រតិបត្តិការកែប្រែទាំងអស់នឹងបង្កើតកំណត់ហេតុសកម្មភាព និងការជូនដំណឹង webhook
