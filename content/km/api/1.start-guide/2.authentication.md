## ក្បាល

មុនពេលអនុវត្តសំណើណាមួយ វាជារឿងសំខាន់ក្នុងការកំណត់ក្បាលសម្រាប់សំណើ API របស់អ្នក។ ក្បាលទាំងនេះនៅស្ថិតស្ថេរនៅលើសំណើទាំងអស់ ដូច្នេះត្រូវការការកំណត់មួយដងប៉ុណ្ណោះ។ វាប្រើសម្រាប់ផ្ទៀងផ្ទាត់សំណើនីមួយៗទៅកាន់ API។

ក្បាលដែលត្រូវការមាន Token-ID និង Secret-ID ដែលតំណាងដោយ `x-bloo-token-id` និង `x-bloo-token-secret` តាមលំដាប់។

ក្រៅពីនេះ សំណើជាក់លាក់ខ្លះអាចត្រូវការការបញ្ចូល Company-ID `x-bloo-company-id` និង Project-ID `x-bloo-project-id` សម្រាប់ប្រតិបត្តិការដែលជាក់លាក់ជាងនេះ។

អ្នកអាចសិក្សាអំពីរបៀបបង្កើតកូដ និងសម្ងាត់នៅក្នុងវីដេអូខាងក្រោម៖

<youtube url="https://www.youtube.com/watch?v=C-q_AqdFUzE" />

## Token ID
Token ID គឺជាអត្តសញ្ញាណដ៏ឯកត្តសាស្ត្ររបស់អ្នកសម្រាប់ការចូលប្រើ API។

ដើម្បីចាប់ផ្តើមប្រើ API របស់យើង សូមចុចលើប្រវត្តិរូបរបស់អ្នកនៅខាងលើស្តាំ ហើយនៅក្រោមម៉ឺនុយប្រវត្តិរូប អ្នកនឹងឃើញផ្ទាប API។

![API 2](/docs/API_2.png "API 2")

នៅក្នុងផ្ទាប API សូមចុចលើប៊ូតុង "បង្កើតកូដ": 

![API 3](/docs/API_3.png "API 3")

ផ្តល់ឈ្មោះសម្រាប់កូដ និងកំណត់កាលបរិច្ឆេទផុតកំណត់ប្រសិនបើចង់។ ប្រសិនបើអ្នកកំណត់កាលបរិច្ឆេទផុតកំណត់ កូដ API នឹងឈប់ធ្វើការបន្ទាប់ពីកាលបរិច្ឆេទនោះ។

![API 4](/docs/API_4.png "API 4")

បន្ទាប់ពីបង្កើត អ្នកនឹងត្រូវបានបង្ហាញ Token ID និង Secret ID។ Token ID នឹងត្រូវបានបន្ថែមជាមុនដោយ `pat_` សម្រាប់ការកំណត់អត្តសញ្ញាណយ៉ាងងាយស្រួល។

## Secret ID

Secret ID មានតួនាទីដូចជាពាក្យសម្ងាត់ ដើម្បីធានាថាការទំនាក់ទំនងជាមួយ API របស់ Blue មានសុវត្ថិភាព។

ចូរចងចាំថា Secret ID ត្រូវបានបង្ហាញតែម្ដងតែប៉ុណ្ណោះសម្រាប់ហេតុផលសុវត្ថិភាព ដូច្នេះសូមរក្សាទុកវាឲ្យមានសុវត្ថិភាពដើម្បីរក្សាភាពចូលដំណើរការទិន្នន័យរបស់អ្នកតាមរយៈ API។ សម្រាប់សុវត្ថិភាពអតិបរិមា Blue រក្សាសម្ងាត់របស់អ្នកដោយប្រើ bcrypt hashing ដែលមានន័យថា ទាំងក្រុមរបស់យើងក៏មិនអាចទាញយកសម្ងាត់អត្ថបទធម្មតារបស់អ្នកបានបន្ទាប់ពីការបង្កើត។

## Company & Project IDs

នេះគឺជាវីដេអូដែលបង្ហាញពីរបៀបរកឃើញ Company ID និង Project ID របស់អ្នកនៅក្នុង Blue៖

<youtube url="https://www.youtube.com/watch?v=zLEvs6zqGTc" />

ក្រុមហ៊ុនគឺជាអង្គភាពកម្រិតខ្ពស់នៅក្នុង Blue។ Company IDs ត្រូវបានប្រើសម្រាប់កំណត់អត្តសញ្ញាណក្រុមហ៊ុននៅក្នុង Blue។ វាត្រូវបានប្រើនៅក្នុង API ដើម្បីកំណត់ថាតើត្រូវអន្តរកម្មជាមួយក្រុមហ៊ុនណា។ ID គឺដូចគ្នាជាមួយ slug ក្រុមហ៊ុននៅក្នុង URL នៅពេលដែលអ្នកចូលទៅកាន់ក្រុមហ៊ុននៅក្នុង Blue៖

```
app.blue.cc/company/{company-id}/
```

សម្រាប់សំណួរ និងការប្រែប្រួលជាក់លាក់ខ្លះ អ្នកនឹងត្រូវការផ្តល់ Project ID។ នេះគឺជាអត្តសញ្ញាណនៃគម្រោងដែលអ្នកចង់អន្តរកម្មជាមួយ។ ID គឺដូចគ្នាជាមួយ slug គម្រោងនៅក្នុង URL នៅពេលដែលអ្នកចូលទៅកាន់គម្រោងនៅក្នុង Blue៖

```
app.blue.cc/company/{company-id}/project/{project-id}/
```
