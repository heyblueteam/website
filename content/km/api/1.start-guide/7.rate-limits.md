នៅទូទៅ, Blue API មិនបានអនុវត្តកំណត់អត្រាដែលរឹងមាំលើប្រតិបត្តិការជាច្រើនទេ។ ទោះយ៉ាងណា, ប្រតិបត្តិការមួយចំនួនដែលមានអារម្មណ៍អាសន្នមានកំណត់អត្រាពិសេសដើម្បីការពារការប្រើប្រាស់អាក្រក់ និងធានាស្ថិរភាពរបស់ប្រព័ន្ធ។

## ប្រតិបត្តិការ​ដែលមាន​កំណត់​អត្រា

ប្រតិបត្តិការទាំងអស់នេះមានកំណត់អត្រាដែលបានអនុវត្ត:

| ប្រតិបត្តិការ | កំណត់អត្រា | វិនាទី | គោលបំណង |
|-----------|------------|---------|---------|
| `signIn` | 5 requests | 60 seconds | ការពារប្រយុទ្ធប្រឆាំងនឹងការប្រើប្រាស់អាក្រក់ |  |
| `signInRequest` | 3 requests | 120 seconds | កំណត់ការប្រឡងសម្គាល់ |  |
| `createDocument` | 5 requests | 60 seconds | ការពារការប្រយុទ្ធប្រឆាំងនឹងអ៊ីមែលស្ដើង |  |
| `sendTestEmail` | 5 requests | 60 seconds | ការពារការប្រើប្រាស់អ៊ីមែលអាក្រក់ |  |
| `submitForm` | 5 requests | 60 seconds | ការពារការប្រយុទ្ធប្រឆាំងនឹងសំណុំបែបបទស្ដើង |  |
| `exportTodos` | 1 request | 50 seconds | កំណត់ការនាំចេញដែលប្រើប្រាស់ធនធានច្រើន |  |
| `deleteCompany` | 3 requests | 60 seconds | ការពារការលុបចោលដោយអកុសល |  |
| `deleteCompanyRequest` | 3 requests | 60 seconds | ការពារការលុបចោលដោយអកុសល |  |
| `updateEmail` | 3 requests | 60 seconds | ការពារការប្រើប្រាស់អ៊ីមែលអាក្រក់ |  |
| `updateEmailRequest` | 3 requests | 60 seconds | ការពារការប្រើប្រាស់អ៊ីមែលអាក្រក់ |  |
| `verifyAcceptInvitation` | 3 requests | 60 seconds | កំណត់ការប្រឡងសម្គាល់ |  |
| `verifySecurityCode` | 3 requests | 60 seconds | កំណត់ការប្រឡងសម្គាល់ |  |

## អាការៈកំណត់អត្រា

- **តាមអ្នកប្រើ**: កំណត់អត្រាត្រូវបានអនុវត្តតាមអ្នកប្រើដែលបានសម្គាល់
- **តាម IP**: សម្រាប់សំណើដែលមិនមានការសម្គាល់, កំណត់ត្រូវបានអនុវត្តតាមអាសយដ្ឋាន IP
- **គ្មានក្បាល**: ព័ត៌មានអំពីកំណត់អត្រាមិនត្រូវបានបញ្ចូលក្នុងក្បាលចម្លើយ

## ចម្លើយកំហុស

ពេលដែលកំណត់អត្រាត្រូវបានលើស, អ្នកនឹងទទួលបានកំហុស GraphQL:

```json
{
  "errors": [{
    "message": "Rate limit exceeded",
    "extensions": {
      "code": "RATE_LIMITED"
    }
  }]
}
```

## អនុសាសន៍ល្អ

1. **ដោះស្រាយកំហុសកំណត់អត្រា**: អនុវត្តការដោះស្រាយកំហុសយ៉ាងត្រឹមត្រូវសម្រាប់ប្រតិបត្តិការដែលមានកំណត់អត្រា
2. **ការប្រឡងត្រឡប់ក្រោយ**: ប្រើការប្រឡងត្រឡប់ក្រោយពេលដែលកំពុងព្យាយាមសំណើដែលមានកំណត់អត្រា
3. **តាមដានការប្រើប្រាស់**: សូមប្រុងប្រយ័ត្នអំពីប្រតិបត្តិការណាដែលមានកំណត់នៅពេលកំពុងកសាងការបញ្ចូល

សម្រាប់សំណួរអំពីកំណត់អត្រា, សូម [ទំនាក់ទំនងក្រុមគាំទ្ររបស់យើង](mailto:support@blue.cc)។
