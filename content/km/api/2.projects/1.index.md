## ទិដ្ឋភាពទូទៅ

គម្រោងគឺជាឯកតាស្ថាបនាដែលសំខាន់នៅក្នុង Blue។ ពួកវាមានបញ្ជី, ការងារ, វាលផ្ទាល់ខ្លួន, ការប្រព្រឹត្តសកម្មភាព, និងទិន្នន័យដែលទាក់ទងនឹងការងារផ្សេងទៀត។ គម្រោង属于ក្រុមហ៊ុន និងមានប្រព័ន្ធអនុញ្ញាត, គំរូ, និងការកំណត់រចនាសម្ព័ន្ធផ្ទាល់ខ្លួន។

## គំនិតសំខាន់

### រចនាសម្ព័ន្ធគម្រោង
- គម្រោង属于ក្រុមហ៊ុន
- គម្រោងមួយអាចមានបញ្ជីច្រើន
- បញ្ជីមានការងារ
- គម្រោងគាំទ្រវាលផ្ទាល់ខ្លួន, ស្លាក, និងការប្រព្រឹត្តសកម្មភាព

### ម៉ូដែលអនុញ្ញាត
គម្រោងមានប្រព័ន្ធអនុញ្ញាតជាច្រើនកម្រិត:

| កម្រិត | អនុញ្ញាត |
|-------|-------------|
| **OWNER** | Full control, can delete project |
| **ADMIN** | Manage project settings, users, and content |
| **MEMBER** | Create and edit content |
| **CLIENT** | Limited edit access |
| **VIEW_ONLY** | Read-only access |
| **COMMENT_ONLY** | Can only comment |

### ប្រភេទគម្រោង
គម្រោងអាចត្រូវបានចាត់ថ្នាក់សម្រាប់ការរៀបចំល្អប្រសើរ:
- CRM
- CROSS_FUNCTIONAL
- CUSTOMER_SUCCESS
- DESIGN
- ENGINEERING
- GENERAL (លំនាំដើម)
- HR
- IT
- MARKETING
- OPERATIONS
- PERSONAL
- PROCUREMENT
- PRODUCT
- SALES

## លំនាំទូទៅ

### បង្កើតគម្រោងមូលដ្ឋាន
```graphql
mutation CreateProject {
  createProject(input: {
    name: "Q1 Marketing Campaign"
    companyId: "company-123"
    category: MARKETING
  }) {
    id
    name
    slug
  }
}
```

### ស្វែងរកគម្រោងជាមួយច្រោះ
```graphql
query GetProjects {
  projectList(
    filter: {
      companyIds: ["company-123"]
      isArchived: false
      categories: [MARKETING, SALES]
    }
    sort: [{ field: updatedAt, direction: DESC }]
    take: 20
  ) {
    items {
      id
      name
      category
      todosCount
      todosDoneCount
    }
    pageInfo {
      hasNextPage
      total
    }
  }
}
```

> **កំណត់ចំណាំ**: ការស្វែងរក `projectList` គឺជាវិធីសាស្ត្រដែលបានណែនាំសម្រាប់ការស្វែងរកគម្រោង។ ការស្វែងរកមួយ `projects` មានស្រាប់ប៉ុន្តែគួរត្រូវបានប្រើសម្រាប់ការអនុវត្តថ្មីៗទេ។

### គ្រប់គ្រងបញ្ជីគម្រោង
```graphql
# Get all lists in a project
query GetProjectLists {
  todoLists(projectId: "project-123") {
    id
    title
    position
    todosCount
  }
}

# Create a new list
mutation CreateList {
  createTodoList(input: {
    projectId: "project-123"
    title: "To Do"
    position: 1.0
  }) {
    id
    title
  }
}
```

## អនុវត្តន៍ល្អបំផុត

1. **ការបោះឈ្មោះគម្រោង**
   - ប្រើឈ្មោះដែលច្បាស់ និងពិពណ៌នា
   - ជៀសវាងតួអក្សរពិសេសដែលអាចប៉ះពាល់ដល់ slug
   - រក្សាឈ្មោះឱ្យនៅក្រោម 50 តួអក្សរ

2. **ការគ្រប់គ្រងអនុញ្ញាត**
   - ចាប់ផ្តើមជាមួយអនុញ្ញាតតិចតួច
   - ប្រើតួនាទី CLIENT សម្រាប់អ្នកចូលរួមក្រៅ
   - ពិនិត្យសិទ្ធិចូលប្រើគម្រោងជាប្រចាំ

3. **ការរៀបចំ**
   - ប្រើប្រភេទដើម្បីក្រុមគម្រោងដែលស្រដៀងគ្នា
   - ការបញ្ជូនគម្រោងដែលបានបញ្ចប់ជំនួសការលុប
   - ប្រើគំរូសម្រាប់ប្រភេទគម្រោងដែលធ្វើឡើងវិញ

4. **សមត្ថភាព**
   - ប្រើការបែងចែកសម្រាប់បញ្ជីគម្រោងធំ
   - ច្រោះតាមស្ថានភាពសកម្ម/បានបញ្ជូន
   - កំណត់ចំនួនបញ្ជីក្នុងគម្រោង (អតិបរិមា 50)

## ការដោះស្រាយកំហុស

កំហុសទូទៅដែលអ្នកអាចប្រឈមមុខ:

| លេខកូដកំហុស | ការពិពណ៌នា | ដំណោះស្រាយ |
|------------|-------------|----------|
| `PROJECT_NOT_FOUND` | គម្រោងមិនមានឬគ្មានការចូលប្រើ | Verify project ID and permissions |
| `COMPANY_NOT_FOUND` | ក្រុមហ៊ុនមិនមាន | Check company ID |
| `FORBIDDEN` | អនុញ្ញាតមិនគ្រប់គ្រាន់ | Ensure proper role level |
| `BAD_USER_INPUT` | កំហុសក្នុងការផ្ទៀងផ្ទាត់ (ឧ. ឈ្មោះយ៉ាងហោចណាស់) | Check input validation requirements |
