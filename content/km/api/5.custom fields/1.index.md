## ទិដ្ឋភាពទូទៅ

វាលកំណត់ផ្ទាល់ខ្លួនអនុញ្ញាតឱ្យអ្នកពង្រីករចនាសម្ព័ន្ធកំណត់ត្រាមាតុភូមិរបស់ Blue ជាមួយវាលទិន្នន័យបន្ថែមដែលសមស្របនឹងតម្រូវការអាជីវកម្មរបស់អ្នក។ វាផ្តល់នូវវិធីដ៏មានអំណាចក្នុងការចាប់យកទិន្នន័យដែលមានរចនាសម្ព័ន្ធលើសពីវាលដែលមាននៅក្នុងដូចជា ចំណងជើង ការពិពណ៌នា និងកាលបរិច្ឆេទផុតកំណត់។

វាលកំណត់ផ្ទាល់ខ្លួនត្រូវបានកំណត់នៅកម្រិតគម្រោង ហើយអាចប្រើបាននៅក្នុងកំណត់ត្រាទាំងអស់ (ការងារ) នៅក្នុងគម្រោងនោះ។ វាលនីមួយៗមានប្រភេទជាក់លាក់ដែលកំណត់ច្បាប់ការត្រួតពិនិត្យ ការបញ្ចូលទ្រង់ទ្រាយ និងអាកប្បកិរិយាការបង្ហាញរបស់វា។

## គំនិតសំខាន់ៗ

### ការកំណត់វាល
- វាលកំណត់ផ្ទាល់ខ្លួនត្រូវបានកំណត់នៅកម្រិតគម្រោង
- វាលនីមួយៗមានឈ្មោះ និងប្រភេទឯកត្តភាព
- វាលអាចរួមបញ្ចូលច្បាប់ការត្រួតពិនិត្យ និងកំណត់
- ជម្រើសកំណត់ខុសគ្នាតាមប្រភេទវាល

### តម្លៃវាល
- តម្លៃត្រូវបានរក្សាទុកនៅលើកំណត់ត្រាទីមួយ (ការងារ)
- កំណត់ត្រានីមួយៗអាចមានតម្លៃខុសគ្នាសម្រាប់វាលដូចគ្នា
- តម្លៃទទេ/សូន្យត្រូវបានអនុញ្ញាតសម្រាប់វាលដែលជាជម្រើស
- តម្លៃត្រូវបានត្រួតពិនិត្យតាមច្បាប់ប្រភេទវាល

### គំរូសិទ្ធិ
វាលកំណត់ផ្ទាល់ខ្លួនគោរពសិទ្ធិនៅកម្រិតគម្រោង:

| តួនាទី | បង្កើតវាល | កែសម្រួលវាល* | កំណត់តម្លៃ | មើលតម្លៃ |
|------|---------------|-------------|------------|-------------|
| **OWNER** | ✅ បាទ | ✅ បាទ | ✅ បាទ | ✅ បាទ |
| **ADMIN** | ✅ បាទ | ✅ បាទ | ✅ បាទ | ✅ បាទ |
| **MEMBER** | ❌ ទេ | ❌ ទេ | ✅ បាទ | ✅ បាទ |
| **CLIENT** | ❌ ទេ | ❌ ទេ | ✅ កំណត់ | ✅ កំណត់ |

* កែសម្រួលវាលមានន័យថាកែប្រែការកំណត់វាល (ឈ្មោះ ប្រភេទ ជម្រើស ច្បាប់ការត្រួតពិនិត្យ) - មិនមែនកំណត់តម្លៃវាលនៅលើកំណត់ត្រាទេ

### សិទ្ធិតួនាទីផ្ទាល់ខ្លួន
- គម្រោងអាចមានតួនាទីផ្ទាល់ខ្លួនដែលមានសិទ្ធិពាក់ព័ន្ធនឹងវាល
- វាលអាចត្រូវបានសម្គាល់ថាជាអាចកែប្រែ/មិនអាចកែប្រែតាមតួនាទី
- តួនាទីផ្ទាល់ខ្លួនអាចកំណត់ការចូលដំណើរការទៅកាន់វាលជាក់លាក់

## លំនាំទូទៅ

### បង្កើតវាលកំណត់ផ្ទាល់ខ្លួនមូលដ្ឋាន
```graphql
mutation CreateCustomField {
  createCustomField(input: {
    name: "Priority Level"
    type: SELECT_SINGLE
    description: "Task priority level"
    customFieldOptions: [
      { title: "Low", color: "#28a745" }
      { title: "Medium", color: "#ffc107" }
      { title: "High", color: "#fd7e14" }
      { title: "Critical", color: "#dc3545" }
    ]
  }) {
    id
    name
    type
    customFieldOptions {
      id
      title
      color
    }
  }
}
```

### កំណត់តម្លៃវាលនៅលើកំណត់ត្រា
```graphql
mutation SetFieldValue {
  setTodoCustomField(input: {
    todoId: "todo_123"
    customFieldId: "field_456"
    customFieldOptionId: "option_789"  # For SELECT_SINGLE
  })
}
```

### ស្វែងរកកំណត់ត្រាជាមួយវាលកំណត់ផ្ទាល់ខ្លួន
```graphql
query GetTodosWithCustomFields {
  todos(projectId: "project_123") {
    id
    title
    customFields {
      id
      customField {
        name
        type
      }
      # Type-specific value fields
      text           # TEXT_SINGLE, TEXT_MULTI, EMAIL, etc.
      number         # NUMBER, CURRENCY, PERCENT, RATING
      selectedOption # SELECT_SINGLE
      selectedOptions # SELECT_MULTI
      checked        # CHECKBOX
      date           # DATE
    }
  }
}
```

### បង្កើតកំណត់ត្រាជាមួយតម្លៃវាលកំណត់ផ្ទាល់ខ្លួន
```graphql
mutation CreateTodoWithCustomFields {
  createTodo(input: {
    title: "New task"
    todoListId: "list_123"
    customFields: [
      { customFieldId: "priority_field", value: "high_priority_option" }
      { customFieldId: "budget_field", value: "5000" }
      { customFieldId: "notes_field", value: "Additional context here" }
    ]
  }) {
    id
    title
    customFields {
      customField { name }
      value
    }
  }
}
```

1. **ប្រើឈ្មោះដែលពិពណ៌នាដែលច្បាស់** - ធ្វើឱ្យគោលបំណងវាលច្បាស់
2. **ជ្រើសប្រភេទដែលសមស្រប** - ត្រូវតាមតម្រូវការទិន្នន័យ
3. **កំណត់ច្បាប់ការត្រួតពិនិត្យ** - ប្រើតម្លៃអប្បបរមា/អតិបរមា វាលដែលត្រូវការនៅក្នុង
4. **រៀបចំដោយមានលក្ខណៈត្រឹមត្រូវ** - រៀបចំវាលដែលពាក់ព័ន្ធគ្នា

### ការពិចារណាអំពីសមត្ថភាព
1. **កំណត់ចំនួនវាល** - វាលច្រើនពេកអាចប៉ះពាល់ដល់សមត្ថភាព
2. **ប្រើការបែងចែកទំព័រ** - នៅពេលស្វែងរកទិន្នន័យធំ
3. **បង្កើនវាលសំខាន់** - សម្រាប់វាលដែលប្រើក្នុងការច្រោះ/តម្រៀប
4. **ចAvoid deep nesting** - រក្សាទំនាក់ទំនងវាលឱ្យសាមញ្ញ

### គុណភាពទិន្នន័យ
1. **ត្រួតពិនិត្យការបញ្ចូល** - ប្រើប្រភេទវាលដែលសមស្របជាមួយការត្រួតពិនិត្យ
2. **ផ្តល់អត្រា** - កំណត់តម្លៃអត្រាដែលមានសមត្ថភាពនៅពេលដែលសមស្រប
3. **ប្រើទ្រង់ទ្រាយដែលស្របគ្នា** - ស្តង់ដារប្រព័ន្ធបញ្ចូលទិន្នន័យ
4. **ការសម្អាតជាប្រចាំ** - លុបវាល និងជម្រើសដែលមិនបានប្រើ

### បទពិសោធន៍អ្នកប្រើ
1. **ការពិពណ៌នាដែលច្បាស់** - ផ្តល់ការពិពណ៌នាវាលដែលមានប្រយោជន៍
2. **ការបញ្ជាក់លំដាប់** - ដាក់វាលនៅក្នុងលំដាប់ធម្មជាតិ
3. **លំដាប់វិជ្ជមាន** - ប្រើពណ៌ និងការបង្ហាញយ៉ាងមានប្រសិទ្ធភាព
4. **ការបង្ហាញជាបន្តបន្ទាប់** - បង្ហាញវាលនៅពេលដែលមានសារៈសំខាន់

## ការដោះស្រាយកំហុស

កំហុសទូទៅនៅពេលធ្វើការជាមួយវាលកំណត់ផ្ទាល់ខ្លួន:

| កូដកំហុស | ការពិពណ៌នា | ដំណោះស្រាយ |
|------------|-------------|----------|
| `CUSTOM_FIELD_NOT_FOUND` | វាលមិនមាន | Verify field ID and project access |
| `VALIDATION_ERROR` | តម្លៃមិនសមស្របនឹងប្រភេទវាល | Check format and validation rules |
| `UNAUTHORIZED` | សិទ្ធិមិនគ្រប់គ្រាន់ | Ensure proper role level |
| `CUSTOM_FIELD_VALUE_PARSE_ERROR` | ទ្រង់ទ្រាយតម្លៃមិនត្រឹមត្រូវ | Review field type requirements |
