title: Administrador del Proyecto
description: 
---

El Administrador del Proyecto es el rol de mayor privilegio dentro de un proyecto de Blue, similar al Propietario del Proyecto. Los Administradores del Proyecto tienen control total sobre el proyecto y sus configuraciones.

**Permisos**

Los Administradores del Proyecto pueden:

- Editar todas las configuraciones del proyecto
- Agregar/quitar miembros del proyecto
- Asignar [roles](/docs/user-management/roles) y [permisos](/docs/user-management/roles/custom-user-roles) a los usuarios
- [Importar](/docs/integrations/csv-import) y [exportar](/docs/integrations/csv-export) datos del proyecto
- Eliminar el proyecto
- Crear [campos personalizados](/docs/custom-fields/introduction)
- Crear roles personalizados
- Configurar [Automatización](/docs/automations/introduction)

Esencialmente, tienen permisos ilimitados dentro del proyecto.

**Casos de Uso**

Usuarios típicos asignados como Administrador del Proyecto:

- Gerentes de proyecto
- Líderes de equipo
- Gerentes de TI
- Propietarios de negocios

Cualquiera que necesite configurar completamente el flujo de trabajo o gestionar miembros.

**Asignación del Rol**

Al invitar a miembros, se puede asignar el rol de Administrador del Proyecto.

Los miembros existentes también pueden ser promovidos a Administrador del Proyecto. Pero siempre debe haber al menos un Administrador del Proyecto en un proyecto.

**Eliminación del Rol**

Para eliminar los permisos de Administrador del Proyecto de un miembro, primero debe ser degradado a un rol personalizado más restringido o a un rol estándar.
