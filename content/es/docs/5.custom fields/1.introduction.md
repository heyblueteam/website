Los campos personalizados son una forma de crear una estructura de datos personalizada dentro de Blue que se adapte perfectamente a los requisitos de su organización. Al crear campos personalizados para sus datos estructurados en lugar de usar comentarios o descripciones, usted aprovecha todo el poder de Blue como plataforma de trabajo. Los datos estructurados pueden convertirse en su única fuente de verdad.



## Puntos Clave

Por defecto, cada registro en Blue viene con campos específicos:

- Nombre del Registro
- [Asignado](/es/docs/records/assignees)
- [Fecha](/es/docs/records/dates)
- [Etiqueta](/es/docs/records/tags)
- [Dependencias](/es/docs/records/dependencies)

Pero usted puede elegir entre numerosos campos adicionales diferentes que puede agregar a todos los registros dentro de un proyecto. Por ejemplo, esto podría ser un campo personalizado de moneda para "Precio de Venta" en un proyecto de ventas.

![Introduction 1](/docs/Introduction_1.png "Introduction 1")

Cada campo personalizado es una columna adicional en la base de datos de su proyecto

Los campos personalizados están estrechamente integrados en Blue. Usando los datos contenidos dentro de cada campo personalizado, usted puede [crear formularios de entrada de trabajo](/es/docs/projects/forms), [importar](/es/docs/integrations/csv-import) y [exportar](/es/docs/integrations/csv-export) datos, [filtrar datos](/es/docs/views/filters), y [crear tableros de informes](/es/docs/dashboards/introduction).

Los campos personalizados se pueden subdividir en dos tipos principales:

- **Campos Personalizados Editables**: Estos son campos personalizados donde los usuarios pueden ingresar datos en el campo. Por ejemplo, un [campo personalizado de Email](/es/docs/custom-fields/email) permite a los usuarios ingresar una dirección de correo electrónico, o un [campo personalizado de Selección Única](/es/docs/custom-fields/single-select) permite a los usuarios elegir una opción del menú desplegable proporcionado.

- **Campos Personalizados de Solo Lectura**: Estos son campos que solo muestran datos. Por ejemplo, el [campo personalizado de ID Único](/es/docs/custom-fields/unique-id) muestra un identificador único generado aleatoriamente para su registro, y el [campo personalizado de Fórmula](/es/docs/custom-fields/formula) muestra el resultado de un cálculo basado en valores de otros campos personalizados.

## Creando Campos Personalizados

Solo los [Administradores de Proyecto](/es/docs/user-management/roles/project-administrator) pueden crear campos personalizados para asegurar que la integridad de la estructura de datos del proyecto se mantenga intacta. Usted puede crear y administrar campos personalizados abriendo un registro en el proyecto y haciendo clic en "Agregar Más Campos"

![Introduction 2](/docs/Introduction_2.png "Introduction 2")

Use el botón "Agregar Más Campos" dentro de un registro para agregar campos personalizados a un proyecto.

::callout
---

icon: lightbulb
---

**Los Campos Personalizados Son de Todo el Proyecto** <br/> <br/>
Cuando usted crea un nuevo campo personalizado, ese campo personalizado está disponible en todos los registros dentro del proyecto específico. Cada registro puede entonces tener diferentes valores para ese campo personalizado. Por ejemplo, el registro "Cliente 1" puede tener un valor de $2000 para el campo personalizado "Precio de Venta", mientras que el registro "Cliente 2" tiene un valor de $15,000 para el campo personalizado "Precio de Venta".<br/> <br/>
Utilizar campos personalizados en todo un proyecto mejora significativamente la consistencia de datos y la eficiencia de gestión. Al estandarizar la estructura de información, los campos personalizados aseguran que todos los registros estén organizados de manera uniforme, haciendo que el seguimiento y análisis de datos sea más simplificado y confiable.
::

## Eliminando Campos Personalizados

Para eliminar un campo personalizado, abra cualquier registro y haga clic en "Agregar Más Campos". Se le presentará una lista de campos personalizados en su proyecto. Junto a cada campo personalizado, habrá un ícono de papelera roja para eliminar ese campo personalizado.

![Introduction 3](/docs/Introduction_3.png "Introduction 3")

El ícono de eliminar para campos personalizados
Usted puede hacer clic en este ícono para eliminar los campos personalizados del proyecto. Tendrá un mensaje de confirmación, y luego puede confirmar y eliminar el campo personalizado.

::callout
---

icon: lightbulb
---

**Eliminar Campos Personalizados Es de Todo el Proyecto y Permanente** <br/><br/>
Cuando usted elimina un campo personalizado, esto elimina el campo personalizado de todos los registros dentro de ese proyecto. Todos los datos contenidos dentro de ese campo personalizado serán eliminados permanentemente y no son recuperables.
::

## Permisos de Usuario

Una de las características más poderosas de Blue es que proporcionamos permisos a nivel de fila para campos personalizados. Esto le permite establecer qué grupos de usuarios pueden ver o editar campos personalizados individuales. [Esto se puede configurar creando un rol de usuario personalizado](/es/docs/user-management/roles/custom-user-roles).
