---
title: Límites de Tasa
description: Directrices para el límite de tasa de la API de Blue
---

En general, la API de Blue no impone límites de tasa estrictos en la mayoría de las operaciones. Sin embargo, ciertas operaciones sensibles tienen límites de tasa específicos para prevenir abusos y garantizar la estabilidad del sistema.

## Operaciones con Límite de Tasa

Las siguientes operaciones tienen límites de tasa impuestos:

| Operación | Límite de Tasa | Ventana | Propósito |
|-----------|----------------|---------|-----------|
| `signIn` | 5 requests | 60 seconds | Prevenir ataques de fuerza bruta |  |
| `signInRequest` | 3 requests | 120 seconds | Limitar intentos de autenticación |  |
| `createDocument` | 5 requests | 60 seconds | Prevenir spam de documentos |  |
| `sendTestEmail` | 5 requests | 60 seconds | Prevenir abuso de correo electrónico |  |
| `submitForm` | 5 requests | 60 seconds | Prevenir spam de formularios |  |
| `exportTodos` | 1 request | 50 seconds | Limitar exportaciones intensivas en recursos |  |
| `deleteCompany` | 3 requests | 60 seconds | Prevenir eliminaciones accidentales |  |
| `deleteCompanyRequest` | 3 requests | 60 seconds | Prevenir eliminaciones accidentales |  |
| `updateEmail` | 3 requests | 60 seconds | Prevenir abuso en el cambio de correo electrónico |  |
| `updateEmailRequest` | 3 requests | 60 seconds | Prevenir abuso en el cambio de correo electrónico |  |
| `verifyAcceptInvitation` | 3 requests | 60 seconds | Limitar intentos de verificación |  |
| `verifySecurityCode` | 3 requests | 60 seconds | Limitar intentos de verificación |  |

## Comportamiento del Límite de Tasa

- **Por Usuario**: Los límites de tasa se aplican por usuario autenticado
- **Por IP**: Para solicitudes no autenticadas, los límites se aplican por dirección IP
- **Sin Encabezados**: La información del límite de tasa no se incluye en los encabezados de respuesta

## Respuesta de Error

Cuando se excede un límite de tasa, recibirás un error de GraphQL:

```json
{
  "errors": [{
    "message": "Rate limit exceeded",
    "extensions": {
      "code": "RATE_LIMITED"
    }
  }]
}
```

## Mejores Prácticas

1. **Manejar Errores de Límite de Tasa**: Implementar un manejo de errores adecuado para operaciones con límite de tasa
2. **Retroceso Exponencial**: Usar retroceso exponencial al reintentar solicitudes con límite de tasa
3. **Monitorear Uso**: Estar al tanto de qué operaciones tienen límites al construir integraciones
