Проекты в Blue уже являются мощным инструментом для управления данными вашего бизнеса и продвижения работы вперед.

Сегодня мы делаем следующий логический шаг и позволяем вам взаимосвязывать ваши данные *между* проектами для максимальной гибкости и мощности.

Взаимосвязь проектов в Blue превращает его в единственный источник правды для вашего бизнеса. Эта возможность позволяет создавать комплексный и взаимосвязанный набор данных, обеспечивая бесшовный поток данных и улучшенную видимость между проектами. Связывая проекты, команды могут достичь единого взгляда на операции, улучшая принятие решений и операционную эффективность.

## Пример

Рассмотрим компанию ACME, которая использует Справочные и Поисковые пользовательские поля Blue для создания взаимосвязанной экосистемы данных в своих проектах по работе с клиентами, продажам и инвентаризации. Записи клиентов в проекте "Клиенты" связаны через Справочные поля с транзакциями продаж в проекте "Продажи". Эта связь позволяет Полям Поиска извлекать связанные данные о клиентах, такие как номера телефонов и статусы аккаунтов, непосредственно в каждую запись о продаже. Кроме того, товары, проданные на складе, отображаются в записи о продаже через Поле Поиска, ссылающееся на данные о Количестве Проданных товаров из проекта "Инвентаризация". Наконец, списания из инвентаря связаны с соответствующими продажами через Справочное поле в проекте "Инвентаризация", указывающее на записи о продажах. Эта настройка обеспечивает полную видимость того, какая продажа вызвала списание из инвентаря, создавая интегрированный 360-градусный обзор между проектами.

## Как работают Справочные поля

Справочные пользовательские поля позволяют вам создавать отношения между записями в разных проектах в Blue. При создании Справочного поля администратор проекта выбирает конкретный проект, который предоставит список справочных записей. Опции настройки включают:

* **Одиночный выбор**: Позволяет выбрать одну справочную запись.
* **Множественный выбор**: Позволяет выбрать несколько справочных записей.
* **Фильтрация**: Установите фильтры, чтобы разрешить пользователям выбирать только записи, соответствующие критериям фильтра.

После настройки пользователи могут выбирать конкретные записи из выпадающего меню в Справочном поле, устанавливая связь между проектами.

## Расширение справочных полей с помощью поисковых полей

Поисковые пользовательские поля используются для импорта данных из записей в других проектах, создавая одностороннюю видимость. Они всегда являются только для чтения и связаны с конкретным Справочным пользовательским полем. Когда пользователь выбирает одну или несколько записей с помощью Справочного пользовательского поля, Поисковое пользовательское поле будет отображать данные из этих записей. Поисковые поля могут отображать такие данные, как:

* Дата создания
* Дата обновления
* Срок выполнения
* Описание
* Список
* Тег
* Ответственный
* Любое поддерживаемое пользовательское поле из справочной записи — включая другие поисковые поля!

Например, представьте сценарий, в котором у вас есть три проекта: **Проект A** — это проект по продажам, **Проект B** — это проект по управлению инвентаризацией, а **Проект C** — это проект по управлению отношениями с клиентами. В Проекте A у вас есть Справочное пользовательское поле, которое связывает записи о продажах с соответствующими записями о клиентах в Проекте C. В Проекте B у вас есть Поисковое пользовательское поле, которое импортирует информацию из Проекта A, такую как количество проданных товаров. Таким образом, когда запись о продаже создается в Проекте A, информация о клиенте, связанная с этой продажей, автоматически извлекается из Проекта C, а количество проданных товаров автоматически извлекается из Проекта B. Это позволяет вам хранить всю соответствующую информацию в одном месте и просматривать ее без необходимости создавать дублирующие данные или вручную обновлять записи между проектами.

Реальный пример этого — это компания электронной коммерции, которая использует Blue для управления своими продажами, инвентаризацией и отношениями с клиентами. В своем проекте **Продажи** у них есть Справочное пользовательское поле, которое связывает каждую запись о продаже с соответствующей записью **Клиента** в их проекте **Клиенты**. В своем проекте **Инвентаризация** у них есть Поисковое пользовательское поле, которое импортирует информацию из проекта Продаж, такую как количество проданных товаров, и отображает ее в записи о товаре на складе. Это позволяет им легко видеть, какие продажи приводят к списаниям из инвентаря, и поддерживать свои уровни инвентаря в актуальном состоянии без необходимости вручную обновлять записи между проектами.

Обеспечивая бесшовные соединения данных и предоставляя видимость в реальном времени между проектами, эти функции трансформируют то, как команды сотрудничают и принимают решения. Независимо от того, управляете ли вы отношениями с клиентами, отслеживаете продажи или контролируете инвентаризацию, Справочные и Поисковые поля в Blue позволяют вашей команде работать умнее, быстрее и эффективнее.
