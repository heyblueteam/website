Пользовательские поля — это способ создать структуру данных в Blue, которая идеально соответствует требованиям вашей организации. Создавая пользовательские поля для структурированных данных вместо использования комментариев или описаний, вы используете всю мощь Blue как рабочей платформы. Структурированные данные могут стать вашим единственным источником истины.

Наши [API](/ru/docs/integrations/api) и [Веб-хуки](/ru/docs/integrations/webhooks) полностью поддерживают пользовательские поля, поэтому используя пользовательские поля в Blue, вы создаете API структурированных данных, который идеально соответствует вашим данным.


## Ключевые моменты

По умолчанию каждая запись в Blue включает определенные поля:

- Название записи
- [Исполнитель](/ru/docs/records/assignees)
- [Дата](/ru/docs/records/dates)
- [Тег](/ru/docs/records/tags)
- [Зависимости](/ru/docs/records/dependencies)

Но вы можете выбрать из множества различных дополнительных полей, которые можно добавить ко всем записям в рамках проекта. Например, это может быть валютное пользовательское поле для "Цены продажи" в проекте продаж.

![Introduction 1](/docs/Introduction_1.png "Introduction 1")

Каждое пользовательское поле — это дополнительный столбец в базе данных вашего проекта

Пользовательские поля тесно интегрированы в Blue. Используя данные, содержащиеся в каждом пользовательском поле, вы можете [создавать формы приема работ](/ru/docs/projects/forms), [импортировать](/ru/docs/integrations/csv-import) и [экспортировать](/ru/docs/integrations/csv-export) данные, [фильтровать данные](/ru/docs/views/filters) и [создавать информационные панели отчетности](/ru/docs/dashboards/introduction).

Пользовательские поля можно разделить на два основных типа:

- **Редактируемые пользовательские поля**: Это пользовательские поля, в которые пользователи могут вводить данные. Например, [пользовательское поле Email](/ru/docs/custom-fields/email) позволяет пользователям вводить адрес электронной почты, а [пользовательское поле одиночного выбора](/ru/docs/custom-fields/single-select) позволяет пользователям выбрать один вариант из предоставленного выпадающего списка.

- **Пользовательские поля только для чтения**: Это поля, которые только отображают данные. Например, [пользовательское поле уникального ID](/ru/docs/custom-fields/unique-id) отображает случайно сгенерированный уникальный идентификатор для вашей записи, а [пользовательское поле формулы](/ru/docs/custom-fields/formula) отображает результат вычисления на основе значений из других пользовательских полей.

## Создание пользовательских полей

Только [Администраторы проекта](/ru/docs/user-management/roles/project-administrator) могут создавать пользовательские поля, чтобы обеспечить сохранение целостности структуры данных проекта. Вы можете создавать и управлять пользовательскими полями, открыв запись в проекте и нажав на "Добавить больше полей"

![Introduction 2](/docs/Introduction_2.png "Introduction 2")

Используйте кнопку "Добавить больше полей" внутри записи, чтобы добавить пользовательские поля в проект.

::callout
---

icon: lightbulb
---

**Пользовательские поля действуют на уровне проекта** <br/> <br/>
Когда вы создаете новое пользовательское поле, это пользовательское поле становится доступным во всех записях в рамках конкретного проекта. Каждая запись может иметь разные значения для этого пользовательского поля. Например, запись "Клиент 1" может иметь значение $2000 для пользовательского поля "Цена продажи", в то время как запись "Клиент 2" имеет значение $15,000 для пользовательского поля "Цена продажи".<br/> <br/>
Использование пользовательских полей в рамках проекта значительно повышает согласованность данных и эффективность управления. Стандартизируя структуру информации, пользовательские поля обеспечивают единообразную организацию всех записей, делая отслеживание и анализ данных более упорядоченными и надежными.
::

## Удаление пользовательских полей

Чтобы удалить пользовательское поле, откройте любую запись и нажмите на "Добавить больше полей." Вам будет представлен список пользовательских полей в вашем проекте. Рядом с каждым пользовательским полем будет красная иконка корзины для удаления этого пользовательского поля.

![Introduction 3](/docs/Introduction_3.png "Introduction 3")

Иконка удаления для пользовательских полей
Вы можете нажать на эту иконку, чтобы удалить пользовательские поля из проекта. У вас будет сообщение с подтверждением, после чего вы можете подтвердить и удалить пользовательское поле.

::callout
---

icon: lightbulb
---

**Удаление пользовательских полей действует на уровне проекта и является необратимым** <br/><br/>
Когда вы удаляете пользовательское поле, это удаляет пользовательское поле из всех записей в рамках этого проекта. Все данные, содержащиеся в этом пользовательском поле, будут безвозвратно удалены и не подлежат восстановлению.
::

## Разрешения пользователей

Одной из самых мощных функций Blue является предоставление разрешений на уровне строк для пользовательских полей. Это позволяет вам установить, какие группы пользователей могут просматривать или редактировать отдельные пользовательские поля. [Это можно настроить, создав пользовательскую роль пользователя](/ru/docs/user-management/roles/custom-user-roles).
