## Обзор

Обложки записей позволяют добавить визуальный контекст к вашим записям, отображая превью изображений непосредственно на карточках записей. Эта функция автоматически извлекает изображения из различных источников в ваших записях и отображает их в виде миниатюр, упрощая идентификацию и организацию визуального контента с первого взгляда.

Независимо от того, управляете ли вы дизайнерскими проектами, календарем контента, каталогом продуктов или любым рабочим процессом, включающим визуальные ресурсы, обложки записей помогают быстро просматривать записи и находить то, что вы ищете. Функция включает отслеживание прогресса в реальном времени и гибкие опции отображения для соответствия вашим конкретным потребностям рабочего процесса.

## Ключевые функции

- **Автоматическое извлечение изображений**: Blue автоматически находит и отображает изображения из выбранного вами источника
- **Множественные источники изображений**: Получение изображений из описаний записей, комментариев или пользовательских файловых полей
- **Опции размещения**: Выберите, как изображения отображаются с помощью режимов Cover или Contain
- **Умный выбор**: Настройте, использовать ли первое или последнее изображение из вашего источника
- **Обновления в реальном времени**: Обложки записей обновляются автоматически при изменении содержимого записей
- **Отслеживание прогресса в реальном времени**: Мониторинг массовой обработки с подписками на прогресс в реальном времени

## Как это работает

### Включение обложек записей

Обложки записей настраиваются на уровне проекта администраторами проекта:

1. Откройте ваш проект и перейдите в **Настройки проекта**
2. Выберите **Настройки записей** 
3. Найдите секцию **Опции обложек изображений**
4. Переключите **Включить обложки записей** для активации функции

### Опции конфигурации

#### Источник изображений
Выберите, где Blue должен искать изображения:
- **Описание**: Извлекает изображения, встроенные в описания записей (по умолчанию)
- **Комментарии**: Получает первое/последнее изображение из комментариев записей
- **Пользовательское поле**: Использует файлы из конкретного файлового пользовательского поля (поддерживает любой тип файлов)

#### Выбор изображения
Когда доступно несколько изображений, выберите какое отображать:
- **Первое изображение**: Использует первое найденное изображение (по умолчанию)
- **Последнее изображение**: Использует самое последнее изображение

#### Размещение отображения
Контролируйте, как изображения появляются в области превью:
- **Cover**: Изображение заполняет всю область превью, может обрезать края для сохранения пропорций (по умолчанию)
- **Contain**: Показывает полное изображение в области превью, может иметь пустое пространство

## Случаи использования

**Дизайнерские проекты** - Отображайте макеты и итерации дизайна непосредственно на карточках задач, упрощая просмотр текущего состояния каждой дизайнерской задачи без открытия отдельных записей.

**Планирование контента** - Для календарей контента и редакционных рабочих процессов показывайте избранные изображения или миниатюры для визуализации вашего конвейера контента с первого взгляда.

**Управление продуктами** - Отображайте фотографии продуктов на инвентарных записях или запросах функций, помогая командам быстро идентифицировать элементы во время обсуждений.

**Недвижимость** - Показывайте фотографии недвижимости на записях листингов, позволяя агентам быстро просматривать объекты визуально.

**Планирование событий** - Отображайте фотографии мест проведения, концепции дизайна или планировки событий на задачах планирования для лучшей визуальной организации.

## Настройка обложек записей

### Шаг 1: Выберите ваш источник изображений

Рассмотрите, где изображения естественно существуют в вашем рабочем процессе:
- Если изображения являются частью описаний задач или брифов, используйте **Описание**
- Если изображения добавляются через командное сотрудничество, используйте **Комментарии**
- Если у вас есть выделенное поле для изображений, используйте **Пользовательское поле**

### Шаг 2: Настройте параметры отображения

1. Выберите предпочтительное **Размещение изображения** на основе ваших типов изображений:
   - Используйте **Cover** для фотографий и дизайнов, где обрезка приемлема
   - Используйте **Contain** для диаграмм или скриншотов, где вам нужно видеть все

2. Выберите **Выбор изображения** на основе вашего рабочего процесса:
   - Используйте **Первое** для рабочих процессов, где основное изображение добавляется изначально
   - Используйте **Последнее** для рабочих процессов, где самое последнее изображение наиболее релевантно

### Шаг 3: Обработайте существующие записи

Когда вы впервые включаете обложки записей или изменяете конфигурацию, Blue автоматически обрабатывает ваши записи:
1. Обновления прогресса в реальном времени показывают статус обработки 
2. Записи обрабатываются пакетами (50 записей на пакет) для оптимальной производительности
3. Вы можете продолжать работать, пока обработка происходит в фоновом режиме
4. Списки автоматически обновятся, когда обработка завершится
5. Обработка автоматически запускается при сохранении изменений конфигурации

## Технические детали

### Поддерживаемые форматы
- При использовании источников Описание или Комментарии:
  - JPEG/JPG изображения
  - PNG изображения (включая прозрачность)
  - GIF изображения (статическое отображение)
- При использовании источника Пользовательское поле:
  - Любой тип файлов может быть отображен (изображения, PDF, документы)
  - Файлы, не являющиеся изображениями, отображают соответствующие иконки типов файлов
- Все файлы должны быть загружены в Blue (внешние URL не поддерживаются)

### Ограничения
- Максимум 6,000 записей на проект для массового генерирования обложек
- Только файловые пользовательские поля могут использоваться как источники
- Обложки записей доступны только для просмотра и не могут редактироваться напрямую

### Автоматические обновления
Обложки автоматически регенерируются, когда:
- Описание записи редактируется (если используется источник Описание)
- Добавляются новые комментарии с изображениями (если используется источник Комментарии)  
- Файлы в пользовательских полях обновляются (если используется источник Пользовательское поле)
- Изменяются настройки конфигурации обложек
- Изменяется опция размещения отображения

### Доступ к API
Обложки записей полностью интегрированы с GraphQL API:
- Настройка через мутацию `setTodoCoverConfig`
- Доступ к данным обложек через поле `cover` в типе Todo
- Мониторинг прогресса с подпиской `subscribeToCoverGenerationProgress`
- URL миниатюр автоматически генерируются по адресу `/uploads/{uid}/thumbnail/cover`


## Интеграция с другими функциями

Обложки записей работают бесшовно с другими функциями Blue:
- **[Формы](/ru/docs/projects/forms)**: Отправленные изображения могут автоматически становиться обложками
- **[API](/ru/docs/integrations/api)**: Полная поддержка GraphQL API для конфигурации и доступа к данным
- **[Пользовательские поля](/ru/docs/projects/custom-fields)**: Любое файловое поле может служить источником обложки
