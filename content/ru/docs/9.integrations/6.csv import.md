Вы можете импортировать файлы CSV (значения, разделенные запятыми). Если у вас есть файл Excel, [вы можете конвертировать его в файл CSV по следующей ссылке](https://cloudconvert.com/xls-to-csv).

## Импорт CSV

Импорт файлов CSV в Blue — это простой процесс. Каждый столбец в вашем файле CSV соответствует стандартному или [настраиваемому полю](/docs/custom-fields/introduction) в Blue.

Для начала вы должны быть администратором проекта, в который хотите импортировать данные. Нажмите на значок настроек проекта (значок шестеренки) и выберите "Импорт"

![Импорт CSV 1](/docs/CSVimport1.png "CSV Import 1")

Функция импорта доступна в настройках проекта для администраторов проекта

На скриншоте выше вы также можете увидеть другую опцию под названием "Скачать шаблон CSV". Это крайне полезно, если вы уже создали свои [настраиваемые поля](/docs/custom-fields/introduction) в вашем проекте Blue, потому что вы можете скачать файл CSV, который включает столбцы для всех ваших настраиваемых полей! Это отлично, когда вы хотите убедиться, что у вас есть точная структура для вашего CSV перед импортом.

Далее просто перетащите ваш файл CSV в Blue:

![Импорт CSV 2](/docs/CSVimport2.png "CSV Import 2")

Перетащите и отпустите, чтобы загрузить ваши данные

В зависимости от размера вашего файла, это может занять несколько секунд для загрузки и анализа Blue. Как только Blue проанализирует данные, вы увидите предварительный просмотр первых десяти строк ваших данных:

![Импорт CSV 3](/docs/CSVimport3.png "CSV Import 3")

Затем вы сможете сопоставить каждый столбец в вашем CSV с полем в Blue.

![Импорт CSV 4](/docs/CSVimport4.png "CSV Import 4")

## Принятые форматы дат

Blue поддерживает широкий спектр форматов дат для импорта. Вот все поддерживаемые форматы:

- **Unix временные метки**:
  ```plaintext
  1511648546 (seconds)
  1620021848429 (milliseconds)
  1620024872717915000 (nanoseconds)
  ```
  
- **RFC стандарты**:
  ```plaintext
  RFC 3339: 2021-05-01T01:17:02.604456Z
  RFC 2822: Wed, 02 Jun 2021 06:31:39 GMT
  ```

- **Форматы баз данных**:
  ```plaintext
  PostgreSQL: 
  2019-11-29 08:08-08
  2019-11-29 08:08:05-08
  2021-05-02 23:31:36.0741-07
  ```

- **Общие форматы**:
  ```plaintext
  Year-Month-Day: 
  2021-04-30 21:14
  2021-02-21
  2021-Feb-21
  
  Month/Day/Year:
  3/31/2014 22:05
  04/02/2014 03:00:51
  
  Day-Month-Year:
  12 Feb 2006, 19:17
  03 February 2013
  ```

- **Часовые пояса**:
  ```plaintext
  2014-04-26 13:13:43 +0800
  2017-11-25 13:31 PST
  2021-02-21 UTC
  ```

- **Естественный язык**:
  ```plaintext
  May 6 at 9:24 PM
  September 17, 2012 at 10:09am PST
  October 7, 1970
  ```

- **Международные форматы**:
  ```plaintext
  Chinese: 2014年04月08日
  Japanese: 2014/04/08 22:05
  ```

- **Только время**:
  ```plaintext
  01:06:06
  4:00pm PST
  6:00 AM UTC
  ```

Все даты будут преобразованы в часовой пояс UTC во время импорта. Компоненты времени являются необязательными - если они не указаны, время будет по умолчанию 00:00:00 UTC.

## Обработка запятых в ваших данных

Если ваши данные содержат запятые (например, в адресах или названиях компаний), вам нужно обернуть эти значения в двойные кавычки, чтобы предотвратить ошибки импорта.

**Пример:**
```
Company,Address
Acme Inc,123 Main St, New York
```

**Должно стать:**
```
Company,Address
"Acme Inc","123 Main St, New York"
```

Это гарантирует, что запятая в "123 Main St, New York" рассматривается как часть данных, а не как разделитель столбцов.
