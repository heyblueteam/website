## Введение

White label превращает Blue в гибкую платформу для управления проектами и процессами, которая выглядит и ощущается как полностью Ваш собственный продукт — без необходимости управлять технологиями, разработчиками, серверами или обеспечивать бесперебойную работу.

С помощью Blue White Label Вы можете:

- **Продемонстрировать Ваш бренд:** Замените название, логотип и цвета Blue на свои собственные, создав единое брендовое восприятие.
- **Работать на Вашем домене:** Запустите Blue на собственном веб-домене для профессионального, целостного присутствия.
- **Использовать Вашу инфраструктуру электронной почты:** Отправляйте письма с Вашего адреса для персонализированного общения.


Для агентств дополнение White Label укрепляет Ваш бренд, расширяет предложения услуг и упрощает сотрудничество с клиентами. Интегрируя Blue в Ваши рабочие процессы под Вашим брендингом, Вы создаете бесшовный, профессиональный опыт, который помогает привлекать больше клиентов и удерживать их, предоставляя уникальное сочетание услуг и программного обеспечения, адаптированного под их потребности.


## Цены

Дополнение white-label стоит $20/месяц на компанию или $200/год на организацию, дополнительно к Вашей основной подписке Blue. Подробнее о ценах можно узнать [здесь](/pricing).

Если Вас интересует покупка white label оптом для нескольких организаций, обратитесь к sales@blue.cc
 

## Пользовательские домены

Для работы White Label необходимо сначала настроить пользовательский домен для веб-приложения. 

Пользовательские домены можно настроить для следующего:

- Веб-приложение (обязательно)
- [Формы](/docs/projects/forms) (опционально)
- [Ссылки для обмена файлами](/docs/projects/file-management) (опционально)
- Документация (скоро)


Для настройки пользовательского домена добавьте CNAME записи в настройки DNS для каждого поддомена. Обратите внимание, что поддерживаются только поддомены, а не корневые домены. Поэтому `app.yourcompany.com` поддерживается, а `yourcompany.com` — нет.


| Тип домена                 | CNAME запись                  | Указывает на               |
|----------------------------|-------------------------------|----------------------------|
| Веб-приложение             | `app.yourcompany.com`         | `wl-app.onrender.com`    |
| Формы                      | `forms.yourcompany.com`       | `wl-forms.onrender.com`    |
| Ссылки для обмена файлами  | `files.yourcompany.com`       | `wl-files.onrender.com`    |


Приведенная выше таблица содержит рекомендуемые CNAME записи, но Вы можете использовать любой поддомен по желанию. 

Например, основное веб-приложение может быть `projects.yourcompany.com`, а ссылки для обмена файлами — `media.yourcompany.com`.


## Брендинг

Настройте следующие элементы для приведения Blue в соответствие с Вашим брендом:

- **Логотип**: Загрузите свой логотип, чтобы заменить брендинг Blue на всей платформе.
- **Favicon**: Добавьте собственный favicon, который будет отображаться во вкладках браузера и закладках.
- **Основной цвет интерфейса**: Установите основной цвет для интерфейса, который будет последовательно применяться для всех пользователей.
- **Название приложения**: Настройте название приложения, отображаемое во вкладках браузера и в настольном приложении.
- **Шрифты**: (Скоро) Настройте шрифты, используемые на платформе, чтобы они соответствовали Вашему бренду из выбранного списка шрифтов Google.

Blue предоставляет живой превью страницы входа и иконки приложения во время настройки параметров брендинга.

Ваш брендинг также будет применяться к [настольному приложению](/docs/start-guide/download-apps#desktop-app), обеспечивая полностью white-label опыт.

Рекомендуемая конфигурация для логотипа и favicon:

| Ресурс  | Рекомендуемая форма | Фон             | Соотношение сторон |
|---------|---------------------|-----------------|-------------------|
| Логотип | Прямоугольник       | Прозрачный      | 1:1.5            |
| Favicon | Квадрат             | Прозрачный      | 1:1              |


## Уведомления по электронной почте

Вы можете использовать SMTP для отправки уведомлений по электронной почте с собственного почтового сервера с Вашим доменом вместо нашего адреса электронной почты.

Вам необходимо предоставить Blue следующую информацию:

| Настройка                            | Описание                                                                                                                                             |
|--------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------|
| **SMTP сервер**                      | Адрес сервера, который будет обрабатывать отправку Ваших писем.                                                                                     |
| **SMTP порт**                        | Номер порта, используемый для подключения к SMTP серверу, обычно 587 для TLS или 465 для SSL.                                                       |
| **Имя пользователя**                 | Имя пользователя для аутентификации на Вашем SMTP сервере, обычно Ваш адрес электронной почты.                                                      |
| **Пароль (хранится в зашифрованном формате)** | Пароль, связанный с именем пользователя, который будет безопасно храниться в зашифрованном формате для защиты Ваших учетных данных.        |
| **Имя отправителя**                  | Имя, которое будет отображаться как отправитель в уведомлениях по электронной почте, помогая получателям идентифицировать источник письма.         |
| **Адрес электронной почты отправителя** | Адрес электронной почты, который будет отображаться как отправитель, гарантируя, что ответы попадут на правильный адрес. Обратите внимание, что это должен быть тот же домен, что и SMTP сервер. |

Blue требует использования TLS/SSL шифрования для SMTP соединения.

Затем Вы можете протестировать соединение с Вашим SMTP сервером, нажав кнопку "Тест соединения". Blue автоматически протестирует соединение с Вашим SMTP сервером и уведомит Вас об успехе или неудаче через уведомление в приложении.

### Рекомендации 

Все транзакционные, уведомляющие и автоматические письма будут отправляться через Ваш SMTP сервер. Единственное исключение — письма, содержащие OTP (одноразовые пароли), которые отправляются через общий адрес электронной почты (notifications@process-system.app) для обеспечения безопасности.

Убедитесь, что Ваш SMTP сервер:

- Может обрабатывать большие объемы электронной почты.
- Обладает хорошей репутацией отправителя, чтобы избежать спам-фильтров.
- Имеет достаточную квоту для обработки ожидаемой нагрузки по электронной почте.


### Пользовательские приглашения по электронной почте

Вы также можете настроить начальное приглашение по электронной почте, которое отправляется новым пользователям при регистрации в Вашем экземпляре Blue. 

### Пользовательские письма для онбординга (Скоро)

Без white label Blue отправляет серию писем для онбординга новым пользователям, когда они присоединяются к компании, чтобы помочь им начать работу. При включении white-label эти письма не отправляются, и в будущем Вы сможете создать собственный набор писем для онбординга для приветствия новых пользователей.

В настоящее время мы не планируем поддерживать white-labelling мобильных приложений, но если Вы хотите иметь свою собственную брендированную версию мобильного приложения в Ваших аккаунтах Apple Appstore и Google Playstore, Вы можете обратиться к sales@blue.cc для обсуждения этого.

## Часто задаваемые вопросы


**Что включено в дополнение white label?**

Дополнение white label включает три основные функции: интеграция пользовательского домена для веб-приложения, форм и обмена файлами; пользовательский брендинг (логотип, favicon, цвета и название приложения); и пользовательские уведомления по электронной почте с использованием собственного SMTP сервера.

**Сколько стоит white label?**

Дополнение white label стоит $20/месяц/компанию или $200/год/компанию сверх Вашей основной подписки Blue. Оптовые скидки будут доступны в будущем для аккаунтов с несколькими компаниями.

**Влияет ли white-labelling на функциональность Blue?**

Нет, все основные функции и возможности Blue остаются абсолютно теми же. White-labelling изменяет только внешний вид и брендинг платформы.


**Могу ли я использовать корневой домен вместо поддомена?**

Нет, Blue поддерживает только поддомены для настройки пользовательского домена.

**Нужно ли мне настраивать все пользовательские домены?**

Нет, обязателен только домен веб-приложения. Домены для форм и обмена файлами опциональны. Поддержка домена документации скоро появится.

**Могу ли я использовать другие названия поддоменов, отличные от предложенных?**

Да, Вы можете использовать любые названия поддоменов по своему усмотрению. Например, Вы можете использовать `projects.yourcompany.com` вместо `app.yourcompany.com`.

**Какие элементы брендинга я могу настроить?**

Вы можете настроить свой логотип, favicon, основной цвет интерфейса и название приложения.

**Будет ли настольное приложение отражать мой брендинг?**

Да, настольное приложение будет полностью white-label с Вашим брендингом.

**Могут ли разные пользователи иметь разные цветовые схемы?**

Нет, выбранный Вами основной цвет будет применяться для всех пользователей Вашего экземпляра Blue.

**Какие детали конфигурации электронной почты мне нужно предоставить?**

Вам нужно будет предоставить адрес SMTP сервера, SMTP порт, имя пользователя, пароль, имя отправителя и адрес электронной почты отправителя. SMTP пароль хранится в зашифрованном формате для защиты Ваших учетных данных.

**Могу ли я протестировать конфигурацию SMTP перед запуском?**

Да, Blue предоставляет кнопку "Тест соединения", которая проверит конфигурацию Вашего SMTP сервера и уведомит Вас о результате.

**Какие типы писем будут отправляться через мой SMTP сервер?**

Все письма будут отправляться через Ваш SMTP сервер, включая транзакционные письма, уведомления, автоматические письма. OTP (одноразовые пароли) письма отправляются через общий адрес электронной почты (notifications@process-system.app) для обеспечения безопасности.


**Могу ли я применить white-label к мобильным приложениям?**

Blue в настоящее время не поддерживает white-labelling мобильных приложений. Однако, если Вам нужна брендированная версия в Apple App Store или Google Play Store, Вы можете связаться с sales@blue.cc для обсуждения вариантов.

**Будет ли поддержка white-label для мобильных приложений доступна в будущем?**

В настоящее время нет планов по поддержке white-labelling мобильных приложений.

**Как получить помощь с настройкой white-label?**

Для общей помощи Вы можете обратиться к support@blue.cc. По вопросам продаж, касающимся пользовательских мобильных приложений, обращайтесь к sales@blue.cc.

**Могу ли я протестировать функции white-label перед их внедрением?**

Да, Blue предоставляет превью страницы входа и иконки приложения во время настройки параметров брендинга.

## Настройки DNS

Мы удобно собрали инструкции по настройке white label доменов для популярных DNS (Domain Name Service) систем:

#### GoDaddy

1. Войдите в свой аккаунт GoDaddy и откройте свой продукт.
2. Перейдите в Центр управления доменами.
3. На странице настроек домена найдите и выберите название своего домена.
4. Прокрутите вниз до раздела "Дополнительные настройки" и выберите "Управление DNS".
5. На странице управления DNS прокрутите вниз до конца раздела "Записи" и нажмите "Добавить".
6. В выпадающем меню "Тип" выберите CNAME и заполните следующие обязательные поля:
    - Хост - Имя хоста или префикс для CNAME. Вы можете использовать точку (.), но не в качестве первого или последнего символа. Последовательные точки (..) не допускаются. Хост не может превышать 25 символов или быть символом @.
    - Указывает на - Установите это значение в `wl-app.onrender.com`
    - TTL - Укажите время, в течение которого сервер должен кэшировать информацию.
7. Нажмите "Сохранить".

#### Bluehost

1. Войдите в свой аккаунт BlueHost.
2. Нажмите меню "Домены", затем "Редактор зон". Появится новая страница.
3. Перейдите к разделу "Добавить DNS запись".
4. В поле "Запись хоста" введите поддомен адреса, который хотите использовать для своего пользовательского домена.
5. В выпадающем меню "Тип" выберите опцию CNAME.
6. В разделе "Указывает на" введите `wl-app.onrender.com`
7. Нажмите "Добавить запись".

#### Cloudflare

1. Войдите в свой аккаунт Cloudflare.
2. В выпадающем меню найдите и выберите свой домен.
3. Нажмите вкладку "Настройки DNS".
4. Нажмите "Добавить запись":
5. Выберите опцию CNAME.
6. Установите "Имя" в свой поддомен
7. Установите "Цель" в `wl-app.onrender.com`
8. Установите переключатель Cloudflare Proxy в серое облако.

#### Domain.com

1. Войдите в свой аккаунт Domain.com.
2. Найдите свой домен и нажмите "Управление".
3. В меню слева нажмите "DNS и Серверы имен".
4. Нажмите "Добавить DNS запись", чтобы добавить CNAME запись.
5. Установите цель в `wl-app.onrender.com`
6. Нажмите "Обновить DNS", чтобы завершить процесс.

#### Dreamhost

1. Войдите в свой аккаунт DreamHost.
2. Слева на странице нажмите ссылку "Домены".
3. В выпадающем меню "Домены" выберите "Управление доменами".
4. Найдите свой домен и нажмите ссылку DNS под вашим доменом.
5. Прокрутите вниз до раздела "Добавить пользовательскую DNS запись в (ваш пользовательский домен)" и отредактируйте следующие поля:
    - Имя - введите поддомен, который хотите использовать
    - Тип - выберите опцию CNAME.
    - Значение - введите `wl-app.onrender.com`
6. Нажмите "Добавить запись сейчас!", чтобы добавить CNAME запись.

#### Google Domains

1. Войдите в свой аккаунт Google Domains.
2. Найдите и нажмите название своего домена.
3. В меню слева нажмите DNS.
4. Перейдите к разделу "Пользовательские записи ресурсов".
5. В поле "Имя" введите поддомен адреса, который хотите использовать для своего пользовательского домена.
6. В выпадающем меню "Тип" выберите опцию CNAME.
7. В поле "Данные" введите `wl-app.onrender.com`
8. Нажмите "Добавить", чтобы завершить процесс.

#### Hostgator

1. Войдите в свой аккаунт HostGator.
2. В левой части страницы нажмите CPanel.
3. Прокрутите вниз до раздела "Домены" и нажмите "Редактор зон".
4. Найдите свой домен и нажмите ссылку "Запись CName" рядом с доменом.
5. В диалоговом окне "Добавить запись CName" заполните следующие пустые поля:
    - Имя - введите выбранный поддомен/доменное имя.
    - CNAME - введите `wl-app.onrender.com`
6. Нажмите "Добавить запись CName". Появится сообщение от HostGator, сообщающее, что Вы успешно добавили новый CNAME в свой аккаунт HostGator.

#### NameCheap

1. Войдите в свой аккаунт NameCheap.
2. В выпадающем меню "Аккаунт" выберите "Панель управления".
3. Нажмите "Список доменов" и найдите свой домен.
4. Нажмите "Управление" рядом со своим доменом.
5. Нажмите "Расширенный DNS" и перейдите к разделу "Записи хостов".
6. Нажмите "Добавить новую запись".
7. В выпадающем меню "Тип" выберите "Запись CNAME" и введите свой поддомен в качестве хоста.
8. Затем введите `wl-app.onrender.com`
9. Нажмите "Сохранить все изменения", чтобы завершить процесс.
