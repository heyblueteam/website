## Panoramica

I tag sono un modo potente per categorizzare e organizzare i record in Blue. Il sistema di tag fornisce funzionalità complete per creare, gestire e filtrare i tag, con supporto per operazioni in blocco e filtraggio avanzato.

## Elenco Tag

Recupera i tag dai progetti con opzioni di filtraggio e ordinamento flessibili.

### Esempio Base

```graphql
query ListTags {
  tagList(
    filter: { 
      projectIds: ["project-123"] 
      excludeArchivedProjects: false 
    }
    first: 50
    orderBy: title_ASC
  ) {
    items {
      id
      uid
      title
      color
      project {
        id
        name
      }
      createdAt
      updatedAt
    }
    pageInfo {
      totalPages
      totalItems
      page
      perPage
      hasNextPage
      hasPreviousPage
    }
    totalCount
  }
}
```

### Esempio di Filtraggio Avanzato

```graphql
query FilterTags {
  tagList(
    filter: {
      projectIds: ["project-123", "project-456"]
      search: "urgent"
      colors: ["#ff0000", "#ffaa00"]
      excludeArchivedProjects: true
    }
    orderBy: createdAt_DESC
    first: 20
  ) {
    items {
      id
      title
      color
      todos {
        id
        title
      }
    }
    totalCount
  }
}
```

## Crea Tag

Crea nuovi tag all'interno di un progetto.

```graphql
mutation CreateTag {
  createTag(
    input: {
      title: "High Priority"
      color: "#ff0000"
    }
  ) {
    id
    uid
    title
    color
    project {
      id
      name
    }
    createdAt
  }
}
```

## Aggiorna Tag

Modifica le proprietà dei tag esistenti.

```graphql
mutation UpdateTag {
  editTag(
    input: {
      id: "tag-123"
      title: "Critical Priority"
      color: "#cc0000"
    }
  ) {
    id
    title
    color
    updatedAt
  }
}
```

## Elimina Tag

Rimuovi i tag dal sistema. Questo rimuoverà anche il tag da tutti i record associati.

```graphql
mutation DeleteTag {
  deleteTag(id: "tag-123")
}
```

## Tagga Record

Associa i tag ai record (todo) utilizzando tag esistenti o creandone di nuovi.

### Utilizzando Tag Esistenti

```graphql
mutation TagRecord {
  setTodoTags(
    input: {
      todoId: "todo-123"
      tagIds: ["tag-456", "tag-789"]
    }
  )
}
```

### Crea Tag Durante la Taggatura

Puoi creare nuovi tag al volo fornendo i titoli dei tag:

```graphql
mutation TagRecordWithNewTags {
  setTodoTags(
    input: {
      todoId: "todo-123"
      tagIds: ["existing-tag-id"]
      tagTitles: ["New Tag", "Another New Tag"]
    }
  )
}
```

## Parametri di Input

### TagListFilter

| Parametro | Tipo | Richiesto | Descrizione |
|-----------|------|----------|-------------|
| `projectIds` | [String!] | No | Filtra i tag per ID o slug del progetto |  |
| `excludeArchivedProjects` | Boolean | No | Escludi i tag dai progetti archiviati |  |
| `search` | String | No | Cerca i titoli dei tag (senza distinzione tra maiuscole e minuscole) |  |
| `titles` | [String!] | No | Filtra per titoli di tag specifici |  |
| `colors` | [String!] | No | Filtra per colori specifici (formato esadecimale) |  |
| `tagIds` | [String!] | No | Filtra per ID di tag specifici |  |

### CreateTagInput

| Parametro | Tipo | Richiesto | Descrizione |
|-----------|------|----------|-------------|
| `title` | String | No | Titolo del tag (generato automaticamente se non fornito) |  |
| `color` | String! | ✅ Sì | Colore del tag in formato esadecimale (es. "#ff0000") |  |

### EditTagInput

| Parametro | Tipo | Richiesto | Descrizione |
|-----------|------|----------|-------------|
| `id` | String! | ✅ Sì | ID del tag da aggiornare |  |
| `title` | String | No | Nuovo titolo del tag |  |
| `color` | String | No | Nuovo colore del tag in formato esadecimale |  |

### SetTodoTagsInput

| Parametro | Tipo | Richiesto | Descrizione |
|-----------|------|----------|-------------|
| `todoId` | String! | ✅ Sì | ID del record da taggare |  |
| `tagIds` | [String!] | No | ID dei tag esistenti da applicare |  |
| `tagTitles` | [String!] | No | Titoli dei nuovi tag da creare e applicare |  |

### TagOrderByInput Values

| Valore | Descrizione |
|-------|-------------|
| `id_ASC` / `id_DESC` | Ordina per ID del tag |  |
| `uid_ASC` / `uid_DESC` | Ordina per identificatore unico |  |
| `title_ASC` / `title_DESC` | Ordina alfabeticamente per titolo |  |
| `color_ASC` / `color_DESC` | Ordina per valore di colore |  |
| `createdAt_ASC` / `createdAt_DESC` | Ordina per data di creazione |  |
| `updatedAt_ASC` / `updatedAt_DESC` | Ordina per data dell'ultimo aggiornamento |  |

## Campi di Risposta

### Tipo di Tag

| Campo | Tipo | Descrizione |
|-------|------|-------------|
| `id` | ID! | Identificatore unico per il tag |  |
| `uid` | String! | Identificatore unico a misura d'utente |  |
| `title` | String! | Titolo/nome del tag |  |
| `color` | String! | Colore del tag in formato esadecimale |  |
| `project` | Project! | Progetto a cui appartiene questo tag |  |
| `todos` | [Todo!]! | Record associati a questo tag |  |
| `createdAt` | DateTime! | Quando è stato creato il tag |  |
| `updatedAt` | DateTime! | Quando è stato aggiornato l'ultimo tag |  |

### Risposta di TagPagination

| Campo | Tipo | Descrizione |
|-------|------|-------------|
| `items` | [Tag!]! | Array di record di tag |  |
| `pageInfo` | PageInfo! | Informazioni di paginazione |  |
| `totalCount` | Int! | Numero totale di tag che corrispondono ai filtri |  |

### Campi PageInfo

| Campo | Tipo | Descrizione |
|-------|------|-------------|
| `totalPages` | Int | Numero totale di pagine |  |
| `totalItems` | Int | Numero totale di elementi in tutte le pagine |  |
| `page` | Int | Numero della pagina corrente |  |
| `perPage` | Int | Numero di elementi per pagina |  |
| `hasNextPage` | Boolean! | Se c'è una pagina successiva |  |
| `hasPreviousPage` | Boolean! | Se c'è una pagina precedente |  |

### Visibilità dei Tag

I tag sono visibili a tutti gli utenti che hanno accesso al progetto in cui è stato creato il tag. Potrebbero applicarsi filtri basati sui ruoli per determinate operazioni.

## Migliori Pratiche

1. **Usa Colori Coerenti**: Stabilire un sistema di codifica a colori per il tuo team
2. **Titoli Descrittivi**: Utilizza nomi di tag chiari e descrittivi che i membri del team comprenderanno
3. **Operazioni in Blocco**: Usa `setTodoTags` per applicare più tag contemporaneamente
4. **Cerca e Filtra**: Usa la funzionalità di ricerca per trovare tag esistenti prima di crearne di nuovi
5. **Gestione dei Tag**: Rivedi e pulisci regolarmente i tag non utilizzati
6. **Convenzioni di Colore**: Considera di utilizzare colori standard per tipi di tag comuni (rosso per urgente, verde per completato, ecc.)

## Funzionalità Avanzate

### Impostazioni Predefinite per la Creazione dei Tag

- I nuovi tag creati tramite `setTodoTags` con `tagTitles` predefiniti al colore `#4a9fff`
- I tag senza colori specificati tornano a `#00a0d2`
- I titoli dei tag vengono automaticamente trimmati dagli spazi bianchi

### Integrazione dell'Automazione

I tag possono essere:
- Applicati automaticamente da regole di automazione
- Utilizzati come trigger per flussi di lavoro di automazione
- Modificati tramite azioni di automazione

### Aggiornamenti in Tempo Reale

Le modifiche ai tag vengono trasmesse tramite sottoscrizioni GraphQL, consentendo aggiornamenti in tempo reale nei client connessi.

## Note Importanti

- Eliminare un tag lo rimuove da tutti i record associati
- I tag sono limitati al progetto e non possono essere condivisi tra progetti
- I valori dei colori devono essere codici esadecimali validi (es. "#ff0000", "#00aa00")
- I titoli dei tag sono sensibili alle maiuscole
- Massimo 100 tag possono essere applicati a un singolo record
- I tag supportano la ricerca full-text tra i titoli
