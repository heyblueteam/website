Puoi importare file CSV (Comma Separated Value). Se hai un file Excel, [puoi convertirlo in un file CSV al seguente link](https://cloudconvert.com/xls-to-csv).

## Importazione CSV

L'importazione di file CSV in Blue è un processo semplice. Ogni colonna nel tuo file CSV corrisponde a un campo predefinito o [campo personalizzato](/docs/custom-fields/introduction) in Blue.

Per iniziare, devi essere l'Amministratore del Progetto del progetto in cui desideri importare i dati. Clicca sull'icona delle impostazioni del progetto (icona dell'ingranaggio) e seleziona "Importa".

![Importazione CSV 1](/docs/CSVimport1.png "CSV Import 1")

La funzione di importazione è disponibile nelle Impostazioni del Progetto per gli Amministratori del Progetto.

Nello screenshot sopra, puoi anche vedere un'altra opzione chiamata "Scarica Modello CSV". Questo è estremamente utile se hai già creato i tuoi [campi personalizzati](/docs/custom-fields/introduction) nel tuo progetto Blue, perché puoi scaricare un file CSV che include colonne per tutti i tuoi campi personalizzati! Questo è ottimo quando vuoi assicurarti di avere la struttura precisa per il tuo CSV prima di importare.

Successivamente, basta trascinare e rilasciare il tuo file CSV in Blue:

![Importazione CSV 2](/docs/CSVimport2.png "CSV Import 2")

Trascina e rilascia per caricare i tuoi dati.

A seconda delle dimensioni del tuo file, questo potrebbe richiedere alcuni secondi per essere caricato e per Blue di analizzare. Una volta che Blue ha analizzato i dati, vedrai un'anteprima delle prime dieci righe dei tuoi dati:

![Importazione CSV 3](/docs/CSVimport3.png "CSV Import 3")

Puoi quindi mappare ogni colonna nel tuo CSV a un campo in Blue.

![Importazione CSV 4](/docs/CSVimport4.png "CSV Import 4")

Una volta completata la mappatura, clicca su "Importa" e i tuoi dati verranno importati. La velocità di importazione tipica in Blue è di 10.000 a 15.000 record al minuto.

## Formati di Data Accettati

Blue supporta una vasta gamma di formati di data per l'importazione. Ecco tutti i formati supportati:

- **Timestamp Unix**:
  ```plaintext
  1511648546 (seconds)
  1620021848429 (milliseconds)
  1620024872717915000 (nanoseconds)
  ```
  
- **Standard RFC**:
  ```plaintext
  RFC 3339: 2021-05-01T01:17:02.604456Z
  RFC 2822: Wed, 02 Jun 2021 06:31:39 GMT
  ```

- **Formati di Database**:
  ```plaintext
  PostgreSQL: 
  2019-11-29 08:08-08
  2019-11-29 08:08:05-08
  2021-05-02 23:31:36.0741-07
  ```

- **Formati Comuni**:
  ```plaintext
  Year-Month-Day: 
  2021-04-30 21:14
  2021-02-21
  2021-Feb-21
  
  Month/Day/Year:
  3/31/2014 22:05
  04/02/2014 03:00:51
  
  Day-Month-Year:
  12 Feb 2006, 19:17
  03 February 2013
  ```

- **Fusi Orari**:
  ```plaintext
  2014-04-26 13:13:43 +0800
  2017-11-25 13:31 PST
  2021-02-21 UTC
  ```

- **Lingua Naturale**:
  ```plaintext
  May 6 at 9:24 PM
  September 17, 2012 at 10:09am PST
  October 7, 1970
  ```

- **Formati Internazionali**:
  ```plaintext
  Chinese: 2014年04月08日
  Japanese: 2014/04/08 22:05
  ```

- **Solo Tempo**:
  ```plaintext
  01:06:06
  4:00pm PST
  6:00 AM UTC
  ```

Tutte le date verranno convertite nel fuso orario UTC durante l'importazione. I componenti temporali sono facoltativi: se non forniti, il tempo predefinito sarà 00:00:00 UTC.

## Gestione delle Virgole nei Tuoi Dati

Se i tuoi dati contengono virgole (ad esempio, negli indirizzi o nei nomi delle aziende), devi racchiudere quei valori tra virgolette doppie per prevenire errori di importazione.

**Esempio:**
```
Company,Address
Acme Inc,123 Main St, New York
```

**Dovrebbe diventare:**
```
Company,Address
"Acme Inc","123 Main St, New York"
```

Questo assicura che la virgola in "123 Main St, New York" venga trattata come parte dei dati piuttosto che come un separatore di colonna.
