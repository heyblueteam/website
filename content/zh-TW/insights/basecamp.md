## Basecamp 介紹

Basecamp 是最早的項目管理系統之一——它於 2004 年推出！

該平台最初作為一個內部項目管理工具，由網頁設計公司 [37signals](https://37signals.com/) 創建，該公司由 [Jason Fried](https://world.hey.com/jason) 和 [David Heinemeier Hansson](https://dhh.dk/) 共同創立。

David 也是 [開源網頁框架 Ruby on Rails](https://rubyonrails.org/) 的創建者，許多公司如 [GitHub](https://github.com/)、[Shopify](https://www.shopify.com/)、[Airbnb](https://www.airbnb.com/)、[Square](https://squareup.com/)、[Coinbase](https://www.coinbase.com/) 和 [Zendesk](https://www.zendesk.com/) 都在使用它。

![](/insights/basecamp-home.png)
<p class="text-sm text-gray-500 text-center -mt-8">圖片描述在此</p>


Basecamp 的開發是為了滿足公司自身對更好項目管理和與客戶溝通的需求。這其實與 [Blue 的故事非常相似！](/about)

在推出後，Basecamp 迅速獲得了人氣，促使創始人全心投入，最終於 2014 年將公司重新品牌為 Basecamp，標誌著產品首次發布十周年。他們的重點，與 [Blue](/) 一樣，是 *“令人耳目一新的簡單項目管理。”*

多年來，Basecamp 經營著一個很棒的博客 [Signal vs Noise](https://signalvnoise.com/)，值得一讀。

Basecamp 團隊還出版了幾本有趣的書籍，充滿了反傳統的思考。

- **[Shape Up](https://basecamp.com/books/shapeup)**：為團隊提供工具，以應對產品開發過程中每個階段的風險和未知數。
- **[It Doesn't Have to Be Crazy at Work](https://basecamp.com/books/calm)**：主張冷靜、沉著和專注的工作環境比混亂和高壓的環境更具生產力和可持續性，強調工作與生活平衡的重要性。
- **[Rework](https://basecamp.com/books/rework)**：挑戰傳統商業實踐，倡導更簡單、更高效和不那麼傳統的創業和工作方式。
- **[Remote](https://basecamp.com/books/remote)**：推廣遠程工作的好處和可行性，強調企業和員工如何在不受傳統辦公環境限制的情況下蓬勃發展。
- **[Getting Real](https://basecamp.com/books/getting-real)**：提供有關構建成功的基於網絡的應用程序的實用建議，專注於簡單性、速度和效率，而不是傳統的複雜方法。

## Blue 介紹
Blue 於 2018 年由 [Emanuele "Manny" Faja](https://www.linkedin.com/in/efaja/) 創立，使命明確：

> 通過建立地球上最好的項目管理平台來組織世界的工作——簡單、強大、靈活，並且對所有人都負擔得起。

有趣的是，Manny 進入項目管理世界的旅程是從使用 Basecamp 來管理他的 [產品設計公司](/insights/agency-success-guide) 開始的。他喜歡 Basecamp 的簡單性（您可以在 Basecamp 網站上找到他的 [Basecamp 評價！](https://basecamp.com/customers)），但隨著他的代理商成長，他需要更強大的工具，因此轉向了 [Asana](https://asana.com)。

然而，簡單性不再，對於大多數客戶來說，它過於複雜！

Manny 想要一個 *與 Basecamp 一樣易於使用*，並且 *與 Asana 一樣強大* 的工具。

與 Basecamp 一樣，Blue 自豪地自籌資金。我們在這個行業中屬於少數，大多數競爭對手已經籌集了數億美元的風險投資。這種方法使 Blue 能夠堅持其願景，並保持對用戶需求和產品質量的強烈關注，而不必在投資者需求和客戶需求之間掙扎。

- **每年計劃：** 每位用戶 $70（這樣您可以享受兩個月的免費）。

### Basecamp 計劃細分

Basecamp 有兩個計劃。

1. **Basecamp**：每位用戶收取 $15，並且不包括 [工時表](https://3.basecamp-help.com/article/897-timesheet) 和 [管理專業包](https://3.basecamp-help.com/article/687-admin-pro-pack)。這些附加功能每個月各 $50。
2. **Basecamp Pro Unlimited**：每月 $349，無限用戶和所有功能。

讓我們進一步探討這兩個計劃。

| 功能 | Basecamp (每位用戶) | Basecamp Pro Unlimited |
|---------|---------------------|------------------------|
| 成本 | 每位用戶每月 $15 | 每月 $299（年付）<br>每月 $349（按月付） |
| 理想對象 | 自由職業者、初創企業或小型團隊 | 成長中的企業和大型團隊 |
| 用戶 | 按用戶付費 | 無限用戶，固定費用 |
| 文件存儲 | 500 GB | 5 TB |
| 項目 | 無限 | 無限 |
| 客戶合作者 | 免費（無需收費） | 免費（無需收費） |
| 客戶支持 | 標準 | 優先 24/7 |
| 上線協助 | 無 | 一對一協助 |
| 計費選項 | 可按月計費 | 可按支票年付 |
| 附加功能 | 訪問所有基本項目管理功能 | 包含 Basecamp 計劃的所有功能 |


因此，Blue 的每位用戶費用超過一半的價格（$7 對 $15），而且 Blue 默認包含所有功能——沒有額外的付費附加功能或追加銷售。

與 Basecamp 的 Pro Unlimited 計劃相比，如果您有 49 位用戶或更少，Blue 是更划算的選擇。對於擁有 50 位或更多用戶的團隊，Basecamp Pro Unlimited 在純價格每位用戶的基礎上變得更具成本效益。然而，值得注意的是，Blue 為某些用戶類型（客戶、僅限評論、僅限查看）提供免費帳戶，這可能使其對於大型團隊來說根據用戶類型的組合變得更具成本效益。

## 文件存儲

在文件存儲方面，Blue 和 Basecamp 採取了不同的路徑。Blue 通過在所有計劃中提供慷慨的文件存儲來保持簡單。這意味著無論您的團隊有多大或項目多麼龐大，您都不必擔心空間不足。此外，Blue 每個文件的上傳限制為 5GB，能夠輕鬆滿足大多數商業需求。

然而，Basecamp 選擇了基於您選擇的計劃的分級存儲系統。他們的基本計劃提供 500GB 的存儲，這對於小型到中型團隊或項目來說應該足夠。如果您的業務有更大的需求，那麼他們的 Pro Unlimited 計劃就能發揮真正的優勢，提供高達 5TB 的存儲。這種靈活的方法使團隊能夠隨著成長擴展存儲。

需要考慮的一件事是，Basecamp 沒有明確說明上傳的最大文件大小。與 Blue 不同，Blue 明確將其限制設置為 5GB，而 Basecamp 在這一細節上則有些模糊。如果您經常處理大型文件，這一遺漏可能會引發一些警示。

您在 Blue 和 Basecamp 之間的選擇可能取決於您的需求。如果您更喜歡無限存儲和明確的文件大小限制，Blue 可能是更好的選擇。擁有不會達到上限的安心感以及合理的文件大小限制，Blue 為大多數情況提供了簡單的解決方案。

另一方面，如果您認為您的存儲需求將符合 Basecamp 的框架，他們的計劃可能適合您。基本計劃的 500GB 對於許多團隊來說相當慷慨，而 Pro Unlimited 計劃的 5TB 提供了充足的增長空間。只需確保在經常處理超過 5GB 的文件時檢查 Basecamp 的上傳限制。

最終，Blue 和 Basecamp 都提供強大的存儲選項。


## API 和 Webhook

### API

Basecamp 擁有 [文檔完善的 Rest API](https://github.com/basecamp/api)，然而，即使是 Basecamp 也承認它並不全面：

> Basecamp API 並不打算成為我們所有應用程序中您可以做的所有事情的完整模型。事實上，少於百分之五的最終用戶利用與 Basecamp 集成的應用程序或編寫自己的集成。如此低的使用率意味著繼續更新 API 以保持與所有新功能的同步並不是 Basecamp 的高優先事項。當我們為那不到百分之五的用戶工作時，我們並沒有幫助其他 99.5%。

少於百分之五的最終用戶利用 API 的事實，可能是因為他們的目標受眾是尋找簡單工具的典型小企業。

[Blue GraphQL API](/platform/api) 100% 覆蓋了 Blue 中的所有功能。

因此，您可以通過我們的界面執行的 *任何* 操作，都可以以編程方式觸發。這是在我們首次創建 API 時的有意設計選擇，我們希望為開發人員提供對所有內容的完全訪問。

每次我們 [發布新功能](/platform/changelog) 時，API 都會更新以支持這些新功能。

### Webhook


Webhook 是在特定事件發生時從一個應用程序自動發送到另一個應用程序的消息。在項目管理軟體的上下文中，Webhook 非常有用，因為它們允許不同工具之間的無縫通信。例如，在項目管理軟體中，Webhook 可以觸發動作，如更新任務狀態、創建通知或實時與其他軟體同步數據。這使得項目管理工具與其他系統之間的集成更為順暢，簡化了工作流程，提高了效率。

Basecamp 曾在 Basecamp 3 中擁有 [Webhook](https://github.com/basecamp/bc3-api/blob/master/sections/webhooks.md)，但在網上沒有提到 Basecamp 4 擁有 Webhook，我們在測試該軟體時也找不到任何。

然而，[Basecamp 確實與第三方軟體有一些集成](https://basecamp.com/extras)。作為比較， [您也可以查看 Blue 提供的集成列表。](/platform/integrations)


## 用戶界面

雖然對用戶界面的評價總是有些主觀，但 Basecamp 和 Blue 之間存在值得討論的顯著差異。

Basecamp 的界面雖然乾淨且簡單，但與更現代的項目管理工具相比，可能會感覺有些過時。這不一定是負面的——許多用戶欣賞它的簡單性和熟悉感。然而，對於習慣於更當代設計的團隊來說，它可能會感覺有些老派。

Basecamp 和 Blue 之間用戶體驗的最顯著差異之一是項目導航。

Blue 還強調其界面中的數據可視化和報告功能。用戶可以直接從項目視圖中快速生成圖表、圖形和報告，提供即時見解，而無需導航到單獨的報告工具。

## 企業結構的差異

在選擇項目管理工具時，重要的是要超越其功能。產品背後的公司也很重要。他們的企業結構和重點可能會影響工具的發展及其長期可持續性。

讓我們深入探討 Basecamp 和 Blue 在這方面的不同之處。

Basecamp 是由 37signals 創建的，這是一家主要由其創始人擁有的公司，有一個有趣的例外：亞馬遜創始人 Jeff Bezos 擁有一小部分股份。[雖然 37signals 喜歡說他們沒有投資者](https://37signals.com/01)，但由於 Bezos 的參與，這並不完全真實。因此，儘管他們享有相當大的獨立性，但仍需考慮外部利益相關者的利益。

37signals 和 Blue 之間的一個主要區別是他們的產品重點。37signals 不僅專注於 Basecamp；他們還創建和維護其他產品，如 [Hey](https://hey.com)，一個電子郵件服務，以及流行的網頁框架 Ruby on Rails。最近，他們增加了 [Once](https://once.com)，包括兩個新產品：[Campfire](https://once.com/campfire)，一個聊天工具，以及 [Writebook](https://once.com/writebook)，用於協作寫作。

這種廣泛的關注意味著 37signals 必須在各種項目和行業之間分配其注意力、資源和專業知識。

而這種分散的注意力確實影響了 Basecamp 的發展。重大更新通常每幾年才會發生一次。雖然這種穩定的方法有其好處，但可能無法跟上項目管理需求或行業新趨勢的快速變化。

許多 Basecamp 用戶發現，即使他們的工作流程和挑戰發生變化，他們仍在使用相同的工具。

在 Blue，我們相信改進是逐步累積的。我們專注於項目管理平台。這使得團隊能夠全力以赴地提升 Blue 的功能。Blue 獨立運營，沒有外部投資者的干擾。這種獨立性，加上單一的焦點，幫助 Blue 快速響應用戶反饋和市場變化。

Blue 的奉獻體現在其開發周期中。該平台每月獲得 [定期更新和增強](/platform/changelog)。這種敏捷的方法意味著 Blue 可以迅速適應用戶建議並滿足新興需求。用戶可以期待頻繁的改進、新功能和精細調整的工具，確保平台保持新鮮和有效。

因此，在 Basecamp 和 Blue 之間做出決定時，請考慮您是否想要一個隨時間變化不大的平台，還是希望一個不斷適應您需求的工具。如果您重視定期更新和不斷提升項目管理體驗的承諾，您可能會發現 Blue 更適合您。

最終，您的決定應該反映您團隊的獨特需求和偏好。在當今快速變化的商業世界中，擁有一個不斷發展並持續增值的項目管理工具可以真正使您脫穎而出。
