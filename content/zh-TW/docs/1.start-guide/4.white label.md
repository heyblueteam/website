## 簡介

白標服務將 Blue 轉換為一個靈活的專案和流程管理平台，看起來完全像是您自己的產品——無需承擔管理技術、開發人員、伺服器或正常運行時間的負擔。

使用 Blue 的白標服務，您可以：

- **展示您的品牌：** 用您自己的名稱、標誌和顏色取代 Blue 的品牌元素，創造無縫的品牌體驗。
- **在您的網域上運行：** 在您自己的網域上操作 Blue，打造專業、一致的形象。
- **運用您的電子郵件基礎設施：** 從您的電子郵件地址發送電子郵件，提供個人化的溝通體驗。


對於代理商而言，白標附加功能能夠強化您的品牌、提升服務內容，並簡化客戶協作。通過在您的品牌下將 Blue 整合到您的工作流程中，您可以創造出無縫、專業的體驗，協助您吸引更多客戶並通過提供量身訂制的服務和軟體組合來留住他們。


## 定價

白標附加功能每個公司每月收費 $20 或每個組織每年收費 $200，這是在您的核心 Blue 訂閱之外的額外費用。您可以在[此處](/pricing)了解更多定價資訊。

如果您有興趣為多個組織批量購買白標服務，請聯繫 sales@blue.cc
 

## 自訂網域

要使白標服務正常運作，您需要首先為網頁應用程式設定自訂網域。

可以為以下項目配置自訂網域：

- 網頁應用程式（必須）
- [表單](/docs/projects/forms)（可選）
- [檔案分享連結](/docs/projects/file-management)（可選）
- 文件（即將推出）


要配置自訂網域，請為每個子網域在您的 DNS 設定中添加 CNAME 記錄。請注意，僅支援子網域，不支援根網域。因此 `app.yourcompany.com` 是支援的，但 `yourcompany.com` 不支援。


| 網域類型                | CNAME 記錄                  | 指向                       |
|----------------------------|-------------------------------|----------------------------|
| 網頁應用程式            | `app.yourcompany.com`         | `wl-app.onrender.com`    |
| 表單                      | `forms.yourcompany.com`       | `wl-forms.onrender.com`    |
| 檔案分享連結         | `files.yourcompany.com`       | `wl-files.onrender.com`    |


上表提供了建議的 CNAME 記錄，但您可以使用任何您想要的子網域。

例如，主要網頁應用程式可以是 `projects.yourcompany.com`，檔案分享連結可以是 `media.yourcompany.com`。


## 品牌設定

自訂以下元素以使 Blue 與您的品牌保持一致：

- **標誌**：上傳您的標誌以取代平台上的 Blue 品牌元素。
- **網站圖示**：添加您自己的網站圖示，它將出現在瀏覽器標籤和書籤中。
- **主要介面顏色**：為介面設定主要顏色，在所有使用者中保持一致應用。
- **應用程式名稱**：自訂在瀏覽器標籤和桌面應用程式中顯示的應用程式名稱。
- **字體**：（即將推出）從精選的 Google 字體清單中自訂平台使用的字體以符合您的品牌。

Blue 在您配置品牌設定時提供登入頁面和應用程式圖示的即時預覽。

您的品牌設定也將應用於[桌面應用程式](/docs/start-guide/download-apps#desktop-app)，確保完全的白標體驗。

以下是標誌和網站圖示的建議配置：

| 資產   | 建議形狀 | 背景      | 長寬比 |
|---------|------------------|-----------------|-------------|
| 標誌    | 矩形        | 透明     | 1:1.5       |
| 網站圖示 | 正方形           | 透明     | 1:1         |


## 電子郵件通知

您可以使用 SMTP 從您自己的電子郵件伺服器發送電子郵件通知，使用您自己的網域而不是我們的電子郵件地址。

您需要向 Blue 提供以下資訊：

| 設定                          | 描述                                                                                                                                         |
|-----------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------|
| **SMTP 伺服器**                   | 處理您電子郵件發送的伺服器地址。                                                                              |
| **SMTP 連接埠**                     | 用於連接到 SMTP 伺服器的連接埠號，通常 TLS 使用 587 或 SSL 使用 465。                                                           |
| **使用者名稱**                      | 用於 SMTP 伺服器身份驗證的使用者名稱，通常是您的電子郵件地址。                                                                  |
| **密碼（以加密格式儲存）** | 與使用者名稱關聯的密碼，將以加密格式安全儲存以保護您的憑證。              |
| **寄件人名稱**                   | 將在電子郵件通知中顯示為寄件人的名稱，幫助收件人識別電子郵件來源。                            |
| **寄件人電子郵件地址**          | 將顯示為寄件人的電子郵件地址，確保回覆發送到正確的地址。請注意，這必須與 SMTP 伺服器相同的網域。 |

Blue 強制對 SMTP 連接使用 TLS/SSL 加密。

然後您可以點擊「測試連接」按鈕來測試與您的 SMTP 伺服器的連接。Blue 將自動測試與您的 SMTP 伺服器的連接，並通過應用程式內通知告知您測試是否成功。

### 注意事項 

所有交易型、通知和自動化電子郵件都將通過您的 SMTP 伺服器發送。唯一的例外是包含 OTP（一次性密碼）的電子郵件，這些電子郵件通過通用電子郵件地址（notifications@process-system.app）發送以確保安全。

確保您的 SMTP 伺服器：

- 能夠處理大量電子郵件。
- 具有良好的發送聲譽以避免垃圾郵件過濾器。
- 有足夠的配額來容納預期的電子郵件負載。


### 自訂邀請電子郵件

您也可以自訂發送給新使用者註冊您的 Blue 實例時的初始邀請電子郵件。

### 自訂入門電子郵件（即將推出）

沒有白標服務時，Blue 會向加入公司的新使用者發送一系列入門電子郵件以幫助他們開始使用。當啟用白標服務時，這些電子郵件不會被發送，將來您將能夠創建自己的一套入門電子郵件來歡迎新使用者。

## 行動應用程式

我們目前不計劃支援行動應用程式的白標服務，但如果您希望在自己的 Apple App Store 和 Google Play Store 帳戶上擁有自己品牌版本的行動應用程式，您可以聯繫 sales@blue.cc 討論此事。

## 常見問題


**白標附加功能包含什麼？**

白標附加功能包含三個主要功能：為網頁應用程式、表單和檔案分享的自訂網域整合；自訂品牌設定（標誌、網站圖示、顏色和應用程式名稱）；以及使用您自己的 SMTP 伺服器的自訂電子郵件通知。

**白標服務費用是多少？**

白標附加功能在您的核心 Blue 訂閱之外，每個公司每月收費 $20 或每個公司每年收費 $200。將來會為擁有多個公司的帳戶提供批量折扣。

**白標服務會影響 Blue 的功能嗎？**

不會，Blue 的所有核心功能都保持完全相同。白標服務只會改變平台的外觀和品牌。


**我可以使用根網域而不是子網域嗎？**

不可以，Blue 僅支援子網域進行自訂網域設定。

**我需要設定所有的自訂網域嗎？**

不需要，只有網頁應用程式網域是必須的。表單和檔案分享網域是可選的。文件網域支援即將推出。

**我可以使用與建議不同的子網域名稱嗎？**

可以，您可以使用任何您喜歡的子網域名稱。例如，您可以使用 `projects.yourcompany.com` 而不是 `app.yourcompany.com`。

**我可以自訂哪些品牌元素？**

您可以自訂您的標誌、網站圖示、主要介面顏色和應用程式名稱。

**桌面應用程式會反映我的品牌嗎？**

是的，桌面應用程式將完全以您的品牌進行白標。

**不同使用者可以有不同的顏色方案嗎？**

不可以，您選擇的主要顏色將在您的 Blue 實例的所有使用者中強制執行。

**我需要提供哪些電子郵件配置詳細資訊？**

您需要提供 SMTP 伺服器地址、SMTP 連接埠、使用者名稱、密碼、寄件人名稱和寄件人電子郵件地址。SMTP 密碼以加密格式儲存以保護您的憑證。

**我可以在正式啟用前測試 SMTP 配置嗎？**

可以，Blue 提供「測試連接」按鈕，將驗證您的 SMTP 伺服器配置並通知您結果。

**哪些類型的電子郵件會通過我的 SMTP 伺服器發送？**

所有電子郵件都將通過您的 SMTP 伺服器發送，包括交易型電子郵件、通知電子郵件、電子郵件自動化。OTP（一次性密碼）電子郵件通過通用電子郵件地址（notifications@process-system.app）發送以確保安全。


**我可以對行動應用程式進行白標嗎？**

Blue 目前不支援行動應用程式的白標。但是，如果您需要在 Apple App Store 或 Google Play Store 上的品牌版本，可以聯繫 sales@blue.cc 討論選項。

**未來會提供行動應用程式的白標支援嗎？**

目前沒有支援行動應用程式白標的計劃。

**如何獲得白標設定的幫助？**

一般幫助請聯繫 support@blue.cc。有關自訂行動應用程式的銷售相關問題，請聯繫 sales@blue.cc。

**我可以在實施前測試白標功能嗎？**

可以，Blue 在您配置品牌設定時提供登入頁面和應用程式圖示的預覽。

## DNS 設定

我們已經方便地收集了常見 DNS（網域名稱服務）系統的白標網域設定說明：

#### GoDaddy

1. 登入您的 GoDaddy 帳戶，並開啟您的產品。
2. 導航到網域控制中心。
3. 在網域設定頁面上，找到並選擇您的網域名稱。
4. 向下滾動到其他設定部分，並選擇管理 DNS。
5. 在 DNS 管理頁面上，向下滾動到記錄部分的底部，並點擊添加。
6. 在類型下拉選單中，選擇 CNAME，並完成以下必填欄位：
    - 主機 - CNAME 的主機名稱或前綴。您可以使用句點（.），但不能作為第一個或最後一個字符。不允許連續句點（..）。主機不能超過 25 個字符或是 @ 符號。
    - 指向 - 將此設定為 `wl-app.onrender.com`
    - TTL - 指定伺服器必須快取資訊的時間量。
7. 點擊儲存。

#### Bluehost

1. 登入您的 BlueHost 帳戶。
2. 點擊網域選單，然後點擊區域編輯器。出現新頁面。
3. 導航到添加 DNS 記錄部分。
4. 在主機記錄欄位中，輸入您要用於自訂網域的地址的子網域。
5. 在類型下拉選單中，選擇 CNAME 選項。
6. 在指向部分，輸入 `wl-app.onrender.com`
7. 點擊添加記錄。

#### Cloudflare

1. 登入您的 Cloudflare 帳戶。
2. 在下拉選單中，找到並選擇您的網域。
3. 點擊 DNS 設定標籤。
4. 點擊「添加記錄」：
5. 選擇 CNAME 選項。
6. 將「名稱」設定為您的子網域
7. 將「目標」設定為 `wl-app.onrender.com`
8. 將 Cloudflare 代理切換圖示設定為灰色雲朵。

#### Domain.com

1. 登入您的 Domain.com 帳戶。
2. 找到您的網域並點擊管理。
3. 在左側選單上，點擊 DNS 和名稱伺服器。
4. 點擊添加 DNS 記錄來添加 CNAME 記錄。
5. 將目標設定為 `wl-app.onrender.com`
6. 點擊更新 DNS 完成流程。

#### Dreamhost

1. 登入您的 DreamHost 帳戶。
2. 在頁面左側，點擊網域連結。
3. 在網域下拉選單中，選擇管理網域。
4. 找到您的網域並點擊您網域下的 DNS 連結。
5. 向下滾動到添加自訂 DNS 記錄到（您的自訂網域）部分並編輯以下欄位：
    - 名稱 - 輸入您要使用的子網域
    - 類型 - 選擇 CNAME 選項。
    - 值 - 輸入 `wl-app.onrender.com`
6. 點擊立即添加記錄！來添加 CNAME 記錄。

#### Google Domains

1. 登入您的 Google Domains 帳戶。
2. 找到並點擊您的網域名稱。
3. 在左側選單上，點擊 DNS。
4. 導航到自訂資源記錄部分。
5. 在名稱欄位中，輸入您要用於自訂網域的地址的子網域。
6. 在類型下拉選單中，選擇 CNAME 選項。
7. 在資料欄位中，輸入 `wl-app.onrender.com`
8. 點擊添加完成流程。

#### Hostgator

1. 登入您的 HostGator 帳戶。
2. 在頁面左側，點擊 CPanel。
3. 向下滾動到網域部分，並點擊區域編輯器。
4. 找到您的網域並點擊網域旁邊的 CName 記錄連結。
5. 在添加 CName 記錄對話框中，完成以下空白欄位：
    - 名稱 - 輸入您選擇的子網域/網域名稱。
    - CNAME - 輸入 `wl-app.onrender.com`
6. 點擊添加 CName 記錄。來自 HostGator 的訊息出現，說明您已成功向您的 HostGator 帳戶添加新的 CNAME。

#### NameCheap

1. 登入您的 NameCheap 帳戶。
2. 在帳戶下拉選單中，選擇儀表板。
3. 點擊網域清單並找到您的網域。
4. 點擊您網域旁邊的管理。
5. 點擊進階 DNS 並導航到主機記錄部分。
6. 點擊添加新記錄。
7. 在類型下拉選單中，選擇 CNAME 記錄並輸入您的子網域作為主機。
8. 然後輸入 `wl-app.onrender.com`
9. 點擊儲存所有變更完成流程。
