Zapier 是一個集成平台，幫助連接不同的網絡應用程序並自動化工作流程。

**使用案例**

以下是 Blue 通過 Zapier 與其他應用程序集成的一些最受歡迎的方式：

**_將 Blue 記錄同步到 Google Sheets_**

隨著 Blue 記錄的創建或更新，自動將它們附加為 Google 表格中的新行，以便集中報告。

**_從 Blue 記錄發送訂單確認_**

當您在 Blue 上選擇某些自定義字段時，自動向您的客戶發送電子郵件以確認訂單。

**_將 Blue 評論添加到 Slack_**

獲取實時 Blue 通知，發佈到正確的 Slack 頻道，以便您的團隊了解重要的記錄更新。

## 主要好處

**跨工具自動化**：在 Zapier 中創建的 Zaps 可以自動在 Blue 和成千上萬的其他應用程序之間發送和接收數據。這消除了在系統之間移動數據的手動工作。

**實時集成**：使用 Blue 的 webhook 觸發器，Zaps 會立即對 Blue 中的事件做出反應，例如新記錄、添加評論、狀態更改等。這使得實時同步和流程自動化成為可能。

**易於構建和維護**：Zaps 提供了一種用戶友好的無代碼方式來集成工具。無需開發工作。它們可以持續運行，無需持續維護。

## 在 Blue 中設置 Webhook

</video>

1. 登錄到您的 Blue 帳戶，進入個人資料設置
2. 前往個人資料設置並點擊「Webhooks」
3. 點擊「添加 Webhook」
4. 為 webhook 命名並輸入 Zapier webhook URL
5. 選擇哪些事件應該觸發 webhook
6. 點擊「創建 Webhook」

## Blue 到 Zapier

1. 登錄到您的 Zapier 帳戶
2. 點擊創建 Zap  
3. 在第一步中，搜索並選擇 ["Webhooks by Zapier" 觸發應用](https://zapier.com/apps/webhook)
4. 配置以接受來自您的 Blue webhook URL 的有效載荷
5. 在第二步中，搜索並選擇要發送數據的目標應用
6. 將 JSON 有效載荷字段映射到目標應用的輸入字段
7. 開啟 Zap

現在，當 Blue 中發生指定事件時，webhook 有效載荷將發送到 Zapier，Zapier 將根據 Zap 設置自動將數據傳遞到您的集成應用。

## Zapier 到 Blue

要將數據從其他應用程序推送到 Blue，請按照以下步驟操作：

**1. 獲取 Blue API 憑證**
- 前往 Blue 設置 > API 令牌
- 創建新令牌或使用現有憑證：
  - 令牌 ID
  - 令牌密鑰
  - 公司 ID
  - 項目 ID

**2. 在 Zapier 中創建 Zap**
1. 設置您的觸發應用（例如，Google Sheets、Slack 等）
2. 添加「Webhooks by Zapier」作為操作應用
3. 選擇「自定義請求」事件類型

**3. 配置 Webhook**
- 方法：POST
- URL：`https://api.blue.cc/graphql`（如有需要可使用 beta 端點）
- 數據通過：FALSE
- 不展平：NO

**4. 設置標頭：**
```json
{
  "Content-Type": "application/json",
  "X-Bloo-Token-ID": "Your_Token_ID",
  "X-Bloo-Token-Secret": "Your_Token_Secret", 
  "X-Bloo-Company-ID": "Your_Company_ID",
  "X-Bloo-Project-ID": "Your_Project_ID"
}
```

**5. 配置查詢（示例變更）：**
```json
{
  "query": "mutation CreateTodo($input: CreateTodoInput!) {\n  createTodo(input: $input) {\n    id\n    title\n    customFields {\n      value\n    }\n  }\n}",
  "variables": {
    "input": {
      "todoListId": "your-todo-list-id",
      "title": "New task from Zapier",
      "customFields": [
        {
          "customFieldId": "priority-field-id",
          "value": "High"
        },
        {
          "customFieldId": "due-date-field-id",
          "value": "2024-03-01"
        }
      ]
    }
  }
}
```

**6. 測試並啟用**
- 首先使用 Blue 的 API Playground 測試變更
- 使用 Zapier 的數據字段映射觸發應用中的動態值
- 測試成功後啟用 Zap
