Blue 允許您使用 Google 地圖作為地圖視圖，而不是默認的 OpenStreetMap。Google 地圖通常對某些區域提供更詳細的地圖和更準確的位置數據。然而，這需要一個 Google 地圖 API 密鑰，並必須由公司擁有者在公司設置中輸入。

當您切換到 Google 地圖時，這會影響 Blue 中的所有地圖視圖，包括公司地圖、[項目中的地圖視圖](/docs/views/map)和[位置自定義字段](/docs/custom-fields/location-map)。

要在 Blue 中使用 Google 地圖，需要在 Google Cloud Console 中啟用以下 API：
- [地圖瓷磚 API](https://console.cloud.google.com/apis/library/tile.googleapis.com)：這是渲染地圖瓷磚所必需的。
- [地點 API](https://console.cloud.google.com/apis/library/places.googleapis.com)：這是搜索地點並獲取其坐標所必需的。
- [地理編碼 API](https://console.cloud.google.com/apis/library/geocoding-backend.googleapis.com)：這是獲取地點坐標所必需的。

您需要註冊一個 [Google Cloud 帳戶](https://console.cloud.google.com/)並創建一個項目以獲取 API 密鑰，具體說明請參見 [這裡](https://developers.google.com/maps/documentation/embed/get-api-key)。

## 限制 API 密鑰使用

出於安全考慮，限制您的 Google 地圖 API 密鑰僅在授權的域名上工作至關重要。這可以防止在密鑰被暴露的情況下未經授權的使用。您應該將 API 密鑰限制為僅在以下域名上工作：

- app.blue.cc - 主要的 Blue 應用程序
- forms.blue.cc - 用於公共表單
- 如果您使用 [白標](/docs/start-guide/white-label)，則為您的自定義域名

要限制您的 API 密鑰：

1. 前往 [Google Cloud Console 憑證頁面](https://console.cloud.google.com/apis/credentials)
2. 找到您的 API 密鑰並點擊“編輯”（鉛筆圖標）
3. 在“應用程序限制”下，選擇“HTTP 引用者（網站）”
4. 添加以下引用者：
   - `*.app.blue.cc/*`
   - `*.forms.blue.cc/*`
   - `*.yourdomain.com/*`（如果使用白標）
5. 點擊“保存”

**關於 Google 地圖收費的重要說明**<br><br>

Google 將單獨對您使用 Google 地圖 API 收取費用，這與您的 Blue 訂閱無關。與您的 Google 地圖帳戶和 API 使用相關的所有費用、開支和費用均由組織管理員全權負責。Blue 對您使用 Google 地圖 API 所產生的任何費用不承擔責任，包括但不限於第三方集成、API 使用或存儲成本。
