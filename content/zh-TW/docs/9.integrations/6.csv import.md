您可以匯入 CSV（逗號分隔值）文件。如果您有 Excel 文件，[您可以在以下鏈接將其轉換為 CSV 文件](https://cloudconvert.com/xls-to-csv)。

## 匯入 CSV

在 Blue 中匯入 CSV 文件是一個簡單的過程。您 CSV 文件中的每一列對應於 Blue 中的默認或 [自定義字段](/docs/custom-fields/introduction)。

首先，您必須是要匯入數據的項目的項目管理員。點擊項目設置圖標（齒輪圖標），然後選擇“匯入”

![CSV 匯入 1](/docs/CSVimport1.png "CSV Import 1")

匯入功能在項目設置下對項目管理員可用

在上面的截圖中，您還可以看到另一個選項，稱為“下載 CSV 模板”。如果您已經在 Blue 項目中創建了您的 [自定義字段](/docs/custom-fields/introduction)，這將非常有用，因為您可以下載一個包含所有自定義字段列的 CSV 文件！這在您想確保在匯入之前擁有精確的 CSV 結構時非常有用。

接下來，只需將您的 CSV 文件拖放到 Blue 中：

![CSV 匯入 2](/docs/CSVimport2.png "CSV Import 2")

拖放以上傳您的數據

根據您的文件大小，上傳和 Blue 分析可能需要幾秒鐘。一旦 Blue 分析了數據，您將看到數據的前十行的預覽：

![CSV 匯入 3](/docs/CSVimport3.png "CSV Import 3")

然後，您可以將 CSV 中的每一列映射到 Blue 中的字段。

![CSV 匯入 4](/docs/CSVimport4.png "CSV Import 4")

## 接受的日期格式

Blue 支持多種日期格式的匯入。以下是所有支持的格式：

- **Unix 時間戳**：
  ```plaintext
  1511648546 (seconds)
  1620021848429 (milliseconds)
  1620024872717915000 (nanoseconds)
  ```
  
- **RFC 標準**：
  ```plaintext
  RFC 3339: 2021-05-01T01:17:02.604456Z
  RFC 2822: Wed, 02 Jun 2021 06:31:39 GMT
  ```

- **數據庫格式**：
  ```plaintext
  PostgreSQL: 
  2019-11-29 08:08-08
  2019-11-29 08:08:05-08
  2021-05-02 23:31:36.0741-07
  ```

- **常見格式**：
  ```plaintext
  Year-Month-Day: 
  2021-04-30 21:14
  2021-02-21
  2021-Feb-21
  
  Month/Day/Year:
  3/31/2014 22:05
  04/02/2014 03:00:51
  
  Day-Month-Year:
  12 Feb 2006, 19:17
  03 February 2013
  ```

- **時區**：
  ```plaintext
  2014-04-26 13:13:43 +0800
  2017-11-25 13:31 PST
  2021-02-21 UTC
  ```

- **自然語言**：
  ```plaintext
  May 6 at 9:24 PM
  September 17, 2012 at 10:09am PST
  October 7, 1970
  ```

- **國際格式**：
  ```plaintext
  Chinese: 2014年04月08日
  Japanese: 2014/04/08 22:05
  ```

- **僅時間**：
  ```plaintext
  01:06:06
  4:00pm PST
  6:00 AM UTC
  ```

所有日期在匯入過程中將轉換為 UTC 時區。時間組件是可選的 - 如果未提供，時間將默認為 00:00:00 UTC。

## 處理數據中的逗號

如果您的數據包含逗號（例如，在地址或公司名稱中），您需要將這些值用雙引號括起來，以防止匯入錯誤。

**示例：**
```
Company,Address
Acme Inc,123 Main St, New York
```

**應變為：**
```
Company,Address
"Acme Inc","123 Main St, New York"
```

這確保了“123 Main St, New York”中的逗號被視為數據的一部分，而不是列分隔符。
