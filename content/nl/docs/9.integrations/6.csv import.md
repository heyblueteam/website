Je kunt CSV (Comma Separated Value) bestanden importeren. Als je een Excel-bestand hebt, [kun je het omzetten naar een CSV-bestand via de volgende link](https://cloudconvert.com/xls-to-csv).

## CSV Importeren

Het importeren van CSV-bestanden in Blue is een eenvoudig proces. Elke kolom in je CSV-bestand komt overeen met een standaard of [aangepast veld](/docs/custom-fields/introduction) in Blue.

Om te beginnen, moet je de Projectbeheerder zijn van het project waarin je gegevens wilt importeren. Klik op het pictogram voor projectinstellingen (tandwielpictogram) en selecteer "Importeren".

![CSV Import 1](/docs/CSVimport1.png "CSV Import 1")

De importfunctie is beschikbaar onder Projectinstellingen voor Projectbeheerders.

In de bovenstaande screenshot zie je ook een andere optie genaamd "CSV-sjabloon downloaden". Dit is uiterst handig als je al je [aangepaste velden](/docs/custom-fields/introduction) in je Blue-project hebt aangemaakt, omdat je een CSV-bestand kunt downloaden dat kolommen bevat voor al je aangepaste velden! Dit is geweldig wanneer je ervoor wilt zorgen dat je de juiste structuur voor je CSV hebt voordat je importeert.

Sleep vervolgens eenvoudig je CSV-bestand naar Blue:

![CSV Import 2](/docs/CSVimport2.png "CSV Import 2")

Sleep en laat vallen om je gegevens te uploaden.

Afhankelijk van de bestandsgrootte kan dit enkele seconden duren om te uploaden en voor Blue om te analyseren. Zodra Blue de gegevens heeft geanalyseerd, zie je een voorbeeld van de eerste tien rijen van je gegevens:

![CSV Import 3](/docs/CSVimport3.png "CSV Import 3")

Je kunt vervolgens elke kolom in je CSV toewijzen aan een veld in Blue.

![CSV Import 4](/docs/CSVimport4.png "CSV Import 4")

## Geaccepteerde Datumformaten

Blue ondersteunt een breed scala aan datumformaten voor import. Hier zijn alle ondersteunde formaten:

- **Unix Timestamps**:
  ```plaintext
  1511648546 (seconds)
  1620021848429 (milliseconds)
  1620024872717915000 (nanoseconds)
  ```
  
- **RFC-standaarden**:
  ```plaintext
  RFC 3339: 2021-05-01T01:17:02.604456Z
  RFC 2822: Wed, 02 Jun 2021 06:31:39 GMT
  ```

- **Databaseformaten**:
  ```plaintext
  PostgreSQL: 
  2019-11-29 08:08-08
  2019-11-29 08:08:05-08
  2021-05-02 23:31:36.0741-07
  ```

- **Algemene formaten**:
  ```plaintext
  Year-Month-Day: 
  2021-04-30 21:14
  2021-02-21
  2021-Feb-21
  
  Month/Day/Year:
  3/31/2014 22:05
  04/02/2014 03:00:51
  
  Day-Month-Year:
  12 Feb 2006, 19:17
  03 February 2013
  ```

- **Tijdzones**:
  ```plaintext
  2014-04-26 13:13:43 +0800
  2017-11-25 13:31 PST
  2021-02-21 UTC
  ```

- **Natuurlijke taal**:
  ```plaintext
  May 6 at 9:24 PM
  September 17, 2012 at 10:09am PST
  October 7, 1970
  ```

- **Internationale formaten**:
  ```plaintext
  Chinese: 2014年04月08日
  Japanese: 2014/04/08 22:05
  ```

- **Alleen tijd**:
  ```plaintext
  01:06:06
  4:00pm PST
  6:00 AM UTC
  ```

Alle datums worden tijdens de import geconverteerd naar UTC-tijdzone. Tijdcomponenten zijn optioneel - als ze niet worden opgegeven, wordt de tijd standaard 00:00:00 UTC.

## Omgaan met Komma's in Je Gegevens

Als je gegevens komma's bevatten (bijvoorbeeld in adressen of bedrijfsnamen), moet je die waarden tussen dubbele aanhalingstekens plaatsen om importfouten te voorkomen.

**Voorbeeld:**
```
Company,Address
Acme Inc,123 Main St, New York
```

**Moet worden:**
```
Company,Address
"Acme Inc","123 Main St, New York"
```

Dit zorgt ervoor dat de komma in "123 Main St, New York" als onderdeel van de gegevens wordt behandeld in plaats van als een kolomscheider.
