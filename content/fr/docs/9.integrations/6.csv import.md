Vous pouvez importer des fichiers CSV (Comma Separated Value). Si vous avez un fichier Excel, [vous pouvez le convertir en fichier CSV au lien suivant](https://cloudconvert.com/xls-to-csv).

## Importation de CSV

L'importation de fichiers CSV dans Blue est un processus simple. Chaque colonne de votre fichier CSV correspond à un champ par défaut ou à un [champ personnalisé](/docs/custom-fields/introduction) dans Blue.

Pour commencer, vous devez être l'administrateur du projet dans lequel vous souhaitez importer des données. Cliquez sur l'icône des paramètres du projet (icône d'engrenage) et sélectionnez "Importer".

![Importation CSV 1](/docs/CSVimport1.png "CSV Import 1")

La fonctionnalité d'importation est disponible dans les paramètres du projet pour les administrateurs de projet.

Dans la capture d'écran ci-dessus, vous pouvez également voir une autre option appelée "Télécharger le modèle CSV". Cela est extrêmement utile si vous avez déjà créé vos [champs personnalisés](/docs/custom-fields/introduction) dans votre projet Blue, car vous pouvez télécharger un fichier CSV qui inclut des colonnes pour tous vos champs personnalisés ! C'est idéal lorsque vous souhaitez vous assurer que vous avez la structure précise pour votre CSV avant l'importation.

Ensuite, il vous suffit de faire glisser et de déposer votre fichier CSV dans Blue :

![Importation CSV 2](/docs/CSVimport2.png "CSV Import 2")

Faites glisser et déposez pour télécharger vos données.

Selon la taille de votre fichier, cela peut prendre quelques secondes pour que vous puissiez télécharger et pour que Blue analyse. Une fois que Blue a analysé les données, vous verrez un aperçu des dix premières lignes de vos données :

![Importation CSV 3](/docs/CSVimport3.png "CSV Import 3")

Vous pouvez ensuite mapper chaque colonne de votre CSV à un champ dans Blue.

![Importation CSV 4](/docs/CSVimport4.png "CSV Import 4")

Une fois le mappage terminé, cliquez sur "Importer" et vos données seront importées. La vitesse d'importation typique dans Blue est de 10 000 à 15 000 enregistrements par minute.

## Formats de date acceptés

Blue prend en charge une grande variété de formats de date pour l'importation. Voici tous les formats pris en charge :

- **Horodatages Unix** :
  ```plaintext
  1511648546 (seconds)
  1620021848429 (milliseconds)
  1620024872717915000 (nanoseconds)
  ```
  
- **Normes RFC** :
  ```plaintext
  RFC 3339: 2021-05-01T01:17:02.604456Z
  RFC 2822: Wed, 02 Jun 2021 06:31:39 GMT
  ```

- **Formats de base de données** :
  ```plaintext
  PostgreSQL: 
  2019-11-29 08:08-08
  2019-11-29 08:08:05-08
  2021-05-02 23:31:36.0741-07
  ```

- **Formats courants** :
  ```plaintext
  Year-Month-Day: 
  2021-04-30 21:14
  2021-02-21
  2021-Feb-21
  
  Month/Day/Year:
  3/31/2014 22:05
  04/02/2014 03:00:51
  
  Day-Month-Year:
  12 Feb 2006, 19:17
  03 February 2013
  ```

- **Fuseaux horaires** :
  ```plaintext
  2014-04-26 13:13:43 +0800
  2017-11-25 13:31 PST
  2021-02-21 UTC
  ```

- **Langage naturel** :
  ```plaintext
  May 6 at 9:24 PM
  September 17, 2012 at 10:09am PST
  October 7, 1970
  ```

- **Formats internationaux** :
  ```plaintext
  Chinese: 2014年04月08日
  Japanese: 2014/04/08 22:05
  ```

- **Temps seulement** :
  ```plaintext
  01:06:06
  4:00pm PST
  6:00 AM UTC
  ```

Toutes les dates seront converties en fuseau horaire UTC lors de l'importation. Les composants temporels sont optionnels - s'ils ne sont pas fournis, l'heure par défaut sera 00:00:00 UTC.

## Gestion des virgules dans vos données

Si vos données contiennent des virgules (par exemple, dans des adresses ou des noms d'entreprise), vous devez envelopper ces valeurs entre guillemets doubles pour éviter les erreurs d'importation.

**Exemple :**
```
Company,Address
Acme Inc,123 Main St, New York
```

**Doit devenir :**
```
Company,Address
"Acme Inc","123 Main St, New York"
```

Cela garantit que la virgule dans "123 Main St, New York" est considérée comme faisant partie des données plutôt que comme un séparateur de colonne.
