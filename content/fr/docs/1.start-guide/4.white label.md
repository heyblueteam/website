## Introduction

White label transforme Blue en une plateforme flexible de gestion de projets et de processus qui ressemble et fonctionne entièrement comme votre propre produit — sans le fardeau de gérer la technologie, les développeurs, les serveurs ou la disponibilité.

Avec le White Label de Blue, vous pouvez :

- **Mettre en valeur votre marque :** Remplacez le nom, le logo et les couleurs de Blue par les vôtres, créant ainsi une expérience de marque cohérente.
- **Fonctionner sur votre domaine :** Exploitez Blue sur votre propre domaine web pour une présence professionnelle et cohérente.
- **Tirer parti de votre infrastructure email :** Envoyez des emails depuis votre adresse pour une communication personnalisée.


Pour les agences, le module White Label renforce votre marque, améliore vos offres de services et simplifie la collaboration avec les clients. En intégrant Blue dans vos flux de travail sous votre marque, vous créez une expérience professionnelle et transparente qui vous aide à conclure plus de clients et à les fidéliser en offrant un mélange unique de services et de logiciels adaptés à leurs besoins.


## Tarification

Le module white-label coûte 20$/mois par entreprise ou 200$/an par organisation, en plus de votre abonnement Blue de base. Vous pouvez en apprendre plus sur la tarification [ici](/pricing).

Si vous êtes intéressé par l'achat de white label en volume pour plusieurs organisations, veuillez contacter sales@blue.cc
 

## Domaines personnalisés

Pour que le White Label fonctionne, vous devez d'abord configurer un domaine personnalisé pour l'application web. 

Les domaines personnalisés peuvent être configurés pour :

- Application web (obligatoire)
- [Formulaires](/docs/projects/forms) (optionnel)
- [Liens de partage de fichiers](/docs/projects/file-management) (optionnel)
- Documentation (bientôt disponible)


Pour configurer un domaine personnalisé, ajoutez des enregistrements CNAME à vos paramètres DNS pour chaque sous-domaine. Veuillez noter que seuls les sous-domaines sont pris en charge, pas les domaines racines. Donc `app.yourcompany.com` est pris en charge, mais `yourcompany.com` ne l'est pas.


| Type de domaine            | Enregistrement CNAME          | Pointe vers                |
|----------------------------|-------------------------------|----------------------------|
| Application Web            | `app.yourcompany.com`         | `wl-app.onrender.com`      |
| Formulaires                | `forms.yourcompany.com`       | `wl-forms.onrender.com`    |
| Liens de partage de fichiers| `files.yourcompany.com`      | `wl-files.onrender.com`    |


Le tableau ci-dessus fournit des enregistrements CNAME suggérés, mais vous pouvez utiliser n'importe quel sous-domaine de votre choix. 

Par exemple, l'application web principale pourrait être `projects.yourcompany.com` et les liens de partage de fichiers pourraient être `media.yourcompany.com`.


## Image de marque

Personnalisez les éléments suivants pour aligner Blue avec votre marque :

- **Logo** : Téléchargez votre logo pour remplacer la marque Blue sur toute la plateforme.
- **Favicon** : Ajoutez votre propre favicon, qui apparaîtra dans les onglets du navigateur et les favoris.
- **Couleur principale de l'interface** : Définissez une couleur principale pour l'interface, appliquée de manière cohérente à tous les utilisateurs.
- **Nom de l'application** : Personnalisez le nom de l'application affiché dans les onglets du navigateur et dans l'application de bureau.
- **Polices** : (Bientôt disponible) Personnalisez les polices utilisées sur toute la plateforme pour correspondre à votre marque à partir d'une liste sélectionnée de polices Google.

Blue fournit un aperçu en direct de la page de connexion et de l'icône de l'application lorsque vous configurez vos paramètres de marque.

Votre image de marque s'appliquera également à [l'application de bureau](/docs/start-guide/download-apps#desktop-app), garantissant une expérience entièrement white-labelisée.

Voici la configuration recommandée pour le logo et le favicon :

| Élément | Forme recommandée | Arrière-plan    | Rapport d'aspect |
|---------|------------------|-----------------|------------------|
| Logo    | Rectangle        | Transparent     | 1:1.5            |
| Favicon | Carré            | Transparent     | 1:1              |


## Notifications par email

Vous pouvez utiliser SMTP pour envoyer des notifications par email depuis votre propre serveur de messagerie avec votre propre domaine au lieu de notre adresse email.

Vous devez fournir les informations suivantes à Blue :

| Paramètre                         | Description                                                                                                                                         |
|-----------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------|
| **Serveur SMTP**                  | L'adresse du serveur qui gérera l'envoi de vos emails.                                                                                             |
| **Port SMTP**                     | Le numéro de port utilisé pour se connecter au serveur SMTP, généralement 587 pour TLS ou 465 pour SSL.                                            |
| **Nom d'utilisateur**             | Le nom d'utilisateur pour l'authentification avec votre serveur SMTP, généralement votre adresse email.                                            |
| **Mot de passe (stocké sous forme cryptée)** | Le mot de passe associé au nom d'utilisateur, qui sera stocké de manière sécurisée sous forme cryptée pour protéger vos identifiants.     |
| **Nom de l'expéditeur**           | Le nom qui apparaîtra comme expéditeur dans les notifications par email, aidant les destinataires à identifier la source de l'email.               |
| **Adresse email de l'expéditeur** | L'adresse email qui sera affichée comme expéditeur, garantissant que les réponses vont à la bonne adresse. Notez que cela doit être le même domaine que le serveur SMTP. |

Blue impose l'utilisation du cryptage TLS/SSL pour la connexion SMTP.

Vous pouvez ensuite tester la connexion à votre serveur SMTP en cliquant sur le bouton "Tester la connexion". Blue testera automatiquement la connexion à votre serveur SMTP et vous informera si elle est réussie ou non via une notification dans l'application.

### Considérations 

Tous les emails transactionnels, de notification et d'automatisation seront envoyés via votre serveur SMTP. La seule exception concerne les emails contenant des OTP (mots de passe à usage unique) qui sont envoyés via une adresse email générique (notifications@process-system.app) pour garantir la sécurité.

Assurez-vous que votre serveur SMTP :

- Peut gérer des volumes d'emails importants.
- A une solide réputation d'envoi pour éviter les filtres anti-spam.
- Dispose d'un quota suffisant pour supporter la charge d'emails prévue.


### Emails d'invitation personnalisés

Vous pouvez également personnaliser l'email d'invitation initial qui est envoyé aux nouveaux utilisateurs lorsqu'ils s'inscrivent à votre instance Blue. 

### Emails d'intégration personnalisés (Bientôt disponible)

Sans white labeling, Blue envoie une série d'emails d'intégration aux nouveaux utilisateurs lorsqu'ils rejoignent une entreprise pour les aider à démarrer. Lorsque le white-label est activé, ces emails ne sont pas envoyés, et à l'avenir vous pourrez créer votre propre ensemble d'emails d'intégration pour accueillir les nouveaux utilisateurs.

## FAQ


**Qu'est-ce qui est inclus dans le module white label ?**

Le module white label comprend trois fonctionnalités principales : l'intégration de domaine personnalisé pour l'application web, les formulaires et le partage de fichiers ; l'image de marque personnalisée (logo, favicon, couleurs et nom de l'application) ; et les notifications par email personnalisées utilisant votre propre serveur SMTP.

**Combien coûte le white label ?**

Le module white label coûte 20$/mois/entreprise ou 200$/an/entreprise en plus de votre abonnement Blue de base. Des remises de volume seront disponibles à l'avenir pour les comptes avec plusieurs entreprises.

**Le white-labeling affecte-t-il les fonctionnalités de Blue ?**

Non, toutes les fonctionnalités principales de Blue restent exactement les mêmes. Le white-labeling ne modifie que l'apparence et l'image de marque de la plateforme.


**Puis-je utiliser un domaine racine au lieu d'un sous-domaine ?**

Non, Blue ne prend en charge que les sous-domaines pour la configuration de domaine personnalisé.

**Dois-je configurer tous les domaines personnalisés ?**

Non, seul le domaine de l'application web est obligatoire. Les domaines des formulaires et du partage de fichiers sont optionnels. Le support du domaine de documentation arrive bientôt.

**Puis-je utiliser des noms de sous-domaines différents de ceux suggérés ?**

Oui, vous pouvez utiliser n'importe quels noms de sous-domaines que vous préférez. Par exemple, vous pourriez utiliser `projects.yourcompany.com` au lieu de `app.yourcompany.com`.

**Quels éléments d'image de marque puis-je personnaliser ?**

Vous pouvez personnaliser votre logo, favicon, couleur principale de l'interface et nom de l'application.

**L'application de bureau reflétera-t-elle mon image de marque ?**

Oui, l'application de bureau sera entièrement white-labelisée avec votre image de marque.

**Les utilisateurs peuvent-ils avoir des schémas de couleurs différents ?**

Non, la couleur principale que vous choisissez sera appliquée à tous les utilisateurs de votre instance Blue.

**Quels détails de configuration email dois-je fournir ?**

Vous devrez fournir l'adresse du serveur SMTP, le port SMTP, le nom d'utilisateur, le mot de passe, le nom de l'expéditeur et l'adresse email de l'expéditeur. Le mot de passe SMTP est stocké sous forme cryptée pour protéger vos identifiants.

**Puis-je tester la configuration SMTP avant la mise en ligne ?**

Oui, Blue fournit un bouton "Tester la connexion" qui vérifiera votre configuration de serveur SMTP et vous informera du résultat.

**Quels types d'emails seront envoyés via mon serveur SMTP ?**

Tous les emails seront envoyés via votre serveur SMTP, y compris les emails transactionnels, les emails de notification, les automatisations d'emails. Les emails OTP (mot de passe à usage unique) sont envoyés via une adresse email générique (notifications@process-system.app) pour garantir la sécurité.


**Puis-je white-labeler les applications mobiles ?**

Blue ne prend actuellement pas en charge le white-labeling des applications mobiles. Cependant, si vous avez besoin d'une version de marque sur l'Apple App Store ou Google Play Store, vous pouvez contacter sales@blue.cc pour discuter des options.

**Le support white-label pour les applications mobiles sera-t-il disponible à l'avenir ?**

Il n'y a actuellement aucun plan pour prendre en charge le white-labeling des applications mobiles.

**Comment puis-je obtenir de l'aide pour la configuration du white-label ?**

Pour une aide générale, vous pouvez contacter support@blue.cc. Pour des questions liées aux ventes concernant les applications mobiles personnalisées, contactez sales@blue.cc.

**Puis-je tester les fonctionnalités white-label avant de les implémenter ?**

Oui, Blue fournit un aperçu de la page de connexion et de l'icône de l'application lorsque vous configurez vos paramètres de marque.

## Paramètres DNS

Nous avons rassemblé de manière pratique des instructions sur la façon de configurer les domaines white label pour les systèmes DNS (Domain Name Service) courants :

#### GoDaddy

1. Connectez-vous à votre compte GoDaddy et ouvrez votre produit.
2. Naviguez vers le centre de contrôle de domaine.
3. Sur la page des paramètres de domaine, trouvez et sélectionnez votre nom de domaine.
4. Faites défiler jusqu'à la section Paramètres supplémentaires et sélectionnez Gérer DNS.
5. Sur la page de gestion DNS, faites défiler jusqu'au bas de la section Enregistrements et cliquez sur Ajouter.
6. Dans le menu déroulant Type, sélectionnez CNAME et complétez les champs obligatoires suivants :
    - Hôte - Le nom d'hôte ou le préfixe pour le CNAME. Vous pouvez utiliser un point (.), mais pas comme premier ou dernier caractère. Les points consécutifs (..) ne sont pas autorisés. L'hôte ne peut pas dépasser 25 caractères ou être le symbole @.
    - Pointe vers - Définissez ceci sur `wl-app.onrender.com`
    - TTL - Spécifiez la durée pendant laquelle le serveur doit mettre en cache les informations.
7. Cliquez sur Enregistrer.

#### Bluehost

1. Connectez-vous à votre compte BlueHost.
2. Cliquez sur le menu Domaines puis cliquez sur Éditeur de zone. Une nouvelle page apparaît.
3. Naviguez vers la section Ajouter un enregistrement DNS.
4. Dans le champ Enregistrement d'hôte, entrez le sous-domaine de l'adresse que vous souhaitez utiliser pour votre domaine personnalisé.
5. Dans le menu déroulant Type, sélectionnez l'option CNAME.
6. Dans la section Pointe vers, entrez `wl-app.onrender.com`
7. Cliquez sur Ajouter un enregistrement.

#### Cloudflare

1. Connectez-vous à votre compte Cloudflare.
2. Dans le menu déroulant, localisez et sélectionnez votre domaine.
3. Cliquez sur l'onglet Paramètres DNS.
4. Cliquez sur "Ajouter un enregistrement" :
5. Sélectionnez l'option CNAME.
6. Définissez "Nom" sur votre sous-domaine
7. Définissez "Cible" sur `wl-app.onrender.com`
8. Définissez l'icône de basculement du proxy Cloudflare sur un nuage gris.

#### Domain.com

1. Connectez-vous à votre compte Domain.com.
2. Localisez votre domaine et cliquez sur Gérer.
3. Dans le menu à gauche, cliquez sur DNS et serveurs de noms.
4. Cliquez sur Ajouter un enregistrement DNS pour ajouter l'enregistrement CNAME.
5. Définissez la cible sur `wl-app.onrender.com`
6. Cliquez sur Mettre à jour DNS pour terminer le processus.

#### Dreamhost

1. Connectez-vous à votre compte DreamHost.
2. À gauche de la page, cliquez sur le lien Domaines.
3. Dans le menu déroulant Domaines, sélectionnez Gérer les domaines.
4. Localisez votre domaine et cliquez sur le lien DNS sous votre domaine.
5. Faites défiler jusqu'à la section Ajouter un enregistrement DNS personnalisé à (votre domaine personnalisé) et modifiez les champs suivants :
    - Nom - tapez le sous-domaine que vous souhaitez utiliser
    - Type - sélectionnez l'option CNAME.
    - Valeur - entrez `wl-app.onrender.com`
6. Cliquez sur Ajouter l'enregistrement maintenant ! pour ajouter l'enregistrement CNAME.

#### Google Domains

1. Connectez-vous à votre compte Google Domains.
2. Trouvez et cliquez sur votre nom de domaine.
3. Dans le menu à gauche, cliquez sur DNS.
4. Naviguez vers la section Enregistrements de ressources personnalisés.
5. Dans le champ Nom, entrez le sous-domaine de l'adresse que vous souhaitez utiliser pour votre domaine personnalisé.
6. Dans le menu déroulant Type, sélectionnez l'option CNAME.
7. Dans le champ Données, entrez `wl-app.onrender.com`
8. Cliquez sur Ajouter pour terminer le processus.

#### Hostgator

1. Connectez-vous à votre compte HostGator.
2. Sur le côté gauche de la page, cliquez sur CPanel.
3. Faites défiler jusqu'à la section Domaines et cliquez sur Éditeur de zone.
4. Trouvez votre domaine et cliquez sur le lien Enregistrement CName à côté du domaine.
5. Dans la boîte de dialogue Ajouter un enregistrement CName, complétez les champs vides suivants :
    - Nom - entrez votre sous-domaine/nom de domaine choisi.
    - CNAME - entrez `wl-app.onrender.com`
6. Cliquez sur Ajouter un enregistrement CName. Un message de HostGator apparaît, indiquant que vous avez ajouté avec succès un nouveau CNAME à votre compte HostGator.

#### NameCheap

1. Connectez-vous à votre compte NameCheap.
2. Dans le menu déroulant Compte, sélectionnez Tableau de bord.
3. Cliquez sur Liste de domaines et trouvez votre domaine.
4. Cliquez sur Gérer à côté de votre domaine.
5. Cliquez sur DNS avancé et naviguez vers la section Enregistrements d'hôte.
6. Cliquez sur Ajouter un nouvel enregistrement.
7. Dans le menu déroulant Type, sélectionnez Enregistrement CNAME et entrez votre sous-domaine comme hôte.
8. Entrez ensuite `wl-app.onrender.com`
9. Cliquez sur Enregistrer toutes les modifications pour terminer le processus.
