## Comment fonctionnent les projets

Chaque projet dans Blue est entièrement autonome. Seuls les utilisateurs qui sont spécifiquement invités dans le projet peuvent voir les données et collaborer. Vous pouvez avoir un nombre illimité de projets dans un compte Blue.

Il existe deux façons de concevoir les projets dans Blue :

1. **Projets** : Un projet a généralement un début et une fin clairement définis, se concentrant sur des objectifs ou des événements spécifiques. Les exemples incluent le lancement d'un événement, une refonte de marque ou la conception d'un site web. Ces projets sont orientés vers des objectifs et ont des objectifs et des échéances définis. Une fois terminés, ils sont souvent [archivés](/docs/projects/archiving-projects).
2. **Processus** : Blue prend également en charge les processus continus qui n'ont pas de date de fin fixe. Ceux-ci peuvent inclure des tâches comme le service client, les opérations de support technique, le suivi des ventes et le recrutement. Ces projets nécessitent une gestion et une organisation régulières mais sont de nature continue.

## Paramètres du projet

Les [Administrateurs de projet](/docs/user-management/roles/project-administrator) verront une icône de paramètres à côté du nom du projet dans la barre supérieure.

Plusieurs paramètres sont disponibles :

- **Modifier les paramètres du projet** : Vous pouvez changer le nom du projet, écrire une description du projet, ajouter une photo ou une icône de projet, et également changer le nom des [Enregistrements](/docs/records/introduction) en autre chose (ex. : Tâche, Vente)
- **Définir l'icône et la couleur** : Vous pouvez choisir une icône et une couleur pour le projet ; celles-ci seront affichées dans la barre latérale pour tous les utilisateurs qui sont dans le projet.
- **Gérer** : Cela ouvre un sous-menu où vous pouvez modifier les fonctionnalités disponibles et les [champs personnalisés](/docs/custom-fields/introduction) dans le projet.
- [Convertir en modèle](/docs/projects/templates) : Cela permet de convertir le projet en un modèle réutilisable.
- [Copier](/docs/projects/copying-projects) : Permet de dupliquer le projet
- [Archiver](/docs/projects/archiving-projects) : Archiver le projet pour référence future, le transformant en projet en lecture seule.
- [Importer](/docs/integrations/csv-import) **&** [Exporter](/docs/integrations/csv-export) : Permet l'importation et l'exportation d'enregistrements depuis le projet
- **Connexions OAuth** : Vous permet de connecter votre projet à un service externe tel que Github ou Quickbooks.
- **Supprimer** : Permet la suppression permanente du projet.
