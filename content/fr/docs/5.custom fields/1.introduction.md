Les champs personnalisés sont un moyen de créer une structure de données personnalisée dans Blue qui correspond parfaitement aux exigences de votre organisation. En créant des champs personnalisés pour vos données structurées au lieu d'utiliser des commentaires ou des descriptions, vous exploitez toute la puissance de Blue en tant que plateforme de travail. Les données structurées peuvent devenir votre source unique de vérité.

Notre [API](/fr/docs/integrations/api) et nos [Webhooks](/fr/docs/integrations/webhooks) prennent entièrement en charge les champs personnalisés, donc en utilisant des champs personnalisés dans Blue, vous créez une API de données structurées qui correspond parfaitement à vos données.


## Points clés

Par défaut, chaque enregistrement dans Blue est livré avec des champs spécifiques :

- Nom de l'enregistrement
- [Assigné](/fr/docs/records/assignees)
- [Date](/fr/docs/records/dates)
- [Tag](/fr/docs/records/tags)
- [Dépendances](/fr/docs/records/dependencies)

Mais vous pouvez choisir parmi de nombreux champs supplémentaires différents que vous pouvez ajouter à tous les enregistrements dans un projet. Par exemple, cela pourrait être un champ personnalisé de devise pour "Prix de vente" dans un projet de vente.

![Introduction 1](/docs/Introduction_1.png "Introduction 1")

Chaque champ personnalisé est une colonne supplémentaire dans votre base de données de projet

Les champs personnalisés sont étroitement intégrés dans Blue. En utilisant les données contenues dans chaque champ personnalisé, vous pouvez [créer des formulaires d'admission de travail](/fr/docs/projects/forms), [importer](/fr/docs/integrations/csv-import) et [exporter](/fr/docs/integrations/csv-export) des données, [filtrer les données](/fr/docs/views/filters), et [créer des tableaux de bord de reporting](/fr/docs/dashboards/introduction).

Les champs personnalisés peuvent être subdivisés en deux types principaux :

- **Champs personnalisés modifiables** : Ce sont des champs personnalisés dans lesquels les utilisateurs peuvent saisir des données dans le champ. Par exemple, un [champ personnalisé Email](/fr/docs/custom-fields/email) permet aux utilisateurs de saisir une adresse email, ou un [champ personnalisé Sélection unique](/fr/docs/custom-fields/single-select) permet aux utilisateurs de choisir parmi une option dans le menu déroulant fourni.

- **Champs personnalisés en lecture seule** : Ce sont des champs qui affichent uniquement des données. Par exemple, le [champ personnalisé ID unique](/fr/docs/custom-fields/unique-id) affiche un identifiant unique généré aléatoirement pour votre enregistrement, et le [champ personnalisé Formule](/fr/docs/custom-fields/formula) affiche le résultat d'un calcul basé sur les valeurs d'autres champs personnalisés.

## Création de champs personnalisés

Seuls les [Administrateurs de projet](/fr/docs/user-management/roles/project-administrator) peuvent créer des champs personnalisés pour s'assurer que l'intégrité de la structure de données du projet soit maintenue intacte. Vous pouvez créer et gérer des champs personnalisés en ouvrant un enregistrement dans le projet et en cliquant sur "Ajouter plus de champs"

![Introduction 2](/docs/Introduction_2.png "Introduction 2")

Utilisez le bouton "Ajouter plus de champs" dans un enregistrement pour ajouter des champs personnalisés à un projet.

::callout
---

icon: lightbulb
---

**Les champs personnalisés sont à l'échelle du projet** <br/> <br/>
Lorsque vous créez un nouveau champ personnalisé, ce champ personnalisé est disponible dans tous les enregistrements du projet spécifique. Chaque enregistrement peut alors avoir différentes valeurs pour ce champ personnalisé. Par exemple, l'enregistrement "Client 1" peut avoir une valeur de 2000 $ pour le champ personnalisé "Prix de vente", tandis que l'enregistrement "Client 2" a une valeur de 15 000 $ pour le champ personnalisé "Prix de vente".<br/> <br/>
L'utilisation de champs personnalisés dans un projet améliore considérablement la cohérence des données et l'efficacité de la gestion. En standardisant la structure d'information, les champs personnalisés garantissent que tous les enregistrements sont organisés de manière uniforme, rendant le suivi et l'analyse des données plus rationalisés et fiables.
::

## Suppression de champs personnalisés

Pour supprimer un champ personnalisé, ouvrez n'importe quel enregistrement et cliquez sur "Ajouter plus de champs". Vous verrez une liste de champs personnalisés dans votre projet. À côté de chaque champ personnalisé, il y aura une icône de corbeille rouge pour supprimer ce champ personnalisé.

![Introduction 3](/docs/Introduction_3.png "Introduction 3")

L'icône de suppression pour les champs personnalisés
Vous pouvez cliquer sur cette icône pour supprimer les champs personnalisés du projet. Vous aurez un message de confirmation, puis vous pourrez confirmer et supprimer le champ personnalisé.

::callout
---

icon: lightbulb
---

**La suppression de champs personnalisés est à l'échelle du projet et permanente** <br/><br/>
Lorsque vous supprimez un champ personnalisé, cela supprime le champ personnalisé de tous les enregistrements dans ce projet. Toutes les données contenues dans ce champ personnalisé seront définitivement supprimées et ne sont pas récupérables.
::

## Permissions utilisateur

Une des fonctionnalités les plus puissantes de Blue est que nous fournissons des permissions au niveau des lignes pour les champs personnalisés. Cela vous permet de définir quels groupes d'utilisateurs peuvent voir ou modifier des champs personnalisés individuels. [Cela peut être configuré en créant un rôle utilisateur personnalisé](/fr/docs/user-management/roles/custom-user-roles).
