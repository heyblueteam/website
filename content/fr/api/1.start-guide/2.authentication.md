## En-têtes

Avant d'exécuter des requêtes, il est essentiel de configurer les en-têtes pour votre requête API. Ces en-têtes restent constants pour toutes les requêtes, nécessitant seulement une configuration unique. Ils servent à authentifier chaque requête à l'API.

Les en-têtes requis sont le Token-ID et le Secret-ID, représentés respectivement par `x-bloo-token-id` et `x-bloo-token-secret`.

De plus, certaines requêtes peuvent nécessiter l'inclusion d'un Company-ID `x-bloo-company-id` et d'un Project-ID `x-bloo-project-id` pour des opérations plus spécifiques.

## Token ID
Le Token ID est votre identifiant unique pour l'accès à l'API.

Pour commencer à utiliser notre API, cliquez sur votre profil en haut à droite, et sous le menu de profil, vous trouverez l'onglet API.

![API 2](/docs/API_2.png "API 2")

Dans l'onglet API, cliquez sur le bouton "Générer un jeton" :

![API 3](/docs/API_3.png "API 3")

Fournissez un nom pour le jeton et définissez une date d'expiration si vous le souhaitez. Si vous définissez une date d'expiration, le jeton API cessera automatiquement de fonctionner après cette date.

![API 4](/docs/API_4.png "API 4")

Après la génération, un Token ID et un Secret ID vous seront affichés. Le Token ID sera préfixé par `pat_` pour une identification facile.

## Secret ID

Le Secret ID agit comme un mot de passe, garantissant une communication sécurisée avec l'API de Blue.

N'oubliez pas que le Secret ID n'est affiché qu'une seule fois pour des raisons de sécurité, donc conservez-le en lieu sûr pour maintenir l'accès à vos données via l'API. Pour une sécurité maximale, Blue stocke votre secret en utilisant le hachage bcrypt, ce qui signifie même notre équipe ne peut pas récupérer votre secret en texte clair après sa création.

![API 5](/docs/API_5.png "API 5")

## Company & Project IDs

Voici une vidéo qui montre comment trouver votre Company ID et votre Project ID dans Blue :

<youtube url="https://www.youtube.com/watch?v=zLEvs6zqGTc" />

Les entreprises sont l'entité de niveau supérieur dans Blue. Les Company IDs sont utilisés pour identifier une entreprise dans Blue. Ils sont utilisés dans l'API pour spécifier avec quelle entreprise interagir. L'ID est le même que le slug de l'entreprise dans l'URL lorsque vous visitez l'entreprise dans Blue :

```
app.blue.cc/company/{company-id}/
```

Pour certaines requêtes et mutations, vous devrez fournir un Project ID. C'est l'ID du projet avec lequel vous souhaitez interagir. L'ID est le même que le slug du projet dans l'URL lorsque vous visitez le projet dans Blue :

```
app.blue.cc/company/{company-id}/project/{project-id}/
```
