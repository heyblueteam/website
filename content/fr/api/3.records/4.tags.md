## Aperçu

Les étiquettes sont un moyen puissant de catégoriser et d'organiser les enregistrements dans Blue. Le système d'étiquettes offre une fonctionnalité complète pour créer, gérer et filtrer les étiquettes, avec un support pour les opérations en masse et le filtrage avancé.

## Lister les Étiquettes

Récupérez les étiquettes des projets avec des options de filtrage et de tri flexibles.

### Exemple de Base

```graphql
query ListTags {
  tagList(
    filter: { 
      projectIds: ["project-123"] 
      excludeArchivedProjects: false 
    }
    first: 50
    orderBy: title_ASC
  ) {
    items {
      id
      uid
      title
      color
      project {
        id
        name
      }
      createdAt
      updatedAt
    }
    pageInfo {
      totalPages
      totalItems
      page
      perPage
      hasNextPage
      hasPreviousPage
    }
    totalCount
  }
}
```

### Exemple de Filtrage Avancé

```graphql
query FilterTags {
  tagList(
    filter: {
      projectIds: ["project-123", "project-456"]
      search: "urgent"
      colors: ["#ff0000", "#ffaa00"]
      excludeArchivedProjects: true
    }
    orderBy: createdAt_DESC
    first: 20
  ) {
    items {
      id
      title
      color
      todos {
        id
        title
      }
    }
    totalCount
  }
}
```

## Créer des Étiquettes

Créez de nouvelles étiquettes au sein d'un projet.

```graphql
mutation CreateTag {
  createTag(
    input: {
      title: "High Priority"
      color: "#ff0000"
    }
  ) {
    id
    uid
    title
    color
    project {
      id
      name
    }
    createdAt
  }
}
```

## Mettre à Jour des Étiquettes

Modifiez les propriétés des étiquettes existantes.

```graphql
mutation UpdateTag {
  editTag(
    input: {
      id: "tag-123"
      title: "Critical Priority"
      color: "#cc0000"
    }
  ) {
    id
    title
    color
    updatedAt
  }
}
```

## Supprimer des Étiquettes

Retirez des étiquettes du système. Cela supprimera également l'étiquette de tous les enregistrements associés.

```graphql
mutation DeleteTag {
  deleteTag(id: "tag-123")
}
```

## Étiqueter des Enregistrements

Associez des étiquettes à des enregistrements (tâches) en utilisant des étiquettes existantes ou en créant de nouvelles.

### Utiliser des Étiquettes Existantes

```graphql
mutation TagRecord {
  setTodoTags(
    input: {
      todoId: "todo-123"
      tagIds: ["tag-456", "tag-789"]
    }
  )
}
```

### Créer des Étiquettes Lors de l'Étiquetage

Vous pouvez créer de nouvelles étiquettes à la volée en fournissant des titres d'étiquettes :

```graphql
mutation TagRecordWithNewTags {
  setTodoTags(
    input: {
      todoId: "todo-123"
      tagIds: ["existing-tag-id"]
      tagTitles: ["New Tag", "Another New Tag"]
    }
  )
}
```

## Paramètres d'Entrée

### TagListFilter

| Paramètre | Type | Requis | Description |
|-----------|------|--------|-------------|
| `projectIds` | [String!] | Non | Filtrer les étiquettes par ID de projet ou slugs |  |
| `excludeArchivedProjects` | Boolean | Non | Exclure les étiquettes des projets archivés |  |
| `search` | String | Non | Rechercher des titres d'étiquettes (insensible à la casse) |  |
| `titles` | [String!] | Non | Filtrer par titres d'étiquettes spécifiques |  |
| `colors` | [String!] | Non | Filtrer par couleurs spécifiques (format hexadécimal) |  |
| `tagIds` | [String!] | Non | Filtrer par ID d'étiquettes spécifiques |  |

### CreateTagInput

| Paramètre | Type | Requis | Description |
|-----------|------|--------|-------------|
| `title` | String | Non | Titre de l'étiquette (généré automatiquement si non fourni) |  |
| `color` | String! | ✅ Oui | Couleur de l'étiquette en format hexadécimal (par exemple, "#ff0000") |  |

### EditTagInput

| Paramètre | Type | Requis | Description |
|-----------|------|--------|-------------|
| `id` | String! | ✅ Oui | ID de l'étiquette à mettre à jour |  |
| `title` | String | Non | Nouveau titre d'étiquette |  |
| `color` | String | Non | Nouvelle couleur d'étiquette en format hexadécimal |  |

### SetTodoTagsInput

| Paramètre | Type | Requis | Description |
|-----------|------|--------|-------------|
| `todoId` | String! | ✅ Oui | ID de l'enregistrement à étiqueter |  |
| `tagIds` | [String!] | Non | IDs des étiquettes existantes à appliquer |  |
| `tagTitles` | [String!] | Non | Titres des nouvelles étiquettes à créer et à appliquer |  |

### TagOrderByInput Valeurs

| Valeur | Description |
|-------|-------------|
| `id_ASC` / `id_DESC` | Trier par ID d'étiquette |  |
| `uid_ASC` / `uid_DESC` | Trier par identifiant unique |  |
| `title_ASC` / `title_DESC` | Trier par ordre alphabétique par titre |  |
| `color_ASC` / `color_DESC` | Trier par valeur de couleur |  |
| `createdAt_ASC` / `createdAt_DESC` | Trier par date de création |  |
| `updatedAt_ASC` / `updatedAt_DESC` | Trier par date de dernière mise à jour |  |

## Champs de Réponse

### Type d'Étiquette

| Champ | Type | Description |
|-------|------|-------------|
| `id` | ID! | Identifiant unique pour l'étiquette |  |
| `uid` | String! | Identifiant unique convivial |  |
| `title` | String! | Titre/nom de l'étiquette |  |
| `color` | String! | Couleur de l'étiquette en format hexadécimal |  |
| `project` | Project! | Projet auquel appartient cette étiquette |  |
| `todos` | [Todo!]! | Enregistrements associés à cette étiquette |  |
| `createdAt` | DateTime! | Date de création de l'étiquette |  |
| `updatedAt` | DateTime! | Date de dernière mise à jour de l'étiquette |  |

### Réponse de TagPagination

| Champ | Type | Description |
|-------|------|-------------|
| `items` | [Tag!]! | Tableau des enregistrements d'étiquettes |  |
| `pageInfo` | PageInfo! | Informations de pagination |  |
| `totalCount` | Int! | Nombre total d'étiquettes correspondant aux filtres |  |

### Champs PageInfo

| Champ | Type | Description |
|-------|------|-------------|
| `totalPages` | Int | Nombre total de pages |  |
| `totalItems` | Int | Nombre total d'éléments sur toutes les pages |  |
| `page` | Int | Numéro de la page actuelle |  |
| `perPage` | Int | Nombre d'éléments par page |  |
| `hasNextPage` | Boolean! | Indique s'il y a une page suivante |  |
| `hasPreviousPage` | Boolean! | Indique s'il y a une page précédente |  |

## Permissions Requises

### Gestion des Étiquettes

| Action | Rôle Requis |
|--------|-------------|
| Create tags | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |
| Edit tags | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |
| Delete tags | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |
| Apply tags to records | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |

### Visibilité des Étiquettes

Les étiquettes sont visibles pour tous les utilisateurs ayant accès au projet où l'étiquette a été créée. Un filtrage basé sur les rôles peut s'appliquer pour certaines opérations.

## Meilleures Pratiques

1. **Utilisez des Couleurs Cohérentes** : Établissez un système de codage couleur pour votre équipe
2. **Titres Descriptifs** : Utilisez des noms d'étiquettes clairs et descriptifs que les membres de l'équipe comprendront
3. **Opérations en Lot** : Utilisez `setTodoTags` pour appliquer plusieurs étiquettes à la fois
4. **Recherche et Filtrage** : Utilisez la fonctionnalité de recherche pour trouver des étiquettes existantes avant d'en créer de nouvelles
5. **Gestion des Étiquettes** : Passez régulièrement en revue et nettoyez les étiquettes inutilisées
6. **Conventions de Couleur** : Envisagez d'utiliser des couleurs standard pour les types d'étiquettes courants (rouge pour urgent, vert pour terminé, etc.)

## Fonctionnalités Avancées

### Valeurs par Défaut de Création d'Étiquettes

- Les nouvelles étiquettes créées via `setTodoTags` avec `tagTitles` par défaut à la couleur `#4a9fff`
- Les étiquettes sans couleurs spécifiées reviennent à `#00a0d2`
- Les titres d'étiquettes sont automatiquement dépouillés des espaces

### Intégration d'Automatisation

Les étiquettes peuvent être :
- Appliquées automatiquement par des règles d'automatisation
- Utilisées comme déclencheurs pour des flux de travail d'automatisation
- Modifiées par des actions d'automatisation

### Mises à Jour en Temps Réel

Les changements d'étiquettes sont diffusés via des abonnements GraphQL, permettant des mises à jour en temps réel dans les clients connectés.

## Remarques Importantes

- La suppression d'une étiquette la retire de tous les enregistrements associés
- Les étiquettes sont limitées au projet et ne peuvent pas être partagées entre les projets
- Les valeurs de couleur doivent être des codes hexadécimaux valides (par exemple, "#ff0000", "#00aa00")
- Les titres d'étiquettes sont sensibles à la casse
- Un maximum de 100 étiquettes peut être appliqué à un seul enregistrement
- Les étiquettes prennent en charge la recherche en texte intégral à travers les titres
