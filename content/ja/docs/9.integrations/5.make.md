[Make.com](https://www.make.com)は、異なるウェブアプリを接続し、ワークフローを自動化するための統合プラットフォームです。Blueは**Make.comと公式に統合**されており、2,000以上のアプリ間でシームレスな自動化を実現します—コーディングは不要です！

![Blue Make.com モジュールの概要](/docs/blue_make_modules_overview.png "Overview of available Blue modules in Make.com")

## ユースケース

以下は、Make.com統合を使用してBlueの機能を拡張するための強力な方法です：
BlueのMake.com統合は、さまざまなビジネス機能にわたる強力な自動化を可能にします。顧客体験のために、Intercom、HubSpot、ZendeskなどのツールとBlueを接続して、包括的な顧客記録を作成し、インタラクションを同期し、SlackやMicrosoft Teamsを通じてリアルタイムのコミュニケーションを維持できます。プロジェクト管理では、BlueはJira、GitHub、Trello、Monday.comと統合され、タスクを同期し、進捗を追跡し、通知を自動化します。

マーケティングと営業では、BlueはMailchimp、Stripe、Calendlyと接続してキャンペーンを追跡し、支払いを管理し、会議のワークフローを自動化します。ドキュメント管理はGoogle Drive、Dropbox、OneDrive、SharePointとの統合を通じて効率化され、Slack、Gmail、WhatsAppなどのコミュニケーションツールはBlueの中央タスク管理システムを通じて統合されます。データ分析機能はGoogle Sheets、Power BI、Airtable、Tableauとの接続を通じて強化され、金融サービス、eコマース、リモートチーム管理向けの業界特化型ソリューションも利用可能です。

## Blueアカウントの接続と認証

BlueをMake.comに安全に接続するには：

1. Blueアカウントにログインし、プロフィールを開いて**API**タブに移動します。
2. **トークンを生成**をクリックし、名前（およびオプションの有効期限）を提供します。
3. 次の両方をコピーします：
   - **トークンID** (`x‑bloo‑token‑id`)
   - **シークレットキー** (`x‑bloo‑token‑secret`)

::callout
---
icon: lightbulb
---
**重要な注意** <br/>
シークレットキーは一度だけ表示されます—安全に保管してください！
::

5. Make.comで新しい接続を作成する際に、次の情報を入力します：
   - **トークンID**
   - **シークレットキー**

## 利用可能なBlueモジュール

シナリオを構築する際にこれらのモジュールを挿入します：

### レコードを作成
カスタムフィールドとメタデータを持つ新しいBlueエントリを追加します。

![レコードを作成モジュール](/docs/create_a_record.png "Create a Record module configuration")

### GraphQLクエリを実行
高度なデータ操作のために認可されたGraphQL操作を実行します。

![GraphQLクエリを実行モジュール](/docs/execute_a_graphql_query.png "Execute GraphQL Query module configuration")

### レコードを取得
IDによってレコードを取得し、すべての関連データを取得します。

![レコードを取得モジュール](/docs/get_a_record.png "Get a Record module configuration")

### プロジェクトにユーザーを招待
チームメンバーにプロジェクト招待を自動的に送信します。

![ユーザーを招待モジュール](/docs/invite_user_to_a_project.png "Invite User to Project module configuration")

### すべてのプロジェクトをリスト
すべてのBlueプロジェクトとその詳細を取得します。

![プロジェクトをリストモジュール](/docs/list_all_projects.png "List All Projects module configuration")

### カスタムフィールドをリスト
Blueプロジェクトのフィールドメタデータを取得します。

![カスタムフィールドをリストモジュール](/docs/list_custom_fields.png "List Custom Fields module configuration")

### プロジェクトのリストをリスト
Blueプロジェクト内のリスト/パイプラインにアクセスします。

![リストをリストモジュール](/docs/list_lists_of_a_project.png "List Lists of a Project module configuration")

### プロジェクト内のタグをリスト
プロジェクト全体でのタグの詳細と使用状況を取得します。

![タグをリストモジュール](/docs/list_tags_within_projects.png "List Tags Within Projects module configuration")

### プロジェクトのレコードを検索
さまざまな基準に基づいてレコードをフィルタリングして検索します。

![レコードを検索モジュール](/docs/search_records_of_a_project.png "Search Records of a Project module configuration")

### イベントを監視
レコードの作成、更新、または削除時にトリガーします。

![イベントを監視モジュール](/docs/watch_events.png "Watch Events module configuration")

## モジュールパラメータリファレンス

### イベントを監視
- イベント（オプション）：マルチセレクトイベントタイプ
- 会社ID（オプション）：マルチセレクト会社IDまたはスラグ
- プロジェクトID（オプション）：マルチセレクトプロジェクトIDまたはスラグ

### レコードを作成
- 会社ID：会社IDまたはスラグ
- プロジェクトID：プロジェクトIDまたはスラグ
- リスト：リストID
- レコード名：レコードの名前
- デフォルトフィールド：デフォルト値
- 単一日付：期日
- 日付範囲：開始日と期日
- 担当者：マルチセレクトユーザー
- タグ：マルチセレクトタグ
- 説明：テキストフィールド、HTMLをサポート
- チェックリスト：チェックリストタイトルの配列

**サポートされているカスタムフィールド：**
| フィールドタイプ | 説明 |
|------------------|------|
| 単一行テキスト | オプションの検証と文字制限を持つ単一行テキストフィールド |
| 複数行テキスト | リッチテキストサポートとフォーマットオプションを持つ複数行テキストフィールド |
| 単一選択 | 単一オプション選択とカスタムオプションを持つドロップダウン |
| 複数選択 | 複数オプション選択とカスタムオプションを持つドロップダウン |
| 位置 | 緯度と経度を持ち、地図ビューと住所検索をサポート |
| チェックボックス | オプションのデフォルト値とカスタムラベルを持つ真偽値トグル |
| 国 | 国名を持つ2文字のISO国コードセレクタ |
| 通貨 | 通貨記号と為替レートサポートを持つ数値フィールド（例：USD200、200USD、KHR4000） |
| 日付 | オプションの時間コンポーネントとタイムゾーンサポートを持つ日付ピッカー |
| 電話 | E.164、国際フォーマット、または国コード + 番号をサポートする電話番号フィールド |
| メール | フォーマット検証とドメイン確認を持つメールフィールド |
| 星評価 | 1-5の数値評価と視覚的な星、カスタムラベル |
| パーセント | パーセント記号（0-100）と小数精度を持つ数値フィールド |
| URL | フォーマット検証とプレビューを持つウェブアドレスフィールド |
| 数値 | オプションの小数点以下と単位表示を持つ数値フィールド |
| 数式 | 他のフィールドに基づく計算フィールドで数学的操作を含む |
| ユニークID | カスタムプレフィックスとフォーマットを持つ自動生成識別子 |
| 参照 | 検索とフィルタオプションを持つ他のレコードへのリンク |
| ルックアップ | マッピングオプションを持つ参照レコードからデータを引き出す動的フィールド |
| 時間の長さ | 時間と分のスパンを持ち、カスタム表示フォーマット |

### レコードを更新
- 会社ID：会社IDまたはスラグ
- プロジェクトID：プロジェクトIDまたはスラグ
- リスト：リストID
- レコードID：更新するレコードのID
- レコード名：レコードの名前
- デフォルトフィールド：デフォルト値
- 単一日付：期日
- 日付範囲：開始日と期日
- 担当者：マルチセレクトユーザー
- タグ：マルチセレクトタグ
- 説明：テキストフィールド、HTMLをサポート
- チェックリスト：チェックリストタイトルの配列

**サポートされているカスタムフィールド：**（「レコードを作成」と同じ）

### レコードを検索
- 会社ID：会社IDまたはスラグ
- プロジェクトID：プロジェクトIDまたはスラグ
- リスト：リストID
- 検索/クエリ：高度なフィルタリングオプションを持つ検索テキスト

### レコードを取得
- 会社ID：会社IDまたはスラグ
- プロジェクトID：プロジェクトIDまたはスラグ
- リスト：リストID
- レコードID：すべての関連データを持つレコードのIDを取得

### ユーザーを招待
- 会社ID：会社IDまたはスラグ
- プロジェクトID：プロジェクトIDまたはスラグ
- メールアドレス：検証付きのユーザーのメールアドレス
- ロールID：権限を持つロールID（デフォルトまたはカスタム）

### カスタムフィールドをリスト
- 会社ID：会社IDまたはスラグ
- プロジェクトID：プロジェクトIDまたはスラグ
- 制限：ページネーション付きで返す最大フィールド数

### リストをリスト
- 会社ID：会社IDまたはスラグ
- プロジェクトID：プロジェクトIDまたはスラグ
- 制限：ソートオプション付きで返す最大リスト数

### すべてのプロジェクトをリスト
- 会社ID：マルチセレクト会社IDまたはスラグ
- 制限：フィルタリング付きで返す最大プロジェクト数

### タグをリスト
- 会社ID：会社IDまたはスラグ
- プロジェクトID：プロジェクトIDまたはスラグ
- 制限：使用統計付きで返す最大タグ数

### GraphQLを実行
- ヘッダー：認証とカスタムヘッダーのためのキー-バリューの配列
- メソッド：リクエストオプション付きのGETまたはPOST
- クエリ：変数サポートを持つGraphQLクエリまたはミューテーション
- 操作名：追跡用の操作の名前
- 変数データソース：生の、フォーム、またはコレクションで型チェック
- 変数：検証付きのGraphQL変数コンテンツ
