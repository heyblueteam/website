CSV（カンマ区切り値）ファイルをインポートできます。Excelファイルがある場合は、[次のリンクでCSVファイルに変換できます](https://cloudconvert.com/xls-to-csv)。

## CSVのインポート

BlueでのCSVファイルのインポートは簡単なプロセスです。CSVファイルの各列は、Blueのデフォルトまたは[カスタムフィールド](/docs/custom-fields/introduction)に対応しています。

まず、データをインポートしたいプロジェクトのプロジェクト管理者である必要があります。プロジェクト設定アイコン（ギアアイコン）をクリックし、「インポート」を選択します。

![CSVインポート 1](/docs/CSVimport1.png "CSV Import 1")

インポート機能は、プロジェクト管理者のプロジェクト設定の下にあります。

上記のスクリーンショットでは、「CSVテンプレートのダウンロード」という別のオプションも見ることができます。これは、Blueプロジェクトで既に[カスタムフィールド](/docs/custom-fields/introduction)を作成している場合に非常に便利です。すべてのカスタムフィールドの列を含むCSVファイルをダウンロードできます！これは、インポート前にCSVの正確な構造を確保したい場合に最適です。

次に、CSVファイルをBlueにドラッグアンドドロップするだけです：

![CSVインポート 2](/docs/CSVimport2.png "CSV Import 2")

データをアップロードするためにドラッグアンドドロップします。

ファイルサイズによっては、アップロードとBlueによる分析に数秒かかる場合があります。Blueがデータを分析すると、データの最初の10行のプレビューが表示されます：

![CSVインポート 3](/docs/CSVimport3.png "CSV Import 3")

その後、CSVの各列をBlueのフィールドにマッピングできます。

![CSVインポート 4](/docs/CSVimport4.png "CSV Import 4")

マッピングが完了したら、「インポート」をクリックすると、データがインポートされます。Blueでの一般的なインポート速度は、1分あたり10,000から15,000レコードです。

## 受け入れられる日付フォーマット

Blueは、インポート用のさまざまな日付フォーマットをサポートしています。サポートされているすべてのフォーマットは以下の通りです：

- **Unixタイムスタンプ**:
  ```plaintext
  1511648546 (seconds)
  1620021848429 (milliseconds)
  1620024872717915000 (nanoseconds)
  ```
  
- **RFC標準**:
  ```plaintext
  RFC 3339: 2021-05-01T01:17:02.604456Z
  RFC 2822: Wed, 02 Jun 2021 06:31:39 GMT
  ```

- **データベースフォーマット**:
  ```plaintext
  PostgreSQL: 
  2019-11-29 08:08-08
  2019-11-29 08:08:05-08
  2021-05-02 23:31:36.0741-07
  ```

- **一般的なフォーマット**:
  ```plaintext
  Year-Month-Day: 
  2021-04-30 21:14
  2021-02-21
  2021-Feb-21
  
  Month/Day/Year:
  3/31/2014 22:05
  04/02/2014 03:00:51
  
  Day-Month-Year:
  12 Feb 2006, 19:17
  03 February 2013
  ```

- **タイムゾーン**:
  ```plaintext
  2014-04-26 13:13:43 +0800
  2017-11-25 13:31 PST
  2021-02-21 UTC
  ```

- **自然言語**:
  ```plaintext
  May 6 at 9:24 PM
  September 17, 2012 at 10:09am PST
  October 7, 1970
  ```

- **国際フォーマット**:
  ```plaintext
  Chinese: 2014年04月08日
  Japanese: 2014/04/08 22:05
  ```

- **時間のみ**:
  ```plaintext
  01:06:06
  4:00pm PST
  6:00 AM UTC
  ```

すべての日付は、インポート中にUTCタイムゾーンに変換されます。時間のコンポーネントはオプションです - 提供されない場合、時間は00:00:00 UTCにデフォルト設定されます。

## データ内のカンマの取り扱い

データにカンマ（たとえば、住所や会社名に）が含まれている場合は、インポートエラーを防ぐために、その値を二重引用符で囲む必要があります。

**例:**
```
Company,Address
Acme Inc,123 Main St, New York
```

**次のようになります:**
```
Company,Address
"Acme Inc","123 Main St, New York"
```

これにより、「123 Main St, New York」のカンマが列の区切りではなく、データの一部として扱われることが保証されます。
