Campos personalizados são uma forma de criar uma estrutura de dados personalizada dentro do Blue que combina perfeitamente com os requisitos da sua organização. Ao criar campos personalizados para seus dados estruturados em vez de usar comentários ou descrições, você aproveita todo o poder do Blue como plataforma de trabalho. Os dados estruturados podem se tornar sua única fonte da verdade.

Nossa [API](/pt/docs/integrations/api) e [Webhooks](/pt/docs/integrations/webhooks) oferecem suporte completo a campos personalizados, então ao usar campos personalizados no Blue, você está criando uma API de dados estruturados que combina perfeitamente com seus dados.


## Pontos Principais

Por padrão, cada registro no Blue vem com campos específicos:

- Nome do Registro
- [Responsável](/pt/docs/records/assignees)
- [Data](/pt/docs/records/dates)
- [Tag](/pt/docs/records/tags)
- [Dependências](/pt/docs/records/dependencies)

Mas você pode escolher entre diversos campos adicionais diferentes que pode adicionar a todos os registros dentro de um projeto. Por exemplo, isto poderia ser um campo personalizado de moeda para "Preço de Venda" em um projeto de vendas.

![Introduction 1](/docs/Introduction_1.png "Introduction 1")

Cada campo personalizado é uma coluna adicional no banco de dados do seu projeto

Campos personalizados estão totalmente integrados ao Blue. Usando os dados contidos em cada campo personalizado, você pode [criar formulários de entrada de trabalho](/pt/docs/projects/forms), [importar](/pt/docs/integrations/csv-import) e [exportar](/pt/docs/integrations/csv-export) dados, [filtrar dados](/pt/docs/views/filters) e [criar painéis de relatórios](/pt/docs/dashboards/introduction).

Campos personalizados podem ser subdivididos em dois tipos principais:

- **Campos Personalizados Editáveis**: Estes são campos personalizados onde os usuários podem inserir dados no campo. Por exemplo, um [campo personalizado de Email](/pt/docs/custom-fields/email) permite que os usuários insiram um endereço de email, ou um [campo personalizado de Seleção Única](/pt/docs/custom-fields/single-select) permite que os usuários escolham uma opção no menu suspenso fornecido.

- **Campos Personalizados Somente Leitura**: Estes são campos que apenas exibem dados. Por exemplo, o [campo personalizado de ID Único](/pt/docs/custom-fields/unique-id) exibe um identificador único gerado aleatoriamente para o seu registro, e o [campo personalizado de Fórmula](/pt/docs/custom-fields/formula) exibe o resultado de um cálculo baseado em valores de outros campos personalizados.

## Criando Campos Personalizados

Apenas [Administradores de Projeto](/pt/docs/user-management/roles/project-administrator) podem criar campos personalizados para garantir que a integridade da estrutura de dados do projeto seja mantida intacta. Você pode criar e gerenciar campos personalizados abrindo um registro no projeto e clicando em "Adicionar Mais Campos"

![Introduction 2](/docs/Introduction_2.png "Introduction 2")

Use o botão "Adicionar Mais Campos" dentro de um registro para adicionar campos personalizados a um projeto.

::callout
---

icon: lightbulb
---

**Campos Personalizados São Para Todo o Projeto** <br/> <br/>
Quando você cria um novo campo personalizado, esse campo personalizado fica disponível em todos os registros dentro do projeto específico. Cada registro pode então ter valores diferentes para esse campo personalizado. Por exemplo, o registro "Cliente 1" pode ter um valor de R$ 2.000 para o campo personalizado "Preço de Venda", enquanto o registro "Cliente 2" tem um valor de R$ 15.000 para o campo personalizado "Preço de Venda".<br/> <br/>
Utilizar campos personalizados em um projeto melhora significativamente a consistência dos dados e a eficiência do gerenciamento. Ao padronizar a estrutura de informações, os campos personalizados garantem que todos os registros sejam organizados uniformemente, tornando o rastreamento e análise de dados mais simplificados e confiáveis.
::

## Excluindo Campos Personalizados

Para excluir um campo personalizado, abra qualquer registro e clique em "Adicionar Mais Campos." Você será apresentado com uma lista de campos personalizados em seu projeto. Ao lado de cada campo personalizado, haverá um ícone de lixeira vermelha para excluir esse campo personalizado.

![Introduction 3](/docs/Introduction_3.png "Introduction 3")

O ícone de exclusão para campos personalizados
Você pode clicar neste ícone para excluir os campos personalizados do projeto. Você terá uma mensagem de confirmação, e então pode confirmar e excluir o campo personalizado.

::callout
---

icon: lightbulb
---

**Excluir Campos Personalizados É Para Todo o Projeto e Permanente** <br/><br/>
Quando você exclui um campo personalizado, isso exclui o campo personalizado de todos os registros dentro desse projeto. Todos os dados contidos dentro desse campo personalizado serão permanentemente excluídos e não são recuperáveis.
::

## Permissões de Usuário

Uma das características mais poderosas do Blue é que fornecemos permissões em nível de linha para campos personalizados. Isso permite que você defina quais grupos de usuários podem visualizar ou editar campos personalizados individuais. [Isso pode ser configurado criando uma função de usuário personalizada](/pt/docs/user-management/roles/custom-user-roles).
