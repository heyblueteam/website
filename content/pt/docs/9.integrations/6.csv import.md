Você pode importar arquivos CSV (Comma Separated Value). Se você tiver um arquivo Excel, [pode convertê-lo em um arquivo CSV no seguinte link](https://cloudconvert.com/xls-to-csv).

## Importando CSV

Importar arquivos CSV no Blue é um processo simples. Cada coluna em seu arquivo CSV corresponde a um campo padrão ou [campo personalizado](/docs/custom-fields/introduction) no Blue.

Para começar, você deve ser o Administrador do Projeto do projeto onde deseja importar dados. Clique no ícone de configurações do projeto (ícone de engrenagem) e selecione "Importar".

![Importação de CSV 1](/docs/CSVimport1.png "CSV Import 1")

O recurso de Importação está disponível nas Configurações do Projeto para Administradores de Projeto.

Na captura de tela acima, você também pode ver outra opção chamada "Baixar Modelo CSV". Isso é extremamente útil se você já criou seus [campos personalizados](/docs/custom-fields/introduction) em seu projeto Blue, pois você pode baixar um arquivo CSV que inclui colunas para todos os seus campos personalizados! Isso é ótimo quando você quer garantir que tenha a estrutura precisa para seu CSV antes de importar.

Em seguida, basta arrastar e soltar seu arquivo CSV no Blue:

![Importação de CSV 2](/docs/CSVimport2.png "CSV Import 2")

Arraste e solte para enviar seus dados.

Dependendo do tamanho do seu arquivo, isso pode levar alguns segundos para você enviar e para o Blue analisar. Assim que o Blue tiver analisado os dados, você verá uma prévia das dez primeiras linhas dos seus dados:

![Importação de CSV 3](/docs/CSVimport3.png "CSV Import 3")

Você pode então mapear cada coluna em seu CSV para um campo no Blue.

![Importação de CSV 4](/docs/CSVimport4.png "CSV Import 4")

## Formatos de Data Aceitos

O Blue suporta uma ampla variedade de formatos de data para importação. Aqui estão todos os formatos suportados:

- **Timestamps Unix**:
  ```plaintext
  1511648546 (seconds)
  1620021848429 (milliseconds)
  1620024872717915000 (nanoseconds)
  ```
  
- **Padrões RFC**:
  ```plaintext
  RFC 3339: 2021-05-01T01:17:02.604456Z
  RFC 2822: Wed, 02 Jun 2021 06:31:39 GMT
  ```

- **Formatos de Banco de Dados**:
  ```plaintext
  PostgreSQL: 
  2019-11-29 08:08-08
  2019-11-29 08:08:05-08
  2021-05-02 23:31:36.0741-07
  ```

- **Formatos Comuns**:
  ```plaintext
  Year-Month-Day: 
  2021-04-30 21:14
  2021-02-21
  2021-Feb-21
  
  Month/Day/Year:
  3/31/2014 22:05
  04/02/2014 03:00:51
  
  Day-Month-Year:
  12 Feb 2006, 19:17
  03 February 2013
  ```

- **Fusos Horários**:
  ```plaintext
  2014-04-26 13:13:43 +0800
  2017-11-25 13:31 PST
  2021-02-21 UTC
  ```

- **Linguagem Natural**:
  ```plaintext
  May 6 at 9:24 PM
  September 17, 2012 at 10:09am PST
  October 7, 1970
  ```

- **Formatos Internacionais**:
  ```plaintext
  Chinese: 2014年04月08日
  Japanese: 2014/04/08 22:05
  ```

- **Somente Hora**:
  ```plaintext
  01:06:06
  4:00pm PST
  6:00 AM UTC
  ```

Todas as datas serão convertidas para o fuso horário UTC durante a importação. Os componentes de hora são opcionais - se não fornecidos, a hora será definida como 00:00:00 UTC.

## Tratando Vírgulas em Seus Dados

Se seus dados contiverem vírgulas (por exemplo, em endereços ou nomes de empresas), você precisa envolver esses valores entre aspas duplas para evitar erros de importação.

**Exemplo:**
```
Company,Address
Acme Inc,123 Main St, New York
```

**Deve se tornar:**
```
Company,Address
"Acme Inc","123 Main St, New York"
```

Isso garante que a vírgula em "123 Main St, New York" seja tratada como parte dos dados e não como um separador de coluna.
