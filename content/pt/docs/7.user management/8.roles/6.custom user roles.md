No Blue, as permissões dos usuários são atribuídas individualmente e podem ser gerenciadas de forma eficiente através de grupos de usuários. Este sistema permite que [Administradores de Projetos](/docs/user-management/roles/project-administrator) agrupem usuários com funções e necessidades semelhantes e, em seguida, atribuam permissões personalizadas a todo o grupo. Isso simplifica o gerenciamento de acesso ao permitir a definição de funções que se alinham com as funções específicas e os requisitos de acesso de diferentes equipes ou departamentos dentro de um projeto.

Com o Blue, você pode:
- **Permissões Baseadas em Grupos**: Criar funções que se aplicam a grupos de usuários, garantindo direitos de acesso consistentes entre os membros da equipe que compartilham as mesmas responsabilidades.
- **Acesso a Atividades e Formulários**: Controlar o acesso do grupo a atividades e formulários do projeto, protegendo assim informações sensíveis e focando os esforços do grupo em tarefas relevantes.
- **Visibilidade de Registros**: Especificar se um grupo pode apenas visualizar registros atribuídos a eles, agilizando seu fluxo de trabalho e mantendo o foco.
- **Controle de Acesso em Nível de Tag**: Conceder ou restringir acesso a registros com base em suas tags, proporcionando filtragem de conteúdo granular além das permissões tradicionais.
- **Capacidades de Convite**: Atribuir o poder de convidar novos usuários para grupos, facilitando a expansão da equipe e mantendo o controle sobre quem pode adicionar membros ao projeto.

## Acesso a Campos Personalizados

Por padrão, os campos personalizados não são visíveis para funções de usuário personalizadas, mas você pode ativar essa opção.

Este é um recurso extremamente poderoso do Blue, pois fornece acesso em nível de coluna aos dados no Blue. Você pode determinar se uma função pode visualizar ou editar campos personalizados específicos, o que é crucial para manter a integridade e relevância dos dados. Isso pode variar de links de URL a documentos, designs ou durações de tempo.

## Gerenciamento de Listas

Atribua permissões específicas para cada lista dentro do projeto, como visualizar, editar ou excluir. Essa granularidade permite que você ajuste funções para gerenciar interações com diferentes listas, alinhando o acesso com os requisitos únicos de cada fase do projeto.

## Permissões em Nível de Tag

Controle o acesso a registros com base em suas tags, fornecendo mais uma camada de gerenciamento de permissões granular. As permissões em nível de tag permitem que você especifique quais registros os usuários podem visualizar e interagir com base nas tags atribuídas a esses registros.

Com permissões em nível de tag, você pode:
- **Permitir Tags Específicas**: Conceder acesso apenas a registros que têm tags específicas atribuídas.
- **Negar Tags Específicas**: Restringir o acesso a registros com certas tags, mesmo que outras permissões permitam o acesso.
- **Configuração Flexível**: Escolher entre tags permitidas ou tags negadas para criar o padrão de acesso que você precisa.
- **Filtragem Dinâmica**: Registros são automaticamente filtrados com base nas permissões de tags sem intervenção do usuário.

### Como Funcionam as Permissões de Tag

Quando as permissões em nível de tag são habilitadas para uma função, você pode escolher uma das duas abordagens:

1. **Tags Permitidas**: Os usuários só podem ver registros que têm pelo menos uma das tags permitidas especificadas.
2. **Tags Negadas**: Os usuários podem ver todos os registros, exceto aqueles que têm qualquer uma das tags negadas especificadas.

**Exemplos:**

**Abordagem de Tags Permitidas:**
- Defina tags permitidas: "Marketing", "Vendas"
- Os usuários só verão registros marcados com "Marketing" ou "Vendas" (ou ambos)
- Todos os outros registros serão ocultados

**Abordagem de Tags Negadas:**
- Defina tags negadas: "Confidencial", "Interno"
- Os usuários verão todos os registros, exceto aqueles marcados como "Confidencial" ou "Interno"
- Registros sem tags ou com outras tags serão visíveis

**Gerenciando Funções Personalizadas**

Criar uma função personalizada no Blue é um processo simplificado voltado para aprimorar os fluxos de trabalho do projeto. Para começar, você simplesmente fornece um nome descritivo e, se necessário, uma breve descrição para a função, estabelecendo seu propósito e escopo. Em seguida, você selecionará entre uma variedade de permissões adaptadas às responsabilidades da função dentro do projeto. Essas permissões ditam o que os usuários nessa função podem visualizar, editar ou excluir, e podem incluir acesso a atividades específicas do projeto, formulários e campos personalizados. Ao delinear essas permissões, você garante que a função esteja equipada com as ferramentas e níveis de acesso adequados para contribuir efetivamente para o projeto.

**Implementando Funções Personalizadas**

Após criar uma função personalizada, você pode atribuí-la a indivíduos ou grupos dentro do seu projeto. Esse processo de atribuição é fundamental, pois equipa os membros da equipe com um conjunto definido de capacidades e limitações que se alinham com os requisitos operacionais do projeto. É uma maneira de impor a governança de dados, agilizar a alocação de tarefas e manter a segurança do projeto. Ao implementar essas funções personalizadas, você pode garantir que cada usuário opere dentro de um ambiente controlado, com permissões que refletem seu envolvimento e necessidades, impulsionando, assim, o projeto em direção aos seus objetivos com precisão e clareza.
