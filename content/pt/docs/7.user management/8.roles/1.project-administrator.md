title: Administrador do Projeto
description: 
---

O Administrador do Projeto é o papel de maior privilégio dentro de um projeto Blue, semelhante ao Proprietário do Projeto. Os Administradores do Projeto têm controle total sobre o projeto e suas configurações.

**Permissões**

Os Administradores do Projeto podem:

- Editar todas as configurações do projeto
- Adicionar/remover membros do projeto
- Atribuir [papéis](/docs/user-management/roles) e [permissões](/docs/user-management/roles/custom-user-roles) aos usuários
- [Importar](/docs/integrations/csv-import) e [exportar](/docs/integrations/csv-export) dados do projeto
- Excluir projeto
- Criar [campos personalizados](/docs/custom-fields/introduction)
- Criar papéis personalizados
- Configurar [Automação](/docs/automations/introduction)

Essencialmente, eles têm permissões ilimitadas dentro do projeto.

**Casos de Uso**

Usuários típicos designados como Administradores do Projeto:

- Gerentes de projeto
- Líderes de equipe
- Gerentes de TI
- Proprietários de negócios

Qualquer pessoa que precise configurar completamente o fluxo de trabalho ou gerenciar membros.

**Atribuindo o Papel**

Ao convidar membros, o papel de Administrador do Projeto pode ser atribuído.

Membros existentes também podem ser promovidos a Administradores do Projeto. Mas deve sempre haver pelo menos um Administrador do Projeto em um projeto.

**Removendo o Papel**

Para remover as permissões de Administrador do Projeto de um membro, ele deve primeiro ser rebaixado para um papel personalizado mais restrito ou um papel padrão.
