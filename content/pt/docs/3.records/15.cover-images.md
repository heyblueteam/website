## Visão Geral

As imagens de capa permitem adicionar contexto visual aos seus registros exibindo visualizações de imagens diretamente nos cartões de registro. Este recurso extrai automaticamente imagens de várias fontes dentro dos seus registros e as exibe como miniaturas, facilitando a identificação e organização de conteúdo visual de forma rápida.

Seja gerenciando projetos de design, calendários de conteúdo, catálogos de produtos ou qualquer fluxo de trabalho envolvendo recursos visuais, as imagens de capa ajudam você a examinar rapidamente os registros e encontrar o que está procurando. O recurso inclui rastreamento de progresso em tempo real e opções de exibição flexíveis para atender às necessidades específicas do seu fluxo de trabalho.

## Principais Recursos

- **Extração Automática de Imagens**: Blue encontra e exibe automaticamente imagens da sua fonte escolhida
- **Múltiplas Fontes de Imagens**: Extraia imagens de descrições de registros, comentários ou campos de arquivo personalizados
- **Opções de Ajuste de Exibição**: Escolha como as imagens aparecem com Cover ou Contain
- **Seleção Inteligente**: Configure se deve usar a primeira ou última imagem da sua fonte
- **Atualizações em Tempo Real**: As imagens de capa são atualizadas automaticamente quando o conteúdo do registro muda
- **Rastreamento de Progresso ao Vivo**: Monitore o processamento em lote com assinaturas de progresso em tempo real

## Como Funciona

### Habilitando Imagens de Capa

As imagens de capa são configuradas no nível do projeto pelos Administradores do Projeto:

1. Abra seu projeto e navegue até **Configurações do Projeto**
2. Selecione **Configurações de Registro**
3. Encontre a seção **Opções de Capa de Imagem**
4. Alterne **Habilitar Imagens de Capa** para ativar o recurso

### Opções de Configuração

#### Fonte da Imagem
Escolha onde Blue deve procurar por imagens:
- **Descrição**: Extrai imagens incorporadas nas descrições dos registros (padrão)
- **Comentários**: Puxa a primeira/última imagem dos comentários do registro
- **Campo Personalizado**: Usa arquivos de um campo personalizado específico do tipo arquivo (suporta qualquer tipo de arquivo)

#### Seleção de Imagem
Quando múltiplas imagens estão disponíveis, escolha qual exibir:
- **Primeira Imagem**: Usa a primeira imagem encontrada (padrão)
- **Última Imagem**: Usa a imagem mais recente

#### Ajuste de Exibição
Controle como as imagens aparecem na área de visualização:
- **Cover**: A imagem preenche toda a área de visualização, pode cortar bordas para manter a proporção (padrão)
- **Contain**: Mostra a imagem completa dentro da área de visualização, pode ter espaço vazio

## Casos de Uso

**Projetos de Design** - Exiba mockups e iterações de design diretamente nos cartões de tarefas, facilitando a visualização do estado atual de cada tarefa de design sem abrir registros individuais.

**Planejamento de Conteúdo** - Para calendários de conteúdo e fluxos de trabalho editoriais, mostre imagens em destaque ou miniaturas para visualizar seu pipeline de conteúdo de forma rápida.

**Gestão de Produtos** - Exiba fotos de produtos em registros de inventário ou solicitações de recursos, ajudando as equipes a identificar rapidamente itens durante discussões.

**Imobiliário** - Mostre fotos de propriedades em registros de listagem, permitindo que agentes naveguem rapidamente por propriedades de forma visual.

**Planejamento de Eventos** - Exiba fotos de locais, conceitos de design ou layouts de eventos em tarefas de planejamento para melhor organização visual.

## Configurando Imagens de Capa

### Passo 1: Escolha Sua Fonte de Imagem

Considere onde as imagens existem naturalmente no seu fluxo de trabalho:
- Se as imagens fazem parte de descrições de tarefas ou briefings, use **Descrição**
- Se as imagens são adicionadas através de colaboração em equipe, use **Comentários**
- Se você tem um campo dedicado para imagens, use **Campo Personalizado**

### Passo 2: Configure as Configurações de Exibição

1. Selecione seu **Ajuste de Imagem** preferido baseado nos seus tipos de imagem:
   - Use **Cover** para fotos e designs onde o corte é aceitável
   - Use **Contain** para diagramas ou capturas de tela onde você precisa ver tudo

2. Escolha **Seleção de Imagem** baseada no seu fluxo de trabalho:
   - Use **Primeira** para fluxos de trabalho onde a imagem principal é adicionada inicialmente
   - Use **Última** para fluxos de trabalho onde a imagem mais recente é mais relevante

### Passo 3: Processe Registros Existentes

Quando você habilita imagens de capa pela primeira vez ou altera a configuração, Blue processa automaticamente seus registros:
1. Atualizações de progresso em tempo real mostram o status do processamento
2. Registros são processados em lotes (50 registros por lote) para performance otimizada  
3. Você pode continuar trabalhando enquanto o processamento acontece em segundo plano
4. Listas serão automaticamente atualizadas quando o processamento for concluído
5. O processamento é acionado automaticamente quando mudanças de configuração são salvas

## Detalhes Técnicos

### Formatos Suportados
- Ao usar fontes de Descrição ou Comentários:
  - Imagens JPEG/JPG
  - Imagens PNG (incluindo transparência)
  - Imagens GIF (exibição estática)
- Ao usar fonte de Campo Personalizado:
  - Qualquer tipo de arquivo pode ser exibido (imagens, PDFs, documentos)
  - Arquivos não-imagem exibem ícones apropriados do tipo de arquivo
- Todos os arquivos devem ser enviados para Blue (URLs externos não são suportados)

### Limitações
- Máximo de 6.000 registros por projeto para geração em lote de capas
- Apenas campos personalizados do tipo arquivo podem ser usados como fontes
- Imagens de capa são somente para visualização e não podem ser editadas diretamente

### Atualizações Automáticas
Capas são regeneradas automaticamente quando:
- Descrição do registro é editada (se usando fonte Descrição)
- Novos comentários com imagens são adicionados (se usando fonte Comentários)
- Arquivos em campos personalizados são atualizados (se usando fonte Campo Personalizado)
- Configurações de capa são alteradas
- Opção de ajuste de exibição é modificada

### Acesso à API
Imagens de capa são totalmente integradas com a API GraphQL:
- Configure via mutação `setTodoCoverConfig`
- Acesse dados de capa através do campo `cover` no tipo Todo
- Monitore progresso com assinatura `subscribeToCoverGenerationProgress`
- URLs de miniaturas geradas automaticamente em `/uploads/{uid}/thumbnail/cover`
