Benutzerdefinierte Felder sind eine Möglichkeit, eine benutzerdefinierte Datenstruktur in Blue zu erstellen, die perfekt zu Ihren organisatorischen Anforderungen passt. Indem Sie benutzerdefinierte Felder für Ihre strukturierten Daten verwenden, anstatt Kommentare oder Beschreibungen zu nutzen, nutzen Sie die volle Leistungsfähigkeit von Blue als Arbeitsplattform. Die strukturierten Daten können zu Ihrer einzigen Quelle der Wahrheit werden.

Unsere [API](/de/docs/integrations/api) und [Webhooks](/de/docs/integrations/webhooks) unterstützen benutzerdefinierte Felder vollständig, sodass Sie durch die Verwendung benutzerdefinierter Felder in Blue eine strukturierte Daten-API erstellen, die perfekt zu Ihren Daten passt.


## Wichtige Punkte

Standardmäßig verfügt jeder Datensatz in Blue über spezifische Felder:

- Datensatzname
- [Bearbeiter](/de/docs/records/assignees)
- [Datum](/de/docs/records/dates)
- [Tag](/de/docs/records/tags)
- [Abhängigkeiten](/de/docs/records/dependencies)

Sie können jedoch aus zahlreichen verschiedenen zusätzlichen Feldern wählen, die Sie zu allen Datensätzen innerhalb eines Projekts hinzufügen können. Dies könnte beispielsweise ein Währungs-Benutzerdefiniertes Feld für "Verkaufspreis" in einem Vertriebsprojekt sein.

![Introduction 1](/docs/Introduction_1.png "Introduction 1")

Jedes benutzerdefinierte Feld ist eine zusätzliche Spalte in Ihrer Projektdatenbank

Benutzerdefinierte Felder sind eng in Blue integriert. Mit den in jedem benutzerdefinierten Feld enthaltenen Daten können Sie [Arbeitsaufnahmeformulare erstellen](/de/docs/projects/forms), Daten [importieren](/de/docs/integrations/csv-import) und [exportieren](/de/docs/integrations/csv-export), [Daten filtern](/de/docs/views/filters) und [Reporting-Dashboards erstellen](/de/docs/dashboards/introduction).

Benutzerdefinierte Felder können in zwei Haupttypen unterteilt werden:

- **Bearbeitbare benutzerdefinierte Felder**: Dies sind benutzerdefinierte Felder, in die Benutzer Daten eingeben können. Beispielsweise ermöglicht ein [E-Mail-Benutzerdefiniertes Feld](/de/docs/custom-fields/email) Benutzern die Eingabe einer E-Mail-Adresse, oder ein [Einzelauswahl-Benutzerdefiniertes Feld](/de/docs/custom-fields/single-select) ermöglicht es Benutzern, eine Option aus dem bereitgestellten Dropdown-Menü zu wählen.

- **Schreibgeschützte benutzerdefinierte Felder**: Dies sind Felder, die nur Daten anzeigen. Beispielsweise zeigt das [Eindeutige ID-Benutzerdefinierte Feld](/de/docs/custom-fields/unique-id) eine zufällig generierte eindeutige Kennung für Ihren Datensatz an, und das [Formel-Benutzerdefinierte Feld](/de/docs/custom-fields/formula) zeigt das Ergebnis einer Berechnung basierend auf Werten aus anderen benutzerdefinierten Feldern an.

## Erstellen benutzerdefinierter Felder

Nur [Projektadministratoren](/de/docs/user-management/roles/project-administrator) können benutzerdefinierte Felder erstellen, um sicherzustellen, dass die Integrität der Datenstruktur des Projekts erhalten bleibt. Sie können benutzerdefinierte Felder erstellen und verwalten, indem Sie einen Datensatz im Projekt öffnen und auf "Weitere Felder hinzufügen" klicken.

![Introduction 2](/docs/Introduction_2.png "Introduction 2")

Verwenden Sie die Schaltfläche "Weitere Felder hinzufügen" innerhalb eines Datensatzes, um benutzerdefinierte Felder zu einem Projekt hinzuzufügen.

::callout
---

icon: lightbulb
---

**Benutzerdefinierte Felder gelten projektweite** <br/> <br/>
Wenn Sie ein neues benutzerdefiniertes Feld erstellen, ist dieses benutzerdefinierte Feld in allen Datensätzen innerhalb des spezifischen Projekts verfügbar. Jeder Datensatz kann dann unterschiedliche Werte für dieses benutzerdefinierte Feld haben. Beispielsweise kann der Datensatz "Kunde 1" einen Wert von 2.000 € für das benutzerdefinierte Feld "Verkaufspreis" haben, während der Datensatz "Kunde 2" einen Wert von 15.000 € für das benutzerdefinierte Feld "Verkaufspreis" hat.<br/> <br/>
Die Verwendung benutzerdefinierter Felder über ein Projekt hinweg verbessert die Datenkonsistenz und Verwaltungseffizienz erheblich. Durch die Standardisierung der Informationsstruktur stellen benutzerdefinierte Felder sicher, dass alle Datensätze einheitlich organisiert sind, wodurch die Datenverfolgung und -analyse rationalisierter und zuverlässiger wird.
::

## Löschen benutzerdefinierter Felder

Um ein benutzerdefiniertes Feld zu löschen, öffnen Sie einen beliebigen Datensatz und klicken Sie auf "Weitere Felder hinzufügen". Ihnen wird eine Liste der benutzerdefinierten Felder in Ihrem Projekt angezeigt. Neben jedem benutzerdefinierten Feld befindet sich ein rotes Papierkorb-Symbol, um dieses benutzerdefinierte Feld zu löschen.

![Introduction 3](/docs/Introduction_3.png "Introduction 3")

Das Lösch-Symbol für benutzerdefinierte Felder
Sie können auf dieses Symbol klicken, um die benutzerdefinierten Felder aus dem Projekt zu löschen. Sie erhalten eine Bestätigungsmeldung und können dann bestätigen und das benutzerdefinierte Feld löschen.

::callout
---

icon: lightbulb
---

**Das Löschen benutzerdefinierter Felder ist projektweite und dauerhaft** <br/><br/>
Wenn Sie ein benutzerdefiniertes Feld löschen, wird dieses benutzerdefinierte Feld aus allen Datensätzen innerhalb dieses Projekts gelöscht. Alle in diesem benutzerdefinierten Feld enthaltenen Daten werden dauerhaft gelöscht und sind nicht wiederherstellbar.
::

## Benutzerberechtigungen

Eine der leistungsstärksten Funktionen von Blue ist, dass wir Berechtigungen auf Zeilenebene für benutzerdefinierte Felder bereitstellen. Dies ermöglicht es Ihnen, festzulegen, welche Benutzergruppen einzelne benutzerdefinierte Felder anzeigen oder bearbeiten können. [Dies kann durch die Erstellung einer benutzerdefinierten Benutzerrolle eingerichtet werden](/de/docs/user-management/roles/custom-user-roles).
