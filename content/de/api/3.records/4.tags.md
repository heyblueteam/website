## Übersicht

Tags sind eine leistungsstarke Möglichkeit, Datensätze in Blue zu kategorisieren und zu organisieren. Das Tagsystem bietet umfassende Funktionen zum Erstellen, Verwalten und Filtern von Tags, mit Unterstützung für Massenoperationen und erweitertes Filtern.

## Tags auflisten

Rufen Sie Tags aus Projekten mit flexiblen Filter- und Sortieroptionen ab.

### Einfaches Beispiel

```graphql
query ListTags {
  tagList(
    filter: { 
      projectIds: ["project-123"] 
      excludeArchivedProjects: false 
    }
    first: 50
    orderBy: title_ASC
  ) {
    items {
      id
      uid
      title
      color
      project {
        id
        name
      }
      createdAt
      updatedAt
    }
    pageInfo {
      totalPages
      totalItems
      page
      perPage
      hasNextPage
      hasPreviousPage
    }
    totalCount
  }
}
```

### Beispiel für erweitertes Filtern

```graphql
query FilterTags {
  tagList(
    filter: {
      projectIds: ["project-123", "project-456"]
      search: "urgent"
      colors: ["#ff0000", "#ffaa00"]
      excludeArchivedProjects: true
    }
    orderBy: createdAt_DESC
    first: 20
  ) {
    items {
      id
      title
      color
      todos {
        id
        title
      }
    }
    totalCount
  }
}
```

## Tags erstellen

Erstellen Sie neue Tags innerhalb eines Projekts.

```graphql
mutation CreateTag {
  createTag(
    input: {
      title: "High Priority"
      color: "#ff0000"
    }
  ) {
    id
    uid
    title
    color
    project {
      id
      name
    }
    createdAt
  }
}
```

## Tags aktualisieren

Ändern Sie die Eigenschaften vorhandener Tags.

```graphql
mutation UpdateTag {
  editTag(
    input: {
      id: "tag-123"
      title: "Critical Priority"
      color: "#cc0000"
    }
  ) {
    id
    title
    color
    updatedAt
  }
}
```

## Tags löschen

Entfernen Sie Tags aus dem System. Dies entfernt das Tag auch aus allen zugehörigen Datensätzen.

```graphql
mutation DeleteTag {
  deleteTag(id: "tag-123")
}
```

## Datensätze taggen

Verknüpfen Sie Tags mit Datensätzen (To-Dos) unter Verwendung vorhandener Tags oder erstellen Sie neue.

### Verwendung vorhandener Tags

```graphql
mutation TagRecord {
  setTodoTags(
    input: {
      todoId: "todo-123"
      tagIds: ["tag-456", "tag-789"]
    }
  )
}
```

### Tags beim Taggen erstellen

Sie können neue Tags im Handumdrehen erstellen, indem Sie Tag-Titel angeben:

```graphql
mutation TagRecordWithNewTags {
  setTodoTags(
    input: {
      todoId: "todo-123"
      tagIds: ["existing-tag-id"]
      tagTitles: ["New Tag", "Another New Tag"]
    }
  )
}
```

## KI-Tag-Vorschläge

Erhalten Sie KI-gestützte Tag-Vorschläge für einen Datensatz:

```graphql
mutation GetTagSuggestions {
  aiTag(
    input: {
      todoId: "todo-123"
      suggestionsCount: 5
    }
  ) {
    suggestions {
      title
      confidence
      color
    }
  }
}
```

## Eingabeparameter

### TagListFilter

| Parameter | Typ | Erforderlich | Beschreibung |
|-----------|------|--------------|-------------|
| `projectIds` | [String!] | Nein | Filtert Tags nach Projekt-IDs oder Slugs |  |
| `excludeArchivedProjects` | Boolean | Nein | Schließt Tags aus archivierten Projekten aus |  |
| `search` | String | Nein | Sucht nach Tag-Titeln (nicht groß-/kleinschreibungsempfindlich) |  |
| `titles` | [String!] | Nein | Filtert nach bestimmten Tag-Titeln |  |
| `colors` | [String!] | Nein | Filtert nach bestimmten Farben (Hex-Format) |  |
| `tagIds` | [String!] | Nein | Filtert nach bestimmten Tag-IDs |  |

### CreateTagInput

| Parameter | Typ | Erforderlich | Beschreibung |
|-----------|------|--------------|-------------|
| `title` | String | Nein | Tag-Titel (automatisch generiert, wenn nicht angegeben) |  |
| `color` | String! | ✅ Ja | Tag-Farbe im Hex-Format (z.B. "#ff0000") |  |

### EditTagInput

| Parameter | Typ | Erforderlich | Beschreibung |
|-----------|------|--------------|-------------|
| `id` | String! | ✅ Ja | ID des zu aktualisierenden Tags |  |
| `title` | String | Nein | Neuer Tag-Titel |  |
| `color` | String | Nein | Neue Tag-Farbe im Hex-Format |  |

### SetTodoTagsInput

| Parameter | Typ | Erforderlich | Beschreibung |
|-----------|------|--------------|-------------|
| `todoId` | String! | ✅ Ja | ID des zu taggenden Datensatzes |  |
| `tagIds` | [String!] | Nein | IDs vorhandener Tags, die angewendet werden sollen |  |
| `tagTitles` | [String!] | Nein | Titel neuer Tags, die erstellt und angewendet werden sollen |  |

### TagOrderByInput-Werte

| Wert | Beschreibung |
|-------|-------------|
| `id_ASC` / `id_DESC` | Nach Tag-ID sortieren |  |
| `uid_ASC` / `uid_DESC` | Nach eindeutiger Kennung sortieren |  |
| `title_ASC` / `title_DESC` | Alphabetisch nach Titel sortieren |  |
| `color_ASC` / `color_DESC` | Nach Farbwert sortieren |  |
| `createdAt_ASC` / `createdAt_DESC` | Nach Erstellungsdatum sortieren |  |
| `updatedAt_ASC` / `updatedAt_DESC` | Nach Datum der letzten Aktualisierung sortieren |  |

## Antwortfelder

### Tag-Typ

| Feld | Typ | Beschreibung |
|-------|------|-------------|
| `id` | ID! | Eindeutige Kennung für das Tag |  |
| `uid` | String! | Benutzerfreundliche eindeutige Kennung |  |
| `title` | String! | Tag-Titel/-Name |  |
| `color` | String! | Tag-Farbe im Hex-Format |  |
| `project` | Project! | Projekt, zu dem dieses Tag gehört |  |
| `todos` | [Todo!]! | Datensätze, die mit diesem Tag verknüpft sind |  |
| `createdAt` | DateTime! | Wann das Tag erstellt wurde |  |
| `updatedAt` | DateTime! | Wann das Tag zuletzt aktualisiert wurde |  |

### TagPagination-Antwort

| Feld | Typ | Beschreibung |
|-------|------|-------------|
| `items` | [Tag!]! | Array von Tag-Datensätzen |  |
| `pageInfo` | PageInfo! | Paginierungsinformationen |  |
| `totalCount` | Int! | Gesamtzahl der Tags, die den Filtern entsprechen |  |

### PageInfo-Felder

| Feld | Typ | Beschreibung |
|-------|------|-------------|
| `totalPages` | Int | Gesamtzahl der Seiten |  |
| `totalItems` | Int | Gesamtzahl der Elemente über alle Seiten |  |
| `page` | Int | Aktuelle Seitenzahl |  |
| `perPage` | Int | Anzahl der Elemente pro Seite |  |
| `hasNextPage` | Boolean! | Ob es eine nächste Seite gibt |  |
| `hasPreviousPage` | Boolean! | Ob es eine vorherige Seite gibt |  |

## Erforderliche Berechtigungen

### Tag-Verwaltung

| Aktion | Erforderliche Rolle |
|--------|---------------------|
| Create tags | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |
| Edit tags | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |
| Delete tags | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |
| Apply tags to records | `OWNER`, `ADMIN`, `MEMBER`, or `CLIENT` |  |

### Tag-Sichtbarkeit

Tags sind für alle Benutzer sichtbar, die Zugriff auf das Projekt haben, in dem das Tag erstellt wurde. Rollenbasierte Filterung kann für bestimmte Operationen gelten.

## Beste Praktiken

1. **Verwenden Sie konsistente Farben**: Etablieren Sie ein Farbcodesystem für Ihr Team
2. **Beschreibende Titel**: Verwenden Sie klare, beschreibende Tag-Namen, die Teammitglieder verstehen
3. **Batch-Operationen**: Verwenden Sie `setTodoTags`, um mehrere Tags auf einmal anzuwenden
4. **Suchen und Filtern**: Verwenden Sie die Suchfunktion, um vorhandene Tags zu finden, bevor Sie neue erstellen
5. **Tag-Verwaltung**: Überprüfen und bereinigen Sie regelmäßig ungenutzte Tags
6. **Farbkonventionen**: Ziehen Sie in Betracht, Standardfarben für gängige Tag-Typen zu verwenden (rot für dringend, grün für abgeschlossen usw.)

## Erweiterte Funktionen

### Standardwerte für die Tag-Erstellung

- Neue Tags, die über `setTodoTags` mit `tagTitles` erstellt werden, haben standardmäßig die Farbe `#4a9fff`
- Tags ohne angegebene Farben fallen auf `#00a0d2` zurück
- Tag-Titel werden automatisch von Leerzeichen befreit

### Automatisierungsintegration

Tags können:
- Automatisch durch Automatisierungsregeln angewendet werden
- Als Auslöser für Automatisierungs-Workflows verwendet werden
- Durch Automatisierungsaktionen geändert werden

### Echtzeit-Updates

Tag-Änderungen werden über GraphQL-Abonnements übertragen, wodurch Echtzeit-Updates in verbundenen Clients ermöglicht werden.

## Wichtige Hinweise

- Das Löschen eines Tags entfernt es aus allen zugehörigen Datensätzen
- Tags sind projektbezogen und können nicht über Projekte hinweg geteilt werden
- Farbwerte müssen gültige Hex-Codes sein (z.B. "#ff0000", "#00aa00")
- Tag-Titel sind groß-/kleinschreibungsempfindlich
- Maximal 100 Tags können auf einen einzelnen Datensatz angewendet werden
- Tags unterstützen die Volltextsuche über Titel
